The notion that executing n threads in parallel should correspond to some interleaved execution of those threads on a single core is called sequential consistency。

 If multico systems implement sequential consistency。

One possible fix is to give up on sequential consistency。

An alternative memory semantics is weak consistency。

 which only requires that the memory operations from each thread appear to be performed in the order issued by that thread。



![](img/cd680dd253017879e7730d053a5140d4_11.png)

In other words， in a weakly consistent system， if a particular thread writes to X and then writes to Y。

In a weakly consistent system， memory operations from other threads may overlap in arbitrary ways。

Note that our multicore cache doesn't itself guarantee even weak consistency。

A thread that executes right X， right Y will update its local cache。

Out of order courses have an extra complication， since there's no guarantee that successive storage instructions will complete in the order they appeared in the program。

These architectures provide a barrier instruction that guarantees that memory operations before the barrier are completed before memory operations executed after the barrier。
