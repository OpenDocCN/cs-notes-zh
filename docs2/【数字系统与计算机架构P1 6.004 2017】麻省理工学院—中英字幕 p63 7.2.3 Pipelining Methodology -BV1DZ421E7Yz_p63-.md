Here's our strategy that will ensure if we add a pipeline register along one path from system inputs to system outputs。

 we will add pipeline registers along every path。Step1 is to draw a contour line that crosses every output in the circuit and markets endpoints as the terminal points for all the other contours will add。

Make sure that every signal connection crosses the new contour line in the same direction。

This means that system inputs will be on one side of the contour and system outputs will be on the other side。

These contours demarcate pipeline stages。Place a pipeline register wherever a signal connection intersects。

 the pipeline and contours。Here we've marked the location of pipeline registers with large black dots。

By drawing the contours from terminal point to terminal point。

 we guarantee that we cross every input output path， thus ensuring our pipeline will be well formed。

Now we can compute the system clock period by looking for the pipeline stage with the longest register to register or input to register Proag delay。

With these contours and assuming ideal zero delay pipeline registers。

This gives a system throughput of one output every eight nanoseconds。Since we drew three contours。

Our usual goal in pipelining a circuit is to achieve maximum throughput using the fewest possible registers。

So our strategy is to find the slowest system component， in our example， the C component。

 and place pipeline registers on its inputs and outputs。

So we drew contours that passed on either side of the C module。

This sets the clock period at eight nanoseconds， so we positioned the contour so that the longest path between any two pipeline registers is at most eight。

There are often several choices for how to draw a contour while maintaining the same throughput and latency。

 For example， we could have included the E module in the same pipeline stage as the F module。

Okay， let's review our pipeing strategy。First， we draw a contour across all the outputs。

 This creates a one pipeline， which， as you can see。

 will always have the same throughput in latency as the original combinational circuit。

Then we draw our next contour trying to isolate the slowest component in the system。

This creates a two pipeline with a clock period of two and hence a throughput of one half or double that of the one pipeline。

We can add additional contours， but note that the two pipeline had the smallest possible clo period。

 So after that， additional contours add stages and hence increase the system's latency without increasing its throughput。

Not illegal， just not a worthwhile investment in hardware。

Note that the signal connection between the A and C module now has two back to back pipelinelining registers。

Nothing wrong with that， it often happens when we pipeline a circuit where the input output paths are of different length。

So our pipelineing strategy will be to pipeline implementations with increasing throughput。

 usually at the cost of increased latency。Sometimes we get lucky and the delays of each pipeline stage are perfectly balanced。

In which case the latency will not increase。Note that a pipeline circuit will never have a smaller latency than the unppeline circuit。

Notice that once we've isolated the slowest component， we can't increase the throughput any further。
