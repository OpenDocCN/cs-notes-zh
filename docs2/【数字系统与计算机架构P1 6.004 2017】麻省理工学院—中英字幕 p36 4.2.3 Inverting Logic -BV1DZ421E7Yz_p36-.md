Why would we ever want to use a slow gate？Remember that the propagation delay of a circuit is set by the longest path in terms of delay from inputs to outputs。

In a complex circuit， there are many input output paths。

 but it's only the components on the longest path that need to be fast in order to achieve the best possible overall TPD。

The components on the other shorter paths can potentially be a bit slower。

And the components on short input output pass can be very slow indeed。

So for the portions of the circuit that aren't speed sensitive。

 it's a good trade off to use slower but smaller gates。 The overall performance isn't effect。

 but the total size is improved。So for faster performance we'll design with inverting gates。

 and for smaller size will design with non inverting gates。

 the creators at the Gate Library design the available gates with this trade off in mind。

The four input inverting gates are also designed with this trade off in mind。

For the ultimate in performance， we want to use a tree circuit of two input gates。

This circuit is a good example of the application of a particular Boolean identity known as De Morgan's Law。

There are two forms of D Morgangan's law， both of which are shown here。

The top form is the one we're interested in for analyzing the lower circuit。

It tells us that the nor of A with B is equivalent to the and of not a with not B。

So the two input Nor gate can be thought of as a two input and gate with inverting inputs。

How does this help？We can now see that the lower circuit is actually a tree of ant gates。

Where the inverting outputs of the first layer match up with the inverting inputs of the second layer。

It's a little confusing the first time you see it， but with practice you'll be comfortable using D Morgan's Law when building trees or chains of inverting logic。

Our G library includes inverting gates with up to four inputs， why stop there？Well。

 the pull down chain of a four input Ngate has four nFs in series。

 and the resistance of the conducting channels is starting to add up。

We could make the nFts wider to compensateate， but then the gate gets much larger and the wider NFs impose a higher capacitive load on the input signals。

The number of possible trade offs between size and speed grows rapidly with the number of inputs。

 so it's usually just best for the library designer to stop at four input gates and let the circuit designer take it from there。
