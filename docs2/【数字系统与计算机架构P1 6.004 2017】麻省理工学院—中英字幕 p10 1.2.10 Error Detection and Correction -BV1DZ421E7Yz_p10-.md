Now， let's think a bit about what happens if there's an error and one or more of the bits in our encoded data gets corrupted。

 we'll focus on single bit errors， but much of what we discuss can be generalized to multi bit errors。

For example， consider encoding the results of some unpredictable event。 For example。

 flipping a fair coin。There are two outcomes， heads encoded as， say，0 and tails encoded as one。



![](img/add746ab863c6a23d11326bb0209e70f_1.png)

Now， suppose some error occurs during processing。 For example。

 the data is corrupted while being transmitted from Bob to Alice。

 Bob intended to send the message heads， but the 0 was corrupted and became a one during transmission。

 So Alice receives one， which he interprets his tales。

So this simple encoding doesn't work very well if there's the possibility of single bit errors。



![](img/add746ab863c6a23d11326bb0209e70f_3.png)

To help with our discussion， we'll introduce the notion of hamming distance defined as the number of positions in which the corresponding digits differ in two encodings of the same length。

For example， here are two seven bit encodings which differ in their third and fifth positions。

 so the hamming distance between the encodings is2。

If someone tells us the heming distance of two encodings is zero。

 then the two encodings are identical。Hamming distanceance is a handy tool for measuring how encodings differ。



![](img/add746ab863c6a23d11326bb0209e70f_5.png)

How does this help us think about single bit errors。

 A single bit error changes exactly one of the bits of an encoding。

 so the hamming distance between a valid binary code word and the same code word where the single bit error is one。



![](img/add746ab863c6a23d11326bb0209e70f_7.png)

The difficulty with our simple encoding is that the two valid code words。

 0 and 1 also have a heming distance of1， so a single bit error changes one valid code word into another valid code word。



![](img/add746ab863c6a23d11326bb0209e70f_9.png)

We'll show this graphically using an arrow to indicate that two encodings differ by a single bit。

 In other words， that the hamming distance between the encodings is one。

The real issue here is that when Alice receives a one。

 she can't distinguish between an uncorrupted encoding of tails and a corrupted encoding of heads。

 She can't detect that an error has occurred。 Let's figure out how to solve her problem。



![](img/add746ab863c6a23d11326bb0209e70f_11.png)

The insight is to come up with a set of valid code words such that a single bit error does not produce another valid code word。

 What we need are code words that differ by at least two bits。 In other words。

 we want the minimum hamming distance between any two code words to be at least2。

If we have a set of code wordss where the minimum heming distance is1。

 we can generate the set we want by adding a pary bit to each of the original code words。

There's even pary and odd pary， using even pary， the additional pary bit is chosen so that the total number of1 bits in the new code word are even。

For example， our original encoding for heads was0， adding an even parody bit gives us 00。

Adding an even parody bit to our original encoding for tails gives us 11。

The minimum hamming distance between code words has increased from1 to2。How does this help？

Consider what happens when there's a single bit error 0，0 would be corrupted to 0，1 or 1，0。

 Neither of which is a valid code word。 aha， We can detect that a single bit error has occurred。

 Similarlyly， single bit errors for  one，1 would also be detected。Note that the valid code word 0。

0 and 1，1， both have an even number of 1 B， but that the corrupted code word 0。

1 or 10 have an odd number of 1 B。 We say that corrupted code words have a parodyier。



![](img/add746ab863c6a23d11326bb0209e70f_13.png)

It's easy to perform a parity check。 Simply count the number of ones in the code word。

 If it's even a single bit error has not occurred。 If it's odd， a single bit error has occurred。

 We'll see in a couple of chapters that the Boolean function exclusive aura can be used to perform parodity checks。

Note that parody won't help us if there's an even number of bit errors or a corrupted code word would have an even number of1 bits and hence appear to be okay。

Pity is useful for detecting single bit errors， will need more sophisticated encoding to detect more errors。



![](img/add746ab863c6a23d11326bb0209e70f_15.png)

In general， to detect some number E of errors， we need a minimum hamming distance of E plus1 between code words。

 We can see this graphically below， which shows how errors can corrupt the valid code words 0，0。

0 and 111， which have a hamming distance of 3。 In theory。

 this means we should be able to detect up to 2 bit errors。



![](img/add746ab863c6a23d11326bb0209e70f_17.png)

Each arrow represents a single bit error， and we can see from the diagram that following any path of length 2 from either 0。

0，0 or 1，1，1 doesn't get us to the other valid code word。 In other words。

 assuming we start with either 0，0，0 or 1，1，1， we can detect the occurrence of up to 2 errors。

Basically， our error detection scheme relies on choosing code wordss far enough apart as measured by hamming distance so that E errorsors can't corrupt one valid code word so that it looks like another valid code word。
