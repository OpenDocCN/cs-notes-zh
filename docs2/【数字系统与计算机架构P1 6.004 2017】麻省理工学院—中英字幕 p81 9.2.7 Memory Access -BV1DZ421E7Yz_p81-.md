Now let's turn our attention to the second class of instructions， load and store。

 which allow the CPU to access values in memory。Note that since the beta is a load store architecture。

 these instructions are the only mechanism for accessing memory values。

The load and store instructions use the same instruction template as the ALU with constant instructions。



![](img/6bb8086d3aea711752e37b9ff9a066ce_1.png)

To access memory we'll need a memory address which is computed by adding the value of the RA register to the sign extended 16 bit constant from the low order 16 bits of the instruction。

This computation is exactly the one performed by the ADC instruction， so we'll reuse that hardware。

And the sum is sent to main memory as the by address of the location to be accessed for the load instruction。

 the data returned by main memory is written to the RC register。



![](img/6bb8086d3aea711752e37b9ff9a066ce_3.png)

The store instruction performs the same address calculation as load。

 then reach the data value from the RSC register and sends both to main memory。

The store instruction is special in several ways。 Its the only instruction that needs to read the value of the RC register。

 so we'll need to adjust the data path hardware slightly to accommodate that need。

AndSince RC is serving as a source operaand， it appears as the first operaand in the symbolic form of the instruction。

 followed by cost and RA， which are specifying the destination address。

Store is the only instruction that does not write a result into the register file at the end of the instruction。



![](img/6bb8086d3aea711752e37b9ff9a066ce_5.png)

Here's the example we saw earlier where we needed to load the value of the variable x from memory。

 multiply it by 37， and write the result back to the memory location that holds the value of the variable Y。

Now that we have actual beta instructions， we have expressed the computation as a sequence of three instructions。

To access the value of variable X， the load instruction adds the contents of R 3 D1 to the constant Hex1008。

 which sums to Hex 1008。 the address we need to access。

The store instruction specifies a similar address calculation to write into the location for the variable y。

The address calculation performed by load and store works well when the locations we need to access have addresses that fit into the 16 bit constant field。

What happens when we need to access locations that addresses higher than Hex 7 FFF？

Then we need to treat those addresses if we would any large constant and store those large addresses in May memory so they can be loaded into a register to be used by load and store。

Okay， but what if the number of large constants we need to store is greater than will fit in low memory。

 I。e。 the addresses that we can access directly。To solve this problem。

 the beta includes a load relative LDR instruction。

 which we'll see in the lecture on the beta implementation。
