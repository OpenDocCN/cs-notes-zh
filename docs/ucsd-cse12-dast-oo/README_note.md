# 数据结构与面向对象编程：第1章：基础数据结构概述

在本章中，我们将学习数据结构的基本概念及其在面向对象编程中的重要性。数据结构是组织和存储数据的方式，它决定了数据的访问和操作效率。理解数据结构是编写高效、可维护程序的关键。

## 什么是数据结构

数据结构是计算机存储、组织数据的方式。它定义了数据元素之间的逻辑关系，以及在这些数据上可以执行的操作。常见的数据结构包括数组、链表、栈、队列、树和图等。

## 为什么需要数据结构

选择合适的数据结构可以显著提高程序的效率。例如，在需要频繁查找元素的场景中，使用哈希表比使用数组更快。数据结构的选择直接影响算法的时间复杂度和空间复杂度。

## 基本数据结构类型

以下是几种最基本的数据结构类型及其核心特征：

*   **数组**：一种线性数据结构，在连续的内存位置存储相同类型的元素。可以通过索引直接访问任何元素，时间复杂度为 **O(1)**。
*   **链表**：由一系列节点组成，每个节点包含数据和指向下一个节点的指针。它允许在非连续的内存空间中存储元素。
*   **栈**：一种后进先出（LIFO）的数据结构。只允许在一端（栈顶）进行插入（压栈）和删除（弹栈）操作。
*   **队列**：一种先进先出（FIFO）的数据结构。允许在一端（队尾）插入元素，在另一端（队首）删除元素。

## 面向对象编程与数据结构

在面向对象编程中，数据结构通常被封装成类。类不仅定义了数据的结构（成员变量），还定义了操作这些数据的方法。这种封装提高了代码的可重用性和安全性。

例如，我们可以用一个 `Stack` 类来实现栈数据结构：

```java
public class Stack {
    private List<Integer> elements; // 内部数据结构

    public void push(int item) { // 压栈方法
        elements.add(item);
    }

    public int pop() { // 弹栈方法
        if (isEmpty()) {
            throw new EmptyStackException();
        }
        return elements.remove(elements.size() - 1);
    }
}
```
