# 计算机系统管理：CS615：Week 3 - 热身练习 2：在 Debian Linux 上调整文件系统大小 🔧💾

在本节课中，我们将学习如何在 Debian Linux 系统上调整文件系统的大小。我们将通过一个完整的操作流程，演示如何扩展和收缩一个文件系统，并验证数据在调整过程中是否得到保留。

上一节我们介绍了在 NetBSD 上使用 `resize_ffs` 工具调整文件系统。本节中，我们来看看在 Linux 系统上完成相同任务的过程。尽管工具不同，但只要工具支持，其核心概念和基本步骤在不同操作系统间是相通的。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_1.png)

## 概述与准备 📋

我们将启动一个 Debian Linux 实例，并执行以下步骤：
1.  创建一个新卷并附加到实例。
2.  创建两个 512 MB 的分区作为起点。
3.  在分区上创建文件系统，挂载并添加文件以证明操作的非破坏性。
4.  将第一个分区扩展到 1 GB，并验证文件可访问性。
5.  将该分区收缩到 256 MB，并观察结果。

以下是具体操作步骤。

## 步骤一：启动实例并附加磁盘 🚀

![](img/6e1e1052ab0b6c945f559886f6d9fb00_3.png)

我们使用 AWS 命令行工具启动一个 Debian 实例，创建一个新卷，并将其附加到该实例。

实例启动并运行后，我们通过 SSH 以管理员用户身份登录，准备操作新增的磁盘。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_5.png)

## 步骤二：分区与创建文件系统 🗂️

![](img/6e1e1052ab0b6c945f559886f6d9fb00_7.png)

首先，我们使用 `fdisk` 工具对新磁盘进行分区。磁盘设备通常为 `/dev/nvme1n1`。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_9.png)

![](img/6e1e1052ab0b6c945f559886f6d9fb00_11.png)

以下是使用 `fdisk` 创建分区的关键命令序列：

![](img/6e1e1052ab0b6c945f559886f6d9fb00_13.png)

```bash
# 启动 fdisk 对指定磁盘进行操作
sudo fdisk /dev/nvme1n1

![](img/6e1e1052ab0b6c945f559886f6d9fb00_15.png)

# 在 fdisk 交互界面中：
# 输入 ‘n’ 创建新分区
# 选择 ‘p’ 创建主分区
# 分区号使用默认值
# 起始扇区使用默认值
# 设置大小为 +512M
# 再次输入 ‘n’ 创建第二个分区
# 使用所有剩余空间
# 输入 ‘p’ 打印分区表
# 输入 ‘w’ 将更改写入磁盘
```

![](img/6e1e1052ab0b6c945f559886f6d9fb00_17.png)

![](img/6e1e1052ab0b6c945f559886f6d9fb00_19.png)

操作完成后，使用 `lsblk` 命令确认分区创建成功。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_21.png)

接下来，在每个分区上创建 ext2 文件系统：

```bash
sudo mkfs.ext2 /dev/nvme1n1p1
sudo mkfs.ext2 /dev/nvme1n1p2
```

## 步骤三：挂载分区并创建测试文件 📁

我们至少挂载第一个分区，并在其上创建一个文件，以证明后续调整操作不会破坏数据。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_23.png)

```bash
# 创建挂载点并挂载分区
sudo mkdir /mnt/test
sudo mount /dev/nvme1n1p1 /mnt/test

![](img/6e1e1052ab0b6c945f559886f6d9fb00_25.png)

# 创建测试文件
sudo touch /mnt/test/our_test_file.txt
```

至此，我们已准备好调整分区大小。

## 步骤四：扩展文件系统 ⬆️

![](img/6e1e1052ab0b6c945f559886f6d9fb00_27.png)

要调整文件系统大小，首先必须卸载它。与在 NetBSD 上类似，我们需要先调整分区大小，然后才能扩展文件系统。

以下是扩展文件系统的步骤：

1.  卸载文件系统：`sudo umount /mnt/test`
2.  使用 `fdisk` 删除现有分区，并创建一个占用整个磁盘空间的新分区。`fdisk` 会警告磁盘上已有文件系统签名，选择继续。
3.  将新分区表写入磁盘。
4.  运行 `e2fsck` 检查并修复文件系统。该工具会检测到分区描述与文件系统不匹配（超级块信息不正确），并自动修复。
5.  使用 `resize2fs` 将文件系统扩展到分区的全部容量：`sudo resize2fs /dev/nvme1n1p1`

操作完成后，重新挂载分区并检查：

![](img/6e1e1052ab0b6c945f559886f6d9fb00_29.png)

```bash
sudo mount /dev/nvme1n1p1 /mnt/test
lsblk /dev/nvme1n1p1
ls /mnt/test/
```

![](img/6e1e1052ab0b6c945f559886f6d9fb00_31.png)

此时应显示文件系统大小约为 1 GB，且之前创建的文件仍然存在。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_33.png)

## 步骤五：收缩文件系统 ⬇️

![](img/6e1e1052ab0b6c945f559886f6d9fb00_35.png)

现在，我们来收缩文件系统。同样，任何文件系统操作都需要先卸载。

以下是收缩文件系统的步骤：

![](img/6e1e1052ab0b6c945f559886f6d9fb00_37.png)

![](img/6e1e1052ab0b6c945f559886f6d9fb00_39.png)

1.  卸载文件系统：`sudo umount /mnt/test`
2.  尝试使用 `resize2fs` 收缩文件系统到 256 MB：`sudo resize2fs /dev/nvme1n1p1 256M`。工具会提示需要先运行 `e2fsck`。
3.  运行文件系统检查：`sudo e2fsck -f /dev/nvme1n1p1`
4.  再次执行收缩命令，此时应成功。
5.  然而，`lsblk` 可能仍显示分区为 1 GB，因为它读取的是分区表信息。我们需要用 `fdisk` 更新分区表以匹配新的文件系统大小。
6.  使用 `fdisk` 删除分区，然后创建一个大小为 256 MB 的新分区。接着，可以创建第二个分区使用剩余空间。
7.  再次运行 `e2fsck`。
8.  挂载缩小后的分区并验证：`sudo mount /dev/nvme1n1p1 /mnt/test`。使用 `df -h` 查看大小，并确认测试文件仍在。

## 总结 📝

本节课中我们一起学习了在 Debian Linux 上调整 ext2 文件系统大小的完整流程。

核心操作可归纳为两个公式：

*   **扩展文件系统**：
    1.  使用 `fdisk` 调整分区大小。
    2.  使用 `e2fsck` 检查文件系统。
    3.  使用 `resize2fs` 扩展文件系统至分区末尾。
    > 注意：此操作会破坏该磁盘上其他分区（如第二个分区）的任何数据。

*   **收缩文件系统**：
    1.  使用 `e2fsck` 检查文件系统。
    2.  使用 `resize2fs` 收缩文件系统。
    3.  使用 `fdisk` 更新分区表以匹配新大小。
    4.  若想使用新释放的空间，需在其上创建新的文件系统。

![](img/6e1e1052ab0b6c945f559886f6d9fb00_41.png)

可以看到，在掌握了基本概念后，在不同类 Unix 系统上执行此类任务并没有本质区别。请务必亲自运行这些示例，或者尝试用不同的方式来完成它。
