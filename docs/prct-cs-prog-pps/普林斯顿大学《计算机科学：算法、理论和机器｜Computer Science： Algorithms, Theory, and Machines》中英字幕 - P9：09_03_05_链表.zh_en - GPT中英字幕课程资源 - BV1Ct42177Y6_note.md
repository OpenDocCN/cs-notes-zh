# 计算机科学：算法、理论和机器：09：链表

## 概述
在本节课中，我们将要学习一种名为**链表**的新数据结构。我们将用它来实现栈和队列，并理解它与数组在内存分配上的根本区别。链表是一种基础且应用广泛的数据结构，掌握它将为我们打开组织数据的新世界。

## 内存分配：顺序与链接
上一节我们介绍了栈和队列的抽象概念，本节中我们来看看如何用具体的数据结构来实现它们。首先，我们需要理解两种基本的内存分配方式。

*   **顺序数据结构（数组）**：在这种结构中，所有对象在计算机内存中**紧挨着**存放。它们一个接一个地连续存储在内存中。这就像一个Java数组，其大小固定，但可以快速访问其中的任何元素。
*   **链接数据结构（链表）**：在这种结构中，每个对象都**关联着一个指向另一个对象的链接**（即内存地址）。对象本身并不在内存中相邻存放。

## 链表的概念
理解了链接分配的概念后，我们来深入探讨链表的具体形式。

链表可以被视为一种**递归数据结构**：一个链表要么是 `null`（空），要么是一个指向**节点**的引用，而节点本身又包含一个指向另一个链表的引用。展开这种递归，我们可以说链表就是一系列节点的序列。

在Java中，我们使用**引用**来表示内存地址。为了表示链表，我们首先定义一个私有的 `Node` 类来实现节点抽象。

以下是 `Node` 类的简单定义：
```java
private class Node {
    String item;
    Node next;
}
```
这个数据类型包含一个字符串 `item` 和一个指向另一个 `Node` 的引用 `next`。我们将在代码中将其作为嵌套类使用，以便直接访问节点内的变量。

下图展示了一个由节点构建的链表抽象表示：第一个变量 `first` 是一个指向节点的引用，该节点的 `item` 是“Alice”，`next` 指向包含“Bob”的节点，依此类推，直到某个节点的 `next` 为 `null`，表示链表结束。

虽然我们目前只讨论每个节点有一个链接的简单链表，但链接的概念非常强大。节点可以指向多个其他节点（形成树），或者链接可以形成环，从而创造出各种复杂的数学结构。我们将在后续课程中探讨具有多个链接的结构。

## 构建链表
现在，让我们看看用Java代码创建链表的具体过程。我们将结合代码和内存的抽象表示来理解。

1.  首先，创建一个节点并赋值：
    ```java
    Node third = new Node(); // 系统在内存中分配空间并返回引用
    third.item = "Carol";
    third.next = null;
    ```
    执行后，我们得到一个包含“Carol”且 `next` 为 `null` 的节点。

2.  接着，创建第二个节点并将其链接到第一个节点之前：
    ```java
    Node second = new Node();
    second.item = "Bob";
    second.next = third; // 将Bob节点链接到Carol节点之前
    ```
    现在，`second` 指向“Bob”，而“Bob”的 `next` 指向“Carol”。

3.  最后，创建第一个节点并将其链接到链表开头：
    ```java
    Node first = new Node();
    first.item = "Alice";
    first.next = second; // 将Alice节点链接到Bob节点之前
    ```
    这样，我们就构建了一个包含 Alice -> Bob -> Carol 的链表。

通过操作这些引用，我们可以构建相当复杂的数据结构。

## 链表的基本操作
在高级语言如Java出现之前，人们用更低级的代码构建链表，并发展出了一套处理单向链表的标准操作。以下是几个核心操作：

以下是几个核心的链表操作示例。

*   **删除并返回链表的首项**：
    ```java
    String item = first.item; // 1. 保存首项的值
    first = first.next;       // 2. 将first指向原第二个节点
    return item;              // 3. 返回保存的值
    ```
    这段代码将 `first` 引用指向原链表的第二个节点，从而“移除”了第一个节点。在Java中，没有引用指向的原节点会被垃圾回收系统自动回收。

*   **在链表开头添加一项**：
    ```java
    Node second = first;          // 1. 保存原链表的引用
    first = new Node();           // 2. 创建新节点并让first指向它
    first.item = item;            // 3. 为新节点赋值
    first.next = second;          // 4. 将新节点链接到原链表
    ```
    关键在于，在重置 `first` 之前，必须保存对原链表的引用（`second`），否则将无法访问链表中的其他元素。

*   **遍历链表**：
    ```java
    for (Node x = first; x != null; x = x.next) {
        System.out.println(x.item);
    }
    ```
    这个简单的循环从 `first` 开始，沿着 `next` 引用逐个访问每个节点，直到遇到 `null`，从而按顺序打印出所有项。

## 链表应用小测
为了巩固理解，这里有两个链表处理代码的例子，它们体现了链表在实现更高级数据结构中的作用。

*   **示例一：反转输入顺序**
    有一段代码从标准输入读取字符串，并通过始终在链表头部插入新节点的方式构建链表，最后遍历输出。结果是，它**以相反的顺序**打印出输入的字符串。这本质上是一个栈的行为。
*   **示例二：保持输入顺序**
    另一段代码同样从标准输入构建链表，但它通过维护一个指向链表末尾节点的引用，始终在链表尾部添加新节点。结果是，它**以相同的顺序**打印出输入的字符串。这本质上是一个队列的行为。

## 总结
本节课中我们一起学习了**链表**这一基础数据结构。我们理解了它与数组在**顺序分配**和**链接分配**上的根本区别，学习了如何用 `Node` 类构建链表，并掌握了在链表头部**添加**、**删除**节点以及**遍历**链表的基本操作。链表通过引用将离散的节点连接起来，提供了灵活且大小可变的序列存储方式，是实现栈、队列等抽象数据类型的核心工具之一，也为理解更复杂的树、图等数据结构奠定了基础。
