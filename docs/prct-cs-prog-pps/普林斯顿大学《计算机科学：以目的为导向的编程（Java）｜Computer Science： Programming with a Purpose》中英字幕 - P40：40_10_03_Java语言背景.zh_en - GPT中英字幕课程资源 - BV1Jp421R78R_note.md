## 🧐 为什么选择Java？

现在让我们看看Java在我们讨论过的其他语言背景下的情况。

这张幻灯片来自本课程早期的一次讲座，它解释了为什么我们使用Java。Java被广泛使用且易于获取，它已经发展了几十年。它拥有所有现代抽象概念，并对程序中的错误进行了大量自动检查，这对所有程序员，特别是初学者程序员非常有用。

Java生态非常庞大，如今被用于各种设备和应用程序，从超级计算机到医疗设备，从手机到火星探测器，拥有数百万开发者和数十亿设备。

那么，我们为什么在本课程中使用它呢？通过与之前描述的其他语言比较，我们有一个清单。这些语言都被广泛使用和获取，尽管MATLAB需要付费。

但它们并非都拥有完整的现代抽象概念集。C语言没有对象。MATLAB有类似对象的概念。Python有现代抽象概念，但你可能不会说它完全拥抱了这些概念，这里不过多展开。

现代库和系统方面，C++或许可以算一个，但随着互联网经济和移动计算等的发展，Java才是真正拥有现代库和系统的语言。MATLAB和Python或许可以通过插件获得一些检查功能。

自动错误检查方面，这些语言都在一定程度上做到了。但我们将编译时类型检查视为必需。C和C++因为内存泄漏问题，在这方面打了折扣。

总之，很多观点是可辩论的，但我们在这门课程中使用Java，是因为从我们的角度来看，它在所有方面都符合要求。

## 🔄 为何学习其他编程语言？

这很好，但Java是完美的吗？我们为什么应该学习另一种编程语言呢？学习新的编程语言有很多理由。

可能是因为它提供了全新的东西。你可能找到一份工作，或加入一个研究小组，那里的其他人都在用那种语言编写程序，你需要与他们对接。因此，你很可能需要为此学习一门新的编程语言，例如在刚入职或换工作时。

可能是因为对于手头的应用，新的编程语言比Java更合适。可能某种语言特别适合或专为你正在从事的应用而构建。

通常，学习新编程语言的一个好理由就是智力挑战。在本课程的第二部分，我们有一个虚构的机器语言。用那种语言编程很有趣，许多编程语言都是如此。

有时，通过学习另一种编程语言或另一种观点，有机会真正了解计算的某些方面。

或者，可能只是有一种你不了解的新的编程风格，会通过这种语言介绍给你。

因此，理由很充分，随着时间的推移，大多数人都可以预期会学习新的编程语言。

以下是我能想到的一些例子。在20世纪60年代，我们用汇编语言编写程序，必须从256条指令中挑选，我们写的每一行代码都对应一条机器指令。因此，C语言在当时绝对是新事物，它是一种高级语言，我们可以编写`if`和`while`这样的语句，可以使用库中的函数，重用代码等等。这绝对值得学习新东西。

到了20世纪90年代，数据抽象的概念变得非常重要，它使我们能够编写真正的大型软件程序，可以共享库并使用广泛的库。

而在本世纪，新的编程语言为Web开发提供了各种功能，它们使用脚本并直接解释行为，适用于Web类型的开发项目。这些只是几个例子。

## 🧠 编程范式简介

我想稍微谈谈编程风格，因为在本讲座结束时我们会讨论一种完全不同的风格。我们已经讨论过最自然的过程式编程，即一条接一条地执行指令。通常我们将程序编译成机器语言，而机器语言本质上是过程式的。这就是C语言的特点，也是我们在引入对象概念之前所教授的Java的特点。

因此，像Ruby这样的现代Web开发语言中的Python通常是解释执行的。它类似于命令行执行，就像我们在命令行输入“编译这个程序”、“移动这个文件”等。我们可以在像Python这样的脚本语言中编写过程式程序，但通常是解释执行的。这只是一种不同的编程风格，取决于你期望如何使用它。

现在还有围绕特定数据类型优化的专用语言，例如Postscript编程语言围绕图形优化，我们将在课程第二部分讨论它；或者MATLAB围绕矩阵优化。如果你写的代码大部分只关心几种数据类型，这将导致完全不同的风格。

然后是我们将在下一节更详细讨论的面向对象编程语言。它确实是一种不同的风格，许多程序员花了很长时间才适应它，现在仍然有很多程序员没有花时间真正学习如何有效利用面向对象编程。因此，通过本课程，你已经领先于大量从事编程工作的人。

最后是函数式语言，我们将在最后一节讨论，它将计算视为函数的求值。这有很多有趣的影响，是一种完全不同的编程风格，如今许多人正在接受它。这有点类似于Java的自动内存管理战胜了C和C++的显式内存管理。人们曾担心效率问题，但最终便利性无疑胜出了。许多人对函数式编程语言也有同样的感觉，在许多方面，用这些语言表达大量应用的计算更容易。因此，这绝对值得一看，我们将在本讲座结束时进行探讨。
