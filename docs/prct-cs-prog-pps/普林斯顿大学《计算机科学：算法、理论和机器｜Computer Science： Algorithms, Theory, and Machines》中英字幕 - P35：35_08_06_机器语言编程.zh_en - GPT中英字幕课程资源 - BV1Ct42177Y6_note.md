## 超越简单加法

上一节我们介绍了TOY计算机的基本指令。本节中我们来看看，仅凭这些简单的指令和开关、灯泡，我们能做的事情远不止加法。

核心概念是，TOY指令集支持的编程结构与Java等高级语言相同。以下是TOY支持的基本编程结构：
*   **基本数据类型与运算**：支持算术运算和位运算。
*   **赋值语句**：通过加载（`load`）和存储（`store`）指令实现。
*   **条件与循环**：通过改变程序计数器（PC）的指令实现。
*   **数组**：通过操作码A和B实现（将在下节课讨论）。
*   **标准输入/输出**：通过特殊的内存地址实现。
*   **高级结构**：同样支持函数和库（操作码E和F），以及链表等数据结构。

书中甚至有一个TOY程序，它使用二叉搜索树对不限长度的输入进行去重。这非常惊人，因为仅用少量指令就能实现这些功能。实际上，许多TOY程序只需10到30条指令就能完成大量工作。研究这些执行常见任务的机器语言程序非常有价值。

## 条件与循环的实现

现在，让我们具体看看条件和循环是如何实现的。

TOY中有几条指令可以改变程序计数器（PC）的值，从而控制指令的执行流程。这些指令测试寄存器的值，并根据结果改变PC。具体是操作码C（为零则跳转）和操作码D（为正则跳转）。

这些指令接收一个寄存器和一个地址作为参数。它们测试该寄存器的值，并根据结果决定是否将PC设置为该地址。

以下是一个计算寄存器A中数字绝对值的简单程序：
```
DA12
```
*   `D`是“为正则跳转”操作码。
*   检查寄存器A是否为正数。如果是，则将PC设置为内存地址12。
*   该指令位于地址10，下一条指令在地址11。跳转到12意味着跳过下一条指令。
*   下一条指令是操作码2（减法），操作数为`0`和`A`。这意味着从0中减去寄存器A的值。如果A原本是负数，这个操作会使其变为正数，并将结果存回寄存器A。
*   这就是一个用两条指令计算绝对值的程序，实现了一个条件判断。

一个典型的循环结构如下：假设寄存器1中存储了值1。操作码C是“为零则跳转”。我们测试寄存器A的值，如果为零，则跳转到地址15（跳出循环）。循环体内是任意指令，之后会有一条减法指令（操作码2），从寄存器A中减去1并将结果存回A（即递减操作）。递减后，对寄存器0（其值恒为0）执行“为零则跳转”指令，该跳转总是成功，会将PC设置回地址10。这样，我们就实现了一个`while`循环，不断递减寄存器A直到其值为零。

仅凭这几条指令，我们就能实现高级语言（如Java）中常用的控制结构。稍后我们将给出一个这样做的程序示例。

正如我们在第二讲首次编程时所见，一旦引入循环，我们就能实现远超没有流程控制时的能力。而实现它并不需要太多，只需几条TOY指令。

## 输入与输出设备

用开关和灯泡实现程序是一回事，但用它们输入数据显然不现实。因此，所有早期计算机最先解决的问题之一就是如何将外部数据输入机器。

以纸带为例，这实际上是我早年使用过的方法。机器侧面会安装一些可以读写纸带的输入/输出设备。我们可以将其理解为你们熟悉的标准输入和标准输出。

纸带是成卷或折叠的，理论上长度无限。数据通过纸带上的孔洞编码，每个16位字占用纸带上的两行。有孔代表1，无孔代表0。写入时，设备根据字的值在对应位置打孔；读取时，设备用光线照射纸带，另一侧感应孔洞来识别0和1。

那么，TOY程序如何访问这个设备呢？我们只需将这个设备连接到内存地址`FF`。当你向地址`FF`存储数据时，计算机会激活纸带打孔机，打出对应数据的孔洞。当你从地址`FF`加载数据时，计算机会向设备发出信号，读取一个字的数据，并像从内存读取一样提供该数据。

这是一个非常简单的设备。实际上，纸带硬件本身并不重要，因为它很快就被能存储更多数据、速度更快的磁带取代了。但从程序的角度看，没有任何变化。程序将输入/输出流视为无限的，只需向`FF`写入或从`FF`读取即可。这极大地扩展了TOY程序的能力。

## 示例：斐波那契数列程序

作为一个结合了流程控制和标准输出的真实程序示例，让我们来看一个打印斐波那契数列的程序。这是一个经典的示例，尽管简单，但在早期计算机时代，能够计算未知的大斐波那契数或其他整数数学性质，曾让人们非常兴奋。

这个程序仅用13条指令就完成了任务。以下是它的工作原理：

1.  **初始化**：使用`load address`指令（操作码7）将常数1加载到寄存器1中。该指令将地址值放入寄存器，并将高位清零，这是快速将值加载到寄存器的方法。
2.  **设置初始值**：寄存器A和B初始值设为1，代表斐波那契数列的前两个数。
3.  **设置循环计数器**：寄存器9从内存地址`4C`加载值。查看地址`4C`，其值为`A`（即十进制的10）。这是我们想要打印的斐波那契数的个数（`n`）。寄存器9将作为循环计数器（对应伪代码中的`i`）。
4.  **循环开始**：测试寄存器9（`i`）是否为0。操作码C（为零则跳转）检查寄存器9，如果为0则跳转到地址`4B`（停机指令）。这构成了我们的`while`循环。
5.  **循环体**：
    *   `9AFF`：将寄存器A的值写入标准输出（地址`FF`），即打印当前的斐波那契数。
    *   计算下一个斐波那契数：将A和B相加，结果存入C（`C = A + B`）。
    *   更新前两个数：将B的值移给A，将C的值移给B（`A = B`, `B = C`）。
    *   递减计数器：寄存器9减1。
6.  **循环继续**：每次循环都重复步骤5，计算新的和并更新数值。
7.  **循环结束与停机**：当计数器减到0时，跳出循环，执行停机指令。

通过这个示例程序和被称为“TOY参考卡”的幻灯片，我相信你已经可以想象如何编写代码来解决许多我们在Java早期课程中遇到的计算问题了。只要涉及流程控制和整数位运算，TOY能做的事情非常多。

关于这张参考卡，需要记住的是，它完整地规定了计算机的行为。任何16位值都可以用这张卡解码，右侧蓝色的伪代码精确说明了TOY执行该指令时会做什么。这是对TOY内部运作的完整描述。多年来，真正的程序员口袋里都装着这样的卡片，作为编程时需要了解的全部信息。

## TOY编程与理解

TOY编程是一项有趣的智力挑战。为了确保你真正理解了所学内容，你需要能够回答以下类型的问题：

1.  **指令解释**：给定一个16位数字（如`1A75`），如果将其解释为TOY指令，它是什么意思？如果解释为二进制补码整数值，又是多少？
    *   `1A75`作为指令：`1`是加法操作码，`A`是结果寄存器，`7`和`5`是操作数寄存器。即：`R[A] = R[7] + R[5]`。
    *   `1A75`作为整数：它是正数，可按十六进制计算：`1*16^3 + 10*16^2 + 7*16 + 5`。
2.  **特殊值**：`0FFF`作为指令是停机（操作码`0`），其余位被忽略。作为整数，它是4095（计算方式：`0x1000 - 1`）。
3.  **位操作技巧**：如何翻转TOY寄存器中的所有位？通过与全1进行异或操作（`XOR`）即可实现，`XOR` 1会翻转位，这是该级别程序员都知道的技巧。
4.  **小程序分析**：你应该能看懂小程序的功能。例如，分析一个循环10次、每次将寄存器2的值翻倍的程序，你会发现它计算的是2^10 = 1024。

## TOY与现实计算机的关系

TOY与你实际的计算机是完全不同的机器，但它们都实现了基本数据类型、条件、循环和其他低级结构。它们都能支持函数、数组等高级构造，都拥有无限的输入/输出流。我们为Java讨论的整个编程模型，TOY和你的笔记本电脑实现得一样好。

你可能会问，256个字的内存真的够做任何有用的事吗？我们保持其小巧是为了便于理解计算机内部原理。实际上，书中所有覆盖广泛范围的示例程序都能装入同一内存。如果你看过图灵理论讲座，就会知道，如果将TOY的I/O设备改为既可读又可写（这无需改变机器本身），那么TOY就变成了一台图灵机，意味着它可以计算任何计算机能计算的东西。

因此，TOY在很强的意义上等同于你的笔记本电脑。过去50年开发的所有计算机都具有这些相同的基本特征，这是一个非常惊人的现象。当然，我们想要更快的速度和更大的内存，这也是计算历史的一部分。但计算机的基本功能并没有太大变化。
