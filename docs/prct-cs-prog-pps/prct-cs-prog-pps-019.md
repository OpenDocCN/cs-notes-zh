在本节课中，我们将学习如何利用自定义函数的能力来处理高斯分布。高斯分布，即著名的钟形曲线，是一个在众多领域成功拟合实验观测数据的数学模型。我们将学习其概率密度函数和累积分布函数的数学定义，并动手编写一个Java库来实现这些函数，从而掌握模块化编程的核心思想。

---

上一节我们介绍了自定义函数的重要性，本节中我们来看看如何应用这一思想来处理高斯分布。

高斯分布是一个数学模型，即所谓的钟形曲线。几个世纪以来，它被成功应用于多种情境下的实验观测数据拟合。

以下是高斯概率密度函数的定义：
**公式：** `φ(x) = (1 / √(2π)) * e^(-x²/2)`

这是该函数的图像，它以零为中心（即均值），标准差衡量其扩散程度，此处为1。

例如，某年SAT考试的分数分布与实验数据非常匹配。如果我们调整参数，取均值1019和实验数据的标准差209，然后用这些参数绘制高斯函数，就能很好地拟合数据。

双参数高斯函数的公式如下：
**公式：** `φ(x; μ, σ) = (1 / (σ√(2π))) * e^(-(x-μ)²/(2σ²))`

该函数通过缩放来匹配实验数据。你只需要知道样本的均值和标准差。由于高斯分布在处理数据的程序中经常出现，我们需要能够使用它。其背后的数学原理很完善，但目前我们只需要知道公式。

高斯分布无处不在。以下是一些来自网络论文的例子：
*   研究甘油中的聚苯乙烯颗粒
*   激光束传播
*   光镊
*   视觉皮层中的氧化酶斑块
*   预测美国石油产量（尽管这个听起来不太符合高斯分布）
*   德国货币上印有高斯曲线（高斯是德国人）

既然高斯分布如此重要，我们自然希望在Java中使用它。那么，Java数学库中是否有现成的 `Math.phi` 函数呢？答案是没有。

原因可能是你可以自己实现它。你可以创建一个名为 `Gaussian` 的类，并定义一个函数。我们将其称为 `pdf`（概率密度函数）。我们需要做的就是实现那个数学公式。

**代码：**
```java
public static double pdf(double x) {
    return Math.exp(-x*x / 2) / Math.sqrt(2 * Math.PI);
}
```
这很简单。我们再次调用了其他模块（函数）。如果你想实现三参数版本（包含均值和标准差），我们可以适当地使用那个单参数版本。

在Java中，如果两个函数签名具有不同数量的参数，它们可以拥有相同的名称。这在当前情况下很有用。三参数版本所做的就是调用单参数版本，并应用参数 `(x - μ) / σ`，然后将结果除以 `σ`。

**代码：**
```java
public static double pdf(double x, double mu, double sigma) {
    return pdf((x - mu) / sigma) / sigma;
}
```
我们稍后会看更多函数。但关键点是，如果你需要任何数学函数，你可以自己定义实现这些函数的库。这是模块化编程的一个例子：你不想在每个处理高斯的程序中都放入这段数学代码，你只希望程序表现得好像这个函数已经实现了一样。因此，我们将其放入一个模块 `Gaussian.java` 中。现在，你可以通过 `Gaussian.pdf` 等方式调用这些函数。

从某种意义上说，通过库，任何用户都可以扩展Java系统，以包含适合当前应用程序的函数。这是一个非常强大的思想。

---

现在，还有另一个高斯函数，称为累积分布函数。如果你在x轴上有一个特定点，你可能会问：小于或等于该点的部分占总体的百分比是多少？这等价于曲线在该点左侧的面积。

这被称为高斯累积分布函数，在应用中非常有用。其数学表达式如下：
**公式：** `Φ(z) = (1 / √(2π)) * ∫（从-∞到z） e^(-x²/2) dx`

它也有一个类似但更简单的三参数版本，你只需要在 `(x - μ) / σ` 处求值即可。

累积分布函数有典型的应用场景。例如，在某一年，你需要考到820分才能成为一级运动员。你可能会问，有多少比例的考生没有资格？答案由该数据（给定均值和标准差）的高斯CDF给出，在这个例子中大约是17%。

因此，你可能想编写使用高斯累积分布函数的代码来计算此类信息。

那么，我们如何实现它呢？该函数没有闭式解，它是以积分形式表达的。

答案是，存在一个非常精确的泰勒级数。实际上，这是大多数数学函数在计算机上实现的方式。数学推导表明，`Φ(z)` 等于 `1/2 + PDF(z) * S`，其中 `S` 是一个级数，其项会迅速变小。

因此，我们可以编写代码来实现这个数学公式。如果 `z` 小于 -8，其值小到可以忽略不计；如果大于 8，其值非常接近1，差值也可以忽略。我们计算这个级数和，只要新加的项不再改变结果，我们就可以停止。然后返回 `0.5 + 计算的和 * PDF(z)`。事实证明，这个结果精确到15位小数。

同样，如果你想要三参数版本，只需调用另一个函数并传入 `(x - μ) / σ`。

只需相当少量的代码，我们就可以实现它。从使用这个库的程序角度来看，它不知道我们是如何实现的，它只想要计算结果。因此，我们能够保护客户端免于处理所有这些细节。这是一种很好的组织计算的方式，将这些知识保存在一个地方。

归根结底，你有上千年的数学公式可以使用，它们不会全部为你实现。但当你需要使用它们时，你可以自己封装它们，以便能够以连贯的方式编写利用这些公式的程序。

---

以下是我们的总结：这是一个用于高斯分布函数的库。我们实现了PDF（两种形式）和CDF（两种形式）。

最佳实践是在每个库中包含一个 `main` 函数，至少测试所有代码，不妨让它做一些有用的事情。在这个例子中，它接受你感兴趣的均值、标准差和截止点作为参数，然后打印出该点的PDF值和CDF值。任何人问你比例，你只需要知道这三个数字，就可以得到百分比。

我们实际上使用这段代码来评估提交程序中的随机性（当它们本应具有随机性时）。但无论如何，真正的要点是，对于你将来拥有的任何程序，如果你有许多类似性质的函数，你可以将它们构建到一个模块中，然后使用它们。这就是关于库的真正重要概念。

---

那么，我们实际上如何使用这个库呢？你自己编写的库，如果它们与你的其他程序在同一工作目录中，那么你只需在那里放一份副本。你也可以通过操作系统设置，只使用一个副本（详情可查阅书籍网站）。如果库在那里，你就可以从该目录中的任何其他模块使用这些函数。我们已经在“播放音符”和其他例子中这样做了。

例如，我想使用 `StdDraw` 绘制它的图像。这只是一个普通的 `main` 函数。我将设置我的绘图，然后遍历并使用那个 `pdf` 函数。我会根据从命令行获取的参数对其进行采样，然后使用 `pdf` 计算X和Y点，从而绘制出图像。例如，采样200个点的高斯PDF图像。

使用你创建的库非常容易。这再次说明，这是任何用户扩展Java系统的一种简单方式。
