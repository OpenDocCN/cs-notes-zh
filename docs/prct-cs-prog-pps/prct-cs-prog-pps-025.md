# 025：动态规划 🧩

![](img/c416eaab9960389e461437176d9ca511_0.png)

在本节课中，我们将要学习一个非常重要的编程范式——动态规划。动态规划广泛应用于运筹学等多个领域。我们无法深入探讨其全部内容，但将通过一个示例来展示其与递归的关系。理解这种关系以及动态规划的基本思想，对任何程序员都至关重要。

我们将动态规划视为递归的一种替代方案，它以一种有组织的方式避免了重复计算。在递归中，计算是自顶向下的：我们从一个大问题开始，将其分解为更小的子问题。而动态规划则代表了自底向上的计算方式：先解决最小的子问题，保存其解，然后利用这些解来构建更大问题的解。这种方法由理查德·贝尔曼在20世纪50年代开创，在众多领域都非常有效。

## 从斐波那契数列看动态规划

在斐波那契数列计算的语境下，动态规划意味着使用一个数组来保存答案。我们保存小规模子问题的解，将数组的前两项设置为已知值，然后遍历数组，对于每个索引 `i`，利用之前计算好的子问题解来计算当前值。这非常简单：先解决小问题，然后用已解决的子问题来构建更大的解。

这种方法显然没有递归可能带来的指数级浪费问题，它仅需线性次数的加法即可计算出斐波那契数。

以下是使用动态规划计算斐波那契数的核心代码逻辑：

```java
int[] fib = new int[n+1];
fib[0] = 0;
fib[1] = 1;
for (int i = 2; i <= n; i++) {
    fib[i] = fib[i-1] + fib[i-2];
}
```

## 最长公共子序列问题

上一节我们介绍了动态规划的基本思想，本节中我们来看一个具体的应用实例：最长公共子序列问题。初看之下，你可能会认为这个问题极难计算。

首先，我们来定义子序列。给定一个字符串 `S`，其子序列是指通过删除 `S` 中的任意个字符（可以不连续），并将剩余字符按原顺序拼接得到的新字符串。例如，字符串 `"GCACACAG"` 的子序列包括 `"CA"`、`"GCA"` 等。一个长度为 `n` 的字符串，其子序列数量高达 `2^n` 个。

最长公共子序列问题是指：给定两个字符串，找出它们共有的、最长的子序列。例如，对于字符串 `S = "GCACACAG"` 和 `T = "GCCCTACCG"`，其最长公共子序列是 `"GCACCG"`，长度为 7。

尽管两个字符串都有指数级数量的子序列，但令人惊讶的是，这个问题可以高效解决。它在科学领域有重要应用，例如在DNA序列比对中衡量两个序列的相似性，或在计算编辑距离时。

## 动态规划解决方案

现在，我们来看看如何使用动态规划解决这个问题。为了简化讲解，我们只计算最长公共子序列的长度。

动态规划的思路是解决小规模子问题，然后组合这些解来解决更大问题。我们将使用一个二维数组 `opt[i][j]`，它表示字符串 `S` 从索引 `i` 开始的子串（后缀）与字符串 `T` 从索引 `j` 开始的子串的最长公共子序列长度。

关键在于，只要我们已计算出更小子问题的答案，那么当前问题的解只取决于三种情况：

1.  **基本情况**：如果任一子串为空（即 `i` 或 `j` 到达字符串末尾），则最长公共子序列长度为 `0`。
2.  **字符匹配**：如果 `S[i]` 等于 `T[j]`，那么这两个字符可以成为公共子序列的一部分。此时，`opt[i][j]` 等于 `opt[i+1][j+1] + 1`。
3.  **字符不匹配**：如果 `S[i]` 不等于 `T[j]`，那么这两个字符不可能同时出现在当前的公共子序列中。此时，`opt[i][j]` 等于 `max(opt[i+1][j], opt[i][j+1])`。

以下是计算最长公共子序列长度的核心逻辑：

```java
int m = S.length();
int n = T.length();
int[][] opt = new int[m+1][n+1]; // 多一行一列用于处理空子串的情况

// 从后向前填充数组
for (int i = m-1; i >= 0; i--) {
    for (int j = n-1; j >= 0; j--) {
        if (S.charAt(i) == T.charAt(j)) {
            opt[i][j] = opt[i+1][j+1] + 1; // 情况2
        } else {
            opt[i][j] = Math.max(opt[i+1][j], opt[i][j+1]); // 情况3
        }
    }
}
// 最终答案存储在 opt[0][0] 中
```

![](img/c416eaab9960389e461437176d9ca511_2.png)

我们通过从后向前（自底向上）的方式填充数组 `opt`，确保在计算 `opt[i][j]` 时，其依赖的 `opt[i+1][j+1]`、`opt[i+1][j]` 和 `opt[i][j+1]` 都已被计算出来。最终，`opt[0][0]` 就存储了整个字符串 `S` 和 `T` 的最长公共子序列长度。

## 递归与动态规划的对比

本节课中我们一起学习了动态规划，并与递归进行了对比。递归和动态规划都不是玩具，它们是解决问题的强大通用方法。

*   **递归**：采用自顶向下的分解。优势在于分解思路通常直观，且易于推理正确性。但可能存在指数级重复计算的浪费，且并非所有问题都容易分解。
*   **动态规划**：采用自底向上的构建。优势在于避免了重复计算，确保每个子问题只解决一次。但需要额外的空间（如数组）存储子问题解，对于连续值参数可能不适用，且有时确定计算顺序具有挑战性。

![](img/c416eaab9960389e461437176d9ca511_4.png)

两者都为我们提供了新的思维方式，是解决复杂问题的有力编程范式，并深刻揭示了计算的本质，数十年来在各种科学应用中取得了巨大成功。作为程序员，你需要同时掌握这两种方法，并将在更高级的计算课程中反复遇到它们。
