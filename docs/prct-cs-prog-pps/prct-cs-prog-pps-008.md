## 概述

在本节课中，我们将要学习编程中的一个重要概念：**嵌套结构**。我们将了解如何将条件语句和循环语句相互嵌套，以构建更复杂、功能更强大的程序。通过一个具体的“赌徒破产问题”模拟案例，我们将看到嵌套结构如何帮助我们解决实际的计算问题。

---

## 嵌套结构的概念

上一节我们介绍了基本的条件语句和循环语句。本节中我们来看看如何将它们组合起来。

嵌套的核心思想是：在条件语句或循环语句中，作为其执行体的语句序列里，任何一条语句本身都可以是另一个条件语句或循环语句。这允许我们构建非常复杂的控制流。虽然理解这类程序更具挑战性，但所有逻辑仍然基于我们已学过的条件判断和循环。

以下是一个嵌套结构的示例，其中`if`语句嵌套在`while`循环内，而`while`循环又嵌套在`for`循环内：

```java
for (int i = 0; i < N; i++) {
    while (condition) {
        if (anotherCondition) {
            // 执行某些操作
        }
    }
}
```

## 嵌套`if-else`语句示例

让我们通过一个简单的税务计算问题来理解嵌套`if-else`语句的应用。

给定一个收入值，需要根据以下税率表计算正确的税率：
*   收入低于 47，450：税率 22%
*   收入在 47，450 到 114，650 之间：税率 25%
*   收入在 114，650 到 174，700 之间：税率 28%
*   收入在 174，700 到 311，950 之间：税率 33%
*   收入高于 311，950：税率 35%

以下是使用嵌套`if-else`语句解决此问题的一种方法：

```java
if (income < 47450) {
    rate = 0.22;
} else {
    if (income < 114650) {
        rate = 0.25;
    } else {
        if (income < 174700) {
            rate = 0.28;
        } else {
            if (income < 311950) {
                rate = 0.33;
            } else {
                rate = 0.35;
            }
        }
    }
}
```

这段代码通过四层嵌套的`if`语句清晰地实现了税率查找逻辑。请注意，每个`else`子句都至关重要。如果省略它们，程序逻辑将完全错误，因为前三个`if`语句的判断会变得无效，最终只会执行最后一个`if`语句。

> **注意**：虽然可以省略大括号来编写链式`if-else`语句，但这有时可能导致代码歧义。为了清晰起见，建议始终使用大括号，或仔细阅读相关规范。

## 应用案例：赌徒破产问题模拟

为了展示嵌套结构在实际中的应用，我们将分析一个经典问题：**赌徒破产问题**。

### 问题描述

想象一个赌徒带着一定数额的赌本（`stake`）进入赌场。他每次进行公平的1美元投注（输赢概率各50%），并持续下注，直到出现以下两种结果之一：
1.  **输**：赌本耗尽（破产）。
2.  **赢**：达到预设的目标金额（`goal`）。

赌徒可能想知道：获胜的概率是多少？平均需要下注多少次？

### 蒙特卡洛模拟方法

我们将使用**蒙特卡洛模拟**来研究这个问题。这意味着我们将通过程序多次重复模拟赌博过程，并统计结果，从而估算获胜概率和平均下注次数。我们将利用之前学过的模拟抛硬币（`if`语句结合随机数）来实现单次下注。

以下是模拟赌徒破产问题的完整代码：

```java
public class GamblerRuin {
    public static void main(String[] args) {
        // 从命令行参数获取输入
        int stake = Integer.parseInt(args[0]); // 初始赌本
        int goal = Integer.parseInt(args[1]);  // 目标金额
        int trials = Integer.parseInt(args[2]);// 模拟实验次数

        int wins = 0; // 记录获胜次数

        // 外层循环：重复进行多次实验
        for (int t = 0; t < trials; t++) {
            int cash = stake; // 每次实验开始时，重置现金为初始赌本

            // 内层while循环：模拟单次实验，直到破产或达成目标
            while (cash > 0 && cash < goal) {
                // 模拟一次公平的抛硬币下注
                if (Math.random() < 0.5) {
                    cash++; // 获胜，现金增加
                } else {
                    cash--; // 失败，现金减少
                }
            }
            // 实验结束，检查是否获胜（现金等于目标）
            if (cash == goal) {
                wins++;
            }
        }

        // 输出结果
        System.out.println(wins + " wins out of " + trials);
        System.out.println("Percent of games won = " + 100.0 * wins / trials);
    }
}
```

### 代码解析

以下是代码各部分的简要说明：
1.  **参数输入**：从命令行获取初始赌本（`stake`）、目标金额（`goal`）和实验次数（`trials`）。
2.  **外层`for`循环**：控制整个模拟实验重复执行的次数。
3.  **内层`while`循环**：模拟单次赌博过程。只要现金大于0且小于目标，就持续进行抛硬币下注。
4.  **下注逻辑**：使用`Math.random()`生成一个`[0， 1)`的随机数。若小于0.5，视为获胜，现金加1；否则视为失败，现金减1。
5.  **胜负判定**：当`while`循环退出后，检查现金（`cash`）是否等于目标（`goal`）。若是，则记录一次获胜。
6.  **结果输出**：打印获胜次数和获胜百分比。

### 运行示例与数学验证

如果我们使用参数`stake=5`， `goal=25`， `trials=1000`运行程序，可能得到类似“191 wins out of 1000”的输出，即获胜概率约为19.1%。

有趣的是，对于这个经典问题，存在精确的数学解：
*   **获胜概率** = `stake / goal`
*   **预期下注次数** = `stake * (goal - stake)`

对于`stake=5`， `goal=25`的情况：
*   数学预测获胜概率 = 5 / 25 = **20%**
*   预期下注次数 = 5 * 20 = **100次**

我们的模拟结果（~19.1%）与数学分析（20%）非常接近，这验证了模拟程序的正确性。

### 模拟的强大之处

这个例子展示了计算机模拟的强大能力。对于简单的公平下注策略，我们可以进行数学分析。但是，如果赌徒采用更复杂的策略（例如，资金翻倍后增加赌注，或接近目标时改变策略），数学分析将变得极其困难甚至不可能。然而，我们只需修改模拟程序中的下注逻辑，就能轻松地对任何策略进行测试和评估。这就是为什么模拟成为科学研究、工程设计和金融分析等领域不可或缺的工具。

本节课中我们一起学习了**嵌套结构**。我们了解到，通过将条件语句（`if-else`）和循环语句（`for`， `while`）相互嵌套，可以构建出处理复杂逻辑的程序。我们通过税务计算的例子熟悉了嵌套`if-else`的写法，并重点剖析了“赌徒破产问题”的蒙特卡洛模拟。这个案例不仅综合运用了嵌套循环和条件判断，还向我们展示了如何用程序模拟随机过程，以及计算机模拟在解决复杂问题上的实用价值。尽管嵌套结构使代码更复杂，但通过分解和理解每个组成部分的目的，我们仍然能够清晰地掌握程序的运行逻辑。
