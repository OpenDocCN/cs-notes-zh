## 挑战的起源 🔍

上一节我们介绍了性能分析的重要性，本节中我们来看看这个挑战的历史渊源。自计算早期以来，性能挑战就一直存在。巴贝奇在建造差分机时就提出了核心问题：如何以最少的计算步骤（转动曲柄的次数）来获得答案。这本质上就是今天我们要讨论的问题：如何确保我们的程序能在合理的时间内得出结果。在编写用于解决科学和商业问题的程序时，这一点至关重要。

## 现代视角下的程序开发 🔄

在程序开发流程中，我们通常经历编辑、编译、运行和测试的循环。然而，即使程序功能正确，如果运行速度太慢、效率太低，它也可能无法解决实际问题。因此，我们今天要讨论如何理解程序的性能特征，以便改进它并真正解决实际问题。

## 性能分析的科学方法 🧪

性能分析背后的关键见解由克努特在20世纪70年代提出。他指出，使用科学方法来理解性能不仅是可行的，而且是富有成效的。这正是我们今天要讨论的内容。

以下是研究程序性能的几个主要原因：
*   **预测程序行为**：我们想知道程序是否会结束，是陷入循环还是仅仅需要很长时间，以及它何时会完成。
*   **比较算法与实现**：我们想知道如何改进现有程序，使其运行得更快。
*   **区分算法与程序**：算法是解决问题的通用方法，适合实现为程序。我们将研究许多算法，性能分析有助于我们理解问题本质，并设计新算法或优化参数。

通过高效的算法，我们可以解决原本无法解决的技术或研究问题。

## 性能问题的具体案例 ⚡

为了进一步说明，这里列举一些具体例子：

**1. N体模拟问题**
天体物理学家希望研究大量天体（n值极大）之间的相互作用。我们之前看到的算法所需时间与 `n²` 成正比，即**二次时间**。在20世纪70年代，这超出了计算机可用时间的极限。然而，巴恩斯-胡特算法以 `N log N` 的步骤解决了该问题，使科学家能够研究更大规模的宇宙问题。这个算法最初由普林斯顿大学的安德鲁·阿佩尔教授在其本科毕业论文中开始研究。

**2. 离散傅里叶变换**
这是信号处理中的一项基本计算，用于将波形分解为其周期分量。同样，解决此问题的直接算法也是二次的。早在20世纪50年代和60年代，这就达到了计算机可用时间的极限。快速傅里叶变换算法以 `N log N` 的步骤解决了问题，成为从MRI机器到音乐播放器、手机等众多现代技术的基础。

这些例子都说明了拥有快速算法的重要性。

## 一个技术要点：二进制对数 📊

在我们的分析中，经常使用二进制对数。二进制对数 `log₂ n` 是满足 `2^x = n` 的数 `x`，它增长非常缓慢。对于计算机科学家来说，这是一个基本概念：
*   `2^10 ≈ 1000`，所以 `log₂(2^10) = 10`
*   `2^20 ≈ 一百万`，`log₂(2^20) = 20`
*   `2^30 ≈ 十亿`，`log₂(2^30) = 30`

因此，当我们说 `log n` 时，对于十亿，想到的是30，这比十亿小得多。`⌊log₂ n⌋`（小于等于 `log₂ n` 的最大整数）就是数字 `n` 的二进制表示所需的位数。这个数经常出现在递归问题的研究中，包括FFT和巴恩斯-胡特算法，其中“对半分治”的思想就会引出对数。

## 案例分析：三数和问题 🎯

我们将使用“三数和”问题作为研究性能的例子。这是一个非常简单的问题：给定 `n` 个整数（可正可负），找出有多少个三元组之和为0。我们暂时只关注计数。

这个问题在计算几何（处理几何算法的计算机算法）中是一个基础问题。例如，判断多个点是否共线，或者一个多边形是否能放入另一个多边形中（在机器人路径规划等应用中很重要），都需要解决三数和问题。这说明了即使是简单的问题，在许多实际应用中也可能非常重要。

**问题设定**
我们想要一个方法 `count`，它以数组作为参数，并返回数组中元素之和为零的三元组数量。测试客户端将读取数组，然后调用 `count` 函数。

例如，对于这六个整数：`30, -30, -20, -10, 40, 0`，有三个三元组之和为0：
1.  `30, -30, 0`
2.  `30, -20, -10`
3.  `-30, -10, 40`

**暴力破解算法实现**
我们将研究一个简单的暴力破解算法，作为理解程序性能的例子。该算法会处理所有可能的三元组，并维护一个计数器，当和为0时递增计数器。

以下是算法的伪代码描述：
```java
int count = 0;
for (int i = 0; i < n; i++) {
    for (int j = i+1; j < n; j++) {
        for (int k = j+1; k < n; k++) {
            if (a[i] + a[j] + a[k] == 0) {
                count++;
            }
        }
    }
}
return count;
```
算法通过三层嵌套循环枚举所有满足 `i < j < k` 的三元组，以避免重复处理。然后检查 `a[i] + a[j] + a[k]` 是否等于0，如果是，则增加计数。

模拟运行显示，该算法会枚举所有三元组并找出和为零的那些。

## 性能分析的关键问题 ❓

我们现在的目标是尝试理解这个程序的性能。我们能否对 `n = 一百万` 运行这个程序？那将需要多少时间？

首先要注意的是，满足 `i < j < k` 的三元组总数是组合数 `C(n, 3)`。这肯定会成为分析的一部分。
