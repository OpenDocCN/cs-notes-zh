## 📚 概述
在本节课中，我们将要学习一个非常重要的主题：如何通过函数和库来组织程序。随着我们处理更复杂的应用，程序会变得越来越大。如何组织程序，以便于维护和添加新代码，就成为一个关键问题。函数和库的机制正是我们用来帮助维护和组织程序的工具。

上一节我们介绍了编程的基本构件，本节中我们来看看如何通过开发独立的模块来复用代码，从而用小块构建大程序。

## 🧩 模块化编程与库
模块化编程是一个长期存在的理念。其核心思想是将程序组织成独立的模块，这些模块协同工作以完成任务。这样做的好处是更容易共享和复用代码，并构建更大的程序。在现实中，人们通过模块化编程构建庞大的程序，团队可以协作开发。我们需要的是能让各个模块良好协作的工具。

对于初学者，我们从一个简单的定义开始。在本讲座中，我们将库定义为一组函数的集合。实际上，一个 `.java` 文件可以包含一组函数，但它的功能不止于此。目前，我们暂时将它们视为等同，以便专注于讨论模块化编程。稍后，我们将学习模块如何实现数据类型，这是一个更通用、能更好支持模块化编程的概念。

## 📝 Java函数（静态方法）
Java函数，也称为静态方法，类似于数学函数。它接收一些输入参数，返回一个输出值，并且可能产生一些副作用。它比数学函数更通用，因为它可以包含副作用。程序员使用函数来构建模块化程序，你既可以用它来计算公式，也可以用它来组织代码结构。

我们已经见过很多例子，例如 `Math.random()`、`Math.abs()`、`Math.sin()`、`Math.cos()`、`Math.tan()` 以及 `Integer.parseInt()` 等，这些都是Java内置的函数。我们也一直在使用我们的IO库，它们就是一组静态方法（函数）的集合。我们讨论过的所有标准输入、输出和绘图方法，都是Java库中函数集的例子。当然，你一直在定义一个名为 `main` 的函数，我们马上会详细讨论它。

## 🔍 函数的结构剖析
让我们看看一个函数（Java静态方法）的解剖结构。以下是我们早期讲座中实现的平方根函数：

```java
public static double sqrt(double c, double eps) {
    double t = c;
    while (Math.abs(t - c/t) > eps * t) {
        t = (c/t + t) / 2.0;
    }
    return t;
}
```

*   **名称**：`sqrt`
*   **参数**：每个参数都需要一个类型和一个名称，它们放在函数名后的括号内。例如 `(double c, double eps)`。
*   **返回值**：函数产生的值，必须有一个类型。本例中返回 `double` 类型。

所有这些，加上前面的 `public static`，被称为**方法签名**。它告诉你在使用或调用它时需要知道的信息。然后是**方法体**，即实现功能的Java代码。代码的最后通常是一个 `return` 语句，后面跟着一个与返回类型匹配的值。

## 📁 库文件：将函数组织在一起
库是一组函数，一个 `.java` 文件就可以是一个库。例如，一个名为 `Newton.java` 的库文件：

```java
public class Newton {
    public static double sqrt(double c, double eps) {
        // ... 平方根计算代码 ...
    }

    public static void main(String[] args) {
        // ... 主程序代码 ...
    }
}
```

文件名与类名相同（这里是 `Newton`）。这个库包含两个函数：`sqrt` 方法和 `main` 方法。我们马上会看它们如何交互。这里使用 `sqrt` 只是为了说明基础概念，实际上你可以直接使用 `Math.sqrt()`。

## 🔄 函数调用与流程控制
本节课的一个关键点是，函数提供了一种全新的控制执行流程的方式。虽然细节看起来有点复杂，但如果你将其与数学函数类比，使用起来就非常自然。我们希望能够推理程序在做什么，而将小的计算片段封装在函数中可以帮助我们做到这一点。

为了真正理解发生了什么，我们必须讨论**作用域**的概念。

## 🎯 变量的作用域
一个变量的**作用域**是指可以通过名称引用该变量的代码范围。当我们有多组函数时，你不能在任何地方引用每一个变量（那样的变量称为全局变量，我们在Java库中不使用）。在Java库中，变量的作用域是**其声明之后、在同一代码块内**的代码。

例如，在 `sqrt` 函数中，参数变量 `c` 和 `eps` 的作用域是该函数内部的所有代码。变量 `t` 的作用域仅限于其声明之后的语句。你不能在声明 `t` 之前的代码中引用它，也不能在 `main` 函数中引用 `sqrt` 中的任何变量，因为它们在不同的代码块中。

在 `main` 中声明的数组 `A` 和整数变量 `i`，你不能在 `sqrt` 中引用它们，它们超出了作用域。我们有两个名为 `i` 的变量（例如在两个不同的 `for` 循环中），它们实际上是不同的变量。

我们的目标是尽可能限制变量的作用域，以便于理解程序在做什么。如果能在很远的、不同代码块中引用一个变量，将很难理解该代码块中发生了什么。因此，我们尽量保持作用域小而集中，函数帮助我们做到了这一点。

## 🚦 函数调用的流程控制
让我们看看函数调用的流程控制。以 `Newton` 程序为例，其 `main` 程序的目的是读入一系列参数，然后为每个参数计算平方根并打印出来。

当你在主程序中调用 `sqrt` 时：
1.  控制权转移到函数代码。
2.  声明参数变量，并用你提供的值初始化它们。
3.  执行所有函数代码。
4.  函数完成后，它返回到调用代码，返回值被赋值给调用代码中函数名所在的位置。

这被称为**按值传递**。其他语言和系统可能使用其他方法，但这是Java中使用的方式。

另外，我们迄今为止编写的所有程序是如何运行的呢？当我们使用 `java` 命令调用程序时，操作系统会调用 `main` 函数。因此，`main` 是一个由操作系统调用的函数，其参数（命令行参数）的提供机制与将 `c` 和 `eps` 的值提供给 `sqrt` 的机制完全相同。

## 📊 流程控制跟踪示例
假设我们用命令行参数 `1`、`2`、`3` 调用程序。
1.  `main` 代码读取命令行参数并放入一个双精度数组中。
2.  进入第二个 `for` 循环，计算并打印每个数字的平方根。
3.  循环开始，`i = 0`，`A[i] = 1.0`。
4.  调用 `sqrt(1.0, ...)`，控制转到 `sqrt` 函数，计算后返回 `1.0`。
5.  返回值赋给 `x`，然后打印出来。
6.  循环继续，`i = 1`，计算 `sqrt(2.0, ...)`，经过多次迭代后返回近似值，赋值并打印。
7.  循环继续，`i = 2`，计算 `sqrt(3.0, ...)`，过程类似。
8.  循环结束，程序执行完毕。

虽然详细跟踪时流程控制看起来更复杂，但理解这段代码在做什么是很自然的：我们在 `main` 中声明，要在循环中打印平方根，很自然地假设会发生的是它将执行代码来计算平方根。

## 💡 参数传递与作用域细节
让我们看几个简单的例子来讨论函数中参数传递的方式和作用域。

**示例1：计算立方**
```java
public static int cube(int i) {
    int j = i * i * i;
    return j;
}
```
这个函数接收一个整数 `i`，计算其立方并返回。你可以在许多应用中实现类似的函数。

**示例2：编译错误 - 重复声明**
```java
public static int cube(int i) {
    int i; // 错误：变量 i 已定义
    return i * i * i;
}
```
这段代码甚至无法编译，因为它在参数中声明了 `int i`，又在函数体内试图再次声明 `int i`，编译器会报错。

**示例3：编译错误 - 缺少返回语句**
```java
public static int cube(int i) {
    int j = i * i * i;
    // 错误：缺少返回语句
}
```
函数签名声明返回 `int`，但函数体中没有 `return` 语句，这会导致编译错误。

**示例4：可行但风格不佳**
```java
public static int cube(int i) {
    i = i * i * i;
    return i;
}
```
这个版本可以工作，`cube` 中的 `i` 是作为参数声明和初始化的，它与 `main` 中的 `i` 是不同的变量。但是，我们不喜欢编写会改变函数参数值的代码，即使它能工作，在某些编程环境中可能不适用，我们称之为**不良风格**。

**示例5：推荐风格**
```java
public static int cube(int i) {
    return i * i * i;
}
```
我们更倾向于这种风格。直接进行计算并返回结果，就像在代码块内声明并使用该变量一样，它产生答案且代码清晰。

## 🎓 总结
本节课中，我们一起学习了模块化编程的核心概念。我们了解到，函数（静态方法）是构建和组织大型程序的基本模块。通过将代码封装在具有明确定义输入（参数）和输出（返回值）的函数中，我们可以限制变量的作用域，使程序更易于理解、维护和复用。我们剖析了函数的结构，跟踪了函数调用时的流程控制，并探讨了参数传递的作用域规则和一些良好的编程风格。掌握这些概念是迈向编写更复杂、更结构化Java程序的重要一步。
