## 什么是复数？

复数是一种数学概念，其形式为 **a + bi**，其中 **a** 和 **b** 是实数，而 **i** 被定义为 **√-1**。这个看似“虚构”的概念可以追溯到欧拉和柯西的时代，它为理解许多现实世界的问题提供了深刻的见解。

### 复数的运算

对复数进行代数运算的规则很简单：使用实数代数，但每当遇到 **i²** 时，就将其替换为 **-1**，然后合并同类项。

*   **加法示例**：
    `(3 + 4i) + (-2 + 3i) = (3 - 2) + (4 + 3)i = 1 + 7i`

*   **乘法示例**：
    `(3 + 4i) * (-2 + 3i) = 3*(-2) + 3*3i + 4i*(-2) + 4i*3i = -6 + 9i - 8i + 12i²`
    由于 `i² = -1`，所以 `12i² = -12`。
    合并实部和虚部：`(-6 - 12) + (9i - 8i) = -18 + i`

### 复数的模

复数的模（或绝对值）定义为 **√(a² + b²)**。例如，复数 `3 + 4i` 的模是 `5`。

## 设计复数抽象数据类型

现在，我们希望在Java程序中操作复数，因此需要设计一个抽象数据类型。

### 值

一个复数的值由其**实部**和**虚部**组成。例如，`3 + 4i` 可以用两个双精度浮点数 `3.0` 和 `4.0` 表示。

### 操作

我们希望复数类型支持以下操作：
*   创建复数。
*   将另一个复数加到当前复数上。
*   将当前复数乘以另一个复数。
*   计算复数的模。
*   生成字符串表示形式。

虽然一些编程语言内置了复数类型，但Java没有。不过，我们可以自己实现一个Java类。

---

## 实现复数类

我们将遵循之前使用的方法论：定义实例变量、构造函数、方法，并编写一个测试客户端。

### 实例变量与构造函数

复数对象一旦创建，其值就不应改变（即**不可变**），就像Java中的整数或双精度数一样。因此，其实部和虚部应声明为 `final`。

```java
public class Complex {
    private final double re; // 实部
    private final double im; // 虚部

    // 构造函数
    public Complex(double real, double imag) {
        re = real;
        im = imag;
    }
}
```

### 方法实现

以下是核心方法的实现：

*   **加法**：将两个复数的实部和虚部分别相加，返回一个新的复数对象。
    ```java
    public Complex plus(Complex b) {
        double real = this.re + b.re;
        double imag = this.im + b.im;
        return new Complex(real, imag);
    }
    ```

*   **乘法**：根据复数乘法公式计算，返回一个新的复数对象。
    ```java
    public Complex times(Complex b) {
        double real = this.re * b.re - this.im * b.im;
        double imag = this.re * b.im + this.im * b.re;
        return new Complex(real, imag);
    }
    ```

*   **模**：计算 `√(re² + im²)`。
    ```java
    public double abs() {
        return Math.sqrt(re*re + im*im);
    }
    ```

*   **字符串表示**：以 `a + bi` 的形式返回字符串。
    ```java
    public String toString() {
        return re + " + " + im + "i";
    }
    ```

### 测试客户端

我们可以编写一个简单的测试程序来验证我们的实现：

```java
public static void main(String[] args) {
    Complex a = new Complex(3.0, 4.0);
    Complex b = new Complex(-2.0, 3.0);
    System.out.println("a = " + a);
    System.out.println("b = " + b);
    System.out.println("a * b = " + a.times(b));
}
```

运行此程序应输出 `a * b = -18.0 + 1.0i`，这与我们之前手工计算的结果一致。

## 应用：可视化曼德博集合 🌀

上一节我们介绍了复数类的基本实现，本节中我们来看看一个引人入胜的应用：绘制曼德博集合。这是一个由算法定义的、具有无限复杂分形结构的数学集合。

### 曼德博集合的定义

对于平面上的一个点（对应复数 `z0`），我们根据以下迭代公式生成一个序列：
`z_{n+1} = (z_n)² + z0`

如果这个序列的模（绝对值）**发散到无穷大**，那么 `z0` **不属于**曼德博集合。否则，`z0` **属于**曼德博集合。

### 绘制挑战与近似方案

直接根据定义绘制面临两个挑战：
1.  我们无法绘制无限多个点。
2.  我们无法通过有限次迭代判断序列是否真的会发散到无穷大。

因此，我们采用以下近似方案：
*   **采样**：在平面上选择一个方形区域，生成一个 `N x N` 的像素网格进行采样。
*   **逃逸时间算法**：对于一个点 `z0`，我们进行迭代。如果某次迭代后 `z_n` 的模大于 `2`，数学上可以证明序列将发散，因此该点不属于集合。我们记录下达到此条件的迭代次数 `t`。
*   **着色**：
    *   如果迭代 `255` 次后模始终未超过 `2`，我们近似认为该点属于集合，将其涂为黑色。
    *   否则，根据逃逸时间 `t`（`t` 越小，发散越快）为其分配一个灰度或颜色值。

### 实现可视化

以下是用于确定像素颜色的核心静态方法：

```java
public static Color mand(Complex z0) {
    Complex z = z0;
    for (int t = 0; t < 255; t++) {
        if (z.abs() > 2.0) return Color.WHITE; // 或根据t返回灰度/颜色
        z = z.times(z).plus(z0); // z = z² + z0
    }
    return Color.BLACK; // 迭代255次未逃逸，认为是集合内的点
}
```

然后，我们可以编写一个绘图程序，遍历每个像素，将其坐标映射为复平面上的点，调用 `mand()` 方法获取颜色，并绘制图像。

### 探索分形之美

通过调整观察窗口（缩放）和着色方案（使用逃逸时间 `t` 映射到颜色表），我们可以得到极其复杂和美丽的图像。曼德博集合的边界具有“自相似”的分形特性，无论放大多少倍，都能看到新的、相似而又不同的精细结构。这一切都源于那个简单的迭代公式 `z = z² + c`。
