## 数据可视化的核心组件

![](img/8133c32f354791c7349a36750b58291b_10.png)

在开始编写代码之前，我们首先需要理解构成一个图表的几个核心组件。这被称为“图形语法”。

![](img/8133c32f354791c7349a36750b58291b_12.png)

![](img/8133c32f354791c7349a36750b58291b_14.png)

![](img/8133c32f354791c7349a36750b58291b_16.png)

### 1. 数据 (Data)
数据是我们想要可视化的基础。没有数据，我们就无法创建图表。例如，我们有一个包含候选人及其得票数的表格：

| 候选人 | 得票数 |
| :--- | :--- |
| Mario | 100 |
| Peach | 200 |
| Bowser | 150 |

![](img/8133c32f354791c7349a36750b58291b_18.png)

### 2. 几何对象 (Geometry)
几何对象决定了我们想要绘制的图表类型。例如：
*   **柱状图 (`geom_col`)**: 适合展示分组数据及其对应的数值。
*   **散点图 (`geom_point`)**: 适合展示两个连续变量之间的关系。
*   **折线图 (`geom_line`)**: 适合展示数据随时间的变化趋势。

对于上面的候选人数据，柱状图是一个很好的选择，因为我们可以为每个候选人绘制一个柱子，柱子的高度代表其得票数。

### 3. 美学映射 (Aesthetic Mapping)
美学映射定义了如何将数据表中的列映射到图表的视觉属性上。最常见的两个映射是：
*   **x轴 (x-axis)**: 通常映射到分类变量或自变量。
*   **y轴 (y-axis)**: 通常映射到连续变量或因变量。

![](img/8133c32f354791c7349a36750b58291b_20.png)

![](img/8133c32f354791c7349a36750b58291b_21.png)

在我们的例子中，我们可以将`候选人`列映射到x轴，将`得票数`列映射到y轴。

![](img/8133c32f354791c7349a36750b58291b_23.png)

![](img/8133c32f354791c7349a36750b58291b_25.png)

## 创建第一个柱状图

![](img/8133c32f354791c7349a36750b58291b_27.png)

![](img/8133c32f354791c7349a36750b58291b_29.png)

上一节我们介绍了构成图表的核心概念，本节中我们来看看如何在R中实现它们。

![](img/8133c32f354791c7349a36750b58291b_34.png)

我们将使用`ggplot2`包，它是`tidyverse`生态系统的一部分。首先，确保你已经加载了`tidyverse`。

```r
library(tidyverse)
```

### 步骤1：准备数据并创建空白画布
我们首先使用`ggplot()`函数创建一个空白的图表，并指定要使用的数据框。

```r
# 假设数据已加载到名为 `votes` 的数据框中
p <- ggplot(data = votes)
```
运行这行代码会生成一个空白的图表区域。

![](img/8133c32f354791c7349a36750b58291b_36.png)

### 步骤2：添加美学映射
接下来，我们需要告诉`ggplot`如何将数据映射到坐标轴上。我们使用`aes()`函数。

![](img/8133c32f354791c7349a36750b58291b_38.png)

```r
p <- ggplot(data = votes, aes(x = 候选人, y = 得票数))
```
现在，图表有了x轴和y轴，但还没有图形。

### 步骤3：添加几何对象（图层）
现在，我们为图表添加一个图层——柱状几何对象。在`ggplot2`中，我们使用`+`号来叠加图层。

```r
p <- p + geom_col()
```
运行这行代码后，我们就得到了一个基本的柱状图！`ggplot2`会自动处理许多细节，比如坐标轴标签和刻度。

![](img/8133c32f354791c7349a36750b58291b_40.png)

## 自定义和美化图表

![](img/8133c32f354791c7349a36750b58291b_42.png)

![](img/8133c32f354791c7349a36750b58291b_44.png)

我们已经创建了一个基础图表，但通常我们需要调整它以使其更清晰、更美观。`ggplot2`的图层系统让我们可以轻松地添加和修改各种元素。

![](img/8133c32f354791c7349a36750b58291b_46.png)

![](img/8133c32f354791c7349a36750b58291b_47.png)

### 调整坐标轴范围
有时默认的坐标轴范围不合适。例如，我们可能希望y轴的上限略高于最高值，以留出一些空间。我们可以使用`scale_y_continuous()`函数来调整连续型y轴的范围。

```r
p <- p + scale_y_continuous(limits = c(0, 250))
```

### 修改标签和标题
默认的坐标轴标签是列名。我们可以使用`labs()`函数来设置更友好的标签和添加标题。

```r
p <- p + labs(
  x = “候选人”,
  y = “得票数”,
  title = “选举结果”
)
```

![](img/8133c32f354791c7349a36750b58291b_49.png)

![](img/8133c32f354791c7349a36750b58291b_50.png)

### 修改颜色
我们可以根据数据为柱子着色。例如，让每个候选人的柱子拥有不同的颜色。这可以通过在`aes()`映射中设置`fill`（填充色）美学来实现。

```r
# 注意：这里的 aes() 是放在 geom_col() 内部的，表示这个映射仅应用于柱子
p <- p + geom_col(aes(fill = 候选人))
```

![](img/8133c32f354791c7349a36750b58291b_52.png)

为了照顾色觉障碍人士，我们可以使用色盲友好的配色方案，如`viridis`色标。

![](img/8133c32f354791c7349a36750b58291b_54.png)

```r
p <- p + scale_fill_viridis_d(name = “候选人”) # ‘d’ 代表离散型
```

![](img/8133c32f354791c7349a36750b58291b_56.png)

![](img/8133c32f354791c7349a36750b58291b_57.png)

![](img/8133c32f354791c7349a36750b58291b_59.png)

![](img/8133c32f354791c7349a36750b58291b_61.png)

### 应用主题
主题可以控制图表的整体外观，如背景色、网格线等。`ggplot2`提供了多种主题。

```r
p <- p + theme_classic()
```

![](img/8133c32f354791c7349a36750b58291b_63.png)

### 移除图例
如果x轴标签已经清楚地标明了分类，右侧的图例可能显得多余。我们可以将其隐藏。

```r
p <- p + geom_col(aes(fill = 候选人), show.legend = FALSE)
```

### 保存图表
最后，我们可以使用`ggsave()`函数将图表保存为图片文件。

![](img/8133c32f354791c7349a36750b58291b_65.png)

```r
ggsave(“选举结果.png”, plot = p, width = 12, height = 9, units = “cm”)
```

## 创建散点图

上一节我们学习了如何创建和自定义柱状图，本节中我们来看看如何使用散点图来探索两个连续变量之间的关系。

假设我们有一个糖果数据集，包含每种糖果的“价格百分位数”和“糖分百分位数”。我们想看看价格和糖分之间是否存在关联。

![](img/8133c32f354791c7349a36750b58291b_69.png)

### 基础散点图
创建散点图的流程与柱状图类似，只是几何对象换成了`geom_point`。

![](img/8133c32f354791c7349a36750b58291b_71.png)

```r
# 假设数据已加载到名为 `candy` 的数据框中
p_scatter <- ggplot(data = candy, aes(x = 价格百分位数, y = 糖分百分位数)) +
  geom_point()
```

### 处理点重叠（抖动）
当多个数据点具有相同的x和y值时，它们会重叠在一起。为了解决这个问题，我们可以使用`geom_jitter()`，它会为每个点添加微小的随机偏移。

![](img/8133c32f354791c7349a36750b58291b_73.png)

![](img/8133c32f354791c7349a36750b58291b_75.png)

```r
p_scatter <- p_scatter + geom_jitter()
```

![](img/8133c32f354791c7349a36750b58291b_77.png)

### 自定义点的外观
我们可以通过设置`geom_jitter()`内部的美学参数来改变点的颜色、大小和形状。

![](img/8133c32f354791c7349a36750b58291b_79.png)

```r
p_scatter <- ggplot(data = candy, aes(x = 价格百分位数, y = 糖分百分位数)) +
  geom_jitter(
    color = “darkorchid4”, # 边框颜色
    fill = “orchid”,       # 填充颜色（需要特定形状，如shape=21）
    shape = 21,            # 形状编号，21是带填充的圆
    size = 2,              # 点的大小
    alpha = 0.7            # 透明度 (0-1)
  ) +
  labs(x = “价格”, y = “糖分”, title = “糖果价格与糖分关系”) +
  theme_classic()
```

## 创建折线图（时间序列）

最后，我们来看看如何可视化数据随时间的变化，例如飓风的风速变化。

### 基础折线图与散点图结合
对于时间序列数据，我们通常同时使用点和线来展示趋势。

![](img/8133c32f354791c7349a36750b58291b_83.png)

![](img/8133c32f354791c7349a36750b58291b_85.png)

```r
# 假设数据已加载到名为 `anita` 的数据框中，包含 `timestamp` 和 `wind` 列
p_line <- ggplot(data = anita, aes(x = timestamp, y = wind)) +
  geom_line() +  # 添加折线
  geom_point()   # 添加数据点
```

![](img/8133c32f354791c7349a36750b58291b_87.png)

### 自定义线条和点
我们可以分别调整线和点的外观。

```r
p_line <- ggplot(data = anita, aes(x = timestamp, y = wind)) +
  geom_line(color = “steelblue”, size = 0.8, linetype = “solid”) + # 线条颜色、粗细、类型
  geom_point(color = “darkorange”, size = 2) + # 点的颜色和大小
  labs(x = “日期”, y = “风速 (节)”, title = “飓风 Anita 风速变化”) +
  theme_classic()
```

![](img/8133c32f354791c7349a36750b58291b_89.png)

### 添加参考线
我们可能想添加一条水平参考线来标记重要阈值，例如飓风形成的风速标准（65节）。

![](img/8133c32f354791c7349a36750b58291b_91.png)

```r
p_line <- p_line + geom_hline(yintercept = 65, linetype = “dashed”, color = “red”, size = 0.5)
```
这条虚线清晰地标出了飓风形成的临界点。

## 总结

在本节课中，我们一起学习了使用`ggplot2`进行数据可视化的核心技能。我们从理解“图形语法”（数据、几何对象、美学映射）开始，逐步实践了三种常见图表的创建：

1.  **柱状图 (`geom_col`)**: 用于比较不同类别的数值。
2.  **散点图 (`geom_point`/`geom_jitter`)**: 用于探索两个连续变量之间的关系。
3.  **折线图 (`geom_line`)**: 用于展示数据随时间或其他连续变量的变化趋势。

![](img/8133c32f354791c7349a36750b58291b_93.png)

![](img/8133c32f354791c7349a36750b58291b_95.png)

我们掌握了通过叠加图层（使用`+`号）来构建和自定义图表的方法，包括调整坐标轴(`scale_*`)、修改标签(`labs`)、应用颜色(`aes(fill/color)`)、选择配色方案(`scale_*_viridis_*`)以及改变整体主题(`theme_*`)。

![](img/8133c32f354791c7349a36750b58291b_97.png)

记住，好的可视化始于清晰的问题和整洁的数据。`ggplot2`的强大之处在于其系统性和灵活性，允许你从简单的草图开始，逐步添加细节，最终创造出既信息丰富又美观的图表。
