## 广播消息 📡

上一节我们学习了如何让角色独立行动。本节中，我们来看看如何让多个角色之间进行**通信与协作**。例如，让小猫说完话后，恐龙能自动回应，而无需手动计算等待时间。

以下是实现角色间通信的步骤：

![](img/5838b48a6a438baf8a3ae75977bb6603_5.png)

![](img/5838b48a6a438baf8a3ae75977bb6603_7.png)

1.  **发送广播消息**：一个角色（发送者）在完成某个动作后，可以广播一条自定义消息。
    ```scratch
    当 ⚑ 被点击
    说 [你好，恐龙！] (2) 秒
    广播消息 [greet]
    ```
2.  **接收广播消息**：另一个角色（接收者）可以监听特定的广播消息，并在收到后触发相应的动作。
    ```scratch
    当接收到消息 [greet]
    说 [你好，小猫！] (2) 秒
    ```
通过这种方式，发送者可以精确地通知接收者“轮到你了”，从而协调多个角色的行为。

广播消息的用途非常广泛：

*   **控制角色行为**：例如，创建两个箭头按钮精灵，点击时分别广播“上”和“下”消息。另一个角色（如鸭子）在接收到这些消息时，改变其Y坐标来实现移动。
    ```scratch
    // 上箭头精灵的代码
    当角色被点击
    广播消息 [up]

    // 鸭子精灵的代码
    当接收到消息 [up]
    y 增加 (10)
    ```
*   **舞台背景触发事件**：舞台背景也可以广播消息。例如，点击舞台任意位置，广播“拜访”消息，让小鱼游到鼠标点击的位置。
    ```scratch
    // 舞台背景的代码
    当舞台被点击
    广播消息 [visit]

    // 小鱼精灵的代码
    当接收到消息 [visit]
    面向 [鼠标指针]
    在 (1) 秒内滑行到 [鼠标指针]
    ```

## 克隆功能 👥

除了让角色通信，有时我们还需要在舞台上创建**同一个精灵的多个副本**。例如，在游戏中生成大量相同的敌人或物品。这时就需要使用克隆功能。

以下是创建和使用克隆的基本方法：

1.  **创建克隆**：一个精灵可以执行“创建自己的克隆体”积木来复制自己。
    ```scratch
    当角色被点击
    创建 [自己] 的克隆体
    ```
2.  **定义克隆体的行为**：克隆体被创建后，会独立运行以“当作为克隆体启动时”开头的代码块，定义它独有的行为。
    ```scratch
    当作为克隆体启动时
    在 (1) 秒内滑行到 [随机位置]
    ```
这样，每次点击星星精灵，都会创建一个新的克隆体，并滑向随机位置。克隆功能是构建包含大量相同元素项目（如射击游戏的子弹、收集类游戏的道具）的强大工具。

## 综合实践：构建“接水晶”游戏 💎

现在，让我们将广播、克隆以及之前学过的所有概念结合起来，从头构建一个游戏。游戏规则是：小猫左右移动，接住从天空随机位置落下的水晶。接到水晶得分，水晶落地则算失误，失误三次游戏结束。

### 第一步：设置小猫角色

![](img/5838b48a6a438baf8a3ae75977bb6603_9.png)

![](img/5838b48a6a438baf8a3ae75977bb6603_11.png)

首先，我们需要让小猫能够移动，并初始化游戏。

1.  **移动控制**：使用左右方向键控制小猫在舞台底部水平移动。
    ```scratch
    当按下 [左移] 键
    x 增加 (-10)

    当按下 [右移] 键
    x 增加 (10)
    ```
2.  **游戏初始化**：当绿旗被点击时，将小猫重置到起始位置，并显示游戏提示。
    ```scratch
    当 ⚑ 被点击
    移到 x: (0) y: (-125) // 舞台底部中央
    说 [接住水晶，别让它们落地！] (4) 秒
    广播消息 [begin] // 提示结束，游戏开始
    ```

### 第二步：创建水晶角色

接下来，我们处理水晶的行为。水晶需要在游戏开始后才出现，并不断下落。

1.  **初始隐藏与监听开始**：游戏开始时隐藏水晶，直到收到小猫广播的“begin”消息。
    ```scratch
    当 ⚑ 被点击
    隐藏

    当接收到消息 [begin]
    重复执行
        创建 [自己] 的克隆体
        等待 (15) 秒 // 每15秒生成一个新水晶，增加游戏难度
    ```
2.  **克隆体的行为**：每个水晶克隆体出现时，应移动到舞台顶部的随机水平位置，然后持续下落。
    ```scratch
    当作为克隆体启动时
    显示
    移到 x: (在 (-200) 到 (200) 间随机选一个数) y: (160) // 随机水平位置，顶部
    重复执行
        y 增加 (-2) // 下落
        如果 <碰到 [Cat] ?> 那么 // 如果被小猫接到
            将变量 [catches] 的值增加 (1) // 得分增加
            创建 [自己] 的克隆体 // 生成新水晶
            删除此克隆体 // 移除被接住的水晶
        如果 <碰到 [舞台边缘] ?> 那么 // 如果落地
            将变量 [misses] 的值增加 (1) // 失误增加
            如果 <(misses) = (3)> 那么 // 如果失误达到3次
                广播消息 [game over] // 广播游戏结束
                停止 [全部] // 停止所有脚本
            创建 [自己] 的克隆体 // 生成新水晶
            删除此克隆体 // 移除落地的水晶
    ```

### 第三步：添加变量与结束逻辑

我们需要变量来跟踪得分和失误，并在游戏结束时显示结果。

1.  **创建变量**：创建两个变量，`catches`（接住数）和`misses`（失误数）。
2.  **初始化变量**：在小猫的初始化代码中，将这两个变量都设为0。
    ```scratch
    当 ⚑ 被点击
    ...
    将变量 [catches] 设为 (0)
    将变量 [misses] 设为 (0)
    ```
3.  **游戏结束处理**：小猫接收到“game over”消息后，显示最终得分。
    ```scratch
    当接收到消息 [game over]
    说 [连接 [你的得分是：] 和 (catches)] (5) 秒
    ```

更重要的是，我们综合运用了本课程所学的所有工具——**事件、控制流、条件判断、变量、广播和克隆**——从头构思并逐步构建了一个完整的“接水晶”游戏。这个过程展示了编程的本质：将复杂想法分解为小步骤，利用基础模块的组合来实现创意。
