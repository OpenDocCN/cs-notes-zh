上一节我们介绍了数据结构的基本概念，本节中我们来看看抽象数据类型。抽象数据类型是一种数据结构的抽象描述，它定义了某些属性和操作，但具体的实现细节由程序员决定。

例如，队列是一种常见的抽象数据类型。在现实生活中，队列（或称为“排队”）遵循“先进先出”的原则。

以下是队列的核心操作：
*   **入队**：将元素添加到队列末尾。
*   **出队**：从队列前端移除元素。

队列可以用数组来实现。我们需要跟踪两个关键信息：
*   一个数组，用于存储元素。
*   一个整数，用于记录队列当前的大小。

**公式**：`队列 = 数组 + 大小`

![](img/3432bd9599b7a4ad0470a83a8599af7a_31.png)

## 栈

与队列不同，栈遵循“后进先出”的原则。想象一下你的电子邮件收件箱，新邮件总是出现在顶部，你很可能先处理最新的邮件。

以下是栈的核心操作：
*   **压栈**：将元素添加到栈顶。
*   **弹栈**：从栈顶移除元素。

![](img/3432bd9599b7a4ad0470a83a8599af7a_33.png)

栈同样可以用数组来实现，其底层结构与队列相似，但操作逻辑（`LIFO` 与 `FIFO`）不同。

**公式**：`栈 = 数组 + 大小`

## 数组的局限性

![](img/3432bd9599b7a4ad0470a83a8599af7a_35.png)

![](img/3432bd9599b7a4ad0470a83a8599af7a_37.png)

上一节我们介绍了用数组实现队列和栈，本节中我们来看看这种方法的局限性。数组要求内存连续分配，并且大小固定。如果我们想向一个已满的数组中添加新元素，就必须：
1.  分配一个更大的新数组。
2.  将旧数组的所有元素复制到新数组。
3.  释放旧数组的内存。

![](img/3432bd9599b7a4ad0470a83a8599af7a_39.png)

![](img/3432bd9599b7a4ad0470a83a8599af7a_41.png)

这个过程的时间复杂度是 **O(n)**，效率较低。此外，如果预先分配过大的数组，又会浪费内存空间。

## 链表

为了解决数组的固有限制，我们引入了链表。链表中的每个“节点”可以分散在内存的任何位置，并通过指针连接起来。

![](img/3432bd9599b7a4ad0470a83a8599af7a_43.png)

![](img/3432bd9599b7a4ad0470a83a8599af7a_45.png)

一个链表节点通常包含两部分：
*   **数据**：存储的实际值（例如一个整数）。
*   **指针**：指向下一个节点的内存地址。

**代码**：
```c
typedef struct node
{
    int number;
    struct node *next;
}
node;
```

链表的优势在于可以动态增长和收缩，无需像数组那样进行昂贵的数据拷贝。然而，查找链表中的元素需要从头遍历，时间复杂度为 **O(n)**。

![](img/3432bd9599b7a4ad0470a83a8599af7a_47.png)

## 二叉搜索树

为了结合数组（快速查找）和链表（动态大小）的优点，我们引入了二叉搜索树。在BST中，每个节点包含：
*   一个值。
*   一个指向左子树的指针（所有值小于当前节点）。
*   一个指向右子树的指针（所有值大于当前节点）。

![](img/3432bd9599b7a4ad0470a83a8599af7a_49.png)

![](img/3432bd9599b7a4ad0470a83a8599af7a_51.png)

**代码**：
```c
typedef struct node
{
    int number;
    struct node *left;
    struct node *right;
}
node;
```

![](img/3432bd9599b7a4ad0470a83a8599af7a_53.png)

在平衡良好的BST中，搜索、插入和删除操作的时间复杂度可以达到 **O(log n)**。但是，如果插入的数据顺序不当（例如已排序序列），BST可能会退化成一条链，时间复杂度降为 **O(n)**。

## 哈希表

我们的目标是实现常数时间 **O(1)** 的操作。哈希表朝这个目标迈进了一步。一个简单的哈希表可以是一个数组，数组的每个位置（称为“桶”）指向一个链表。

![](img/3432bd9599b7a4ad0470a83a8599af7a_55.png)

**工作原理**：
1.  使用哈希函数将输入（例如名字）转换成一个数组索引。
2.  将该元素放入对应索引的链表中。

![](img/3432bd9599b7a4ad0470a83a8599af7a_57.png)

**代码**（简单哈希函数示例）：
```c
unsigned int hash(const char *word)
{
    return toupper(word[0]) - 'A';
}
```

![](img/3432bd9599b7a4ad0470a83a8599af7a_59.png)

在理想情况下，元素均匀分布，每个链表很短，操作接近 **O(1)**。但在最坏情况下（所有元素哈希到同一位置），哈希表退化为链表，时间复杂度为 **O(n)**。

![](img/3432bd9599b7a4ad0470a83a8599af7a_61.png)

## 字典树

![](img/3432bd9599b7a4ad0470a83a8599af7a_63.png)

字典树是一种可以实现真正常数时间查找的数据结构，特别适用于字符串键。字典树的每个节点是一个数组，数组的每个元素对应一个字符，并指向下一个节点。

![](img/3432bd9599b7a4ad0470a83a8599af7a_65.png)

**查找过程**：要查找一个长度为 *k* 的单词，只需要沿着路径走 *k* 步。因为 *k* 对于具体单词是常数，所以时间复杂度是 **O(1)**。

![](img/3432bd9599b7a4ad0470a83a8599af7a_67.png)

![](img/3432bd9599b7a4ad0470a83a8599af7a_69.png)

![](img/3432bd9599b7a4ad0470a83a8599af7a_70.png)

然而，字典树的空间消耗非常大，因为即使很多路径是空的，也需要为每个可能字符分配指针。

## 总结
本节课中我们一起学习了多种数据结构。我们从简单的抽象数据类型（队列和栈）及其数组实现开始，认识到数组在动态扩容上的局限性。接着，我们引入了链表，它支持动态增长但查找效率低。为了提升查找效率，我们探讨了二叉搜索树，它在平衡时能有对数级的性能。最后，我们研究了哈希表和字典树，它们旨在追求接近常数时间的操作，但分别在时间（冲突）和空间（稀疏）上做出了权衡。理解这些数据结构的特性和权衡，是设计高效程序的关键。
