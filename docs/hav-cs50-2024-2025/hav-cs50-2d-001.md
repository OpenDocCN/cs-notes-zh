## 游戏开发与编程的故事

关于游戏开发和编程，我有个小故事。多年前，让我开始编程的原因是，我小时候非常喜欢玩电子游戏。所以有一天，我决定去书店，寻找看起来最吸引人、或者最有可能帮助我学习视频游戏开发的书。于是我偶然发现了这本书，尽管它叫《3D 游戏编程大全》，但它是一本包罗万象的书，教的是一个叫 Torque 的引擎。我们在这里可以看到它的图片，它在某种意义上像是如今流行的 Unity 和 Unreal 引擎的前身。但在那时，你知道，这是我第一次看到类似的东西。

我清楚地记得打开这本书，也看了一些源代码。在那之前，我从未真正过多地思考过编程或游戏开发到底意味着什么。我只知道我想制作游戏，我喜欢玩游戏。至少可以说，当我打开书，看到美元符号、与符号、竖线等等这些东西时，我感到非常惊讶，这让我不知所措。不幸的是，我最终把书收了起来，没有再看过。

## 核心工具与概念

我们将使用的两个主要关键工具是 **Lua** 和 **Love**。Lua 是我们要使用的编程语言，Love 是一种微引擎式的框架，一个使用 Lua 的运行时环境。但 Love 赋予我们在屏幕上绘制图形和处理音频的能力，这是游戏或游戏引擎通常需要的所有功能。

我们将学习以下基础知识：
*   **形状和文本**：在屏幕上显示实际视觉效果所需的基础。
*   **增量时间和速度**：如何移动物体，我们将在屏幕上看到这一点。游戏是实时的，物体总是在移动。
*   **状态**：以便我们可以持久化像分数这样的信息。
*   **面向对象编程**：以便我们可以将游戏对象、我们的实际数据表示为真实的对象。
*   **碰撞检测**：可能是最重要的部分，2D 碰撞检测，将在所有其他示例中使用，它允许我们在游戏中实现交互。
*   **音效**：最后，为了趣味性，也为了完善整个体验，我们将学习音效。今天讲座中的声音以及未来讲座中的音乐。我们将学习一些免费的、基于浏览器的工具，可以用来创建我们自己的自定义声音，而不需要复杂的工具。

我们不会有一个详尽的教程，但会快速介绍一些要点和事实。Lua 是一个葡萄牙语名字，意思是“月亮”。它在工业中经常使用，非常灵活，非常轻量级。很多引擎，我自己也遇到过很多，都大量使用它进行脚本编写。如果有人用过 JavaScript，它的语法实际上与 JavaScript 非常相似，尽管它引入了自己的语法。

然后，就像 JavaScript 一样，它非常容易存储数据。JavaScript 有 JSON。Lua 本身，所有东西都存储在所谓的 **表** 中，这有点像 JavaScript 的对象，它使得信息的存储非常简洁、容易。

Love 本身，我已经提到过一些，它是一个非常快速、轻量级的 C++ 编译框架，是一个用于开发游戏的二进制程序，但它是用 Lua 编写/脚本化的。它拥有 2D 游戏开发所需的所有模块：图形、键盘输入等等。幸运的是，因为它自带 Lua，它甚至还有所有这些数学模块，并且在即将发布的一些版本中，它甚至将开始包含网络功能。当然，它是完全免费、开源、可移植的。这是一个很棒的框架，我也很喜欢它有一个非常易读的 API，我们将会看到，它非常适合学习和实际项目。

事实上，这款游戏现在非常流行，它叫《Balatro》，去年发布，现在仍然很受欢迎，我当时也玩了不少。CS50 的 Ian Sexton 在我准备这些幻灯片时给我发了这篇文章，所以感谢 Ian。《Balatro》是用 Love 编写的。我不想绝对肯定地说，但它可能是目前 Love2D 中最著名、最知名的游戏。当然还有其他一些，你现在可以在 Love2D 的主页上看到这个列表。这是一款很棒的游戏，它证明了你可以用 Love2D 制作完整的游戏，而不仅仅是原型。同样有趣且酷的是，还有一个 3D 引擎。有过一些这样的实验，但这个仓库特别有一些非常酷的演示，展示了有人在 Love2D 之上构建了他们自己的 Love 3D 引擎。我想这是很多人（包括我自己）一直以来的愿望，因为 Love 的 API 是如此友好、简洁、易用，以至于有时没有它的 3D 版本会让人感到遗憾。所以我认为这个项目在朝那个方向努力方面做得非常出色。这看起来几乎像是在 Unity 中构建的，是一个非常酷的项目。

关于实际工具、Love 和 Lua 就讲这么多，它们非常容易下载和配置。所以今天的目标，我们将专注于 Pong。表面上，Pong 看起来是一个非常简单的游戏，我甚至会说简单得有点滑稽，尤其是与现代游戏相比。但有几件不同的事情值得注意。这只是一个 Pong 的代表性示例的截图，我稍后会去演示我们的实际仓库，但仅仅看这个游戏，你可以看到我们基本上只有一堆形状和文本。但我们之前列出的所有基础知识都在这里：我们有可移动的矩形，这里的球是一个圆形（在我们的示例中会是一个正方形，但两种方式都可以实现）。球需要与球拍碰撞，所以我们需要碰撞检测。当它碰撞到某物时，也许我们想让它播放声音。我们希望在屏幕上显示分数。我们需要游戏中的状态来更新。我们会有获胜条件。所有贯穿我们将在本课程其余部分看到的所有 2D 游戏的基础知识，都存在于 Pong 中。

所以我要快速演示一下，以便我们可以看到它的样子，然后我们将开始深入探讨我们将要看的细节。我将查看课程的仓库，我们将发布所有这些源代码，我们将通过一系列步骤来实现 Pong。我们会从头开始实现其中的一部分，直到某个点，然后我们会开始从更高层次看代码。但为了让你对我们将要看的内容有个概念，这是 Pong 最终版中的 main.lua。需要注意的一点是，你在 Love2D 中运行的所有东西，如果你查看他们的文档也会看到，就像你在 C 或 C++ 中编程一样，在大多数语言中你都会有一个 main 函数。在 Lua 中有点不同，在 Love 和 Lua 中有点不同，你有一个 main.lua，其中包含引导程序的所有代码。我们将更仔细地看看这个以及它的含义。为了示例，我将使用一个扩展在 VS Code 中加载 main.lua，我稍后会展示这个扩展。但为了不花太长时间展示这个示例，我现在就开始。

## 本讲核心内容

![](img/c3d56f9872493c0a867c3020e139985c_64.png)

让我们看看我们将在本讲座中详细研究的一些内容，很多我们刚刚列举和看过的东西。

![](img/c3d56f9872493c0a867c3020e139985c_66.png)

*   **形状**：我们看到了矩形，这实际上是绘制东西到屏幕上的重要第一步，不仅仅是空白屏幕，而是实际有形状。
*   **移动**：这些物体可以移动，所以我们需要能够控制它们的 2D 位置，这样它们就不是静态的，而是能响应键盘输入上下移动。
*   **碰撞检测**：非常重要，整个游戏都由碰撞检测驱动。无论是屏幕的顶部或底部用于从墙壁弹开，还是左侧和右侧，这本质上是任一球拍的失败条件，或者更准确地说，也许不是失败条件，但肯定是当球能够越过球拍到达另一边时，对手得分。
*   **音效**：正如我们所见。
*   **记分**：一些基本的状态。

第一步是**处理输入**。这意味着我们需要检查在每一帧按下了哪些键。游戏每秒大约运行 1 到 60 帧（理想情况下，在大多数计算机上）。所以，游戏本质上是一种幻觉，下一张幻灯片可能会有这个，但它本质上就像一本你不断翻动的翻页书。我也认为这是一个非常可爱的翻页书例子，我不会剧透。我很喜欢它。它很可爱，是个求婚场景，但这本质上就是我们在游戏中做的事情：给用户一本他们通过行动控制外观的翻页书。

## 2D 坐标系

为了在我们实际编程之前让这一切完全说得通，我们还需要理解，当我们处理 2D 游戏时，我们处理的是 2D 坐标系。我想大多数人以前可能在几何学或其他地方见过类似的东西，它们非常普遍。但这本质上只是一种将世界视为一系列 XY 坐标对的方式。不同的引擎、不同的框架会以不同的方式实现这一点。有些会让 Y 轴向上而不是向下，有些会让 Y 轴向下而不是向上，等等。

在我们的情况下，在我们的轴系统中，y 轴向下，x 轴向右和向左。负 y 向上，所以正 y 向下，负 y 向上。正 x 向右，负 x 向左。这很重要，因为所有的 2D 编程，特别是当我们处理坐标时，它本质上映射到你屏幕上的实际像素。所以，为了在你的屏幕上绘制任何东西，无论是精灵、文本还是形状，你屏幕上的像素本质上都遵循这个坐标系。有不同的方式可以将你的屏幕视为坐标，以及你从哪里设置你的轴、你的原点。但一般来说，你可以看到这些都是 RGB 三元组，我们将在整个课程中看到 RGB 颜色在游戏中非常重要，并且要正确存储 2D 游戏。

我们看这个示例时提到的一组重要函数中，有一个是 `love.window.setMode`，它接受几个参数，前两个是窗口的宽度和高度。所以我们会说 `WINDOW_WIDTH`、`WINDOW_HEIGHT`。然后它接受一个表。如果你熟悉 JavaScript 或 Python，在 JavaScript 中这些被称为对象，在 Python 中被称为字典。在 Lua 中，同样的东西被称为表，这是它的全能数据结构。所以我们要在这里声明一组键值对。我们要说 `resizable = false`、`vsync = true` 和 `fullscreen = false`。所以我们不希望它是全屏的，我们确实希望启用垂直同步。垂直同步是你在游戏中随处可见的重要设置，几乎每个游戏都有垂直同步开关。它只是将游戏的渲染与显示器的刷新率同步，这样你就不会看到屏幕撕裂（即屏幕的一部分似乎有点错位，因为渲染在中间被打断）。我们不希望那样。人们对它有不同的看法，我通常喜欢垂直同步，我不喜欢屏幕撕裂，但在某些情况下你肯定不希望它开启，比如为了性能。但我们不必担心那些东西。所以我们将用这些参数设置窗口模式。

![](img/c3d56f9872493c0a867c3020e139985c_175.png)

![](img/c3d56f9872493c0a867c3020e139985c_177.png)

然后在 `love.draw` 中，我们也要调用函数 `love.graphics.printf`。这是我们之前看到的 Love2D 提供的一个渲染函数。它有几个不同的参数。所以我们要说“Hello Pong”。这是我们作为第一个参数传递的字符串，这将是实际打印到屏幕上的内容。然后它接受 x 和 y 参数，指定它实际开始渲染的位置。我们希望这个文本正好在屏幕中间，水平和垂直都居中。但奇怪的是，因为这是一个格式化文本函数，我们在这里指定 x 参数的方式会与我们预期的有点不同。我们要说 0，然后我们传入 y。所以这将是 `WINDOW_HEIGHT / 2`，大约在一半的位置，但我们要向上移动 6 个像素（减去 6）以考虑字体的高度。然后我们要说，我们必须传入一个格式化宽度，即文本实际将在其中格式化的宽度，这将应用于函数的最后一个参数。所以我们会说 `WINDOW_WIDTH` 作为格式化宽度，然后是 `'center'`。我们也可以在这里看到单引号和双引号的不同用法，Lua 支持两者，所以用你喜欢的。

![](img/c3d56f9872493c0a867c3020e139985c_179.png)

这个 `WINDOW_WIDTH` 将把居中对齐的样式参数/格式化参数应用到从 x 值 0 开始的整个窗口宽度，所以它有效地从窗口左侧开始，占用整个宽度，然后对其应用居中，使其在窗口的整个宽度内居中。现在，假设我没有打错字，我们将用 `Cmd+L` 运行它，因为我已经设置好了扩展。我们确实有了我们非常基础的窗口，上面只写着“Hello Pong”，字体非常非常小，它以原生分辨率渲染，12 像素字体，但它完美居中。我们本质上有了 2D 游戏开发的“Hello World”，在这种情况下定制为“Hello Pong”，但我认为它作为一个极好的、激励人心的示例，可以让我们做更多。

![](img/c3d56f9872493c0a867c3020e139985c_181.png)

我注意到在你的代码中，你使用 `function` 和 `end` 来开始和结束函数，但像 `setMode` 这样的方法像其他语言一样使用括号。你怎么知道什么时候用 `end`，什么时候用括号？

![](img/c3d56f9872493c0a867c3020e139985c_183.png)

![](img/c3d56f9872493c0a867c3020e139985c_184.png)

所以，`end` 通常用于定义块并终止块的定义。所以，一个块是当你有一个像 if 语句或函数体这样的东西时，定义自己作用域的东西通常由末尾的 `end` 界定。而这些是函数调用，它们为了可读性有多行，但本质上这只是一个表，一个匿名表，只是作为我们函数中的一个参数内联，它有几个键值对，并没有实际定义一个新的作用域，它只是一段数据。同样，这个有括号来实际触发函数调用。所以我们在这里触发 `love.window.setMode`。这些括号和那些括号之间的区别在于，在函数的情况下，函数期望有括号，里面是它的参数列表。所以对于 `love.load`，它没有参数；`love.draw` 没有参数；但 `love.update` 例如（我们将会看到）有一个参数，按照惯例称为 `dt`。这就是函数定义（也带有 `end`）和函数调用之间的括号区别：一个是函数定义，另一个是函数调用，我们实际上是在告诉它用那个函数做某事。

![](img/c3d56f9872493c0a867c3020e139985c_186.png)

当你像在 `setMode` 中那样传递一个表作为参数时，键值对的顺序重要吗？

![](img/c3d56f9872493c0a867c3020e139985c_188.png)

![](img/c3d56f9872493c0a867c3020e139985c_190.png)

不，不重要。是的，如果你传递一个表给任何东西作为一段数据，键和键值对可以是你想要的任何顺序。最终，它只会查看所有键值对，本质上只是将它们全部存储在数据中，供以后的函数引用，而不会以任何有意义的方式考虑顺序。

好的，那就是 Pong 0，也就是第零天更新。现在我要说的一件事是，看那个示例，最让我印象深刻的是，有几件事，但肯定是屏幕中间分辨率极低的文本，几乎看不清，相当难以辨认，而且不太合适。当我开始接触游戏时，我认为美学很重要。像素化对于那种复古怀旧的外观很重要，特别是如果我们做 2D 游戏开发。这是非常流行的东西。你当然不需要它。你当然可以做完全原生分辨率的 2D 游戏，大多数手机游戏都是这样，它们看起来很棒。但 Pong 尤其是一款较老的游戏，我们看的许多游戏都是非常像素化风格的游戏。但如果没有特定的工具或渲染方式，用游戏引擎或框架通常不太容易做到这一点，因为默认情况下，如果你渲染一个纹理、字体、任何东西、任何形状，它都只是你屏幕的原生分辨率。

![](img/c3d56f9872493c0a867c3020e139985c_193.png)
