## 概述

![](img/20091dc4a9e0cdefa2017570ff993dee_6.png)

编写代码时，难免会引入错误。这些错误可能是语法错误，导致程序无法运行；也可能是逻辑错误，导致程序输出与预期不符。当仅凭肉眼阅读代码无法找出问题时，我们就需要借助调试工具。本节课将介绍两种实用的调试方法。

## 使用 `print` 函数进行调试

`print` 函数是最简单直接的调试工具。通过在代码中临时插入 `print` 语句，我们可以输出变量的值或程序执行到特定位置的信息，从而帮助我们理解程序的运行状态。

让我们通过一个打印金字塔图案的程序来演示。我们的目标是输入一个高度值，然后打印出相应高度的金字塔，例如高度为3时，输出应为：
```
#
##
###
```

以下是我们最初的、有错误的代码：

```python
def pyramid(n):
    for i in range(n):
        print("#" * i)

def main():
    height = int(input("Height: "))
    pyramid(height)

if __name__ == "__main__":
    main()
```

运行此程序并输入高度 `3`，我们得到的输出是：
```

#
##
```

这与预期不符。第一行是空的，我们只得到了两行“砖块”。

为了找出问题，我们可以在循环中临时添加一个 `print` 语句来查看变量 `i` 的值：

```python
def pyramid(n):
    for i in range(n):
        print(i, end=" ")  # 打印i的值，并以空格结尾而不是换行
        print("#" * i)
```

再次运行程序，输出变为：
```
0
1 #
2 ##
```

现在问题变得清晰了：`range(n)` 生成的序列是从 `0` 开始的，所以第一次循环时 `i` 为 `0`，导致打印了 `0` 个 `#`，即一个空行。

找到问题后，我们修复代码并移除临时的 `print` 语句：

```python
def pyramid(n):
    for i in range(n):
        print("#" * (i + 1))  # 将 i 加 1
```

现在程序可以正确运行了。

虽然 `print` 调试法简单有效，但在复杂程序中，到处插入 `print` 语句会显得混乱且难以管理。此时，我们需要更强大的工具。

## 使用调试器进行调试

集成开发环境（如 VS Code）通常内置了调试器。调试器允许我们逐行执行代码，在任意位置暂停（设置断点），并实时查看所有变量的值。

上一节我们通过 `print` 找到了一个简单的错误，本节中我们来看看如何使用调试器更系统、更清晰地完成同样的任务。

首先，我们恢复有错误的原始代码，并在 VS Code 中设置一个断点。断点告诉调试器在何处暂停执行。

1.  在代码行号左侧点击，会出现一个红点，这表示设置了一个断点。我们可以在程序开始执行的 `main()` 调用处设置断点。
2.  点击 VS Code 侧边栏的“运行和调试”图标（一个带虫子的播放按钮）。
3.  点击“运行和调试”按钮，程序开始运行，并在断点处暂停。当前要执行的行会高亮显示。

调试器界面提供了几个控制按钮：
*   **继续**：执行到下一个断点或程序结束。
*   **单步跳过**：执行当前行，但不进入该行调用的函数内部。
*   **单步进入**：执行当前行，如果该行调用了函数，则进入该函数内部。

以下是使用调试器的步骤：

1.  点击“单步进入”按钮，进入 `main` 函数。
2.  在 `height = int(input("Height: "))` 这一行，点击“单步跳过”。程序会在终端等待我们输入。输入 `3` 并回车后，在调试器的“变量”面板中，可以看到变量 `height` 被创建，其值为 `3`。
3.  接下来是 `pyramid(height)` 调用。由于 `pyramid` 是我们自己写的函数，我们点击“单步进入”，进入其内部。
4.  现在位于 `for i in range(n):` 这一行。点击“单步跳过”执行它。在变量面板中，可以看到局部变量 `i` 被创建，其初始值为 `0`。
5.  点击“单步跳过”执行 `print("#" * i)`。观察终端输出，会发现光标下移了一行，但什么也没打印。这直观地揭示了错误原因：当 `i` 为 `0` 时，打印了空字符串。

通过调试器，我们无需修改代码就能清晰地观察到程序每一步的执行状态和变量变化，从而精准定位问题。

## 总结

本节课我们一起学习了两种调试代码的方法。

1.  **`print` 调试法**：通过临时插入打印语句来输出关键信息，简单快捷，适合快速排查简单问题。
2.  **调试器调试法**：利用 IDE 内置的调试工具，可以设置断点、逐行执行、实时查看变量，是一种更强大、更系统的调试方式，尤其适用于复杂的程序或他人编写的代码。
