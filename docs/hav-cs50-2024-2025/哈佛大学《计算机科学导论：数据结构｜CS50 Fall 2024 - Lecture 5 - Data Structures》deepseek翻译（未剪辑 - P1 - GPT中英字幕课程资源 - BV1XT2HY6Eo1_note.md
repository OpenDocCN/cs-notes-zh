## 队列与栈：抽象数据类型

![](img/856d59aba1fd5b6138475965475677ef_7.png)

![](img/856d59aba1fd5b6138475965475677ef_9.png)

队列，或称队伍，在现实世界中很常见，例如在电影院或餐厅排队。在计算机世界中，打印任务通常也存储在队列中，以确保按顺序打印。队列遵循**先进先出**的原则。

以下是队列的基本操作：
*   **入队**：将元素添加到队列末尾。
*   **出队**：从队列前端移除元素。

栈是另一种抽象数据类型。想象一下叠放的衣服或餐厅里叠放的餐盘。新邮件通常也出现在收件箱顶部。栈遵循**后进先出**的原则。

以下是栈的基本操作：
*   **压入**：将元素添加到栈顶。
*   **弹出**：从栈顶移除元素。

队列和栈是抽象数据类型，描述了数据如何进出，但不规定具体的实现方式。你可以用数组或链表等数据结构来实现它们。

## 数组的局限性

上一节我们了解了抽象数据类型。本节中，我们来看看实现它们的基础数据结构，首先从数组开始。

数组是一种简单的数据结构，要求所有元素在内存中**连续**存储。你必须在创建时就决定数组的大小。

![](img/856d59aba1fd5b6138475965475677ef_35.png)

```c
int list[3]; // 创建一个大小为3的整数数组
```

![](img/856d59aba1fd5b6138475965475677ef_37.png)

![](img/856d59aba1fd5b6138475965475677ef_39.png)

数组的局限性在于，如果你之后需要存储更多数据，可能会“把自己逼入死角”。例如，如果你有一个大小为3的数组 `[1, 2, 3]`，想添加数字4，但紧邻的内存位置可能已被其他数据占用。

解决这个问题的一种方法是分配一个更大的新数组，将旧数据复制过去，然后添加新元素。但这需要**线性时间**来复制数据。

![](img/856d59aba1fd5b6138475965475677ef_41.png)

另一种方法是预先分配远超所需的内存，但这会**浪费空间**。

这里存在一个典型的权衡：时间、空间和代码复杂度。

## 链表：动态数据结构

![](img/856d59aba1fd5b6138475965475677ef_43.png)

![](img/856d59aba1fd5b6138475965475677ef_44.png)

上一节我们看到了数组在动态增长时的局限。本节中，我们来看看链表，它提供了一种更灵活的解决方案。

![](img/856d59aba1fd5b6138475965475677ef_46.png)

![](img/856d59aba1fd5b6138475965475677ef_47.png)

链表中的元素（称为节点）可以存储在内存的任何位置。每个节点不仅包含数据，还包含一个指向下一个节点的**指针**。这样，节点就像被“缝合”在一起。

```c
typedef struct node
{
    int number;
    struct node *next;
}
node;
```

链表由一个指向第一个节点（称为头节点）的指针来跟踪。

**优点**：链表可以动态增长和缩小，无需移动现有数据，能更有效地利用零散的内存。
**缺点**：每个节点需要额外的空间来存储指针。搜索链表需要**线性时间**，因为你必须从头开始遍历。

## 实现链表

![](img/856d59aba1fd5b6138475965475677ef_51.png)

![](img/856d59aba1fd5b6138475965475677ef_53.png)

![](img/856d59aba1fd5b6138475965475677ef_54.png)

上一节我们了解了链表的概念。本节中，我们来看看如何在C语言中实现它。

首先，我们定义一个节点结构体。然后，我们使用 `malloc` 动态分配内存来创建新节点，并设置其数据和指针字段。

以下是将新节点插入链表前端的代码示例（前插法）：
```c
node *n = malloc(sizeof(node));
if (n == NULL)
{
    return 1;
}
n->number = 2; // 假设获取的值为2
n->next = list; // 新节点指向原链表头
list = n; // 更新链表头为新节点
```

![](img/856d59aba1fd5b6138475965475677ef_56.png)

要遍历链表并打印所有值，可以使用循环：
```c
for (node *ptr = list; ptr != NULL; ptr = ptr->next)
{
    printf("%i\n", ptr->number);
}
```

最后，使用完链表后，必须遍历并 `free` 所有节点，以避免内存泄漏。

![](img/856d59aba1fd5b6138475965475677ef_58.png)

## 二叉搜索树

![](img/856d59aba1fd5b6138475965475677ef_60.png)

![](img/856d59aba1fd5b6138475965475677ef_62.png)

上一节我们实现了线性链表。本节中，我们来看看如何构建二维数据结构：**二叉搜索树**。

二叉搜索树是一种节点结构，其中每个节点包含：
*   一个值。
*   一个指向**左子树**的指针（该子树所有值小于当前节点）。
*   一个指向**右子树**的指针（该子树所有值大于当前节点）。

![](img/856d59aba1fd5b6138475965475677ef_64.png)

这种结构允许我们进行**二分查找**。从根节点开始，比较目标值，根据比较结果进入左子树或右子树，从而在 **O(log n)** 时间内完成搜索。

![](img/856d59aba1fd5b6138475965475677ef_66.png)

然而，如果插入的数据顺序不当（例如按升序插入），树可能会退化成一条链，搜索时间会恶化到 **O(n)**。因此，需要额外的算法（如旋转）来保持树的平衡。

## 字典与哈希表

上一节我们讨论了树。本节中，我们来看看另一种非常重要的抽象数据类型：**字典**。

字典存储**键值对**，例如（姓名，电话号码）。我们的目标是实现快速查找。

一种实现方式是**哈希表**。哈希表的核心是一个数组，数组的每个位置称为一个“桶”。我们使用一个**哈希函数**将键（如姓名）映射到数组的某个索引。

例如，一个简单的哈希函数可以取姓名的首字母，将其转换为0到25之间的数字。

![](img/856d59aba1fd5b6138475965475677ef_68.png)

```c
unsigned int hash(const char *word)
{
    return toupper(word[0]) - 'A';
}
```

![](img/856d59aba1fd5b6138475965475677ef_70.png)

**碰撞**发生在两个不同的键被哈希到同一个桶时。解决碰撞的常见方法是在每个桶位置存储一个**链表**，将所有哈希到该位置的键值对链接起来。

在理想情况下，哈希表能提供接近**常数时间**的查找。但在最坏情况下（所有键都碰撞），它会退化为 **O(n)**。

## 字典树

![](img/856d59aba1fd5b6138475965475677ef_76.png)

![](img/856d59aba1fd5b6138475965475677ef_78.png)

上一节我们看到了哈希表如何接近常数时间。本节中，我们来看看一种理论上能实现真正常数时间查找的数据结构：**字典树**。

![](img/856d59aba1fd5b6138475965475677ef_80.png)

字典树是一种树形结构，其中每个节点包含一个指针数组（例如，对应26个字母）。要存储一个键（如姓名），你沿着字母对应的指针路径向下，在键的末尾节点存储值。

查找时间仅取决于**键的长度**，而与字典树中存储的键总数无关。因此，查找时间是 **O(k)**，其中k是键的长度。由于键长通常有上限，这可以被视为常数时间 **O(1)**。

![](img/856d59aba1fd5b6138475965475677ef_82.png)

![](img/856d59aba1fd5b6138475965475677ef_83.png)

![](img/856d59aba1fd5b6138475965475677ef_85.png)

**代价**：字典树可能消耗大量内存，因为即使许多指针路径从未被使用，也需要为它们分配空间。
