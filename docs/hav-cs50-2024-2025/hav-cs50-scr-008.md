# 008：抽象化 🧩

![](img/357452e78b26a3dd6b24618bb17f43cc_0.png)

在本节课中，我们将学习如何通过创建自定义代码块来改进Scratch项目的设计。这种方法被称为“抽象化”，它能减少代码重复，使程序更易于阅读和维护。

---

## 回顾与引入

上一节我们探讨了Scratch中的循环、变量和条件等核心功能。本节中，我们将看看如何通过“抽象化”来优化代码的组织方式。

## 一个简单的游戏示例

让我们从一个简单的游戏项目开始。这个游戏的目标是控制一只恐龙去触碰一颗星星。

### 控制恐龙移动

首先，我们添加恐龙角色，并编写代码使其响应键盘的上下左右箭头键。

```scratch
当 [上移键] 被按下
y坐标增加 (10)

当 [下移键] 被按下
y坐标增加 (-10)

当 [右移键] 被按下
x坐标增加 (10)

当 [左移键] 被按下
x坐标增加 (-10)
```

### 添加胜利条件

接下来，我们添加星星角色，并希望当恐龙触碰到星星时，游戏报告所用时间。

以下是实现这一逻辑的初始方法：在每个移动事件后，都检查是否触碰到星星。

```scratch
当 [上移键] 被按下
y坐标增加 (10)
如果 <碰到 [星星 v] ?> 那么
  说 (计时器) (2) 秒
结束
```

然而，为了确保无论按哪个方向键都能检查胜利条件，我们必须在**所有四个**移动脚本中都复制这段检查代码。这导致了代码重复。

## 问题：代码重复

当您发现自己在项目中多次复制粘贴相同的代码块时，这通常意味着存在更好的解决方案。在我们的例子中，修改功能（例如将计时器结果四舍五入）需要在四个地方进行相同的更改，既繁琐又容易出错。

## 解决方案：创建自定义代码块

Scratch的“我的代码块”功能允许我们创建自定义的代码块，将一系列操作封装成一个有意义的名称。

### 第一步：定义“检查胜利”块

1.  点击“我的代码块”类别。
2.  点击“新建代码块”。
3.  将代码块命名为 `检查胜利`。
4.  在 `定义 检查胜利` 下方，放置我们重复的检查逻辑。

```scratch
定义 检查胜利
如果 <碰到 [星星 v] ?> 那么
  说 (计时器) (2) 秒
结束
```

### 第二步：使用自定义块

现在，我们可以用这个单一的 `检查胜利` 块替换每个移动脚本中冗长的检查代码。

```scratch
当 [上移键] 被按下
y坐标增加 (10)
检查胜利
```

我们对其他三个方向键也进行同样的替换。现在，每个移动脚本都变得简洁明了。

### 第三步：轻松修改功能

如果需要修改胜利时的行为（例如将时间四舍五入），我们只需在 `定义 检查胜利` 中修改一次即可。

```scratch
定义 检查胜利
如果 <碰到 [星星 v] ?> 那么
  说 (四舍五入 (计时器)) (2) 秒
结束
```

这一改动会自动应用到所有使用 `检查胜利` 块的地方，极大地提高了代码的可维护性。

## 另一个示例：给气球充气和放气

让我们通过另一个例子来巩固这个概念。我们编写一个让气球先充气、等待、再放气的脚本。

初始代码可能如下：

```scratch
当 ⚑ 被点击
将大小设为 (50) %
重复 (10) 次
  将大小增加 (10)
结束
等待 (1) 秒
重复 (10) 次
  将大小增加 (-10)
结束
```

虽然这段代码能工作，但阅读者需要仔细计算循环次数才能理解其意图。我们可以通过创建有意义的代码块来使其更清晰。

### 创建“充气”和“放气”块

我们创建两个自定义块：

```scratch
定义 充气
重复 (10) 次
  将大小增加 (10)
结束

定义 放气
重复 (10) 次
  将大小增加 (-10)
结束
```

然后，主脚本可以重写为：

```scratch
当 ⚑ 被点击
将大小设为 (50) %
充气
等待 (1) 秒
放气
```

现在，代码的意图一目了然：设置初始大小，充气，等待，然后放气。这大大提升了代码的可读性。

## 进阶：为自定义块添加输入参数

自定义块可以变得更加灵活，它们能够接受输入参数，从而根据不同的输入值改变行为。

### 让“充气/放气”块可定制

我们可以编辑 `充气` 块，为其添加一个名为 `次数` 的输入参数。

1.  右键点击 `充气` 块，选择“编辑”。
2.  点击“添加输入（数字或文本）”。
3.  将输入命名为 `次数`，并添加标签文字，例如“充气 次数 次”。
4.  在定义中，使用这个输入值来控制循环次数。

```scratch
定义 充气 (次数) 次
重复 (次数) 次
  将大小增加 (10)
结束
```

对 `放气` 块进行同样的操作。现在，我们可以这样使用它们：

```scratch
当 ⚑ 被点击
将大小设为 (50) %
充气 (20) 次 // 充气更多
等待 (1) 秒
放气 (20) 次
```

我们甚至可以在其他事件中使用带不同参数的块：

```scratch
当角色被点击
充气 (2) 次 // 每次点击只充气一点点
```

### 包含多个输入参数的例子

自定义块可以接受多个输入。例如，我们可以为小熊创建一个“走路”块，指定走多少步以及用几只脚走路（对应不同的造型）。

1.  新建一个代码块，命名为 `走路`。
2.  添加第一个数字输入 `步数`，标签为“走路 步数 步”。
3.  添加第二个数字输入 `脚数`，标签为“用 脚数 只脚”。
4.  在定义中，使用这些输入。

```scratch
定义 走路 (步数) 步 用 (脚数) 只脚
换成 (脚数) 造型 // 假设造型名称为数字“2”或“4”
重复 (步数) 次
  移动 (1) 步
结束
```

然后，我们可以这样编写清晰的主脚本：

```scratch
当 ⚑ 被点击
走路 (30) 步 用 (4) 只脚
走路 (30) 步 用 (2) 只脚
```

## 总结

本节课中我们一起学习了Scratch中“抽象化”的强大功能。通过创建自定义代码块，我们可以：
*   **减少代码重复**：将常用操作封装成块，避免复制粘贴。
*   **提高可读性**：使用有意义的块名（如`充气`、`检查胜利`）使代码逻辑更清晰。
*   **增强可维护性**：修改功能时只需在一个地方（块定义处）进行，改动会自动全局生效。
*   **增加灵活性**：通过为块添加输入参数，可以创建出能适应不同场景的可重用代码。

抽象化是编程中的核心思想之一，它能帮助你构建更复杂、组织更良好且更易于理解的Scratch项目。
