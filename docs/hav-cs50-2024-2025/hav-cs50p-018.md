## 概述
在本节课中，我们将要学习Python函数中一个重要的概念：**副作用**。我们将了解什么是副作用，它与函数的返回值有何不同，并通过具体的代码示例来探索两种常见的副作用类型。

## 什么是副作用？
上一节我们介绍了函数的基本概念，即接收输入并产生输出。本节中我们来看看函数的另一种输出形式——副作用。

函数的**返回值**是函数执行完毕后明确返回给调用者的数据。而**副作用**则是指函数在运行过程中，除了返回值之外，对程序状态造成的任何其他改变。例如，在屏幕上打印内容或修改一个全局变量的值，都属于副作用。

## 第一个副作用示例：打印输出
让我们通过一个具体的程序来理解副作用。以下是一个名为 `machine.py` 的程序，它模拟了一个有情绪状态的机器。

```python
emoticon = ":("

def main():
    say("Is anyone there?")
    say("Oh hi")

def say(phrase):
    print(phrase + " " + emoticon)

if __name__ == "__main__":
    main()
```

在这个程序中，我们定义了一个函数 `say`。它的作用是打印一句话，并在后面附上代表机器情绪的 `emoticon`。

以下是 `say` 函数的关键点：
*   `phrase` 是一个局部变量，作为输入传递给函数。
*   `emoticon` 是一个在文件顶部定义的**全局变量**，因此可以在 `say` 函数内部被访问。
*   函数 `say` 的主要行为是使用 `print` 函数将内容输出到终端。

运行这个程序，你会在终端看到两行输出，但表情都是悲伤的 `:(`。这里的 `print` 语句就是函数 `say` 产生的一个**副作用**——它改变了终端显示的内容，而不是返回一个值。

## 第二个副作用示例：修改全局变量
现在，我们尝试在 `main` 函数中改变机器的情绪，让它从悲伤变得快乐。我们希望在说“Oh hi”时使用笑脸 `:D`。

直觉上，我们可能会在 `main` 函数中直接修改 `emoticon` 变量：

```python
def main():
    say("Is anyone there?")
    emoticon = ":D"  # 尝试修改全局变量
    say("Oh hi")
```

然而，运行这段代码后，你会发现表情并没有改变。这是因为在函数内部，Python默认将赋值语句左侧的变量名视为**局部变量**。我们无法直接修改函数外部的全局变量。

为了在函数内部修改全局变量，我们需要使用 `global` 关键字进行显式声明。

以下是修正后的代码：

```python
emoticon = ":("

def main():
    global emoticon  # 声明我们将要修改全局变量 emoticon
    say("Is anyone there?")
    emoticon = ":D"  # 现在可以成功修改全局变量
    say("Oh hi")

def say(phrase):
    print(phrase + " " + emoticon)

if __name__ == "__main__":
    main()
```

运行修正后的程序，输出将变为：
```
Is anyone there? :(
Oh hi :D
```

在 `main` 函数中使用 `global emoticon` 声明后，对 `emoticon` 的赋值操作就变成了修改全局变量。这个修改操作本身是 `main` 函数运行期间产生的另一个**副作用**。

## 关于副作用的注意事项
虽然副作用功能强大，但在编程中需要谨慎使用。

*   **调试难度**：当多个函数都能修改同一个全局变量时，程序的状态会变得难以追踪，增加调试的复杂性。
*   **设计决策**：在编写程序时，需要仔细考虑使用副作用是否是良好的设计选择。通常，应优先考虑使用明确的返回值来传递信息，仅在必要时（如记录日志、更新用户界面等）使用副作用。

除了打印输出和修改全局变量，副作用还包括其他类型，例如读写文件、发送网络请求等，这些我们将在后续课程中探讨。

## 总结
本节课中我们一起学习了Python函数的**副作用**。

我们明确了：
*   副作用是函数运行时对程序状态（非返回值）的改变。
*   我们通过代码实践了两种常见的副作用：
    1.  使用 `print` 函数向终端输出内容。
    2.  使用 `global` 关键字声明并修改函数外部的全局变量。
*   最后，我们讨论了过度使用副作用（尤其是修改全局变量）可能带来的代码可维护性问题。
