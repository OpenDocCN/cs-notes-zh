# 入门 92：MySQL中的约束 🛡️

在本节课中，我们将要学习MySQL中的约束概念。约束是用于确保数据库表中数据有效性和完整性的规则。我们将通过小柠檬餐厅的案例，了解约束的主要类型、功能以及`ON DELETE CASCADE`和`ON UPDATE CASCADE`选项的作用。

## 什么是约束？🔍

当创建表时，你可能需要为某些列设定规则。例如，你可能要求表中的每一行在某个列（如电话号码）上都必须持有唯一的值。

![](img/442417eae3c2fd95396f01081fe8caf6_1.png)

你可以通过使用**唯一约束**来强制执行此规则。该约束能防止在数据库中插入或更新数据时违反此规则。

![](img/442417eae3c2fd95396f01081fe8caf6_3.png)

![](img/442417eae3c2fd95396f01081fe8caf6_4.png)

## MySQL约束的主要类型 📊

MySQL数据库中有三种主要的约束类型，用于强制执行数据规则：

1.  **键约束**：应用于键类型的规则。
2.  **域约束**：用于管理特定列可以存储的值的规则。
3.  **参照完整性约束**：为参照键（外键）建立的规则。

![](img/442417eae3c2fd95396f01081fe8caf6_6.png)

![](img/442417eae3c2fd95396f01081fe8caf6_7.png)

接下来，我们将逐一探讨这三种MySQL约束类型，并了解小柠檬餐厅如何在他们的数据库中使用它们。

![](img/442417eae3c2fd95396f01081fe8caf6_8.png)

![](img/442417eae3c2fd95396f01081fe8caf6_10.png)

## 键约束 🔑

![](img/442417eae3c2fd95396f01081fe8caf6_11.png)

在之前的课程中我们了解到，所有表都包含不同类型的键，例如主键和外键。

![](img/442417eae3c2fd95396f01081fe8caf6_13.png)

![](img/442417eae3c2fd95396f01081fe8caf6_14.png)

你可以使用约束为这些键建立规则。例如，**主键约束**可用于指定一个或多个列的值必须始终唯一，并且不能接受NULL值。

![](img/442417eae3c2fd95396f01081fe8caf6_16.png)

小柠檬餐厅的数据库中有一个名为`Customers`的表。该表使用主键约束来记录客户预订的关键数据。该表有三列：`CustomerID`、`FullName`和`PhoneNumber`。

`CustomerID`被定义为主键，它标识表中唯一的记录。由于主键约束，该列的值必须始终唯一，并且绝不能接受NULL值。换句话说，该列中的每一行都必须有一个客户ID，并且所有客户ID都必须是唯一的。

![](img/442417eae3c2fd95396f01081fe8caf6_18.png)

## 域约束 📏

正如前面提到的，域约束是为特定列可以存储的值定义的特殊规则。

小柠檬餐厅的数据库包含一个`Bookings`表，用于记录客户预订数据。然而，餐厅每次预订最多只能容纳8位客人，因此他们在`NumberOfGuests`列上实施了SQL的**检查约束**。

![](img/442417eae3c2fd95396f01081fe8caf6_20.png)

![](img/442417eae3c2fd95396f01081fe8caf6_21.png)

![](img/442417eae3c2fd95396f01081fe8caf6_22.png)

这限制了可以放入该列的值范围，意味着该表会拒绝任何大于8的数值。

![](img/442417eae3c2fd95396f01081fe8caf6_24.png)

## 参照完整性约束 🔗

前面提到，这种类型的约束为参照键（外键）建立规则，但它具体是如何工作的呢？

![](img/442417eae3c2fd95396f01081fe8caf6_26.png)

基本上，在参照完整性约束中，涉及两种类型的表：一个包含主键的**参照表**和一个包含外键的**被参照表**。

![](img/442417eae3c2fd95396f01081fe8caf6_28.png)

![](img/442417eae3c2fd95396f01081fe8caf6_29.png)

参照表中存在的外键列的值，必须始终存在于被参照表中。否则，两个表之间就无法建立连接。

为了更好地理解这一点，让我们通过实体关系图来探索小柠檬餐厅数据库中相关表的例子。

![](img/442417eae3c2fd95396f01081fe8caf6_31.png)

小柠檬餐厅的数据库包含两个相关的表：存储客户数据的`Customers`表和记录客户餐厅预订信息的`Bookings`表。

![](img/442417eae3c2fd95396f01081fe8caf6_32.png)

`Bookings`表中的每条预订都必须关联到`Customers`表中的特定客户，否则餐厅无法确定是谁进行了预订。这也意味着，每个客户必须先注册在`Customers`表中，才能在`Bookings`表中进行预订。

![](img/442417eae3c2fd95396f01081fe8caf6_34.png)

`Bookings`表中的`CustomerID`列被定义为**外键**。这是将两个表连接在一起并在它们之间建立依赖关系的属性。

![](img/442417eae3c2fd95396f01081fe8caf6_35.png)

![](img/442417eae3c2fd95396f01081fe8caf6_37.png)

这意味着，如果在`Customers`表中修改或删除一行数据，那么这个操作会破坏`Bookings`表中相关的数据行。

![](img/442417eae3c2fd95396f01081fe8caf6_39.png)

换句话说，从`Customers`表中删除一行数据会违反参照完整性规则，这将导致MySQL返回错误消息，警告该操作会直接影响`Bookings`表。

![](img/442417eae3c2fd95396f01081fe8caf6_41.png)

![](img/442417eae3c2fd95396f01081fe8caf6_42.png)

## 使用CASCADE选项 🔄

那么，如何在不违反参照完整性约束的情况下对`Bookings`表进行必要的更改呢？

![](img/442417eae3c2fd95396f01081fe8caf6_44.png)

![](img/442417eae3c2fd95396f01081fe8caf6_45.png)

在这种情况下，你可以使用`ON DELETE CASCADE`选项。此选项会自动从`Bookings`表中删除相关的数据行。

如果你想更新`Customers`表中的主键值，可以使用`ON UPDATE CASCADE`选项来自动更新`Bookings`表中的相关行。

![](img/442417eae3c2fd95396f01081fe8caf6_47.png)
