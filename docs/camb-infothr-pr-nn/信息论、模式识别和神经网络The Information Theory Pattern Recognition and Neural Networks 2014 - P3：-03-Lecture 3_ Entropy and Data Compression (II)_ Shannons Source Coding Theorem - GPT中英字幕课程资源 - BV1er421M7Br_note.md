## 案例研究：游戏“63”

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_6.png)

为了进一步检验香农信息量的合理性，我们来看一个简单的游戏：“63”。在这个游戏中，我从0到63中选一个数字，你通过提问我只能回答“是”或“否”的问题来猜出这个数字。

一个高效的策略是每次都将可能的结果集对半分。例如，第一个问题可以是：“数字是否大于等于32？”根据回答，你可以将搜索范围减半，并继续类似地提问。

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_8.png)

以下是提问序列的一个示例：
1.  数字 ≥ 32？ (回答：是)
2.  数字 mod 32 ≥ 16？ (回答：否)
3.  数字 mod 16 ≥ 8？ (回答：是)
4.  数字 mod 8 ≥ 4？ (回答：否)
5.  数字 mod 4 ≥ 2？ (回答：是)
6.  数字 mod 2 ≥ 1？ (回答：否)

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_10.png)

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_12.png)

最终，回答序列“是，否，是，否，是，否”对应二进制数 `101010`，即十进制42。

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_14.png)

**关键观察**：
*   要唯一确定一个来自大小为 `S` 的集合中的结果，在最坏情况下需要 **`ceil(log₂ S)`** 个二进制问题（比特）。
*   在上述游戏中，`S = 64`，所以需要 `log₂ 64 = 6` 比特。
*   每个问题的答案（是/否）概率各为1/2，因此每个答案的香农信息量恰好是 `log₂(1 / 0.5) = 1` 比特。六个答案总共提供了6比特信息，恰好编码了结果。
*   这个策略之所以最优，是因为每次对半分的提问能最大化**期望信息增益**（即熵）。如果提问不能将概率均分（例如问“是质数吗？”），则期望信息量会减少，可能导致需要更多提问。

这个简单的游戏表明，香农信息量与用二进制问题识别结果所需的最小比特数是一致的。

## 案例研究：游戏“潜艇”

接下来，我们考虑一个概率不均等的场景：“潜艇”游戏。一个潜艇隐藏在64个方格中的某一个。你每次选择一个方格攻击，并得到“命中”或“未命中”的答复。

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_16.png)

假设我们采用一个非常低效的策略：从第一个方格开始，按顺序攻击每个方格，直到命中潜艇。

**关键观察**：
*   攻击一个方格并得到“未命中”的概率很高（例如第一次是63/64），其香农信息量 `log₂(64/63) ≈ 0.0227` 比特，非常小。
*   随着攻击继续，每次“未命中”的信息量会缓慢增加（例如 `log₂(63/62) ≈ 0.0230` 比特）。
*   当你最终命中潜艇时，这个稀有事件的概率很低（例如1/剩余格子数），其香农信息量非常大（例如 `log₂(32/1) = 5` 比特）。
*   **神奇的是**，无论潜艇藏在哪，将所有“未命中”和最终“命中”的香农信息量相加，其总和总是恰好等于 **6 比特**（`log₂ 64`）。

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_18.png)

这个结果并非巧合。它强有力地表明，尽管单个事件的香农信息量看起来可能不合理（一个常见的“未命中”只带来极少信息，一个罕见的“命中”带来大量信息），但它们的总和始终精确地等于唯一确定结果所需的总信息量（6比特）。这进一步支持了 `log(1/p)` 作为信息度量单位的合理性。

## 本特币彩票问题与典型集

现在，我们回到之前提出的“本特币彩票”问题，它将引导我们理解信源编码定理。

**问题设定**：
*   一枚偏置硬币，正面（1）概率 `f = 0.1`，反面（0）概率 `0.9`。
*   抛掷该硬币 `N = 1000` 次，生成一个长度为1000的0/1序列作为“中奖号码”。
*   你可以购买彩票（即特定的0/1序列），每张价格1英镑。
*   黑帮老大要求你：**用最少的钱购买一组彩票，确保你有至少99%的概率持有中奖彩票**。

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_20.png)

**问题分解**：
1.  **如果只买一张票，买哪张？**
    应购买**最可能出现的序列**，即全部为0的序列 `000...0`。虽然它在单次实验中不太可能出现，但比其他任何特定序列的概率都高。

2.  **要保证99%的中奖概率，应该买哪些票？**
    直觉是购买所有“看起来典型”的序列。对于这枚偏置硬币，典型的序列中“1”的个数大约在期望值 `N*f = 100` 附近。
    通过计算可知，“1”的个数 `R` 近似服从二项分布，其标准差为 `σ = sqrt(N*f*(1-f)) ≈ sqrt(90) ≈ 9.5`。
    为了覆盖99%的概率，我们需要购买所有 `R` 不超过某个上限 `R_max` 的序列。`R_max` 大约为 `100 + 2.3σ ≈ 122`（2.3个标准差以满足99%置信度）。
    因此，策略是：**购买所有包含0到122个“1”的所有可能序列**。

3.  **需要买多少张票？**
    需要购买的彩票总数 `M` 是二项式系数之和：
    **`M = Σ_{r=0}^{R_max} C(N, r)`**，其中 `C(N, r)` 是组合数。
    当 `N` 很大时，这个求和式中最大的项是最后一项 `C(1000, 122)`。利用斯特林公式近似，可以证明：
    **`log₂ M ≈ N * H₂(f) + O(√N)`**
    其中 `H₂(f)` 是二元熵函数：**`H₂(f) = -f log₂ f - (1-f) log₂ (1-f)`**。
    代入 `f=0.1`，`H₂(0.1) ≈ 0.469`。因此，`log₂ M ≈ 1000 * 0.469 + 小量 ≈ 469 + 小量`。
    实际上，`C(1000, 122)` 约等于 `2^{530}`，所以 `M ≈ 2^{530}`，即大约需要购买 `2^{530}` 张彩票。

## 香农信源编码定理的“证明”

![](img/07ac39b7b8f6edf52bdc3e8aa08d1aeb_22.png)

上述彩票问题的构造，实际上隐含地**证明**了香农信源编码定理的核心思想。

**压缩思路**：
1.  **构建“典型集”**：我们购买的这 `M ≈ 2^{N*H}` 张彩票，实际上定义了信源的**典型集**。对于这个偏置硬币源，当 `N` 很大时，产生的序列几乎总是（以超过99%的概率）属于这个集合。
2.  **建立压缩映射**：由于典型集中只有 `M` 个序列，我们可以为每个序列分配一个唯一的、较短的二进制名称（编号）。因为共有 `M` 个序列，所以每个短名称的长度 `L` 只需满足 `2^L ≥ M`，即 **`L ≈ log₂ M ≈ N * H₂(f)`**。
3.  **实现压缩与解压**：
    *   **压缩器**：当收到一个长的 `N` 比特序列时，检查它是否在典型集中（即“1”的个数是否 ≤122）。如果在，则输出其对应的短名称（`L ≈ 530` 比特）；如果不在（极小概率事件），可以约定输出一个错误标志或原序列。
    *   **解压器**：收到短名称（`L` 比特）后，根据映射表查找对应的原始 `N` 比特序列并输出。

**定理陈述**：
对于一个独立同分布的信源，当序列长度 `N` 足够大时，信源输出的序列可以以任意高的概率被压缩成长度约为 **`N * H`** 比特的码字，其中 `H` 是信源的熵；反之，如果压缩后的平均长度小于 `N * H`，则必然会导致信息丢失或解码错误。

在我们的例子中：
*   原始序列长度：`N = 1000` 比特。
*   压缩后长度：`L ≈ N * H₂(0.1) ≈ 469` 比特（理论极限），我们实现了约530比特。
*   压缩效率：约 `(530/1000) ≈ 53%`，接近熵极限 `46.9%`。随着 `N` 增大，`O(√N)` 项的影响相对变小，压缩率将无限接近 `H`。

这个“证明”虽然针对特定信源，但阐明了通用证明的关键：**定义典型集、计算其大小、并利用其概率接近1的特性**。严格的通用证明详见教材。

## 总结与展望

本节课中，我们一起学习了：
1.  通过“63”和“潜艇”游戏，我们获得了更多支持**香农信息量**作为基本信息度量单位的直观证据。
2.  通过深入分析“**本特币彩票问题**”，我们引出了**典型集**的概念。
3.  基于典型集，我们阐释并初步“证明”了**香农信源编码定理**：一个熵为 `H` 的信源，其长度为 `N` 的输出序列，可以以高概率压缩至约 `N*H` 比特，且这是压缩的极限。

然而，我们构建的压缩器（存储所有典型序列的映射表）是**不切实际**的，因为典型集的大小 `2^{N*H}` 随 `N` 指数增长，即使对于中等大小的 `N`，这个表也会大到无法存储。

因此，下一个核心问题是：**如何设计出实用、高效的压缩算法，使其性能接近香农极限？** 这将是后续课程要探讨的内容。
