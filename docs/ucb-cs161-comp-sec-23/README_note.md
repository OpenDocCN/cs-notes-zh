# UCB CS161 计算机安全：1：课程概述与内存安全基础

在本节课中，我们将要学习计算机安全的基本概念，并重点探讨内存安全这一核心议题。内存安全问题是许多安全漏洞的根源，理解它是构建安全系统的第一步。

## 什么是计算机安全？

计算机安全的目标是保护计算机系统及其数据的**机密性**、**完整性**和**可用性**。这通常被称为**CIA三要素**。

*   **机密性**：确保信息不被未授权的个人或系统访问。
*   **完整性**：确保信息在存储或传输过程中不被未授权地篡改。
*   **可用性**：确保授权用户在需要时可以访问信息和资源。

## 内存安全的重要性

上一节我们介绍了安全的基本目标，本节中我们来看看实现这些目标时面临的一个根本挑战：内存安全。内存安全漏洞是攻击者最常利用的弱点之一，可能导致程序崩溃、数据泄露或被完全控制。

内存安全的核心问题是程序错误地访问了不应访问的内存区域。这通常源于编程语言（如C/C++）赋予程序员直接操作内存的强大能力，但同时也要求程序员承担正确管理内存的全部责任。

以下是几种常见的内存安全漏洞类型：

*   **缓冲区溢出**：当程序向一个缓冲区（如数组）写入的数据量超过了其预分配的大小时，多余的数据就会“溢出”到相邻的内存区域，可能覆盖重要的数据或代码。
*   **释放后使用**：程序释放了一块内存（如通过`free()`函数），但后续又错误地继续使用指向该内存的指针。
*   **双重释放**：程序对同一块内存进行了两次释放操作。
*   **空指针解引用**：程序试图通过一个值为`NULL`（空）的指针来访问内存。

## 一个简单的缓冲区溢出示例

为了更好地理解，让我们看一段存在缓冲区溢出风险的C语言代码。

```c
#include <string.h>

void vulnerable_function(char *input) {
    char buffer[16];
    strcpy(buffer, input); // 危险！没有检查输入长度
}

int main() {
    char large_input[256];
    memset(large_input, 'A', 255);
    large_input[255] = '\0';
    vulnerable_function(large_input);
    return 0;
}
```

在这段代码中，`vulnerable_function`定义了一个大小为16字节的字符数组`buffer`。`strcpy`函数会将`input`参数的内容复制到`buffer`中，但它不会检查`input`的长度。如果`input`的长度超过15个字符（加上结尾的空字符`\0`），就会发生缓冲区溢出，覆盖`buffer`之后的内存。

## 攻击者如何利用内存错误？

攻击者利用这些内存错误，精心构造输入数据（通常称为**攻击载荷**或**Exploit**），以达到其目的。例如，在缓冲区溢出中，攻击者可能：

1.  用特定的机器指令地址覆盖函数的返回地址。
2.  当函数执行完毕试图返回时，程序就会跳转到攻击者指定的地址去执行。
3.  该地址可能指向攻击者提前植入在内存中的恶意代码（**Shellcode**）。

这个过程可以用一个简单的公式来描述攻击的核心：**控制流劫持 = 覆盖关键数据 + 指向恶意代码**。

## 如何防御内存安全问题？

防御内存安全问题需要从多个层面入手。

以下是几种主要的防御策略：

*   **使用内存安全的语言**：采用如Java、Python、Rust、Go等语言，它们通过自动内存管理（垃圾回收）或所有权模型，在语言层面消除了大部分内存错误。
*   **启用编译器和操作系统保护机制**：
    *   **栈保护符 / Canaries**：编译器在栈上的返回地址前放置一个随机值（canary），函数返回前检查该值是否被改变。
    *   **数据执行保护 / DEP**：标记内存页为不可执行，防止攻击者运行注入到数据区的代码。
    *   **地址空间布局随机化 / ASLR**：随机化程序关键数据在内存中的地址，增加攻击者猜测地址的难度。
*   **安全的编程实践**：在使用C/C++等语言时，始终使用长度受限的安全函数（如`strncpy`替代`strcpy`），并进行严格的边界检查。
