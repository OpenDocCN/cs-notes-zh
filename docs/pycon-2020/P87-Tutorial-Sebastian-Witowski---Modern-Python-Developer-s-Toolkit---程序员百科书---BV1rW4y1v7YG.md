如果你有一个命令来使用 app get 安装一些包，而你没有至少修改它，Docker 将重用缓存层，而不是重新安装所有包。一旦 Docker 到达一个无法使用缓存的步骤，它将重新运行此步骤及后续的每个命令。因此，了解缓存的工作原理以及如何利用它是非常重要的。
