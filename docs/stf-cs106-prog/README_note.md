# 斯坦福 CS106A/B/L/X 编程入门笔记：第1课 - 编程基础与Karel机器人 🧑‍💻

在本节课中，我们将要学习编程的基本概念，并通过一个名为Karel的虚拟机器人来实践这些思想。Karel是一个简单的环境，它帮助我们理解如何通过编写指令来控制一个“机器人”完成任务，而无需一开始就陷入复杂的语法细节。

## 什么是编程？🤔

上一节我们介绍了本课程的目标，本节中我们来看看编程的本质。编程是向计算机提供一系列精确指令的过程。这些指令被称为**代码**，它们告诉计算机执行什么操作。你可以把编程看作是给一个非常听话但理解能力有限的伙伴下达命令。

## 认识Karel机器人 🤖

Karel是一个存在于网格世界中的机器人。它的世界由水平的街道（行）和垂直的大道（列）组成。Karel只能执行几个基本操作，这让我们可以专注于解决问题的逻辑。

以下是Karel能够理解的核心命令：

*   `move();` - Karel向前移动一格。
*   `turnLeft();` - Karel向左转90度。
*   `pickBeeper();` - Karel拾起脚下的一颗“哔哔”声元件。
*   `putBeeper();` - Karel在脚下放置一颗“哔哔”声元件。

## 编写你的第一个程序 💻

现在，让我们尝试用Karel的命令来编写一个简单程序。假设Karel需要移动到一个角落并拾起一个元件。

一个有效的程序必须包含一个特殊的 `run` 函数，这是Karel开始执行任务的起点。

```java
function run() {
    move();
    move();
    turnLeft();
    move();
    pickBeeper();
}
```

这段代码的含义是：Karel先向前移动两格，然后左转，再向前移动一格，最后拾起脚下的元件。

## 控制流：循环与条件 🔄

如果让Karel移动100次，写100行`move();`显然不现实。这时我们需要**循环**。例如，`for`循环可以重复执行一段代码。

```java
for (var i = 0; i < 10; i++) {
    move();
}
```

这段代码会让Karel向前移动10次。`i`是一个计数器，从0开始，每次循环后增加1，直到不再小于10。

有时，我们需要根据情况做决定，这就需要**条件语句**。Karel可以检查周围环境。

```java
if (frontIsClear()) {
    move();
}
```

这段代码的意思是：**如果**（`if`）Karel前方是畅通的，**那么**就执行`move();`命令。`frontIsClear()`是一个返回**布尔值**（`true`或`false`）的条件函数。

## 定义新函数：分解问题 🧩

当任务变复杂时，我们可以将一系列指令打包成一个**新函数**。这就像教Karel一个新技能。

例如，我们可以定义一个让Karel向右转的函数。由于Karel只会左转，向右转可以通过三次左转实现。

```java
function turnRight() {
    turnLeft();
    turnLeft();
    turnLeft();
}
```

定义了这个函数后，我们在`run`函数中就可以直接使用`turnRight();`，使代码更清晰、更易读。

## 解决一个完整问题 🎯

让我们综合运用以上知识。假设任务要求Karel走一个正方形的路径，并在每个角落放置一个元件。

以下是解决此问题的一种方法：

```java
function run() {
    // 使用循环，让“移动-放置元件-右转”这个模式重复4次
    for (var i = 0; i < 4; i++) {
        move();
        putBeeper();
        turnRight();
    }
}

// 定义右转函数
function turnRight() {
    for (var i = 0; i < 3; i++) {
        turnLeft();
    }
}
```

本节课中我们一起学习了编程的核心基础。我们了解到编程就是编写精确的指令（代码）。通过Karel机器人，我们实践了基本命令、控制流（循环`for`和条件`if`）以及如何通过定义函数来分解复杂问题。记住，编程的关键在于将大问题拆解成计算机能够一步步执行的小指令。在接下来的课程中，我们将把这些逻辑思维应用到更通用的编程语言中。
