# UCB CS161 计算机安全：1：课程概述与基础概念

在本节课中，我们将要学习计算机安全的基本定义、核心目标以及一些基础概念。这些知识是理解后续更复杂安全问题的基石。

计算机安全的核心目标是保护计算机系统及其数据的**机密性**、**完整性**和**可用性**。这三个目标通常被称为**CIA三元组**。

*   **机密性**：确保信息不被未授权的个人、实体或过程访问或泄露。
*   **完整性**：保护信息和处理方法的准确性和完备性，防止未授权的篡改。
*   **可用性**：确保授权用户在需要时可以访问和使用信息和相关资产。

---

上一节我们介绍了安全的基本目标，本节中我们来看看实现这些目标时面临的常见威胁。

安全威胁主要分为几类。以下是主要威胁类型的简要说明：

*   **拦截**：攻击者未经授权访问数据，例如窃听网络通信。这破坏了**机密性**。
*   **篡改**：攻击者修改数据或系统状态。这破坏了**完整性**。
*   **中断**：攻击者使系统或服务不可用。这破坏了**可用性**。
*   **伪造**：攻击者伪造信息或身份，例如创建虚假的网站或数据。

---

理解了威胁，我们需要相应的防护措施。安全控制是用于对抗威胁、降低风险的方法。

安全控制可以分为不同类型。以下是基于其功能的分类：

*   **预防性控制**：旨在防止安全事件发生，例如使用强密码和防火墙。
*   **检测性控制**：用于发现正在发生或已经发生的安全事件，例如入侵检测系统。
*   **纠正性控制**：在安全事件发生后，用于减轻影响并恢复系统，例如数据备份和恢复流程。

---

在安全领域，我们经常需要评估潜在的问题。风险是安全事件发生的可能性及其可能造成的影响的结合。

一个简单的风险模型可以用以下公式表示：
**风险 = 可能性 × 影响**
安全工作的核心就是管理风险，通过实施控制措施来降低可能性或减轻影响。

---

本节课中我们一起学习了计算机安全的基础。我们明确了安全的三大目标——机密性、完整性和可用性，认识了拦截、篡改、中断和伪造这四类主要威胁，并了解了预防、检测和纠正这三类安全控制措施。最后，我们引入了风险的基本概念，它是可能性与影响的乘积。这些基础概念将贯穿我们整个计算机安全的学习过程。
