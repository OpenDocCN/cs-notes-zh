## 数据库的抽象层次 🏗️

数据库系统通过不同层次的抽象来管理数据，这有助于简化复杂操作并提高数据独立性。

以下是数据库系统的三个主要抽象层次：

1.  **视图层**
    *   这是最高层次的抽象。
    *   不同用户可以看到为其定制的数据视图，而无需了解底层数据的完整结构和存储细节。

2.  **逻辑层**
    *   数据库管理员在此层工作。
    *   它描述了数据库中存储了哪些数据以及这些数据之间存在何种关系。

3.  **物理层**
    *   这是最低层次的抽象。
    *   它描述了数据在存储介质（如磁盘）上是如何实际存储的，包括文件组织和索引结构。

上一节我们介绍了数据库的抽象模型，本节中我们来看看数据库中最核心的数据组织方式——关系模型。

---

## 关系模型 📊

关系模型使用**表**（称为关系）来组织和表示数据。每个表由行（元组）和列（属性）构成。

以下是关系模型中的几个关键概念：

*   **关系**：一个具有行和列的二维表。例如，一个`学生`表。
*   **元组**：表中的一行，代表一个实体或一条记录。
*   **属性**：表中的一列，代表实体的一个特征。每个属性都有一个名称和数据类型。
*   **域**：每个属性允许取值的集合。例如，`年龄`属性的域可能是正整数。
*   **模式**：关系的结构描述，包括关系名和其属性列表。例如，`学生(学号，姓名，年龄)`。
*   **键**：能够唯一标识关系中一个元组的一个或多个属性。**主键**是选定的一个候选键。

关系模型的核心优势在于其坚实的数学基础，数据操作可以通过关系代数来描述。接下来，我们将学习如何使用SQL语言来操作这些关系表。

---

## SQL 基础 🛠️

SQL是用于管理和操作关系数据库的标准语言。它主要包含数据定义语言和数据操作语言。

### 数据定义语言

DDL用于定义和修改数据库模式。

创建一个新表的语法是：
```sql
CREATE TABLE 表名 (
    列名1 数据类型1,
    列名2 数据类型2,
    ...
    约束条件
);
```
例如，创建一个`学生`表：
```sql
CREATE TABLE Students (
    sid INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);
```

### 数据操作语言

DML用于查询和修改表中的数据。

最核心的语句是`SELECT`，用于从表中检索数据。其基本结构是：
```sql
SELECT 列名
FROM 表名
WHERE 条件;
```
例如，查询所有年龄大于20岁的学生姓名：
```sql
SELECT name
FROM Students
WHERE age > 20;
```
