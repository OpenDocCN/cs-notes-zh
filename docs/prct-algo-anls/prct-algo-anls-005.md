## 📊 概述
在本节课中，我们将要学习**递推关系**，这是为计算机程序性能建立数学模型的第一步。我们将首先探讨如何使用计算机来计算递推关系的数值，这对于验证假设和理解算法行为至关重要。

## 🔍 什么是递推关系？
递推关系是一个递归地定义序列的方程。这对于计算机科学家来说很容易理解。一个简单的例子是斐波那契数列，数学家们对它也很熟悉。

递推关系是：
**`f_n = f_{n-1} + f_{n-2}`**

这是一个递归定义，序列中的每一项都由其前两项定义。然而，正如每个程序员所知，在递归程序中需要指定初始条件。同样，在递推关系中，也必须仔细指定初始条件，并确保对所有 `n` 值都有定义。

对于斐波那契数列，我们定义 `f_0 = 0`，`f_1 = 1`，并要求方程对 `n >= 2` 成立。因此，`f_2 = 0 + 1 = 1`，`f_3 = 1 + 1 = 2`，依此类推。我们通过将前两项相加得到序列中的每一项。

这就是递推关系的一个例子。现在，一个自然而然的问题是：给定递推关系，我们能否找到一个简单的公式，将 `f_n` 描述为 `n` 的简单函数？这正是我们将要解决的问题。

## 🧮 递推关系与程序成本
正如上一讲所见，递推关系直接为程序的成本建模。例如，我们讨论过的快速排序算法，其递推关系要复杂得多，但它仍然具有相同的性质：序列中的每一项（在本例中，序列定义了快速排序的运行时间，即比较次数）都由序列中更早的项定义。

在这种情况下，结果不是整数。我们可以计算出 `C_0 = 0`，`C_1 = 2` 等等，这就是快速排序对 `n` 个不同元素进行排序所需的比较次数。我们记得，这个递推关系是从程序中推导出来的，它是程序运行时间的数学模型，具体来说，是对 `n` 个不同元素的随机有序数组进行排序所需的比较次数。

## 💻 使用计算机计算递推值
如今，处理递推关系时的一个常识性规则是：直接使用计算机计算数值，看看是否能理解这些数值的含义以及发生了什么。甚至在开始数学推导之前就这样做会更好，因为它可能会揭示一些用数学方法难以发现的规律，而且这样做非常容易。

你可能会问的第一个问题是：为什么不使用递归程序呢？我们在每个初级编程课程中都教导，不要这样做。尝试用递归程序计算这种递推关系的值是一个非常糟糕的主意，因为它需要指数级的时间。

例如，要计算 `F(50)`，我们需要计算 `F(49)` 和 `F(48)`；要计算 `F(49)`，又需要计算 `F(48)` 和 `F(47)`，依此类推。观察这个计算过程，你会发现我们在不断地重复计算值：`F(48)` 两次，`F(47)` 三次，`F(46)` 五次，等等。实际上，计算这个需要指数级时间，所以即使是计算 `F(50)` 也无法完成，速度太慢了。

因此，虽然考虑使用递归程序是好的，但在实践中我们不这样做。相反，我们做的是将所有值保存在一个数组中。我们需要为要计算的每个值准备一个数组条目，但如今这已不是问题。

在这种情况下，如果你想计算 `F(50)`，就创建一个大小为 51 的数组，根据初始条件设置前两个值，然后简单地遍历序列中的每个值，根据前两个值计算其值。这是处理任何递推关系的一种常识性方法：直接使用数组。
