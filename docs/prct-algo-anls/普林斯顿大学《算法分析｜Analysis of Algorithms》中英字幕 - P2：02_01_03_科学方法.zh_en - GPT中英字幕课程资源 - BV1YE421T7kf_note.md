# 算法分析：02：科学方法 🔬

在本节课中，我们将详细探讨现代算法分析中使用的科学方法。我们将了解为何传统的“大O”符号不足以预测实际性能，并介绍一种更实用的“波浪号”表示法。同时，我们将构建一个完整的科学分析框架，从实现、建模到验证，帮助你理解如何科学地分析和比较算法。

---

## 算法理论中的符号 📝

在算法理论中，当研究最坏情况性能的上界时，会使用大O、大Ω和大Θ符号来描述增长阶数，并以此对算法进行分类。

*   如果 **G(n) = O(F(n))**，意味着当 n 趋于无穷大时，比值 **G(n)/F(n)** 存在一个上界。
*   如果 **G(n) = Ω(F(n))**，意味着该比值存在一个下界。
*   如果 **G(n) = Θ(F(n))**，意味着该比值同时存在上界和下界。

具体来说，O 表示存在一个常数 c，使得 **G(n) ≤ c * F(n)**；Θ 表示存在常数 c1 和 c2，使得 **c1 * F(n) ≤ G(n) ≤ c2 * F(n)**。这允许我们根据函数增长阶对算法进行分类，例如归并排序是 **n log n**，而快速排序是 **n²**。

这是文献中描述算法性能的常见符号。然而，正如之前提到的，大O符号存在风险，我们稍后会详细讨论。

---

## 大O符号的局限性 ⚠️

使用大O符号来比较算法是不科学的。你不能仅仅因为一个算法的运行时间是 **O(n log n)** 就断定它比 **O(n²)** 的算法好，这对于预测性能毫无用处。

大O符号只是最坏情况的一个上界，实际性能可能远好于最坏情况。甚至实际的上界也可能比大O给出的更小。它适用于对算法进行初步分类，但不适用于直接比较。

---

## 引入波浪号表示法 🔄

我们通常使用的是所谓的“波浪号”表示法。我们会说一个算法的运行时间近似于（~）一个常数乘以 n 的某个函数，其中 n 是输入规模。例如：**运行时间 ~ a * N**。这为预测性能提供了一条有效的路径，稍后我会展示一些例子。

因此，除了在讨论渐近近似时的一种特定技术含义外，我们并不常用大O、大Θ、大Ω符号。

大O符号因许多原因而有其用处，它可以追溯到几个世纪前，我们在数学中也使用它。但认为大O符号可用于预测性能是一个常见的错误，我想在一开始就杜绝这个问题。

---

## 关于“银河算法” 💫

另一个需要注意的概念是“银河算法”。这个想法由 Dick Lipton 提出，他定义那些永远不会被使用的算法为“银河算法”。它们之所以永远不会被使用，是因为在这个银河系中，没有人会注意到这个算法带来的任何效果。因为任何节省都发生在输入规模大到在本银河系中不可能出现的程度。

一个例子是 Chazelle 的线性时间三角剖分算法。证明该问题可以在线性时间内解决是一个理论上的壮举，但该方法过于复杂，以至于没有人会去实现它。即使有人实现了，实现的成本也肯定会超过任何节省，直到 N 大到需要另一个银河系来处理它。

我认为问题在于，许多发表在文献中的算法都属于这一类。Lipton 提出这个概念后，他博客的一位贡献者估计，理论计算机科学会议中大约 75% 到 95% 的论文都是“银河算法”。实践者可能没有意识到它们是“银河算法”，可能会尝试使用它们，而一个相对简单的分析就会指出这对实践者毫无意义。

为基础研究设定议程是可以的，尝试找到具有最佳最坏情况性能上界的算法也没有错。但我们必须纠正人们认为“银河算法”实际上有用的常见错误。

## 科学方法的具体步骤 🧪

接下来，我们来具体谈谈科学方法，即现代版本的 Knuth 所教授的方法，这种方法在我与 Kevin Wayne 合著的《算法》（第四版）中用于分析许多算法。该书的 1.4 节对此有详细描述和示例。

正如 Knuth 所说，我们从一个可以测试的完整实现开始。这样，我们就能够对该实现的性能提出假设并进行检验。

然后，我们会尝试避免 Knuth 方法中的一些细节，通过寻找内循环中的抽象操作来分析算法——即执行次数多于任何其他操作的操作。

我们还需要为程序的输入开发一个现实的模型，这仍然是一个难点。接着，我们只需分析该操作对于输入规模 n 的执行频率。我们的假设是：实际运行时间与这个频率乘以一个常数成正比。

假设可能是错误的，但我们可以检验它。我们使用波浪号操作，稍后我会具体展示如何进行检验。实际上，未知常数是一个令人讨厌但并非无法处理的问题，因为它确实允许我们进行具体的数学计算。

---

## 验证与迭代 🔁

一旦我们有了假设，我们将通过以下步骤验证它：
首先，根据模型生成一些大型输入。例如，对于排序算法，模型是数据随机排列且互异。因此，编写程序来生成大型的随机排列互异文件很容易。

然后，我们可以运行程序处理大型输入来计算常数 a 的估计值。实际上，我们甚至可以跳过这一步。但通过这种方式，我们肯定能得到 a 的估计值。

这为我们提供了一个模型，可以用来预测更大输入下的性能，并检验我们的假设。这就是算法分析的科学方法。

之后，我们还需要找到一个实际使用真实数据的应用程序，并在应用上下文中验证模型。这实际上是最困难的部分，如今常常被忽视。

与任何应用科学方法一样，我们将根据需要，或者随着对问题的深入了解，来完善、重复和修订模型与算法。正如我之前提到的，这个过程的一个巨大好处是，我们经常能了解到关于算法的信息，从而能够开发出改进的版本。

我们在排序、搜索、图算法、字符串处理、数据压缩等许多应用中都有大量成功应用此方法开发良好实现并理解其性能的例子。

## 本课程的核心 📚

本课程的核心正是这种执行频率的分析，数学部分就在这里。但我想给出完整的背景，以便大家理解我们为何要如此费心。

如前所述，我们不常使用大O、Ω、Θ符号，那是算法理论的内容。相反，我们将使用波浪号表示法。波浪号表示法意味着：如果当 n 趋于无穷大时，两个函数 **G(n)** 与 **F(n)** 的比值趋于 1，则 **G(n) ~ F(n)**。通常，**F(n)** 是一个常数乘以 n 的标准函数。

这就是我们使用的符号，它简化了问题，让我们专注于基本组成部分。

---

## 算法分析的基本组成部分 🧩

算法分析的一个基本组成部分涉及编程。进行算法分析的人需要能够熟练地实现算法、运行算法，并至少能够计数操作。因此，需要一个良好的实现。也许有人已经完成了实现，但你仍然需要在自己的计算机上体验它，以测试你可能有的各种假设。

其次是数学挑战，这将是本课程的重点。我们需要建立一个模型，并在模型中分析算法。本课程将讨论我们确实需要做数学，而这正是我即将要谈到的数学。

然后是运行算法解决实际问题，并检查与模型是否一致的科学过程。在完成所有其他步骤之前，你无法真正比较两种算法。因此，理解这个背景概念非常重要，它能帮助我们理解为何我们如此热衷于在不涉及过多细节的情况下正确地完成数学分析。

---

## 潜在挑战与优势 ⚖️

当然，这种方法仍然存在潜在的缺点。算法理论仍然涉足我们无法触及的领域。

首先，模型可能不现实，这在每个科学学科中都是一个挑战。但我们在计算机科学中有一个巨大优势：我们可以通过随机化来使模型完全有效，我稍后会展示一个例子。如果我们没有现实的模型，也许可以通过随机化来获得一个现实的模型，这是一个非常强大的概念。

其次，数学可能过于困难，这在任何科学学科中都是一个巨大挑战。统计物理学等许多领域都需要数学的新发展才能进行科学研究，这里当然也是如此。实际上，解析组合学作为算法分析的微积分，正是本课程的真正动机。

第三，实验可能过于困难，无法真正验证模型。同样，与其他科学学科相比，这对我们来说问题不大。我们可以轻松地将算法运行一百万次，而且我们经常这样做。然而，对于许多试图研究算法的人来说，这可能是一个障碍，因为他们实际上试图研究的是“银河算法”，实现起来可能极其困难。如果你无法实现它，又何必去分析它呢？为了智力挑战而研究是可以的，但同样，有些人认为你开发的算法在实践中是有用的，在让可怜的程序员面对它们之前，确实应该用科学方法进行验证。
