## 概述
在本节课中，我们将要学习排列（Permutations）这一基础组合结构。排列在算法分析中有着广泛的应用。我们将从标记结构的角度，使用指数生成函数来研究排列的基本性质、表示方法及其在算法（如排序和密码学）中的应用。

---

## 排列的基本定义与表示

上一节我们介绍了无标记结构（如树）及其普通生成函数。本节中，我们来看看标记结构——排列，并将使用指数生成函数进行分析。

排列可以理解为对一组元素的重新排序。一个经典的比喻是：有N个编号为1到N的学生参加派对，派对结束后，每个学生随机进入一个编号为1到N的房间。按学生编号顺序记录其所在的房间号，就得到了一个1到N的随机序列，即一个排列。

从解析组合学的角度看，排列是由标记原子构成的序列，其中每种不同的顺序都视为不同的排列。因此，**n个元素的排列总数是 n!**。

从生成函数的角度，排列的**指数生成函数（EGF）** 为：
```
P(z) = Σ_{n≥0} n! * (z^n / n!) = Σ_{n≥0} z^n = 1 / (1 - z)
```
这是因为计数序列是 n!，而EGF中的归一化因子也是 1/n!，两者相消。

## 排列的逆

排列可以看作是从集合 {1, 2, ..., n} 到自身的一个一一映射（双射）。例如，学生到房间的映射。这个映射的逆，称为**排列的逆**，它描述了房间到学生的映射。

计算排列的逆是一个简单的过程。以下是其核心思想与代码实现：

**核心思想**：如果原排列 `a` 表示 `学生i` 去了 `房间a[i]`，那么其逆排列 `b` 就表示 `房间j` 里住的是 `学生b[j]`。我们只需遍历原排列，将值作为索引，将索引作为值填入新数组即可。

**代码示例（基于0索引数组调整）**：
```java
int[] inverse(int[] a) {
    int n = a.length;
    int[] b = new int[n];
    for (int i = 0; i < n; i++) {
        // a[i] 是1-based的房间号，转换为0-based索引
        // i+1 是1-based的学生号，存入逆数组
        b[a[i] - 1] = i + 1;
    }
    return b;
}
```

**计算示例**：对于排列 `a = [8, 1, 3, 7, 6, 2, 9, 4, 5]`（表示1->8, 2->1, ...），其逆排列 `b` 的计算过程如下：
1.  `a[0]=8` -> `b[7] = 1`
2.  `a[1]=1` -> `b[0] = 2`
3.  `a[2]=3` -> `b[2] = 3`
4.  依此类推，最终得到逆排列 `b = [2, 6, 3, 8, 9, 5, 4, 1, 7]`。

## 排列的应用：简单替换密码

排列的逆在密码学中有直接应用，例如简单的**替换密码**。

**加密过程**：
1.  生成一个字母表（如A-Z）的随机排列作为密钥。
2.  将明文中的每个字母，根据该排列映射为另一个字母，生成密文。

**解密过程**：
1.  接收方持有相同的随机排列（密钥）。
2.  为了解密，需要使用该排列的**逆排列**。
3.  将密文中的每个字母，根据逆排列映射回原来的字母，即可恢复明文。

**示例**：
*   密钥（随机排列）：A->W, B->J, C->L, D->N, E->I, ..., Z->?
*   明文：`ATTACK AT DAWN`
*   加密（A->W, T->P, C->L）：密文为 `WPPWKL WN NW?S`
*   解密：使用逆排列（W->A, P->T, L->C, ...）将密文恢复为明文。

> **注意**：单纯的替换密码由于每个明文字符总是映射到同一个密文字符，容易通过频率分析被破解，因此现代密码系统中它通常只作为复杂算法的一部分。

## 排列的网格表示与逆

排列可以用一个 **N×N 的网格**来直观表示：

**表示方法**：对于排列 `a`（第i列的学生去了第a[i]行的房间），我们在网格的第 `a[i]` 行、第 `i` 列放置一个标记块。

**性质**：如果我们按**行**顺序读取标记块所在的**列**，读出的序列正好是原排列的**逆排列**。

**原理**：网格表示本质上是原排列的矩阵表示。**求排列的逆，在几何上等价于对这个矩阵进行转置**。原矩阵中 `(行, 列) = (a[i], i)` 的块，在转置后变为 `(i, a[i])`，这正好对应逆排列中“房间a[i]对应学生i”的关系。

## 排列的循环分解

在解析组合学中，我们介绍过排列的**循环分解**。这是分析排列性质的一个强大工具。

**概念**：从一个位置 `i` 开始，跟随映射 `i -> a[i] -> a[a[i]] -> ...`，最终会回到 `i`，形成一个循环。对排列中所有元素进行此操作，可将整个排列分解为若干个不相交的循环。

**示例**：排列 `a = [9, 12, 11, 10, 5, 6, 15, 4, 1, 14, 3, 2, 16, 13, 7, 8]` 可以分解为循环：`(1 9)(2 12)(3 11)(4 10 14 13 16 8)(5)(6)(7 15)`。

循环分解有助于我们分析排列的许多性质，例如后续将讨论的排序算法分析。

## 符号方法与生成函数

我们使用**符号方法**来系统地研究排列及其变体。符号方法为标记组合类建立了一系列组合构造与指数生成函数运算之间的对应关系。

对于排列类 `P`，其组合构造可以描述为：**一个排列要么是空的，要么是由一个原子（最小元素）与一个排列的“星积”构成**。
用符号表示为：`P = ε + Z ★ P`

这个构造直接翻译成指数生成函数的方程：
```
P(z) = 1 + z * P(z)
```
解这个方程，得到：
```
P(z) = 1 / (1 - z)
```
这与我们之前得到的指数生成函数一致，并且提取系数 `[z^n] P(z) = n!`，验证了排列的计数。

## 排列在排序算法分析中的应用

排序算法（如快速排序、归并排序）是排列的一个重要应用场景。要精确分析这些算法的性能，我们需要一个合理的输入模型。

**核心模型**：假设算法的输入是一个**随机排列**。即，所有 n! 种可能的输入顺序出现的概率相等。

**模型合理性**：即使原始输入是有序或逆序的，我们也可以在排序前先对输入进行一次**随机排列**。这样，排序算法实际处理的输入就严格符合随机排列模型。这使得基于排列的数学分析能够精确预测算法的平均性能。

**生成随机排列**：正确生成均匀随机排列至关重要。一个经典且正确的方法是 **Knuth Shuffle**（或称 Fisher-Yates Shuffle）。

**算法步骤**：
1.  初始化数组为 `[1, 2, ..., n]`。
2.  从 `i = 0` 遍历到 `n-2`：
    *   在 `[i, n-1]` 范围内随机选择一个索引 `r`。
    *   交换数组中 `i` 和 `r` 位置的元素。

**代码实现**：
```java
void shuffle(int[] a) {
    int n = a.length;
    for (int i = 0; i < n; i++) {
        // 随机生成一个介于 i 和 n-1 之间的整数 r
        int r = i + (int) (Math.random() * (n - i));
        // 交换 a[i] 和 a[r]
        int temp = a[i];
        a[i] = a[r];
        a[r] = temp;
    }
}
```

**算法正确性**：第一次循环，任何元素被选到第一位的概率是 `1/n`。第二次循环，在剩余元素中任何元素被选到第二位的概率是 `1/(n-1)`，依此类推。因此，生成**任何特定排列的概率都是 1/n!**，满足均匀随机的要求。
