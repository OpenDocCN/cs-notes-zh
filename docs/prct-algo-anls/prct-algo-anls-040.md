# 040：哈希表 🗂️

![](img/8a42d604ec36f45949ac823605546965_0.png)

在本节课中，我们将学习哈希表的基本原理及其背后的概率模型分析。哈希表是计算机科学中一种高效的数据结构，其性能分析紧密关联于我们之前讨论过的“球与瓮”模型。我们将探讨两种主要的哈希冲突解决策略：分离链接法和线性探测法，并分析它们的性能。

---

上一节我们介绍了“球与瓮”模型及其在组合数学中的经典应用。本节中，我们来看看如何将这个模型应用于哈希表的分析。

哈希表的核心思想是使用一个哈希函数，将键映射到一个固定范围内的索引值（例如 `0` 到 `M-1`）。这类似于将 `N` 个球（键）随机地放入 `M` 个瓮（哈希表的槽位）中。我们假设哈希函数满足**均匀哈希假设**，即每个键被独立、均匀地映射到 `M` 个槽位中的任何一个。

以下是两种主要的冲突解决策略：

## 分离链接法

在分离链接法中，每个哈希表的槽位（瓮）都维护一个链表。所有哈希到同一位置的键（球）都被放入对应的链表中。

以下是该方法的实现示意图：
```
哈希表索引: 0 -> [键A, 键B]
哈希表索引: 1 -> [键C]
哈希表索引: 2 -> [空]
...
哈希表索引: M-1 -> [键Z]
```

搜索一个键时，我们需要遍历其所在槽位的链表。因此，我们关心链表的平均长度，以及链表长度过长的概率。

*   平均链表长度显然是 `N/M`（记作 `α`）。
*   更关键的是，我们需要知道链表长度远超平均值的概率。这可以通过经典的占位分布来分析。

对于一个给定的槽位，其包含 `k` 个键的概率服从二项分布：
```
P(槽位有k个球) = C(N, k) * (1/M)^k * (1 - 1/M)^(N-k)
```
当 `N/M = α` 固定且 `k` 较小时，这个分布可以用泊松分布很好地近似：
```
P(槽位有k个球) ≈ e^(-α) * (α^k) / k!
```

利用这个近似和斯特林公式进行推导，可以证明，当 `α` 取一个适中的值（例如 `10`）时，链表长度超过 `5α` 的概率是极其微小的（例如 `10^-15` 量级）。这为程序员提供了性能保证。

## 线性探测法

在线性探测法中，哈希表是一个简单的数组，每个槽位只能存放一个键。当发生冲突（即两个键哈希到同一位置）时，算法会顺序地向后（或向前）扫描，直到找到一个空槽位。

以下是该方法的操作示例：
```
插入键X（哈希到位置2）: [空, 空, X, 空, ...]
插入键Y（也哈希到位置2）: 位置2已满，扫描到位置3 -> [空, 空, X, Y, ...]
```

分析线性探测法的平均搜索成本更为复杂。唐纳德·克努特在约50年前首次给出了精确解。他证明，在均匀哈希假设下，成功查找一个键所需的平均探测次数为：
```
平均探测次数 = Σ (k>0) [ (N/M) * ((N-1)/M) * ... * ((M - k + 1)/M) ]
```
这个结果是一个里程碑。它表明，如果保持哈希表的负载因子 `α = N/M` 不超过 `1/2`，那么平均探测次数将大约为 `2` 次，这是一个非常实用的结果。

![](img/8a42d604ec36f45949ac823605546965_2.png)

关于线性探测还有一个有趣的故事。尽管其平均性能的公式看似简洁，类似于生日问题或赠券收集问题，但使用我们课程中介绍的符号方法等现代分析工具来重新推导它，却曾是一个长期挑战。这个问题最终引向了与随机图、赌徒破产问题等领域的深刻联系，并由一种称为“艾里定律”的分布所描述。这说明了算法分析领域的深度与魅力。

本节课中我们一起学习了哈希表的两种基本实现：分离链接法和线性探测法。我们看到了“球与瓮”这一经典概率模型如何为分析哈希表性能提供强大的工具。对于分离链接法，我们利用占位分布和泊松近似分析了其最坏情况概率；对于线性探测法，我们介绍了克努特给出的经典平均性能分析结果。这些分析为设计高效、可靠的哈希表提供了理论基础。
