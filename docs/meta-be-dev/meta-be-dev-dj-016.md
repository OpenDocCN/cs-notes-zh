# 16：了解URL 🔗

在本节课中，我们将要学习什么是URL（统一资源定位符），并详细解析构成URL的各个部分。URL是我们在互联网上访问任何资源时使用的地址，理解其结构对于后端开发和网络通信至关重要。

![](img/dd7c0b0f189545407b76be2eea8f461f_1.png)

当您从设备访问网络时，可能会点击一个网页链接或输入一个地址。这个地址被称为URL，它链接或指向一个特定的资源或网页。

## 什么是URL？ 🧩

互联网上的每一个资源都由一个称为统一资源定位符（URL）的地址来定位。每次您点击网页链接或输入网址时，都是在访问一个URL。

![](img/dd7c0b0f189545407b76be2eea8f461f_3.png)

URL由多个部分组合而成。其核心组成部分包括：
*   **协议**
*   **域名**
*   **文件路径**
*   **参数**

![](img/dd7c0b0f189545407b76be2eea8f461f_5.png)

接下来，我们将逐一探索这些部分。

![](img/dd7c0b0f189545407b76be2eea8f461f_7.png)

## 协议（Scheme） 🌐

![](img/dd7c0b0f189545407b76be2eea8f461f_9.png)

URL的第一部分是协议，有时也称为方案。

它位于任何URL地址的开头，通常标识为 `HTTP` 或 `HTTPS`。

![](img/dd7c0b0f189545407b76be2eea8f461f_11.png)

`HTTP` 这几个字符代表超文本传输协议。该协议决定了数据传输和交换的规则集。

![](img/dd7c0b0f189545407b76be2eea8f461f_13.png)

`HTTP` 和 `HTTPS` 之间的区别在于安全性，其中的 `S` 代表安全（Secure）。
*   `HTTP` 以纯文本形式发送数据，这可能会暴露发送者的信息。
*   `HTTPS` 对所有数据进行加密，这意味着数据不是可读的格式，因此更加安全。

HTTP协议也负责在网络上发送和接收HTML。HTTP及其扩展HTTPS是网络上最广泛使用的协议。

![](img/dd7c0b0f189545407b76be2eea8f461f_15.png)

## 子域名与域名 🏷️

![](img/dd7c0b0f189545407b76be2eea8f461f_17.png)

上一节我们介绍了协议，本节中我们来看看子域名和域名。子域名位于域名之前，通常包含主页和其他重要页面。

![](img/dd7c0b0f189545407b76be2eea8f461f_19.png)

最常见的子域名是万维网，用 `WWW` 表示。

需要注意的是，一些浏览器会用符号（例如加密页面的锁形图标）来替换协议和子域名。这样做是为了让URL更易于阅读和理解。

![](img/dd7c0b0f189545407b76be2eea8f461f_21.png)

![](img/dd7c0b0f189545407b76be2eea8f461f_22.png)

接下来是域名，它由两部分组成：二级域名和顶级域名。

![](img/dd7c0b0f189545407b76be2eea8f461f_24.png)

以下是这两部分的简要说明：
*   **二级域名**：指一个组织或公司的名称。例如，名为“Little Lemon”的公司的二级域名就是 `littlelemon`。
*   **顶级域名**：位于二级域名之后，是域名的结尾部分。它用于表示您组织所属的国家或类别。例如，`.com` 地址可以表示商业实体，`.org` 表示组织，而国家代码（如 `.uk`）则代表一个国家、主权国家或属地。

![](img/dd7c0b0f189545407b76be2eea8f461f_26.png)

![](img/dd7c0b0f189545407b76be2eea8f461f_28.png)

## 文件路径 📁

![](img/dd7c0b0f189545407b76be2eea8f461f_30.png)

了解了域名的构成后，让我们来探索文件路径。

![](img/dd7c0b0f189545407b76be2eea8f461f_32.png)

文件路径，也称为页面路径，用于引导您找到资源的位置。该资源可以是任何可以通过HTTP传输的文件类型，例如网页文档、图像文件和元数据。

![](img/dd7c0b0f189545407b76be2eea8f461f_34.png)

请记住，URL只是一个存储文件的地址。这些文件可以是本地的，也可以是基于网络的（可能位于域之外的外部服务器上）。
*   **本地托管**：资源（如图像和网页文件）位于用户的设备或服务器上。
*   **网络托管**：资源存在于用户的远程网络服务器或外部服务器上。

例如，Django 4.1版本发布的基于网络的URL，就有一个包含发布年份、月份和日期的特定文件路径。

![](img/dd7c0b0f189545407b76be2eea8f461f_36.png)

## URL参数（查询字符串） 🔍

![](img/dd7c0b0f189545407b76be2eea8f461f_38.png)

![](img/dd7c0b0f189545407b76be2eea8f461f_40.png)

URL参数，也称为查询字符串，用于在URL内构建附加信息。

![](img/dd7c0b0f189545407b76be2eea8f461f_41.png)

您可以使用URL参数来捕获URL的任何部分，并传递其值以供处理。

![](img/dd7c0b0f189545407b76be2eea8f461f_43.png)

查询字符串的格式类似，它以问号 `?` 符号开始，并放置在URL路径之后。

![](img/dd7c0b0f189545407b76be2eea8f461f_45.png)

它包含以键值对形式表示的参数，这些参数可以出现在URL路径中。

![](img/dd7c0b0f189545407b76be2eea8f461f_47.png)

您可以通过在参数之间添加与符号 `&` 来添加多个参数。

![](img/dd7c0b0f189545407b76be2eea8f461f_49.png)

![](img/dd7c0b0f189545407b76be2eea8f461f_51.png)

Django可以处理这两种类型的URL，但主要使用URL参数。

![](img/dd7c0b0f189545407b76be2eea8f461f_53.png)
