# PostgreSQL 入门课程：课程3：文本与JSON数据处理 📚

![](img/85fe5973cd8ba80e3bf4ec550162aaae_0.png)

在本课程中，我们将重点学习两个核心主题：文本（Text）和JSON数据处理。计算机擅长处理数字，例如我们使用数字作为外键，因为64位整数能被计算机高效地存储、移动和比较。然而，实际应用中存储的大量数据是字符串，如姓名、评论、博客文章等。虽然数字也用于表示平均气温等，但像邮政编码、电话号码这类数据本质上是字符串。因此，文本和自然语言处理非常重要，尤其在对话系统和搜索引擎中。JSON则是近25年来关系型数据库领域最重要的进展之一，它允许应用程序灵活演进数据结构。传统数据建模要求预先精确定义列及其大小，这虽然有利于效率和存储优化，但对某些应用场景限制较大。JSON提供了灵活性，允许临时存储一些数据，同时通过索引特定部分，也能高效用于查询。本课程将探讨如何在保持关系数据库高性能的同时，有效处理半结构化数据。

## 计算机的数据处理能力 💻

上一部分我们概述了课程重点，现在我们来深入了解计算机处理不同类型数据的能力。计算机非常擅长处理数字，这是我们在数据库设计中大量使用数字（如整数）的原因。

以下是计算机高效处理数字的几个关键点：
*   64位整数能被计算机高效地存储和移动。
*   计算机能高效地进行数字比较运算。
*   数字类型（如外键）为数据库的关联和索引提供了性能基础。

## 文本数据的重要性 📝

上一节我们讨论了计算机对数字的处理优势，本节中我们来看看文本数据。尽管数字很重要，但在实际工作中，存储的绝大部分数据是文本形式。

文本数据涵盖广泛，包括：
*   姓名、地址等个人信息。
*   用户评论、博客文章等内容。
*   邮政编码、电话号码（本质上是字符串）。
*   自然语言文本，如对话记录。

因此，有效地存储、查询和处理文本是数据库应用中的一个核心需求。

## JSON：关系数据库的演进 🔄

前面我们强调了文本处理的重要性，现在让我们聚焦于一种特殊的结构化文本格式——JSON。JSON是近25年来关系型数据库领域一次重大的功能演进。

JSON为应用程序带来了关键优势：
*   **结构灵活性**：允许数据结构随时间演进，无需预先严格定义。
*   **适用场景**：适合存储无需排序或作为主要查询条件的辅助数据。
*   **性能保持**：通过对JSON中的特定字段建立索引，可以使其在`WHERE`子句等查询中保持高性能。

简而言之，JSON在提供灵活性的同时，并未牺牲关系型数据库通过精心设计模式所获得的高性能优势。

## 课程总结 🎯

本节课我们一起学习了PostgreSQL课程第三部分的核心内容。我们认识到，虽然计算机擅长处理数字，但文本才是数据存储的主要形式。同时，JSON作为一种灵活的半结构化数据格式，极大地扩展了关系型数据库的应用边界，使其能在保持高性能的前提下，适应快速变化的业务需求。在接下来的课程中，我们将深入实践如何操作和处理这些数据。
