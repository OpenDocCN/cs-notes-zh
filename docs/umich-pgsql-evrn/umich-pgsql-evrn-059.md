在本节课中，我们将学习如何在 PostgreSQL 中使用正则表达式对查询结果进行解析和操作。我们将从简单的列值提取开始，逐步深入到更复杂的模式匹配和数组提取，最后以一个解析电子邮件地址的完整示例作为总结。

## 从列中提取数据

上一节我们介绍了如何使用 `WHERE` 子句和正则表达式来筛选行。本节中，我们将看看如何对筛选出的行中的列数据进行实际处理，即解析查询结果。

通常，如果你只需要整个列，直接使用 `SELECT email` 即可。但我们也可以使用正则表达式来解析列中的数据。

### 提取数字

以下是提取电子邮件地址中第一组连续数字的示例：

```sql
SELECT SUBSTR(email, ‘[0-9]+’) FROM table_name WHERE condition;
```

*   **`[0-9]`**：这是一个字符集，匹配任意一个数字（0到9）。
*   **`+`**：这是一个修饰符，表示前面的模式（即数字）出现**一次或多次**。它匹配的是连续的字符。
*   如果使用 `*`，则表示零次或多次，这可能会匹配到所有内容，因为零个数字也符合条件。
*   这个表达式会找到第一组连续的数字。例如，从 `user79@example.com` 中会提取出 `79`。

### 提取域名

以下是一个更复杂的例子，用于从电子邮件地址中提取域名：

```sql
SELECT SUBSTR(email, ‘@(.+)$’) FROM table_name;
```

这个表达式需要稍作解释：
*   **`.`**：匹配任意单个字符。
*   **`+`**：修饰前面的 `.`，表示任意字符出现一次或多次。
*   **`@`**：匹配字面量的“@”符号。
*   整个 `.*+` 组合的作用是：扫描字符串，直到找到“@”符号为止。
*   **`(`**：括号表示开始提取。提取将从“@”符号之后开始。
*   **`.*`**：匹配任意字符零次或多次。
*   **`$`**：匹配字符串的结尾。
*   因此，这个正则表达式的含义是：**扫描整行直到找到“@”符号，然后提取从“@”之后到行尾的所有内容**，即域名。

你可以在 Python 中使用完全相同的正则表达式，如果你上过我的“Python for Everybody”课程，应该见过这个例子。

### 获取唯一的域名列表

我们可以使用 `DISTINCT` 关键字来获取所有不重复的域名：

```sql
SELECT DISTINCT SUBSTR(email, ‘@(.+)$’) FROM table_name ORDER BY 1;
```

这行代码会从数据库中提取所有电子邮件地址的域名，并按字母顺序排序。

## 分组与计数

现在我们可以进行更复杂的操作。以下查询看起来有点复杂，但功能强大：

```sql
SELECT SUBSTR(email, ‘@(.+)$’) AS domain, COUNT(email) FROM table_name GROUP BY domain;
```

*   `SUBSTR(email, ‘@(.+)$’)` 用于提取域名。
*   `COUNT(email)` 用于统计每个域名出现的次数。
*   `GROUP BY` 子句使用相同的域名提取表达式进行分组。
*   这可以看作是 `DISTINCT` 的一个变体，但它不仅列出唯一的域名，还会记住每个域名出现的次数。例如，`app.co` 出现了2次，`umich.edu` 出现了1次。

你可以将这些表达式放在 `SELECT` 语句的不同部分，虽然最终写出来可能看起来有些晦涩，但数据库引擎能够很好地处理。

## 提取所有匹配项（数组）

之前介绍的方法只会提取文本列中的**第一个**匹配项，即使正则表达式可以匹配到多个，也会忽略后面的匹配。

有时我们需要获取**所有**匹配项。这时可以使用 `REGEXP_MATCHES` 函数，它会返回一个匹配项的数组。在应用程序中处理这些数组结果时，代码需要稍作调整。我们将以提取推文中的话题标签（hashtag）为例。

假设我们有一些包含话题标签的推文：

```
#SQL is great! #learning
Check out #UMich #Python courses.
#databases #SQL
```

### 筛选包含特定标签的推文

我们可以使用 `WHERE` 子句来查找包含特定标签的推文，这和我们之前做的类似：

```sql
SELECT * FROM tweets WHERE content ~ ‘#SQL’;
```

这会找到所有内容中包含“#SQL”的推文。我们也可以用 `LIKE` 子句实现。

### 提取所有话题标签

现在，我们使用 `REGEXP_MATCHES` 来提取所有标签：

```sql
SELECT REGEXP_MATCHES(content, ‘#([A-Za-z0-9_]+)’, ‘g’) FROM tweets;
```

*   **`#`**：匹配字面量的“#”号。
*   **`(`**：开始提取。
*   **`[A-Za-z0-9_]`**：一个字符集，匹配大小写字母、数字和下划线。我们定义话题标签由“#”号后跟至少一个这些字符组成。
*   **`+`**：修饰前面的字符集，表示至少出现一次。
*   **`)`**：结束提取。
*   **`‘g’`**：标志位，表示进行**全局**匹配，即找出该行中所有符合模式的子串。
*   因为我们把“#”号放在了括号外面，所以提取结果中不包含“#”号，只得到标签本身（如 `SQL`， `learning`）。这是有意为之的选择。

### 获取唯一标签列表及关联信息

我们可以结合 `DISTINCT` 来获取所有不重复的标签：

```sql
SELECT DISTINCT REGEXP_MATCHES(content, ‘#([A-Za-z0-9_]+)’, ‘g’) FROM tweets;
```

![](img/846e113b61aaaa8e71763f1a5440a1a9_3.png)

我们还可以进行更复杂的查询，例如，同时获取标签以及出现该标签的推文ID：

```sql
SELECT id, REGEXP_MATCHES(content, ‘#([A-Za-z0-9_]+)’, ‘g’) FROM tweets;
```

这样，你就能知道哪个标签出现在哪条推文中，便于后续处理。`REGEXP_MATCHES` 函数的核心作用是扫描整行，找出所有符合模式的匹配项，并以数组形式返回。

## 总结与预告 🚀

本节课中我们一起学习了在 PostgreSQL 中应用正则表达式进行高级字符串操作。我们掌握了如何使用 `SUBSTR` 提取列中的特定部分（如数字和域名），以及如何使用 `GROUP BY` 和 `COUNT` 进行数据统计。

更重要的是，我们认识了 `REGEXP_MATCHES` 函数，它能够提取一行文本中**所有**符合模式的匹配项，并以数组形式返回，这大大扩展了模式匹配的能力。
