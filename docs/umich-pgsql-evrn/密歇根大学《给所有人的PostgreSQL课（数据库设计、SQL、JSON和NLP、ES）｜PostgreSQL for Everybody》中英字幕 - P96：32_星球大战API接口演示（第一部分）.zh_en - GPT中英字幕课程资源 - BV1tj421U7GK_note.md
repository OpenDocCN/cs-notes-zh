## 概述
在本节课中，我们将学习如何从外部API获取JSON数据，并将其存储到PostgreSQL数据库中。我们将使用一个名为“星球大战API”的公开REST API作为数据源，编写一个Python程序来“爬取”数据。这个程序会像网络爬虫一样工作：从已知的URL开始，读取返回的JSON文档，从中提取出新的URL，然后继续抓取，直到获取所有相关数据。

## 创建数据表

首先，在PSQL命令行中执行以下命令，删除可能已存在的旧表（如果不存在也不会报错）：
```sql
DROP TABLE IF EXISTS swapi;
```

接着，运行我们下载的Python程序来创建表结构：
```bash
python3 swapi.py
```
程序启动后，会首先检查并创建名为 `swapi` 的数据表。这个表的结构设计用于存储我们的爬取任务和结果。

让我们查看一下这个表的结构和初始数据。表结构包含以下核心字段：
*   `id`：自增序列ID。
*   `url`：需要抓取的目标URL，具有唯一性约束。
*   `status`：HTTP状态码，例如200代表成功。初始为空。
*   `jsonb`：存储API返回的JSON数据。
*   `created_at` 和 `updated_at`：时间戳，记录创建和更新时间。

程序初始化时，会向表中插入几个已知的起始URL，例如 `films/1/`、`species/1/` 和 `people/1/`。

我们可以用SQL查询来验证数据是否已插入：
```sql
SELECT url, status FROM swapi;
```
此时，`status` 字段应为空，表示这些URL尚未被处理。

## 程序运行逻辑解析

现在我们已经有了数据表和一些待处理的URL，接下来深入了解一下 `swapi.py` 程序的核心运行逻辑。

程序启动后，首先会调用一个状态函数来汇报当前进度。这个函数会执行几条SQL查询：

```python
# 示例状态查询逻辑
SELECT COUNT(*) FROM swapi;                    -- 总任务数
SELECT COUNT(*) FROM swapi WHERE status IS NULL; -- 待处理数
SELECT COUNT(*) FROM swapi WHERE status = 200;   -- 成功数
SELECT COUNT(*) FROM swapi WHERE status NOT IN (200, NULL); -- 错误数
```

状态汇报完毕后，程序会进入主循环。它会询问用户本次想要抓取多少个文档（例如输入5）。然后，程序执行以下步骤：
1.  从 `swapi` 表中选取一个 `status` 为空的URL。
2.  使用 `requests` 库向该URL发送HTTP请求。
3.  获取响应的状态码（如200）和文本内容（JSON格式）。
4.  更新数据库：将该URL的 `status` 设置为获得的状态码，并将JSON文本存入 `jsonb` 字段。**这个更新操作确保了同一个URL不会被重复抓取。**
5.  程序会解析返回的JSON数据，寻找其中包含的新URL（例如在 `films`、`species`、`characters` 等键对应的列表中）。
6.  将这些新URL尝试插入到 `swapi` 表中。由于 `url` 字段有唯一约束，重复的URL会被忽略（`ON CONFLICT (url) DO NOTHING`）。
7.  每处理完一批记录（例如25个），程序会提交事务（`commit`）并暂停一秒，以便观察和控制。

**一个关键点**：初始版本的代码在每次插入新URL后都立即提交，这会导致性能较慢。优化后，我们将提交操作移到循环外部，每处理完一个主URL（即完成一次API调用和解析）后再提交，这样可以显著提升运行速度。

随着程序运行，“待处理”的URL数量会动态变化。当程序从API响应中发现大量新链接时，待处理数会快速增长；随着这些链接被逐个抓取完毕，待处理数会逐渐减少，直到所有关联数据都被获取。

## 运行演示与结果

了解了程序逻辑后，让我们实际运行并观察其过程。

在命令行中运行程序并输入抓取数量后，你会看到类似以下的输出，它实时显示了抓取进度：
```
Retrieved [URL] status=200 (42 left)
```
这表示成功抓取了一个URL，并且当前还有42个URL等待处理。

我们可以随时在另一个PSQL窗口中使用SQL监控数据变化：
```sql
SELECT COUNT(*) FROM swapi WHERE status IS NULL; -- 查看剩余任务数
```
随着抓取进行，表中的总行数会增加，成功（`status=200`）的行也会越来越多。

当程序最终完成时（例如抓取了200多个文档后），它会输出总结信息，表明所有数据已加载完毕。此时，查询数据库可以看到所有URL的状态都已更新，并且 `jsonb` 字段中存储了压缩后的JSONB格式数据。

需要注意的是，数据库中的 `jsonb` 是PostgreSQL解析和优化后的二进制格式，它与原始API返回的带有缩进格式的JSON字符串不同，但包含完全相同的信息。
