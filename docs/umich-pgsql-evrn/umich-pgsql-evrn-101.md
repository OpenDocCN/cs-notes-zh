# 101：欢迎学习本课程 🎉

![](img/2b3ac3ccc328464aa9623ce7c96b024b_0.png)

在本节课中，我们将开启《给所有人的PostgreSQL》系列课程的第四部分。我们将深入探讨数据库的工作原理、相关元问题，以及数据库技术如何随着时代变迁而演进。

你可能会问，为什么这些内容没有放在课程最开始讲解。实际上，我们在课程初期已经涉及了历史背景，讨论了标准是如何形成的。数据库一直是我们构建的每一个应用程序的核心组成部分。我们将回顾数据库如何随着市场、世界以及互联网和云计算的发展而演变。

## 课程内容概览

上一节我们回顾了数据库的发展历程，本节中我们将具体了解其内部机制。

首先，我们将探讨关系型数据库的工作原理。随后，我们会分析关系型数据库的优势与劣势，并学习如何构建、部署关系型数据库，以及如何实现其大规模运行。

值得注意的是，这里的“规模”定义也在逐年变化，从最初的10个用户、数百用户，发展到数千、数百万乃至数十亿用户。

## NoSQL 运动的兴起

NoSQL运动是上述演变故事的一部分。你或许认为NoSQL运动与关系型数据库运动是相互对立的，但答案在某种程度上是它们非常互补。在许多时刻，关系型数据库需要提升自身能力，而NoSQL正好揭示了关系型数据库在某些方面的弱点。

在后续课程中，我们将讨论部署、可扩展性等主题，并解释诸如**主-主复制**和**主-从复制**等术语。我们将了解这些机制如何运作、如何影响性能，以及哪些方案有效、哪些无效。

## 实践环节：使用 NoSQL 数据库

课程最后，我们将实际使用一个NoSQL数据库。我选择向你们展示 **Elasticsearch**。它是众多NoSQL数据库中的一种，其特点是你基本上通过 **JSON** 格式与之通信。

我选择Elasticsearch的原因是，构建**混合型应用**已非常普遍且历时已久。这类应用通常使用Elasticsearch来支持搜索功能，同时使用如MySQL或PostgreSQL这类关系型数据库来处理应用程序的其他部分。因此，即使你在职业生涯中主要与关系型数据库打交道，将来也很有可能会遇到Elasticsearch。

本课程包含了许多精彩内容，希望你享受学习过程。
