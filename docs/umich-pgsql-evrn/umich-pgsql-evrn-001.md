在本节课中，我们将要学习这门课程的背景、目标以及为什么选择 PostgreSQL 作为我们的核心数据库技术。

大家好，欢迎来到《给所有人的PostgreSQL》课程。这是一门我梦想教授了超过十年的课程。你可能在另一门课程中认识我，例如《给所有人的Python》、《给所有人的JavaScript》或《给所有人的Web应用》。

你会注意到，在所有这些课程中，我都必须教授 SQL。事实证明，我一遍又一遍地教授基础的 SQL，这就像是一名幼儿园老师。因为我无法深入讲解我想要的细节，因为在每一门课程中，我都有另一个主题需要覆盖。

因此，我只能教授 CRUD（增删改查）、什么是索引、什么是连接、什么是外键，然后就转向该课程的其他主题。多年来，许多同学告诉我，他们喜欢课程中的 SQL 部分，并希望能在 SQL 方面进行更多编程。

实际上，在我构建在线教学应用时，SQL 部分几乎是最具创造性的部分。我的意思是，其他人可能更擅长设计漂亮的用户界面，但对我来说，SQL 是应用本质和性能的核心所在。

还有太多东西需要学习，而且使用 SQL 非常有趣。所以现在是我的时刻，我可以开设一门只专注于数据库的系列课程，随心所欲地教授你们相关知识。我不必教授数据库以外的任何东西，这真的让我非常兴奋。

现在你可能会问，为什么选择 PostgreSQL？我读作 “Post-gres”。这是它的吉祥物，一只小象。好的开源项目都需要吉祥物，这是我的 Sakai 项目的吉祥物——袋獾，以及 PostgreSQL 的吉祥物。就是这样。

那么为什么选择 PostgreSQL？我确实喜欢开源，但你可能会问，为什么不选 MySQL？我也曾有过同样的想法。我使用 MySQL 很长时间了，我热爱开源，构建了许多开源软件。你们将用来上这门课的自动评分器实际上就在使用 MySQL。

但问题是，过去很容易知道该使用哪个数据库。如果你在企业环境中，公司选择了 Oracle，你可能会用 Oracle。如果你在开源环境中，并坚持使用免费软件，你可能会用 MySQL。例如，如果你在一个微软技术栈的组织里，你可能会用 SQL Server。所以，答案似乎取决于你的起点，非常明显。

然而，一段时间以前，Oracle 收购了 MySQL。他们并没有收购 MySQL 的全部，因为 MySQL 是开源的，但他们收购了足够多的核心开发团队，以至于 Oracle 掌握了 MySQL 的未来命运。这让一些人感到担忧，因为我们很多人选择 MySQL 正是为了避开 Oracle——我们不想每次安装东西都要付费。

那时，有人获取了 MySQL 的开源代码，复制了一份，并称之为 MariaDB。其想法是，当 MySQL 继续发展时，开源社区会继续发布 MariaDB。但问题在于，大约在同一时期，正如我们稍后将在课程中看到的，数据库的竞争环境变得越来越具有挑战性。

它面临着来自 NoSQL 存储的挑战，以及存储 JSON 的需求——这是经典关系型数据库之前没有太多考虑，但事实证明非常重要的功能。随着像 MongoDB 和 Cassandra 这样的 NoSQL 数据库出现，像 SQL、Oracle 和 PostgreSQL 这样的关系型数据库感到其市场地位正在下滑。

像 SQL、Oracle 和 PostgreSQL 这样的关系型数据库必须做出反应。问题是，MySQL 5 是在所有这些市场反应和担忧之前发布的。因此，Oracle 必须投资 MySQL，使其能与所有这些其他产品竞争，最终他们发布了 MySQL 8。

而 MariaDB 社区将不得不审视他们在多大程度上跟上了 MySQL 8 的步伐。我相信现在是 MariaDB 10。如果我们有足够多的开源人士转向 MariaDB，那可能会成功。但现在的问题是，MariaDB 对于 MySQL 用户来说是一个微小改变，也许 MariaDB 会是答案。

但与此同时，PostgreSQL 已经成为一个有趣的选择，因为它是一个坚定、纯粹的开源项目。你可能听说过，亚马逊在 90 年代末刚起步时选择了 Oracle，因为如果你想要一个快速的数据库，又不想自己成为一家数据库公司（亚马逊不想），那么这就是当时的做法。因此，他们在 Oracle 数据库上构建了许多东西。

问题是，这让他们花费了大量金钱。随着亚马逊规模越来越大，他们的 Oracle 账单也越来越大。这让 Oracle 的创始人拉里·埃里森变得非常、非常富有，以至于钢铁侠的角色就是以他为原型塑造的。

因此，亚马逊投入巨资停止使用 Oracle，转而使用 PostgreSQL。我相信他们之前已经宣布，亚马逊内部不再使用 Oracle，全部转向了 PostgreSQL。这一切让许多开源人士和其他人开始关注 PostgreSQL，你也应该关注它。

我已经仔细研究过 PostgreSQL，并且对它的表现感到满意。PostgreSQL 社区是一个非常优秀的开源社区，我是它的忠实粉丝，在很多方面甚至有些羡慕。他们规模大，人才济济，已经在这个领域深耕了超过 20 年。它是一个先进、稳定、可扩展的 SQL 数据库，亚马逊的基础设施就构建在 PostgreSQL 之上。

所以，我认为现在是时候让我们认真审视 PostgreSQL 了。我很高兴我的第一次高级 SQL 探索之旅是向你们介绍 PostgreSQL。我希望在这门课程结束时，你们能像我一样成为 PostgreSQL 的忠实粉丝。

本节课中我们一起学习了这门课程的起源、教学目标，以及选择 PostgreSQL 作为核心学习对象的历史和现实原因。我们了解到 PostgreSQL 是一个强大、成熟且纯粹的开源关系型数据库，在当今技术生态中占据重要地位。
