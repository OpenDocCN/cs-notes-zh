# 080：16_PostgreSQL与Python集成讲座

## 概述

在本节课中，我们将学习如何将Python编程语言与PostgreSQL数据库集成。我们将了解如何通过Python代码连接数据库、执行SQL命令以及处理查询结果，从而利用Python强大的数据处理能力来操作数据库。

## 从PgSQL到Python

到目前为止，我们一直通过PgSQL客户端输入SQL命令。实际上，模拟PgSQL的行为在Postgres中相当容易，因为PgSQL只是一个客户端。Postgres服务器位于网络上的某个位置，拥有IP地址、用户名和密码。我们登录后，向它发送命令。PgSQL只是众多能够接收SQL命令并将其发送到服务器的软件之一，另一个能够做到这一点的软件就是Python。

由于Python非常擅长处理文本，我们将进行一系列操作，包括使用Python读取文本、与API交互、解析数据，然后将数据存入数据库。我们可以在SQL中进行数据清洗，但使用Python时，你将拥有比SQL更强大的能力。

## 连接PostgreSQL与Python

首先，我们需要了解Python连接PostgreSQL的关键组件。你需要一个名为`psycopg2`的库，它是Python连接Postgres的适配器。安装后，你就可以建立连接了。

建立连接本质上就是登录Postgres服务器，这个过程可能会失败，例如用户名、密码错误或主机名不正确。在我的在线示例中，我们会将包含敏感信息的文件隐藏起来，这样你就不必在代码文件中直接写入密码。

连接建立后，你可以通过它发送命令。但通常，我们可能希望同时处理多个客户端请求，因此我们使用一个称为“游标”的概念。

## 理解游标

游标类似于文件句柄或网络编程中的套接字。你向连接请求一个游标，然后开始通过这个游标发送SQL命令。

游标是一个对象，你调用它的`execute`方法来发送SQL命令字符串。这些命令就是普通的字符串，不一定是常量，你可以拼接字符串来构建更复杂的命令。我们稍后会看到如何安全地使用参数化查询。

以下是一个示例代码片段：

```python
# 获取一个游标
cur = conn.cursor()
# 执行SQL命令
cur.execute("DROP TABLE IF EXISTS example;")
cur.execute("CREATE TABLE example (id SERIAL PRIMARY KEY, data TEXT);")
```

你可以重复使用同一个游标执行多个命令。执行命令后，通过游标可以获取一个称为“记录集”的结果。然后，你可以通过循环来逐行处理结果。

## 处理查询结果

当你执行一个`SELECT`命令并返回多行数据时，在Python中，你需要通过游标来访问这些行。因为结果可能是一行，也可能是成千上万行，所以通常需要编写一个循环来处理，除非你确定只返回一行。

`fetchone()`方法用于获取下一行数据。一行数据是一个元组。如果没有更多行，则返回`None`。

处理方式非常简单，你可以参考以下示例代码结构：

```python
cur.execute("SELECT * FROM example;")
row = cur.fetchone()
while row is not None:
    # 处理这一行数据
    print(row)
    row = cur.fetchone()
```

## 实践项目：加载并索引电子书文本

我们将进行一个较大的实践项目：读取一本电子书的文本，将其加载到数据库中，然后构建全文索引并进行查询。

我们将使用古登堡计划提供的免费开源电子书，例如查尔斯·狄更斯的《圣诞颂歌》。项目步骤如下：

1.  下载文本文件。
2.  运行Python脚本（`loadbook.py`）处理文件。
3.  脚本会读取文件，识别段落（将多行合并为一个长文本行），并清理多余的空白。
4.  将每个段落作为一行数据插入数据库的一个`text`类型字段（例如`body`）中。
5.  最终，数据库中的每一行对应书中的一个段落。

![](img/0f14806c655d6c07ee7a23027fab6be8_1.png)

运行`loadbook.py`脚本时，你可以同时在另一个PgSQL窗口观察数据库的变化。脚本每插入50条记录会提交一次，所以你会在数据库中周期性地看到新数据。

![](img/0f14806c655d6c07ee7a23027fab6be8_3.png)

数据加载完成后，我们就可以对`body`字段创建全文索引（使用`TSVECTOR`），并利用`TSQUERY`进行丰富的全文搜索，体验在更大数据集上进行有趣查询的能力。
