## 数据来源与格式 📥

上一节我们介绍了课程的整体目标，本节中我们来看看具体的数据来源及其格式。

示例代码从 `dr-chuck.net/saki/devel` 在线地址抓取一系列邮件信息。邮件数据采用 **Mbox** 格式存储。这种格式的特点如下：

*   每条邮件信息以字符串 `From `（注意后面有一个空格）作为分隔符。
*   每条邮件包含一个**邮件头**和一个**邮件正文**。
*   邮件头由一系列键值对组成，格式为 `Key: Value`，例如 `From: user@example.com`。
*   邮件头与邮件正文之间通过一个**空行**进行分隔。
*   邮件正文内部也可能包含空行，但只要遇到新行以 `From ` 开头，就标志着一个新邮件的开始。如果正文中本身包含“From ”字符串，则会有特殊的转义处理。

以下是一个简化的Mbox格式示意：
```
From sender@example.com
From: sender@example.com
To: receiver@example.com
Date: Tue, 01 Jan 2024 12:00:00 +0000
Subject: Sample

This is the message body.
It can have multiple lines.

From sender2@example.com
From: sender2@example.com
...
```

## 代码设计与演化过程 🔄

以下是示例代码 `gmane.py` 的核心设计思路与开发哲学。

这段代码并非一蹴而就。它经历了数周的迭代开发，期间需要处理网络错误、服务器限流、数据格式不一致等各种意外情况。其设计体现了**渐进式**和**容错性**编程思想：

*   **基础功能**：首先构建一个能够抓取数据并解析基础格式的简单脚本。
*   **错误处理**：在运行中遇到问题（如服务器无响应、达到访问频率限制、数据格式异常）后，逐步增加相应的错误处理与重试逻辑。
*   **数据清理**：使用正则表达式等技术，统一清理邮件中的日期、电子邮件地址等信息的格式，确保存入数据库的数据规范一致。
*   **状态持久化**：利用数据库作为临时存储，记录抓取状态。这样程序可以随时停止，并在之后从断点恢复，这对于处理大量数据或不可靠的网络环境非常有用。

例如，处理服务器限流的伪代码逻辑可能如下：
```python
if response.status_code == 429: # Rate limit hit
    log("Rate limit exceeded. Pausing for 1 hour.")
    time.sleep(3600)
    # 之后可以选择重试或停止
```

## 核心技术与步骤 🛠️

在了解了整体设计后，我们来看看实现邮件归档系统所涉及的具体技术步骤。

以下是构建该系统的关键步骤与技术要点：

1.  **使用正则表达式进行数据解析与清理**
    *   用于从邮件头中提取键值对。
    *   用于标准化日期和时间字符串的多种格式。
    *   用于清理和验证电子邮件地址格式。

2.  **建立数据库模型并创建索引**
    *   设计存储邮件头（发件人、收件人、主题、日期等）和正文的表结构。
    *   为常用的查询字段（如 `Sender`， `Date`）创建索引以提升性能。
    *   为邮件正文创建**全文检索索引**，以支持复杂的文本搜索。

3.  **实现增量式数据抓取（爬虫）**
    *   编写代码从源URL分页或按序列抓取邮件数据。
    *   将已抓取邮件的ID记录在数据库，避免重复抓取。
    *   实现优雅的暂停与重启机制。

4.  **执行高级数据库查询**
    *   在清理并导入数据后，利用PostgreSQL的强大功能进行数据分析。
    *   例如，使用全文检索查找包含特定关键词的邮件。
    *   例如，使用SQL聚合函数分析邮件发送频率。

一个创建全文检索索引的SQL命令示例：
```sql
CREATE INDEX idx_messages_fts ON messages USING gin(to_tsvector('english', body));
```
