# 051：SQL外连接详解 🧩

在本节课中，我们将要学习SQL中的外连接。外连接不仅返回匹配的行，还会返回不匹配的行，这对于数据合并与分析至关重要。

上一节我们介绍了内连接，本节中我们来看看外连接的不同类型及其应用。

## 外连接类型概述

外连接主要分为三种类型：左外连接、右外连接和全外连接。以下是这三种连接的核心定义：

*   **左外连接**：返回**左表**的所有行，以及**右表**中与左表匹配的行。如果右表中没有匹配项，则结果中对应列显示为缺失值。
    *   **SQL代码示例**：`SELECT * FROM table1 LEFT JOIN table2 ON table1.key = table2.key;`
*   **右外连接**：返回**右表**的所有行，以及**左表**中与右表匹配的行。如果左表中没有匹配项，则结果中对应列显示为缺失值。
    *   **SQL代码示例**：`SELECT * FROM table1 RIGHT JOIN table2 ON table1.key = table2.key;`
*   **全外连接**：返回左表和右表中的所有行。当某一行在另一个表中没有匹配时，另一个表的对应列将显示为缺失值。
    *   **SQL代码示例**：`SELECT * FROM table1 FULL JOIN table2 ON table1.key = table2.key;`

## 外连接语法解析

理解了外连接的类型后，我们来看看它的具体语法结构。

在SQL中，外连接的`FROM`子句会列出两个表名，并在它们之间使用关键字来指明连接类型。请注意，这些关键字中不包含“outer”一词。

以下是外连接语法的核心组成部分：

1.  **连接类型关键字**：直接使用 `LEFT JOIN`、`RIGHT JOIN` 或 `FULL JOIN`。
2.  **ON子句**：用于指定连接条件，例如 `ON table1.column = table2.column`。

在确定“左表”和“右表”时，需要依据它们在`FROM`子句中的位置。在左外连接或右外连接中：

*   **左表** 是`FROM`子句中列出的第一个表。结果表中，来自左表的列将首先出现。
*   **右表** 是`FROM`子句中列出的第二个表。结果表中，来自右表的列将随后出现。
