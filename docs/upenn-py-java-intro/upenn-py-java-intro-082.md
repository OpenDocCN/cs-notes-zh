# 082：字符串与列表对比

在本节课中，我们将要学习Python中两个重要的数据类型：字符串和列表。我们将重点探讨它们的相似之处与关键区别，特别是字符串的不可变性。

![](img/436ed40e5f7e21e974f840827171f653_0.png)

## 字符串与列表的相似性

上一节我们介绍了列表的基本概念，本节中我们来看看字符串。字符串可以被视为一个字符序列。从这个角度看，字符串与列表非常相似。你可以想象字符串就是一个由字符组成的列表。

![](img/436ed40e5f7e21e974f840827171f653_2.png)

例如，我们可以像访问列表元素一样，使用方括号和索引来访问字符串中的单个字符。

```python
my_restaurant_choice = "pizza"
first_character = my_restaurant_choice[0]  # 获取第一个字符 'p'
```

## 字符串与列表的关键区别：不可变性

虽然字符串和列表在访问元素上很相似，但它们有一个根本性的区别：**字符串是不可变的**。这意味着一旦一个字符串被创建，你就无法修改其中的单个字符。

相比之下，列表是可变的，我们可以轻松地更新列表中的元素。

![](img/436ed40e5f7e21e974f840827171f653_4.png)

以下是展示这一区别的核心示例：

![](img/436ed40e5f7e21e974f840827171f653_5.png)

对于列表，我们可以进行读取和更新操作：
```python
my_menu_choices = [“pizza”, “burger”, “salad”]
# 1. 读取元素
first_item = my_menu_choices[0]  # 获取 ‘pizza’
# 2. 更新元素
my_menu_choices[0] = “pasta”     # 将第一个元素改为 ‘pasta’，这是允许的
```

然而，对于字符串，我们只能进行读取操作，尝试更新会导致错误：
```python
my_restaurant_choice = “pizza”
# 1. 读取字符
first_char = my_restaurant_choice[0]  # 获取 ‘p’，这是允许的
# 2. 尝试更新字符（这将导致错误）
my_restaurant_choice[0] = “P”         # 错误！字符串不可变
```

尝试执行上述字符串更新代码会引发 `TypeError`，因为Python不允许直接修改字符串的某个部分。

## 如何“改变”字符串

既然字符串不可变，我们该如何修改它们呢？方法不是直接修改原字符串，而是创建一个新的字符串。

以下是几种常见的操作：

![](img/436ed40e5f7e21e974f840827171f653_7.png)

*   **替换字符或子串**：使用 `.replace()` 方法会返回一个新的字符串。
    ```python
    new_string = my_restaurant_choice.replace(‘p’, ‘P’)  # 得到 ‘Pizza’，原字符串不变
    ```
*   **拼接字符串**：使用 `+` 运算符或 `.join()` 方法创建新字符串。
    ```python
    greeting = “Hello” + “ “ + “World”  # 创建新字符串 “Hello World”
    ```
*   **转换大小写**：使用 `.upper()`, `.lower()`, `.title()` 等方法也会返回新字符串。
    ```python
    loud_string = my_restaurant_choice.upper()  # 得到 ‘PIZZA’
    ```

![](img/436ed40e5f7e21e974f840827171f653_8.png)

本节课中我们一起学习了字符串和列表的核心异同。我们了解到字符串可以像列表一样通过索引访问，但其**不可变性**是它与列表最本质的区别。记住，任何看似“修改”字符串的操作，实际上都是创建了一个全新的字符串对象。理解这一点对于避免程序错误和编写高效的Python代码至关重要。
