# 026：完整开发流程总结 🎮

![](img/04ac20b27193dff2ac4a92a77496c329_0.png)

在本节课中，我们将回顾一个完整的游戏开发项目。该项目通过24场直播，从零开始构建了一款游戏，并最终在Steam平台发布。我们将按直播顺序梳理开发过程中的主要任务和核心概念，为初学者提供一个清晰的、循序渐进的开发路线图。

![](img/04ac20b27193dff2ac4a92a77496c329_2.png)

![](img/04ac20b27193dff2ac4a92a77496c329_4.png)

## 概述

本项目始于一个简单的打砖块（Breakout）克隆游戏，通过逐步迭代，最终演变为一款融合了多种街机游戏创意的完整作品。整个开发过程历时24场直播，总计约81小时，涵盖了从项目初始化、核心玩法实现、引擎构建、视听效果优化到最终发布的全流程。

### 从零开始用C语言制作游戏：P26-1：项目启动与基础框架 🏗️

上一节我们介绍了项目的整体情况，本节中我们来看看项目是如何启动并搭建起基础框架的。

![](img/04ac20b27193dff2ac4a92a77496c329_9.png)

![](img/04ac20b27193dff2ac4a92a77496c329_10.png)

![](img/04ac20b27193dff2ac4a92a77496c329_11.png)

在第一场直播中，我们首先建立了游戏代码的结构，并通过一个“Hello World”程序测试了构建流程。核心任务是创建Windows应用程序的入口点，打开游戏窗口，并实现渲染器、输入处理和计时系统的第一个版本。

以下是初始阶段完成的关键步骤：
*   **项目初始化**：创建代码结构，测试构建流程。
*   **窗口创建**：查阅Win32文档，为游戏添加入口点，成功打开应用程序窗口。
*   **基础系统实现**：实现了渲染器、输入处理和计时系统的雏形。

### 从零开始用C语言制作游戏：P26-2：核心玩法实现 🎯

有了基础框架后，我们开始着手实现游戏的核心玩法。

![](img/04ac20b27193dff2ac4a92a77496c329_15.png)

在第二场直播中，我们立即开始制作打砖块克隆的游戏逻辑，并着手组织代码以支持后续的砖块生成。我们实现了玩家挡板和球之间的简单**AABB（轴对齐包围盒）碰撞检测**，以及球与砖块碰撞的第一个版本。

![](img/04ac20b27193dff2ac4a92a77496c329_16.png)

核心概念公式：
*   **AABB碰撞检测**：通过比较两个矩形的边界坐标来判断是否重叠。
    ```c
    // 伪代码示例
    if (rectA.x < rectB.x + rectB.width &&
        rectA.x + rectA.width > rectB.x &&
        rectA.y < rectB.y + rectB.height &&
        rectA.y + rectA.height > rectB.y) {
        // 发生碰撞
    }
    ```

### 从零开始用C语言制作游戏：P26-16：性能分析与优化 🚀

当游戏内容越来越丰富时，性能问题开始显现，我们需要进行优化。

第十六场直播充满了挑战。我们首先解决了一些Bug，然后开始深入进行性能优化。首先，我编程实现了一个**性能分析器**，帮助我们了解游戏在哪些地方运行缓慢。接着我们讨论了一些优化可能性，并对渲染器进行了优化。我们将菜单在4K分辨率下的运行时间从每帧66毫秒（约15 FPS）优化到了每帧16毫秒（60 FPS），并且仍有很大的改进空间。
