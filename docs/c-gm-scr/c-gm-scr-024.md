大约两个月前，我在我的YouTube频道（YouTube.com/DZDan）上开始直播编写这款游戏。整个项目完全在直播中构建，你可以从第一集开始观看每一行代码的诞生。我们首先搭建了平台层和软件渲染器，然后加入了核心玩法。我们专注于游戏的核心——能够玩不同的街机游戏，比如打砖块。接着我们打磨并改进了游戏，加入了粒子系统、旋转矩形等“果汁”效果。我们改进了渲染，加入了位图和亚像素渲染，还做了一些像素艺术。之后我们实现了音频系统和混音器，创建了一个带任务系统的多线程系统，并让音频也实现了多线程。一路走来，我们修复了许多Bug，直到第23集。

![](img/6957357f2033e4641f328a38ff390d7a_45.png)

![](img/6957357f2033e4641f328a38ff390d7a_47.png)

这个项目对我来说是一个完美的学习挑战。它在我技能水平的“舒适区边缘”——足够有挑战性，让我能学到新东西（比如软件渲染位图和旋转矩形），但又没有难到无法完成。关键在于尽快专注于核心玩法，即使最初的版本看起来和最终版相差甚远。在第二集，我们就已经有了游戏玩法的第一个版本，虽然看起来很简陋，但已经可以玩，并能让人了解游戏的大致想法。

![](img/6957357f2033e4641f328a38ff390d7a_49.png)

![](img/6957357f2033e4641f328a38ff390d7a_51.png)

对于初学者，我的建议是：开始时你并不需要很多系统，只需要最简单的、能让你开始构建游戏的基础。之后，你可以逐步添加多线程、存档系统、菜单、音效等。如果你一开始就花10集的时间来搭建引擎，你可能不会完成太多内容，也会失去动力。先做最小可行产品，然后逐步改进。

```
locked_fps = false    // 设置为 false 以解锁帧率
mouse_sensitivity = 1.5 // 调整鼠标灵敏度
windowed = true       // 设置为 true 以窗口模式启动
```

发布流程全部完成后，是时候庆祝了！我们品尝了蛋糕，并试玩了刚刚发布的《Break Arcade Games Out》。我们体验了不同的游戏模式，如打砖块、Pong和太空侵略者，回顾了这两个月来添加的各种效果和优化。

此外，我们还简要试玩了我的上一款游戏《Aeolis Hunt》，这款游戏我投入了三年半的时间。通过对比，我分享了从大型商业项目到小型趣味项目的心态转变。这次轻松的发布体验与第一次发布时的紧张和空虚形成了鲜明对比。
