## 什么是图？ 📊

图是一种用于表示对象之间关系的数据结构。它由**顶点**（或节点）和连接这些顶点的**边**组成。图无处不在，例如互联网网站之间的链接、社交网络中的好友关系、大脑的神经元连接以及航空公司的航线网络。

图分为两种主要类型：
*   **无向图**：边没有方向，关系是双向的。例如，Facebook上的好友关系（如果A是B的好友，那么B也是A的好友）。
*   **有向图**：边有方向，从一个顶点指向另一个顶点。例如，Twitter上的关注关系（A关注B，并不意味着B关注A）。

我们通常用 **`V`** 表示顶点集合，用 **`E`** 表示边集合。图的规模由顶点数 **`n = |V|`** 和边数 **`m = |E|`** 来描述。

## 如何表示图？ 💾

为了在计算机中存储和处理图，我们需要高效的数据结构。主要有两种表示方法：邻接矩阵和邻接列表。

### 邻接矩阵

邻接矩阵是一个 `n x n` 的二维数组（矩阵）。如果图中存在从顶点 `i` 到顶点 `j` 的边，那么矩阵中 `matrix[i][j]` 的位置就标记为1（或边的权重），否则标记为0。

**公式表示：**
`AdjacencyMatrix[i][j] = 1`，如果边 `(i, j) ∈ E`，否则为 `0`。

**示例代码（Python风格）：**
```python
# 假设有n个顶点，编号从0到n-1
adj_matrix = [[0 for _ in range(n)] for _ in range(n)]
# 添加一条从u到v的边
adj_matrix[u][v] = 1
# 对于无向图，还需要添加对称的边
adj_matrix[v][u] = 1
```

### 邻接列表

邻接列表为每个顶点维护一个列表（通常是链表），用于存储与该顶点直接相连的所有邻居顶点。

**示例代码（Python风格）：**
```python
# 使用列表的列表来表示邻接表
adj_list = [[] for _ in range(n)]
# 添加一条从u到v的边
adj_list[u].append(v)
# 对于无向图，还需要在v的列表中添加u
adj_list[v].append(u)
```

### 两种表示法的比较

上一节我们介绍了图的两种表示方法。本节中我们来看看它们在空间消耗和常见操作效率上的权衡。

以下是两种表示法的关键操作复杂度对比：

| 操作 | 邻接矩阵 | 邻接列表 |
| :--- | :--- | :--- |
| **存储空间** | `O(n²)` | `O(n + m)` |
| **检查边 (u, v) 是否存在** | `O(1)` | `O(deg(u))` 或 `O(deg(v))` |
| **遍历顶点 u 的所有邻居** | `O(n)` | `O(deg(u))` |

**说明：**
*   `deg(u)` 表示顶点 `u` 的度数（邻居数量）。
*   对于稀疏图（`m` 远小于 `n²`），邻接列表在空间上优势巨大（例如社交网络图）。
*   邻接矩阵在检查任意两个顶点间是否存在边时速度极快。
*   邻接列表在遍历某个顶点的所有邻居时更高效，因为它只存储实际存在的边。

在实践中，对于大多数算法，尤其是处理大型稀疏图时，我们更常使用**邻接列表**。

## 探索图：深度优先搜索（DFS） 🧗

知道了如何表示图，接下来我们关心如何系统地“探索”或“遍历”一个图。深度优先搜索是一种经典的图遍历算法，其核心思想是“尽可能深”地探索图的分支，直到无法继续，然后回溯并探索其他分支。

### DFS 算法思想

DFS 算法可以类比于探索迷宫：
1.  **粉笔（标记）**：用一个 `visited` 数组记录哪些顶点已经被访问过，避免重复访问。
2.  **绳子（递归栈）**：通过函数的递归调用（或显式栈）来记录当前的探索路径，用于回溯。

算法从某个起始顶点开始，将其标记为已访问，然后递归地访问其每一个未被访问的邻居。当一个顶点的所有邻居都被访问完毕后，算法回溯到上一个顶点。

### DFS 代码实现

以下是基于邻接列表的 DFS 递归实现的核心代码框架：

```python
def dfs(graph, start_vertex):
    """
    graph: 邻接表表示的图
    start_vertex: 起始顶点
    """
    visited = [False] * len(graph) # “粉笔”，标记访问状态

    def explore(v):
        """探索顶点v及其可达的所有顶点"""
        visited[v] = True # 标记为已访问
        print(f"Visiting vertex {v}") # 处理当前顶点（例如打印）

        # 递归地探索所有未访问的邻居
        for neighbor in graph[v]:
            if not visited[neighbor]:
                explore(neighbor)
        # 回溯发生在这里（函数返回）

    explore(start_vertex)
```

### DFS 的性质与应用：连通分量

上面的 `explore` 函数会访问从起始顶点 **`u`** 出发，通过路径所能到达的所有顶点。这个顶点集合被称为 **`u`** 所在的**连通分量**。

对于一个图，可能包含多个互不连通的子图（连通分量）。为了找到图的所有连通分量，我们可以对 DFS 稍作修改：

```python
def find_connected_components(graph):
    n = len(graph)
    visited = [False] * n
    component_id = [0] * n # 记录每个顶点属于哪个连通分量
    current_component = 0

    def explore(v, cid):
        visited[v] = True
        component_id[v] = cid
        for neighbor in graph[v]:
            if not visited[neighbor]:
                explore(neighbor, cid)

    for v in range(n):
        if not visited[v]:
            current_component += 1
            explore(v, current_component)

    return component_id, current_component
```
这个算法遍历所有顶点。每当遇到一个未访问的顶点，就说明发现了一个新的连通分量，于是启动一次 `explore` 调用，并给该分量内的所有顶点分配相同的ID。最终，`component_id` 数组和 `current_component` 计数就完整描述了图的连通结构。

### DFS 树与边分类

在 DFS 遍历过程中，实际经过的边构成了一棵或多棵**DFS树**（如果图不连通，则是DFS森林）。根据边在DFS过程中的作用，我们可以将其分类：

*   **树边**：DFS中用于递归探索新顶点的边，是构成DFS树的骨架。
*   **后向边**：从后代顶点指向其祖先顶点的边。
*   **前向边**：从祖先顶点指向非直系后代顶点的边。
*   **交叉边**：连接DFS树中两个不存在祖先-后代关系的顶点的边。

**一个重要结论**：在**无向图**的DFS中，只存在**树边**和**后向边**，不存在前向边和交叉边。

### 时间戳与区间表示

为了更精细地分析DFS过程，我们引入**时间戳**概念：
*   **pre[v]**：记录第一次访问（开始探索）顶点 `v` 的时刻。
*   **post[v]**：记录完成对顶点 `v` 的所有邻居探索（结束探索）的时刻。

我们可以将每个顶点的活动期看作一个时间区间 `[pre, post]`。这些时间戳揭示了图的重要结构性质：
*   对于树边或前向边 `(u, v)`，有 `pre[u] < pre[v] < post[v] < post[u]`。`v` 的整个活动区间都包含在 `u` 的活动区间内。
*   对于后向边 `(u, v)`，有 `pre[v] < pre[u] < post[u] < post[v]`。`u` 的活动区间包含在 `v` 的活动区间内。
*   对于交叉边 `(u, v)`，有 `pre[v] < post[v] < pre[u] < post[u]`。两个区间完全不相交，且 `v` 的区间整体在 `u` 的区间之前。

### DFS 的复杂度

DFS 需要检查每个顶点和每条边。每个顶点被访问一次，每条边被检查两次（在无向图的邻接表中）。因此，其时间复杂度为 **`O(n + m)`**，这对于遍历整个图来说是最优的。
