## 现实世界的例子：智能手机 📱

上一讲我们开始讨论分解和抽象的概念。今天，我们将通过一个现实世界的例子来具体理解它们，然后看看如何在编程中实现。

智能手机对我们许多人来说就像一个“黑盒子”。我们通过其**输入**（如按键、触摸）和**输出**（如屏幕显示、声音）与它交互，但并不需要了解其内部硬件和软件是如何协同工作的。这种隐藏复杂内部细节，只暴露必要接口的方式，就是**抽象**。

抽象使得**分解**成为可能。手机由数百个不同的部件组成，不同的制造商可以独立生产这些部件。他们只需要遵循一套标准化的**接口规范**（即输入和输出的定义），而无需知道其他部件是如何工作的。只要接口匹配，这些独立的部件就能组合在一起，共同实现手机的功能。

这个原理不仅适用于硬件，同样适用于软件。在编程中，我们将通过**函数**来实现分解和抽象。

## 编程中的抽象与函数

在编程中，我们也希望隐藏细节。我们通过编写**函数**来实现这一点。函数是一段执行特定有用任务的代码块。一旦我们编写并调试好一个函数，就可以将其视为一个“黑盒子”。之后，我们只需要知道它的**输入**（参数）和**输出**（返回值），而无需关心其内部实现。

事实上，我们已经使用过Python内置的函数，例如：
*   `max(a, b)`：输入两个值，返回其中较大的一个。
*   `abs(x)`：输入一个数字，返回其绝对值。
*   `len(s)`：输入一个字符串，返回其长度。

本节课的重点是学习如何**编写你自己的函数**。

## 函数的组成部分

一个函数具有以下特征：
*   **名称**：一个描述其功能的标识符（通常是动词）。
*   **参数**：函数接收的输入（可以有零个或多个）。
*   **文档字符串**：对函数的说明，是一种“契约”，描述了输入、功能以及输出。
*   **函数体**：执行具体任务的Python代码。
*   **返回语句**：使用 `return` 关键字将结果返回给调用者。

![](img/beffc0efe7cfdccbad125a1af0e69bcf_6.png)

![](img/beffc0efe7cfdccbad125a1af0e69bcf_8.png)

以下是编写函数时的最佳实践步骤：
1.  **理解问题**：明确函数需要完成什么任务。
2.  **确定接口**：为函数起名，并确定它需要哪些输入（参数）。
3.  **编写文档字符串**：在 `"""` 之间写明输入、功能和输出。
4.  **编写函数体**：用代码实现功能。
5.  **返回结果**：使用 `return` 语句输出最终值。

## 定义与调用函数

理解**定义函数**和**调用函数**的区别至关重要。

![](img/beffc0efe7cfdccbad125a1af0e69bcf_10.png)

![](img/beffc0efe7cfdccbad125a1af0e69bcf_12.png)

**定义函数**只是告诉Python这段代码的存在，并不会立即执行它。其语法如下：
```python
def is_even(i):
    """
    输入：i，一个正整数。
    功能：判断数字是否为偶数。
    返回：如果i是偶数则返回True，否则返回False。
    """
    return i % 2 == 0
```

![](img/beffc0efe7cfdccbad125a1af0e69bcf_14.png)

![](img/beffc0efe7cfdccbad125a1af0e69bcf_16.png)

**调用函数**才是实际执行函数体内的代码。调用时，需要提供函数名和具体的参数值。函数调用本身是一个**表达式**，它会被其返回值所替代。
```python
result = is_even(3)  # 函数调用被替换为 False，然后赋值给 result
print(result)        # 输出：False
print(is_even(4))    # 输出：True
```

![](img/beffc0efe7cfdccbad125a1af0e69bcf_18.png)

![](img/beffc0efe7cfdccbad125a1af0e69bcf_20.png)

当调用 `is_even(3)` 时，Python会执行以下步骤：
1.  将形式参数 `i` 绑定到实际参数 `3`。
2.  进入函数体，执行代码 `3 % 2 == 0`，得到结果 `False`。
3.  遇到 `return False`，立即退出函数，并将 `False` 返回给调用处。
4.  整个调用表达式 `is_even(3)` 在程序中被替换为值 `False`。

## 函数在内存中的表示

我们可以将函数定义理解为在内存中创建了一个名为 `is_even` 的“变量”，它指向一段可执行的代码（函数对象）。在定义之后、调用之前，这段代码只是静静地待在那里。

![](img/beffc0efe7cfdccbad125a1af0e69bcf_22.png)

![](img/beffc0efe7cfdccbad125a1af0e69bcf_24.png)

只有当进行函数调用时，程序才会“跳转”到那段代码并执行它，然后将返回值带回调用点，用于后续的计算或赋值。

## 实战示例：求区间内奇数的和

![](img/beffc0efe7cfdccbad125a1af0e69bcf_30.png)

让我们通过一个更复杂的例子来巩固所学，并展示**渐进式开发**和**测试**的良好实践。

![](img/beffc0efe7cfdccbad125a1af0e69bcf_32.png)

**问题**：编写函数 `sum_odds(a, b)`，求 `a` 和 `b` 之间（包含两端）所有奇数的和。

**开发思路**：
1.  **先用简单例子思考**：例如 a=2, b=4。手动列出数字 2,3,4。我们的“配方”是：遍历每个数字，如果是奇数就加到总和里。
2.  **先解决一个更简单的问题**：先编写求 `a` 到 `b` 所有数字之和的代码，确保循环逻辑正确。
    ```python
    def sum_all(a, b):
        total = 0
        for i in range(a, b + 1): # 注意 range 的终点不包含，所以要 b+1
            total += i
        return total
    ```
3.  **测试简单版本**：用 `sum_all(2,4)` 测试，结果应为 9。
4.  **添加核心逻辑**：在循环中加入判断，只累加奇数。
    ```python
    def sum_odds(a, b):
        total = 0
        for i in range(a, b + 1):
            if i % 2 == 1: # 或者 if not is_even(i):
                total += i
        return total
    ```
5.  **用更多例子测试**：测试 `sum_odds(2,7)`，结果应为 15 (3+5+7)。

这种分步的方法能帮助你将复杂问题分解，并及早发现和修复错误。

## 总结

![](img/beffc0efe7cfdccbad125a1af0e69bcf_34.png)

本节课我们一起学习了：
*   **分解**：将大问题拆分成小的、独立的模块。
*   **抽象**：隐藏模块的内部细节，只暴露清晰的接口。
*   **函数**：在Python中实现分解和抽象的核心工具。函数通过 `def` 定义，通过函数名和参数调用，并使用 `return` 返回值。
*   **定义 vs 调用**：定义函数是创建功能，调用函数是使用功能。
*   **良好实践**：为函数编写清晰的文档字符串，采用渐进式开发并勤于测试。
