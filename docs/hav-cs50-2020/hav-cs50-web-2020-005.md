## 什么是Git？ 🤔

![](img/930d8b513effaf77964ac86b639a5fa8_10.png)

![](img/930d8b513effaf77964ac86b639a5fa8_12.png)

![](img/930d8b513effaf77964ac86b639a5fa8_14.png)

Git是一个命令行工具，用于跟踪代码的更改。在开发初期，开发者可能通过复制文件来保存旧版本，但这会迅速导致混乱。Git通过保存代码在不同时间点的快照（称为**提交**），使我们能够清晰地跟踪所有修改。

![](img/930d8b513effaf77964ac86b639a5fa8_16.png)

![](img/930d8b513effaf77964ac86b639a5fa8_18.png)

![](img/930d8b513effaf77964ac86b639a5fa8_20.png)

此外，Git在团队协作开发中至关重要。它允许多人同时在同一个项目上工作，并轻松地同步彼此的更改。核心的工作流程涉及一个在线的**代码仓库**，团队成员可以从中**拉取**最新代码，进行修改后再**推送**回去。

![](img/930d8b513effaf77964ac86b639a5fa8_22.png)

Git还提供了其他强大功能，例如：
*   **分支**：允许在不影响主代码的情况下测试新功能。
*   **回退**：可以轻松地将代码恢复到之前的某个版本。

![](img/930d8b513effaf77964ac86b639a5fa8_24.png)

## 开始使用GitHub 🌐

Git项目需要存储在一个中心位置以便协作，GitHub就是这样一个流行的托管平台。一个**GitHub仓库**就像一个包含项目所有文件和修订历史的文件夹。

![](img/930d8b513effaf77964ac86b639a5fa8_26.png)

首先，你需要在 [github.com](https://github.com) 注册一个免费账户。

![](img/930d8b513effaf77964ac86b639a5fa8_28.png)

![](img/930d8b513effaf77964ac86b639a5fa8_30.png)

以下是创建一个新仓库的步骤：
1.  访问 `github.com/new`。
2.  为仓库命名（例如：`hello`）。
3.  （可选）添加描述。
4.  选择仓库可见性（公开或私有）。
5.  点击“Create repository”按钮。

![](img/930d8b513effaf77964ac86b639a5fa8_32.png)

![](img/930d8b513effaf77964ac86b639a5fa8_34.png)

![](img/930d8b513effaf77964ac86b639a5fa8_36.png)

创建后，你会看到一个空的仓库页面，接下来我们需要将其下载到本地电脑进行操作。

![](img/930d8b513effaf77964ac86b639a5fa8_38.png)

## 核心Git命令 💻

### 克隆仓库

![](img/930d8b513effaf77964ac86b639a5fa8_40.png)

要将在线仓库下载到本地计算机，我们使用 `git clone` 命令。这会在你的电脑上创建该仓库的一个完整副本。

**命令格式**：
```bash
git clone <仓库的URL>
```

![](img/930d8b513effaf77964ac86b639a5fa8_42.png)

例如，在终端中执行：
```bash
git clone https://github.com/你的用户名/hello.git
```
这会在当前目录下创建一个名为 `hello` 的文件夹，里面包含了仓库的所有内容（初始时为空）。

![](img/930d8b513effaf77964ac86b639a5fa8_44.png)

### 跟踪与保存更改

Git不会自动保存每一个更改。你需要明确地告诉Git哪些文件的变化需要被记录，然后创建一个**提交**来保存当前状态的快照。

![](img/930d8b513effaf77964ac86b639a5fa8_46.png)

![](img/930d8b513effaf77964ac86b639a5fa8_48.png)

这个过程通常分为两步：
1.  **`git add`**：将文件添加到“暂存区”，标记哪些更改将被包含在下次提交中。
2.  **`git commit`**：创建一个提交，永久记录暂存区中文件的当前状态，并附上一条描述信息。

**命令示例**：
```bash
# 将特定文件添加到暂存区
git add hello.html

![](img/930d8b513effaf77964ac86b639a5fa8_50.png)

# 提交更改，并添加描述信息
git commit -m “添加了hello.html文件”
```
你也可以使用快捷命令一次性添加所有已修改的文件并提交：
```bash
git commit -am “添加了标题”
```

![](img/930d8b513effaf77964ac86b639a5fa8_52.png)

![](img/930d8b513effaf77964ac86b639a5fa8_54.png)

### 与远程仓库同步

到目前为止，所有操作都只影响你本地的仓库。为了与GitHub上的远程仓库同步，需要使用另外两个命令。

![](img/930d8b513effaf77964ac86b639a5fa8_56.png)

*   **`git push`**：将你本地的提交上传到远程仓库（如GitHub）。
*   **`git pull`**：从远程仓库下载最新的提交，更新你的本地副本。

这是团队协作的核心：你`pull`获取同事的最新工作，进行自己的修改并`commit`，然后`push`分享你的工作成果。

![](img/930d8b513effaf77964ac86b639a5fa8_58.png)

![](img/930d8b513effaf77964ac86b639a5fa8_62.png)

**命令示例**：
```bash
# 将本地提交推送到GitHub
git push

# 从GitHub拉取最新更改到本地
git pull
```

![](img/930d8b513effaf77964ac86b639a5fa8_64.png)

![](img/930d8b513effaf77964ac86b639a5fa8_66.png)

![](img/930d8b513effaf77964ac86b639a5fa8_68.png)

### 查看状态

在操作过程中，你可以随时使用 `git status` 命令来查看工作目录和暂存区的状态，例如哪些文件已被修改但未暂存，或者本地分支比远程分支领先多少个提交。

![](img/930d8b513effaf77964ac86b639a5fa8_70.png)

**命令示例**：
```bash
git status
```

## 工作流程示例 🔄

![](img/930d8b513effaf77964ac86b639a5fa8_78.png)

![](img/930d8b513effaf77964ac86b639a5fa8_80.png)

![](img/930d8b513effaf77964ac86b639a5fa8_82.png)

让我们串联起上述命令，看一个完整的本地修改并同步到GitHub的流程：

1.  **克隆仓库**：`git clone <url>`
2.  **进入目录**：`cd hello`
3.  **创建文件**：`touch hello.html`
4.  **编辑文件**：用编辑器编写HTML代码。
5.  **添加文件**：`git add hello.html`
6.  **提交更改**：`git commit -m “初始HTML页面”`
7.  **推送更改**：`git push`
8.  （之后修改文件）
9.  **提交所有修改**：`git commit -am “添加了h1标题”`
10. **再次推送**：`git push`
11. **获取他人更新**：`git pull`
