## 概述

网络模拟器3（NS-3）是一个用于网络研究和教育的离散事件模拟器。与图形界面工具不同，NS-3主要通过终端命令行进行操作。因此，熟悉Linux终端命令是使用NS-3的重要前提。在之前的课程中，我们已经学习了25个基本的Linux命令。本节我们将进入NS-3的目录，了解其结构，并运行第一个“Hello Simulator”程序。

![](img/fbab64eb7b37d00189561b799ee3bf46_5.png)

## 进入工作目录

首先，我们需要进入NS-3的工作目录。所有NS-3的操作都将在 `ns-allinone-3.27/ns-3.27` 这个文件夹中进行。

![](img/fbab64eb7b37d00189561b799ee3bf46_7.png)

```bash
cd ns-allinone-3.27/ns-3.27
```

> **符号说明**：在后续教程中，`~/ns-allinone-3.27/ns-3.27` 将简写为 **`ns-3.27`**，而 `~/ns-allinone-3.27` 将简写为 **`ns-allinone-3.27`**。请理解这种简写方式。

## 核心文件夹结构

![](img/fbab64eb7b37d00189561b799ee3bf46_15.png)

![](img/fbab64eb7b37d00189561b799ee3bf46_17.png)

进入 `ns-3.27` 目录后，使用 `ls` 命令可以查看其包含的多个文件夹。每个文件夹都有其特定的用途。

以下是主要文件夹及其功能的介绍：

![](img/fbab64eb7b37d00189561b799ee3bf46_19.png)

*   **`scratch/`**：这是**执行文件夹**。所有需要运行的示例程序或我们自己编写的代码，都需要复制到这个文件夹中才能执行。
*   **`src/`**：这是**源代码文件夹**。NS-3所有模块的C++源代码都存放在这里。
*   **`build/`**：这是**编译文件夹**。当我们编译NS-3或自己的程序时，生成的可执行文件和库文件会存放在这里。
*   **`examples/`**：这是**示例文件夹**。NS-3官方提供了大量涵盖各种网络场景（如TCP、UDP、无线网络等）的示例程序，供我们学习和参考。

![](img/fbab64eb7b37d00189561b799ee3bf46_21.png)

## 验证安装与运行第一个示例

![](img/fbab64eb7b37d00189561b799ee3bf46_23.png)

在开始复杂模拟之前，我们需要验证NS-3是否安装成功。NS-3自带了一个简单的测试程序。

### 步骤1：运行Hello Simulator

在 `ns-3.27` 目录下，执行以下命令：

```bash
./waf --run hello-simulator
```

![](img/fbab64eb7b37d00189561b799ee3bf46_25.png)

![](img/fbab64eb7b37d00189561b799ee3bf46_27.png)

这个命令会编译并运行一个名为 `hello-simulator` 的简单程序。如果终端输出 “Hello Simulator”，则表明NS-3安装和基本环境配置成功。

### 步骤2：运行教程示例

![](img/fbab64eb7b37d00189561b799ee3bf46_29.png)

![](img/fbab64eb7b37d00189561b799ee3bf46_31.png)

上一节我们验证了基础环境，本节我们来看看如何运行一个真正的网络模拟示例。NS-3在 `examples/tutorial/` 目录下提供了一系列循序渐进的教程。

![](img/fbab64eb7b37d00189561b799ee3bf46_33.png)

我们以第一个教程文件 `first.cc` 为例。运行它的标准流程是将其复制到 `scratch/` 文件夹。

**操作流程如下：**

![](img/fbab64eb7b37d00189561b799ee3bf46_35.png)

1.  **定位源文件**：`first.cc` 文件位于 `ns-3.27/examples/tutorial/` 目录下。
2.  **复制到scratch**：使用 `cp` 命令将其复制到 `scratch/` 文件夹。
    ```bash
    cp examples/tutorial/first.cc scratch/
    ```
    （视频中也演示了在图形界面中直接复制粘贴的方法。）
3.  **编译并运行**：在 `ns-3.27` 目录下，执行以下命令：
    ```bash
    ./waf --run scratch/first
    ```
    命令执行后，NS-3会编译 `scratch/` 文件夹下的程序并运行。你将看到模拟输出的结果，例如客户端发送和接收数据包的时间与大小信息。

本节课中，我们一起学习了NS-3的入门第一步。我们明确了核心工作目录 `ns-3.27`，了解了 `scratch/`, `src/`, `build/`, `examples/` 等关键文件夹的作用。最重要的是，我们通过运行 `hello-simulator` 验证了安装，并掌握了将示例程序（如 `first.cc`）复制到 `scratch/` 文件夹后，使用 `./waf --run scratch/<程序名>` 命令来运行模拟的标准流程。这为我们后续深入学习NS-3的编程和模拟打下了坚实的基础。在下一节，我们将详细分析 `first.cc` 的源代码结构。
