# 软件工程：1：软件工程基础

在本节课中，我们将要学习软件工程的核心概念，理解其与简单编程的区别，并掌握现代软件开发的基本流程。

## 概述

软件工程是应用系统化、规范化、可量化的方法来开发、运行和维护软件。它不仅仅是编写代码，更关注于整个软件生命周期的管理，包括需求分析、设计、实现、测试、部署和维护。与个人编程相比，软件工程强调团队协作、可维护性、可靠性和满足用户需求。

上一节我们介绍了软件工程的定义，本节中我们来看看其核心原则。

## 核心原则

软件工程建立在几个关键原则之上，这些原则指导着开发过程。

以下是几个核心原则：

*   **抽象**：隐藏复杂的实现细节，只暴露必要的接口。例如，使用函数或类来封装功能。
*   **模块化**：将系统分解为独立的、功能明确的部件（模块）。这降低了复杂性，便于团队分工。
*   **封装**：将数据和操作数据的方法捆绑在一起，并对外隐藏内部细节。在面向对象编程中，这通过**类**来实现。
*   **关注点分离**：将程序划分为不同的部分，使得每一部分只负责一个特定的功能或“关注点”。

理解了基本原则后，我们来看看如何将这些原则应用于实际的开发流程中。

## 软件开发流程

一个结构化的流程有助于管理复杂性和风险。常见的模型包括瀑布模型和敏捷开发。

以下是两种主要模型：

*   **瀑布模型**：一种线性顺序的流程，包括需求分析、设计、实现、测试、部署和维护。前一阶段完成后才能进入下一阶段。
*   **敏捷开发**：一种迭代和增量的方法，强调快速交付可工作的软件、客户协作和响应变化。Scrum和极限编程（XP）是流行的敏捷框架。

流程确定了工作的大致阶段，而在每个阶段，我们都需要使用具体的工具和方法来保证质量。接下来，我们将探讨一些关键实践。

## 关键实践

现代软件工程依赖于一系列实践来提高效率和质量。

以下是几项重要实践：

*   **版本控制**：使用如Git的工具来管理代码变更历史，支持团队协作和代码回滚。基本命令包括 `git commit` 和 `git push`。
*   **测试**：编写自动化测试（如单元测试、集成测试）来验证代码行为，确保其正确性并防止回归错误。
*   **持续集成**：频繁地将代码集成到共享主干，并通过自动化构建和测试来快速发现错误。
*   **代码审查**：团队成员相互检查代码，以提升代码质量、分享知识和保持代码风格一致。

这些实践都服务于一个共同的目标：构建高质量的软件。那么，如何衡量软件的质量呢？

## 软件质量属性

优秀的软件应具备一系列质量属性，这些属性需要在开发过程中被设计和构建进去。

以下是一些关键的质量属性：

*   **正确性**：软件行为是否符合规格说明。
*   **可靠性**：软件在指定条件下无故障运行的能力。
*   **可维护性**：软件易于被修改以纠正错误、改进性能或适应环境变化。
*   **可用性**：软件易于学习和使用。
*   **性能**：软件对系统资源（如时间、内存）的利用效率。

## 总结

本节课中我们一起学习了软件工程的基础知识。我们首先明确了软件工程是一门关于构建和维护高质量软件的学科。然后，我们探讨了抽象、模块化等核心原则，介绍了瀑布模型和敏捷开发等流程，并了解了版本控制、测试等关键实践。最后，我们讨论了正确性、可靠性等一系列软件质量属性。掌握这些基础是进一步学习特定方法、技术和工具的前提。
