## 概述

![](img/c7a34747b511647d7bcfff9d3319c533_8.png)

UART是一种常见的串行通信接口。在发送数据前，程序必须检查UART的发送缓冲区是否已满。如果缓冲区已满，则需要等待（即进入“自旋循环”），直到缓冲区空闲后才能发送新数据。本节课我们将编写一个实现此逻辑的子程序。

## 子程序设计与逻辑

![](img/c7a34747b511647d7bcfff9d3319c533_16.png)

上一节我们介绍了UART的基本概念。本节中，我们来看看如何具体实现字符发送功能。

![](img/c7a34747b511647d7bcfff9d3319c533_18.png)

我们将创建一个子程序，它接收一个参数（要发送的字符），并通过UART将其发送出去。核心逻辑是持续检查状态寄存器，直到发送缓冲区空闲为止。

以下是子程序 `uart_write_char` 的主要步骤：

![](img/c7a34747b511647d7bcfff9d3319c533_20.png)

1.  **读取状态寄存器**：加载UART状态寄存器的地址，并读取其当前值。
2.  **检查发送缓冲区标志**：使用“与”操作检查状态寄存器中的“发送缓冲区满”标志位。
3.  **等待循环**：如果标志位显示缓冲区已满，则跳回检查步骤，形成等待循环。
4.  **写入数据**：当缓冲区空闲时，将要发送的字符写入UART的数据寄存器。
5.  **返回**：完成发送后，使用 `BX LR` 指令返回调用处。

## 代码实现详解

![](img/c7a34747b511647d7bcfff9d3319c533_28.png)

以下是 `uart_write_char` 子程序的完整代码实现，我们将逐部分进行解释。

```assembly
uart_write_char:
    BX LR                         ; 子程序返回
```

![](img/c7a34747b511647d7bcfff9d3319c533_30.png)

![](img/c7a34747b511647d7bcfff9d3319c533_32.png)

首先，我们定义了子程序的标签 `uart_write_char` 和返回指令 `BX LR`。接下来填充其具体逻辑。

![](img/c7a34747b511647d7bcfff9d3319c533_34.png)

![](img/c7a34747b511647d7bcfff9d3319c533_36.png)

```assembly
uart_write_char:
    LDR R1, =UART2_SR             ; 步骤1：加载状态寄存器地址到R1
    LDR R2, [R1]                  ; 读取状态寄存器的值到R2

![](img/c7a34747b511647d7bcfff9d3319c533_38.png)

check_tx_flag:
    AND R2, R2, #UART_TX_BUFFER_FLAG ; 步骤2：检查发送缓冲区满标志位
    CMP R2, #0                    ; 比较结果是否为0（缓冲区是否空闲）
    BEQ check_tx_flag             ; 步骤3：如果为0（缓冲区满），则跳回继续检查

    UXTB R1, R0                   ; 步骤4：将R0中的字符零扩展后存入R1
    LDR R2, =UART2_DR             ; 加载数据寄存器地址到R2
    STR R1, [R2]                  ; 将R1中的字符写入数据寄存器

    BX LR                         ; 步骤5：返回
```

![](img/c7a34747b511647d7bcfff9d3319c533_40.png)

**代码关键点说明：**
*   `LDR R1, =UART2_SR`：这是一个伪指令，用于将标签 `UART2_SR`（代表状态寄存器内存地址）加载到寄存器R1中。
*   `LDR R2, [R1]`：这是一条加载指令，读取R1中地址所指向的内存内容（即状态寄存器的值），并存入R2。
*   `AND R2, R2, #UART_TX_BUFFER_FLAG`：**与操作**。将R2的值与一个掩码（`UART_TX_BUFFER_FLAG`）进行按位与运算，目的是只保留我们关心的“发送缓冲区满”标志位，结果存回R2。
*   `CMP R2, #0` 与 `BEQ check_tx_flag`：**比较与条件分支**。`CMP` 指令比较R2是否等于0。`BEQ`（Branch if EQual）指令在相等时（即标志位为0，缓冲区满）跳转到标签 `check_tx_flag` 处，形成循环等待。
*   `UXTB R1, R0`：**零扩展字节指令**。将R0中的低8位数据（一个字节的字符）零扩展为32位后，存入R1。这确保了数据格式正确。
*   `STR R1, [R2]`：**存储指令**。将R1中的32位数据，写入R2中地址所指向的内存位置（即UART数据寄存器）。执行此操作后，UART硬件便会开始发送该字符。

## 总结

![](img/c7a34747b511647d7bcfff9d3319c533_48.png)
