## 数据结构回顾

![](img/f387e97312d3a877230d6843892be7f0_12.png)

首先，快速回顾一下我们决定使用的三种字典数据结构：

![](img/f387e97312d3a877230d6843892be7f0_14.png)

*   **名称-评分字典**：将餐厅名称与其评分（喜欢该餐厅的人数百分比）关联起来。
    *   **公式/代码**：`name_to_rating: {str: int}`
*   **价格-名称字典**：将价格范围与该价格范围内的餐厅名称列表关联起来。
    *   **公式/代码**：`price_to_names: {str: list of str}`
*   **菜系-名称字典**：将菜系类型与提供该菜系的餐厅名称列表关联起来。
    *   **公式/代码**：`cuisine_to_names: {str: list of str}`

我们还设计了一个例子：对于价格“$”和菜系“中餐”、“泰餐”，程序应输出一个包含两个子列表的列表：`[[82, 'Queen Street Cafe'], [71, 'Dumplings Are Us']]`。注意，评分在前，餐厅名在后，这样设计的原因将在后续说明。

## 主函数设计

我们的主函数目标是：根据文件内容、用户选择的价格和菜系列表，返回一个符合条件的餐厅列表，并按评分排序。

**函数头设计如下：**
```python
def recommend(file: TextIO, price: str, cuisines_list: list) -> list:
```
这个函数接收一个已打开的文件对象、用户选择的价格字符串和一个菜系列表，返回一个形如`[评分, 餐厅名]`的列表的列表，并按评分排序。

## 程序设计步骤

以下是实现推荐功能的主要步骤：

1.  **读取文件并构建数据结构**：读取文件，构建上述三个字典。
2.  **按价格筛选**：根据用户选择的价格，从`price_to_names`字典中获取对应价格区间的餐厅名称列表。
3.  **按菜系筛选**：从上一步得到的餐厅列表中，筛选出提供用户指定菜系之一的餐厅。
4.  **构建评分列表并排序**：根据筛选后的餐厅名称列表，从`name_to_rating`字典中查找评分，构建`[评分, 餐厅名]`的列表，并按评分排序。
5.  **返回结果**：返回排序后的列表。

这些精心设计的步骤也为我们提供了检查点，可以在程序运行过程中验证每一步是否正确。

接下来，让我们逐一仔细分析这些步骤。

## 步骤一：读取文件并构建数据结构

读取文件并构建三个字典（`name_to_rating`, `price_to_names`, `cuisine_to_names`）听起来有些复杂。为了保持函数简短且易于测试，我们将为此编写一个独立的函数。

我们将其命名为`read_restaurants`，它接收一个文件对象作为参数，并返回包含上述三个字典的元组。

**函数头设计如下：**
```python
def read_restaurants(file: TextIO) -> (dict, dict, dict):
```
这个函数的工作原理是逐行读取文件，并为每一家餐厅将相关信息累积到对应的字典中。我们需要三个累加器变量来分别构建这三个字典。

![](img/f387e97312d3a877230d6843892be7f0_16.png)

我们建议你先尝试实现构建`name_to_rating`字典的部分，确保工作正常后，再添加构建`price_to_names`和`cuisine_to_names`字典的逻辑。注意，你只需要遍历文件一次。在解析逗号分隔的菜系列表以构建`cuisine_to_names`字典时，可能会遇到一些困难，考虑为此编写一个辅助函数。

## 步骤二：按价格筛选

![](img/f387e97312d3a877230d6843892be7f0_18.png)

这一步相对简单直接。我们已经有了`price_to_names`字典和用户选择的价格`price`，只需从字典中查找对应价格键的值即可获得餐厅名称列表。

## 步骤三：按菜系筛选

在得到符合价格要求的餐厅名称列表后，我们需要从中筛选出提供用户指定菜系之一的餐厅。为此，我们设计一个`filter_by_cuisine`函数。

我们需要哪些数据？文件已读取完毕，不再需要。价格信息也已使用过。我们需要的是：符合价格的餐厅名称列表`names_matching_price`、菜系-名称字典`cuisine_to_names`以及用户指定的菜系列表`cuisines_list`。

**函数头设计如下：**
```python
def filter_by_cuisine(names_matching_price: list, cuisine_to_names: dict, cuisines_list: list) -> list:
```
这个函数返回一个字符串列表，包含所有既在指定价格区间内，又提供指定菜系之一的餐厅名称。

其算法是：遍历`names_matching_price`列表中的每个餐厅名，检查它是否出现在`cuisines_list`中任何一个菜系对应的餐厅列表里（通过查询`cuisine_to_names`字典）。如果是，则将其加入结果列表。

## 步骤四：构建评分列表并排序

现在，我们有了符合条件的餐厅名称列表。接下来需要构建`[评分, 餐厅名]`格式的列表的列表，并按评分排序。我们设计一个`build_rating_list`函数来完成这个任务，并让它同时负责排序。

我们需要传入`name_to_rating`字典和筛选后的最终餐厅名称列表`names_final`。

**函数头设计如下：**
```python
def build_rating_list(name_to_rating: dict, names_final: list) -> list:
```
这个函数返回一个按评分百分比排序的`[评分百分比, 餐厅名]`对的列表。

其算法是：遍历`names_final`列表，对于每个餐厅名，在`name_to_rating`字典中查找其评分，创建一个包含`[评分, 餐厅名]`的新列表，并将其添加到一个累加器变量中。最后，对这个累加器列表调用`.sort()`方法进行排序。

现在，我们可以解释为什么将评分放在列表首位：Python列表的`.sort()`方法在排序列表的列表时，默认依据每个子列表的第一个元素进行排序。因此，将评分放在前面，我们只需对整个结果列表调用一次`.sort()`即可实现按评分排序。

## 步骤五：整合与返回

最后，在我们的主`recommend`函数中，我们只需要按顺序调用这些辅助函数，并返回`build_rating_list`函数的结果即可。
