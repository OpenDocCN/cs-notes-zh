## 什么是异常？🤔

在编程中，我们经常会遇到错误。例如，尝试除以零就会引发一个错误。

![](img/6a6401b7237c31f750a969e1e59d9643_6.png)

```python
result = 10 / 0
```

![](img/6a6401b7237c31f750a969e1e59d9643_8.png)

当错误发生时，Python会生成一个回溯信息。这个信息告诉我们调用栈上的函数和方法列表。通常，第一行显示问题发生的Python语句，最后一行显示错误的类型和描述信息。

![](img/6a6401b7237c31f750a969e1e59d9643_10.png)

![](img/6a6401b7237c31f750a969e1e59d9643_11.png)

## 常见的异常类型

以下是几种常见的异常类型及其触发场景。

![](img/6a6401b7237c31f750a969e1e59d9643_13.png)

### ValueError（值错误）

![](img/6a6401b7237c31f750a969e1e59d9643_15.png)

当我们尝试执行一个操作，但提供的参数值不合适时，就会引发`ValueError`。

![](img/6a6401b7237c31f750a969e1e59d9643_17.png)

例如，尝试在字符串中查找不存在的子串索引：
```python
index = ‘ABC‘.index(‘Q‘)
```

![](img/6a6401b7237c31f750a969e1e59d9643_19.png)

或者尝试将非数字字符串转换为整数：
```python
num = int(‘Uga‘)
```

![](img/6a6401b7237c31f750a969e1e59d9643_21.png)

虽然异常类型相同，但错误信息会具体描述问题所在。`ValueError`意味着函数或方法被要求执行一个它无法完成的操作。

### ZeroDivisionError（除零错误）

当我们要求Python进行除以零或取模零的运算时，就会引发`ZeroDivisionError`。

```python
result = 10 / 0
```

`ZeroDivisionError`是`ArithmeticError`（算术错误）的一个子类。

## Python的异常处理语句

Python提供了`try...except`语句来处理异常。我们可以尝试执行一段代码，如果出现问题，则执行`except`子句中的代码。

![](img/6a6401b7237c31f750a969e1e59d9643_23.png)

以下是`try...except`语句的基本结构：

![](img/6a6401b7237c31f750a969e1e59d9643_25.png)

![](img/6a6401b7237c31f750a969e1e59d9643_27.png)

```python
try:
    # 尝试执行的代码块
    result = 10 / 0
    print("这行不会被执行")
except ZeroDivisionError:
    # 当发生ZeroDivisionError时执行的代码块
    print("捕获到除零错误！")
```

当异常被引发时，同一代码块中后续的代码将不会被执行。在`except`子句中，我们可以指定我们感兴趣的异常类型。我们也可以使用多个`except`子句来处理不同类型的异常，Python会按顺序尝试匹配每个子句。

![](img/6a6401b7237c31f750a969e1e59d9643_29.png)

```python
try:
    # 可能引发多种异常的代码
    value = int(‘abc‘)
except ValueError:
    print("值错误")
except ZeroDivisionError:
    print("除零错误")
```

![](img/6a6401b7237c31f750a969e1e59d9643_31.png)

每个异常都是一个对象，我们可以给它指定一个变量名，以便在`except`块中使用。

![](img/6a6401b7237c31f750a969e1e59d9643_33.png)

```python
try:
    result = 10 / 0
except ZeroDivisionError as zde:
    print(f“捕获到异常：{zde}”)
```

我们还可以使用`raise`语句主动引发异常。

```python
def raise_an_exception(value):
    if value < 0:
        raise ValueError(“值不能为负数”)
```

![](img/6a6401b7237c31f750a969e1e59d9643_35.png)

## 使用断言检查前置条件

![](img/6a6401b7237c31f750a969e1e59d9643_37.png)

断言是一种用于调试的语句，用于检查程序中的某个条件是否为真。如果条件为假，则会引发`AssertionError`。

```python
x = 3
assert x == 3  # 条件为真，无输出
assert x == 4  # 条件为假，引发AssertionError
```

我们可以使用`assert`语句来检查函数的前置条件是否被满足。

```python
def factorial(n):
    assert n >= 0, “n必须是非负整数”
    # 计算阶乘的代码...
```

许多程序员使用断言来确认前置条件是否得到满足。如果违反前置条件，程序会立即停止并给出错误信息。

## 异常处理的最佳实践

对Python异常的全面讲解可能需要一整周的课程，但本节旨在为你提供一个概览，以便你能阅读和理解异常处理代码。以下是几个在自行探索异常时可以牢记的要点：

1.  **异常用于处理异常情况**：即程序无法完成其预定任务的情况。
2.  **只捕获你知道如何处理的异常**：这通常发生在用户界面代码中，当用户输入无效值时，我们可以捕获异常并提示用户重新输入。
3.  **“低抛高接”原则**：错误通常在程序深处被发现，而该处的程序员往往不知道如何恢复。因此，他们会抛出一个异常。被抛出的异常会沿着调用栈向上传播，直到被某个`except`子句捕获。让程序直接崩溃通常不是好主意，这意味着在程序非常靠前的位置（高层）应该有一个通用的`try...except`来捕获错误，并至少为用户提供信息丰富的消息。
4.  **使用断言检查前置条件**：许多程序员认为使用`assert`语句来确认前置条件是否得到满足是明智的做法。
