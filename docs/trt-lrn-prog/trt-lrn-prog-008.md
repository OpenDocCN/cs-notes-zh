# 008：使用 doctest 进行自动化测试 🧪

![](img/a37c1b81fa14384784903ec50252c7b9_0.png)

在本节课中，我们将学习如何使用 Python 的 `doctest` 模块来自动化执行函数设计流程中的示例调用测试。这将帮助我们更高效、更准确地验证代码功能。

---

## 概述

![](img/a37c1b81fa14384784903ec50252c7b9_2.png)

作为函数设计流程的一部分，我们会在函数的文档字符串中包含一两个示例调用。设计流程的最后一步是执行这些函数调用。到目前为止，我们一直通过复制粘贴示例到 Python 交互式环境（Shell）中，然后手动比较返回值与预期值来执行测试。

本节将介绍 `doctest` 模块，它允许我们以更自动化的方式运行这些测试。

![](img/a37c1b81fa14384784903ec50252c7b9_4.png)

## 示例一：测试元音收集函数

在屏幕右侧，有一个名为 `vowels.py` 的模块，其中包含两个函数定义：`collect_vowels` 和 `count_vowels`。

每个函数的文档字符串中都包含两个示例调用。

为了执行这些示例调用，我们需要运行该模块，然后将每个函数调用复制粘贴到 Python Shell 中。执行函数调用后，我们比较返回值与预期值，以确保它们匹配。

![](img/a37c1b81fa14384784903ec50252c7b9_8.png)

与其逐个调用每个示例，不如能够一次性运行所有测试。Python 的 `doctest` 模块正好可以实现这一点。

以下是使用 `doctest` 的步骤：

![](img/a37c1b81fa14384784903ec50252c7b9_10.png)

1.  首先导入 `doctest` 模块。
2.  然后调用 `doctest.testmod()` 函数来运行该模块的所有测试。

```python
import doctest
doctest.testmod()
```

![](img/a37c1b81fa14384784903ec50252c7b9_12.png)

执行后，我们会看到测试结果。在这个例子中，四个测试都已执行，并且全部通过。

`doctest` 的一个优点是它能自动比较函数调用的返回值与我们期望的值。这使得测试更不容易出现人为错误，因为我们无需自己进行这两个值的比较。

![](img/a37c1b81fa14384784903ec50252c7b9_14.png)

## 示例二：测试除数获取函数

现在，让我们考虑另一个例子。

![](img/a37c1b81fa14384784903ec50252c7b9_18.png)

在屏幕右侧，有一个名为 `divisors.py` 的模块，其中包含一个名为 `get_divisors` 的函数。该函数有两个参数：一个数字 `number` 和一个该数字的可能除数列表 `possible_divisors`。

函数返回列表中所有能被该数字整除的项。

例如，对于数字 8 和可能除数列表 `[1, 2, 3]`，函数调用将返回除数列表 `[1, 2]`。8 不能被 3 整除，因此 3 不包含在函数返回的列表中。

![](img/a37c1b81fa14384784903ec50252c7b9_20.png)

这个函数的文档字符串中还有另一个示例。让我们通过运行模块并调用 `doctest` 来执行这两个示例调用。

![](img/a37c1b81fa14384784903ec50252c7b9_22.png)

这次从 `doctest` 返回的信息比元音示例长得多。在元音示例中，所有四个测试用例都通过了。而这次，执行了两个示例测试用例，但其中一个失败了。

![](img/a37c1b81fa14384784903ec50252c7b9_24.png)

测试失败意味着函数调用的实际返回值与预期值不匹配。

![](img/a37c1b81fa14384784903ec50252c7b9_26.png)

在这个案例中，是第二个示例调用失败了。对于 `get_divisors(4, [-2, 0, 2])`，执行函数调用时发生了错误。

错误是 `ZeroDivisionError`，在我们的函数中某处除以了 0。让我们更仔细地查看代码，看看这是在哪里发生的。

从错误信息可以看出，零除错误发生在第 15 行，也就是这个 `if` 语句的条件判断处。

![](img/a37c1b81fa14384784903ec50252c7b9_28.png)

```python
if number % item == 0:
```

![](img/a37c1b81fa14384784903ec50252c7b9_30.png)

![](img/a37c1b81fa14384784903ec50252c7b9_31.png)

这个布尔表达式将数字除以可能的除数，以检查是否能整除。此时，我们可以查看我们的示例。在可能的除数列表中有一个 0，这意味着在第 15 行，我们正在将数字除以 0。

既然找到了问题，就该修复它。我们需要确保数字只被非零数除。为此，我们将在 `if` 语句中添加一个额外的布尔表达式。

```python
if item != 0 and number % item == 0:
```

只要 `item` 不等于 0 并且数字能被该项整除，它就会被添加到函数最后返回的除数列表中。

回想一下，Python 使用惰性求值来评估 `and` 操作。也就是说，如果第一个操作数为假，则整个 `and` 表达式为假，Python 不会继续计算第二个操作数。换句话说，当 `item` 等于 0 时，`and` 右侧的表达式不会被执行，因此不会发生除以零的情况。

现在是时候重新运行测试了。我们将重新运行模块以获取该函数的最新定义，然后从 `doctest` 模块运行 `testmod`。

虽然我们不再得到零除错误，但测试用例仍然失败。我们期望得到一个包含整数 2 的列表，但实际上我们得到了一个包含 `-2` 和 `2` 的两项列表。

这次，问题不在于函数定义，而在于测试用例本身。`-2` 是 4 的除数，它本应被包含在列表中。我们现在将其添加进去。

接下来，我们需要重新运行模块并重新测试。这次，两个测试用例都通过了。

![](img/a37c1b81fa14384784903ec50252c7b9_33.png)

## 将测试集成到模块中

除了在 Shell 中导入 `doctest` 并运行 `testmod`，我们还可以在 `divisors` 模块内部完成这个操作。

![](img/a37c1b81fa14384784903ec50252c7b9_37.png)

现在，每次运行 `divisors` 模块时，测试用例也会被执行。如果测试用例通过，Shell 中不会有输出——没有消息就是好消息。

![](img/a37c1b81fa14384784903ec50252c7b9_39.png)

```python
if __name__ == '__main__':
    import doctest
    doctest.testmod()
```

## 总结

![](img/a37c1b81fa14384784903ec50252c7b9_41.png)

在本节课中，我们一起学习了如何使用 Python 的 `doctest` 模块来自动化测试函数。我们了解了如何将示例调用写入文档字符串，如何使用 `doctest.testmod()` 运行所有测试，以及如何解读测试结果（通过或失败）。通过修复一个因除以零和测试用例不完整而导致的错误，我们实践了调试和修正代码的过程。最后，我们还学习了如何将测试代码集成到模块中，实现每次运行模块时自动执行测试。自动化测试是编写高质量、可靠代码的重要工具。
