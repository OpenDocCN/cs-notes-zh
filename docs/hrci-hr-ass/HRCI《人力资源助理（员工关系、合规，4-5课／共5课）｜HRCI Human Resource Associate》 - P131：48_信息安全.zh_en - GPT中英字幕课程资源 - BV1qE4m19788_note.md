## 🎭 社会工程学攻击方式  



![](img/657e248acd84c7c33ef8ae303dfa6433_2.png)

上一节我们回顾了尾随进入（Tailgating）和冒充（Impersonation）等物理安全漏洞。本节将在此基础上，进一步介绍其他常见的社会工程学攻击手段。  



以下是几种常见的社会工程学攻击方式：  



### 1️⃣ 网络钓鱼（Phishing）  



**定义：**  

```text
Phishing = 伪装成可信来源 + 发送欺诈信息 + 诱导泄露敏感数据
```  



网络钓鱼是一种欺骗手段，网络犯罪分子通过电子邮件、电话录音或社交媒体信息发送看似合法可信的消息。  

这些信息如同“诱饵”，诱导受害者泄露有价值的信息。  

受害者误以为信息真实可信，从而主动提供个人数据。  



---  



### 2️⃣ 语音钓鱼（Vishing）  



**定义：**  

```text
Vishing = Voice + Phishing
```  



![](img/657e248acd84c7c33ef8ae303dfa6433_4.png)

语音钓鱼是一种通过电话实施的网络犯罪。  

例如，你可能会接到一通预录语音电话，声称来自你的银行。  

电话要求你验证账户号码等敏感信息。  

录音听起来十分真实，从而诱导你输入个人信息。  

这种欺骗可能导致犯罪分子获得未经授权的账户访问权限，进而造成财务损失或身份盗用。  



---  



### 3️⃣ 交换条件攻击（Quid Pro Quo Attack）  



![](img/657e248acd84c7c33ef8ae303dfa6433_6.png)

**定义：**  

```text
Quid Pro Quo = 提供帮助 + 换取敏感信息
```  



攻击者随机拨打组织内部电话，假装来自技术支持团队。  

他们提供技术帮助，以换取敏感信息。  

例如，犯罪分子声称正在回拨电话，希望联系到真正需要技术协助的人。  

![](img/657e248acd84c7c33ef8ae303dfa6433_8.png)

攻击者伪装成专业技术人员，指导受害者输入命令，从而安装恶意软件或授予其对计算机及组织网络的非法访问权限。  



---  



### 4️⃣ 垃圾翻找（Dumpster Diving）  



**定义：**  

```text
Dumpster Diving = 在丢弃的垃圾中寻找敏感信息
```  



黑客可能通过翻找垃圾获取：  

![](img/657e248acd84c7c33ef8ae303dfa6433_10.png)

- 门禁代码  
- 写在便签上的密码  
- 未被粉碎的敏感文件  
- 日历  
- 电话列表  
- 组织结构图  



---  



![](img/657e248acd84c7c33ef8ae303dfa6433_12.png)

### 5️⃣ 诱饵攻击（Baiting）  



**定义：**  

```text
Baiting = 投放诱饵设备 + 用户插入设备 + 恶意软件感染
```  



假设你在地上发现一个U盘，将其插入电脑查看归属。  

![](img/657e248acd84c7c33ef8ae303dfa6433_14.png)

这种看似无害的行为就是诱饵攻击。  

攻击者故意投放U盘、闪存盘、光盘等可移动设备。  

当有人将其插入电脑时，设备可能感染恶意软件，导致敏感数据泄露。  



---  



### 6️⃣ 肩窥攻击（Shoulder Surfing）  



**定义：**  

```text
Shoulder Surfing = 通过观察获取信息
```  



攻击者可能站在你身后，在你使用ATM、电脑或电子设备时窃取信息。  

![](img/657e248acd84c7c33ef8ae303dfa6433_16.png)

攻击者甚至可能使用望远镜或其他视觉增强设备，从远距离进行观察。

## 🛡 访问控制机制  



了解了社会工程学攻击方式之后，本节将介绍如何通过访问控制来降低风险。  



![](img/657e248acd84c7c33ef8ae303dfa6433_18.png)

**访问控制的核心目标：**  

```text
在正确的时间，让正确的人，访问正确的信息
```  



访问控制解决两个关键问题：  

![](img/657e248acd84c7c33ef8ae303dfa6433_20.png)

1. 谁可以访问哪些信息  
2. 访问者是否为其声称的身份  



以下是访问控制的关键组成部分：  



### 1️⃣ 身份识别（Identification）  



**定义：**  

```text
Identification = 声明身份（用户名 / 账户编号）
```  



身份识别是访问控制的第一步。  

用户通过账户名或编号声明身份。  

一旦建立身份，系统便分配相应的访问级别，从而维持组织的信息安全与控制。  



---  



### 2️⃣ 身份验证（Authentication）  



**定义：**  

```text
Authentication = 验证身份真实性
```  



组织通过以下方式验证身份：  

- 密码或PIN码  
- 身份证件或令牌  
- 指纹识别  



**双因素认证（2FA）：**  

```text
2FA = 知识因素（密码） + 持有因素（手机验证码）
```  



用户除输入密码外，还需输入发送至智能手机的验证码。  

另一种方式是使用软硬件令牌生成与系统同步的唯一代码。  

该代码与密码一起输入，增加额外安全层。  



---  



### 3️⃣ 授权（Authorization）  



**定义：**  

```text
Authorization = 决定访问权限范围
```  



系统判断用户是否具有执行特定操作或访问资源的权限。  

权限可能根据：  

- 工作职责  
- 职级  
- 组织分类  



进行限制。  



---  



### 4️⃣ 审计日志（Audit Logs）  



**定义：**  

```text
Audit Log = 记录 + 监控 + 追踪用户行为
```  



审计日志通过记录用户操作，建立责任追踪机制。  

![](img/657e248acd84c7c33ef8ae303dfa6433_22.png)

组织可以确保员工按照政策使用系统。  

审计日志还能识别未经授权的访问或可疑行为，从而威慑恶意活动。
