# 152：风险管理政策示例

![](img/b94d8376aa2b73307d655e72a2993554_0.png)

在本节课中，我们将通过一个真实案例，学习一家组织如何制定风险管理政策。我们将跟随人力资源专业人士Neri，了解她为Urban Attire公司制定业务连续性计划以应对网络攻击的全过程。

![](img/b94d8376aa2b73307d655e72a2993554_2.png)

## 案例背景：Urban Attire公司

上一节我们介绍了风险管理的基本概念，本节中我们来看看一个具体的应用实例。

Urban Attire是一家专注于现代都市休闲服饰的中型企业。公司拥有许多员工，其人力资源团队正在更新适用于全体成员的风险管理政策。

具体而言，人力资源专员Neri正在为Urban Attire制定一项业务连续性计划。近期，Urban Attire的一家竞争对手遭遇了网络犯罪团伙的黑客攻击。黑客入侵了其系统，挟持了数据以索要赎金，并导致该公司在两周内无法在线下门店或线上渠道接受信用卡支付。

此次数据泄露对竞争对手造成了巨大影响，他们不得不在第二周关闭了部分门店。尽管最终通过更换系统恢复了运营，但该公司蒙受了巨大的经济损失，品牌信誉也严重受损。

## 制定业务连续性计划

![](img/b94d8376aa2b73307d655e72a2993554_4.png)

Neri对此事件感到震惊，并意识到类似情况也可能发生在Urban Attire。她决定是时候制定一项专门针对网络攻击的业务连续性计划了。

以下是Neri制定计划所遵循的步骤：

**第一步：进行影响分析**

Neri首先进行了影响分析。在零售业中，经济损失的多少取决于事发时间。因此，她回顾了公司的淡季、平季和旺季，以了解在不同时期发生销售中断可能意味着什么。

Neri还与IT团队、管理团队以及第三方网络安全专家进行了沟通，以确定所需的应急资源和响应措施，尤其是在关键信息或基础设施被黑客入侵的情况下。毋庸置疑，处理此类攻击成本高昂且极其耗时。

**第二步：生成恢复策略**

![](img/b94d8376aa2b73307d655e72a2993554_6.png)

接下来，Neri开始制定恢复策略。

她首先与IT团队合作，了解哪些系统可以进行备份、外部存储以及在遭遇黑客攻击后能够重新安装。

同时，她与一家网络安全公司签订了合同。该公司将主动探测系统漏洞，并在黑客攻击发生时提供协助。

**第三步：起草业务连续性计划**

之后，Neri起草了业务连续性计划。该计划包含了标准操作程序、恢复团队、联络人以及Urban Attire在遭遇网络犯罪时可以使用的消息模板。

![](img/b94d8376aa2b73307d655e72a2993554_8.png)

Neri将计划提交给人力资源部门主管以及高管团队审议。经过一些细微修改后，该计划获得了批准。

**第四步：开展演练与更新**

最后，Neri与第三方网络安全公司以及Urban Attire的IT团队合作，设计了一些实践演练场景。

![](img/b94d8376aa2b73307d655e72a2993554_10.png)

所有参与黑客攻击演练的人员都对培训课程感到兴奋和感兴趣。他们甚至发现了一些意外情况，促使Neri对计划进行了更新。

## 总结与意义

随着计划的落实，Neri确信Urban Attire已经为应对网络攻击做好了准备。当然，此类风险无法被完全消除，但拥有完善的计划将有助于至少减轻部分风险。

Neri的案例到此告一段落。研究、编写和演练风险管理政策是人力资源部门的一项关键职责。一项优秀的计划甚至能够化解巨大且代价高昂的风险，但这需要深思熟虑和努力才能创建。

![](img/b94d8376aa2b73307d655e72a2993554_12.png)

本节课中，我们一起学习了通过具体案例制定风险管理政策的过程，重点关注了业务连续性计划的四个核心步骤：**影响分析**、**恢复策略制定**、**计划起草**以及**演练更新**。理解并实施这些步骤，是有效管理组织风险、保障业务持续运营的基础。
