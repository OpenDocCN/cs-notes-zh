# 086：原子操作

![](img/dd058d8035a0d50441bb9abd64ef48a2_0.png)

在本节课中，我们将要学习计算机体系结构中的原子操作。原子操作是实现互斥锁和信号量等同步原语的基础。我们将探讨几种常见的原子操作指令，如测试与设置、比较与交换、交换以及加载链接/条件存储，并分析它们的工作原理、实现方式以及性能特点。

## 信号量的实现

上一节我们介绍了信号量的概念。信号量是实现互斥和同步的一种机制。然而，要实现信号量，我们首先需要一种方法来保证互斥访问。

### 原子操作的必要性

实现互斥访问，理论上仅使用普通的加载和存储指令也可以做到，但这种方式通常非常慢。从实现的角度来看，人们通常追求更快的速度。因此，计算机体系结构或指令集架构会引入特殊的指令，允许我们以原子方式执行操作。

所谓“原子操作”，意味着在执行该操作期间，不会有其他任何操作被穿插进来。一个简单的解决方案是添加一个原子操作，该操作可以原子性地完成读取、修改和写入这三个步骤。

回到锁和信号量的实现，在P操作和V操作的内部，我们需要以某种方式原子性地修改信号量S。我们需要能够读取S，也需要能够写入S。实现这一点的关键原语是拥有某种“读-修改-写”操作。

## 常见的原子操作

以下是几种常见的原子操作指令。

### 测试与设置

测试与设置是最基本的原子操作之一，x86架构就有这条指令。许多其他架构也有类似的指令。我们用一段伪代码来描述它：

```c
int TestAndSet(int *lock) {
    int old = *lock; // 原子性地读取
    *lock = 1;       // 原子性地写入1
    return old;      // 返回旧值
}
```

这段代码在现实中是一条指令，相对于系统中的所有其他操作，它是原子执行的。

测试与设置指令接收一个内存地址（例如锁或信号量的地址）和一个寄存器。它无条件地将内存地址的值加载到寄存器中，然后检查该值是否为0。如果是0，则向该内存地址写入1。关键在于，架构保证所有这些步骤是原子发生的。

从硬件实现角度看，这需要执行一次加载和一次存储。一种早期的实现方式（例如在一些基于x86的架构上）是使用总线上的“锁定位”。当一个处理器要执行原子操作时，它会向系统中的所有其他处理器广播信号，阻止它们发出任何内存指令，然后执行加载、测试和设置，最后释放广播位。这是一种比较“粗暴”的实现，现代技术已经有了很大改进。

### 获取并加

获取并加是另一种原子操作。它同样原子性地完成读取、修改和写入。

```c
int FetchAndAdd(int *value, int increment) {
    int old = *value;          // 原子性地读取
    *value = old + increment; // 原子性地写入新值
    return old;               // 返回旧值
}
```

它接收一个内存地址和一个增量值，将该增量值加到内存地址处的值上，并返回旧值。返回旧值在实现信号量时非常有用。历史上，人们曾尝试实现更复杂的原子操作，如获取并乘、获取并除等，但后来普遍认为过于复杂并不好。

### 交换

交换操作接收一个寄存器值和一个内存地址，然后交换两者的内容。

```c
void Swap(int *mem, int *reg) {
    int temp = *mem;
    *mem = *reg;
    *reg = temp;
}
```

由于使用起来不太方便，它并不非常常见。更常见的是比较与交换操作。

### 比较与交换

比较与交换操作介于测试与设置和交换之间，功能更强大。

```c
int CompareAndSwap(int *mem, int expected, int new_value) {
    if (*mem == expected) {
        *mem = new_value;
        return 1; // 成功
    } else {
        return 0; // 失败
    }
}
```

它接收一个内存地址和两个寄存器值。首先比较内存中的值是否与第一个寄存器值相等。如果相等，则将第二个寄存器值写入内存，并返回成功状态；否则，不进行任何写入并返回失败。从硬件实现角度看，它只需要在内存控制器中添加一个比较器，比实现一个完整的加法器要简单。

## 使用原子操作实现锁

现在，我们回到多消费者从队列中出队的问题，看看如何使用原子操作实现锁。

### 使用测试与设置实现自旋锁

我们将使用“互斥锁”这个术语来指代只允许一个线程进入的信号量。以下是使用测试与设置实现自旋锁的伪代码：

```c
// 初始化：mutex = 0

// 获取锁（进入临界区）
while (TestAndSet(&mutex) != 0) {
    // 忙等待（自旋）
}
// 临界区代码...
// 释放锁（离开临界区）
mutex = 0;
```

在开始时，有一个紧密循环不断调用`TestAndSet`。如果`mutex`为0，表示锁未被占用，`TestAndSet`会将其原子性地设置为1并返回0，循环结束，线程进入临界区。如果`mutex`为1，表示锁已被占用，`TestAndSet`返回1，线程继续循环等待。

释放锁时，只需简单地将`mutex`存储为0即可，这不需要原子操作，因为此时持有锁的线程是唯一能修改它的线程。

这里存在一个问题：如果一个线程获取锁后崩溃或被长时间挂起，其他线程将永远自旋，导致系统“挂起”。在非抢占式操作系统中这尤其危险。一个解决技巧是在自旋循环中加入“让出”操作，允许其他线程运行。

### 使用比较与交换实现无锁同步

我们也可以使用比较与交换来实现一种非阻塞同步。以下是一个尝试无锁更新队列头指针的例子：

```c
do {
    old_head = head; // 加载当前头指针
    if (old_head == NULL) {
        // 队列为空，处理错误
        break;
    }
    new_head = old_head->next; // 计算新头指针
} while (CompareAndSwap(&head, old_head, new_head) == 0);
// 如果成功，则 old_head 指向的节点已出队
```

线程先加载当前头指针，计算出新头指针，然后尝试用`CompareAndSwap`原子地将头指针从`old_head`更新为`new_head`。如果在此期间头指针被其他线程修改，`CompareAndSwap`会失败，线程则回到循环开始重试。这被称为“无锁同步”或“乐观并发”。

这种方法需要注意“ABA问题”：如果头指针从A变为B，又变回A，那么`CompareAndSwap`会错误地认为它没有被修改过。在循环缓冲区等场景中需要特别处理。

### 加载链接与条件存储

另一种强大的无锁同步原语是加载链接和条件存储指令对。它涉及两条新指令：
*   **加载链接**：从内存地址加载值，并在硬件中秘密设置一个与该地址关联的标志位。
*   **条件存储**：尝试向同一地址存储值。仅当从上次加载链接后，没有其他处理器对该地址进行过条件存储（或其他使标志位失效的操作）时，存储才会成功。

使用它们实现出队操作：

```c
do {
    old_head = LoadLinked(&head); // 加载链接
    if (old_head == NULL) break;
    new_head = old_head->next;
} while (StoreConditional(&head, new_head) == 0); // 条件存储
```

如果在此期间其他线程修改了`head`（并执行了条件存储），当前线程的条件存储会失败，循环重试。这种方法的优点是，在无竞争的情况下，加载链接可以将数据读入本地缓存并设置标志，后续的条件存储只需检查本地缓存标志，无需产生总线流量，性能很高。它也能避免ABA问题。

## 性能考量

不同原子操作的性能特征不同：
*   **阻塞式原子操作**：如基于测试与设置的自旋锁，在高竞争时会产生大量内存流量（不断轮询），性能较差。
*   **非阻塞式原子操作**：如比较与交换或加载链接/条件存储，在无竞争或低竞争时性能更好，因为它们可以减少不必要的通信。但比较与交换可能需要重做推测性工作。
*   **仅用普通加载/存储的算法**：如Dekker算法，性能通常不佳。
*   **竞争程度**：任何同步原语的性能都高度依赖于竞争程度。高竞争会显著降低性能。
*   **与缓存和乱序执行集成**：现代处理器需要仔细设计，使这些原子操作能与缓存一致性协议以及处理器的乱序执行内存系统良好协作。
