# 087：内存屏障与顺序一致性

![](img/0dabe707195268ad53c7f40d5075e67d_0.png)

在本节课中，我们将探讨顺序一致性的实现问题，以及如何在追求性能的乱序处理器和内存系统中维持程序正确性。我们将介绍内存屏障的概念，并了解几种常见的内存模型。

## 概述

顺序一致性要求所有处理器看到的内存操作顺序与程序顺序一致。然而，现代处理器为了提高性能，普遍采用乱序执行和缓存技术，这破坏了顺序一致性。因此，我们需要引入更弱的内存模型和内存屏障指令，在性能与正确性之间取得平衡。

## 顺序一致性的挑战

上一节我们讨论了顺序一致性的概念。本节中我们来看看实现它面临的挑战。

乱序处理器和内存系统会打破顺序一致性。具体来说：
*   处理器可以重排序对不同地址的加载操作。
*   处理器可以重排序对不同地址的存储和加载操作。
*   处理器可以重排序对不同地址的存储操作。

此外，缓存的存在带来了数据可见性问题。如果一个处理器将数据写入自己的缓存，其他处理器可能无法立即看到这个更新。

因此，几乎没有处理器实现完全的顺序一致性内存模型，它们都采用了更弱的模型。

## 弱内存模型与内存屏障

由于完全的顺序一致性会严重限制性能，我们转向更弱的内存模型。在这些模型中，程序员需要决定何时允许或禁止内存操作的重排序。

我们引入**内存屏障**（Memory Fence）或**内存栅栏**（Memory Barrier）的概念。它是一个序列化点，确保：
*   在屏障**之前**发出的所有内存访问操作，在屏障**完成之前**必须全部完成。
*   在屏障**之后**发出的所有内存访问操作，在屏障**完成之前**都**不能**开始。

其核心作用是阻止内存操作跨越屏障进行重排序。公式化描述其语义为：
```
// 假设屏障指令为 MFENCE
// 操作 A 在 MFENCE 之前发出
// 操作 B 在 MFENCE 之后发出

执行顺序必须保证为：A -> MFENCE -> B
// 即 A 对全局内存的效应在 B 执行前必须可见
```

为了追求更高性能，架构师还设计了更细粒度的屏障，例如：
*   **加载屏障**：仅屏障之前的加载操作。
*   **存储屏障**：仅屏障之后的存储操作。

## 常见内存模型分类

以下是几种业界常见的内存模型分类：

**完全存储定序（Total Store Order, TSO）**
这是最接近顺序一致性的模型。在TSO中：
*   加载与加载之间不能重排序。
*   加载与后续存储之间不能重排序。
*   **存储与后续加载之间可以重排序**。
*   存储与存储之间可以重排序。

如果需要保证一个存储操作在其后的加载操作之前完成，必须插入内存屏障：
```asm
STORE [address], value
MFENCE
LOAD value, [another_address]
```

**部分存储定序（Partial Store Order, PSO）**
比TSO更弱。在PSO中：
*   加载与加载之间不能重排序。
*   加载与后续存储之间可以重排序。
*   存储与后续加载之间可以重排序（需要屏障保证顺序）。
*   存储与后续存储之间可以重排序（需要屏障保证顺序）。

**弱定序（Weak Ordering）**
这是目前大多数处理器采用的内存模型类别。它非常灵活，几乎允许所有类型的重排序，但相应地，程序员或编译器需要在需要保证顺序的地方显式地插入内存屏障。

内存屏障的优点是“按需付费”。只有在需要保证操作顺序时（如多线程同步），才需要插入屏障并承担其性能开销。在不需要严格顺序的单线程代码段中，硬件可以自由重排序以获得最佳性能。

## 内存屏障应用示例

让我们通过一个生产者-消费者模型的例子，看看如何应用内存屏障。

**场景1：保证存储顺序**
在生产者代码中，我们需要确保数据`data`的存储操作在更新`flag`标志的存储操作**之前**对全局内存可见。为了防止这两个存储被重排序，我们可以在它们之间插入一个存储屏障（或通用屏障）：
```c
// 生产者
buffer[head] = data; // 存储数据
STORE_BARRIER();     // 确保上一存储完成后，才进行下一存储
flag = 1;            // 更新标志
```

**场景2：保证加载顺序**
在消费者代码中，我们可能希望先检查`flag`标志，再读取`data`数据。但在弱内存模型下，这两个加载操作可能被重排序。为了保证正确性，我们可以在检查`flag`后、读取`data`前插入一个加载屏障（或通用屏障）：
```c
// 消费者
while (flag == 0) { // 加载标志
    // 等待
}
LOAD_BARRIER();      // 确保上一加载完成后，才进行下一加载
value = buffer[tail]; // 加载数据
```

编译器有时会辅助插入屏障。例如，当它识别出代码中的原子操作（如锁、信号量P/V操作）或`test-and-set`指令时，知道这些点需要内存同步，可能会自动添加必要的屏障。

## 总结

本节课中我们一起学习了：
1.  顺序一致性在乱序处理器和缓存系统中难以直接实现。
2.  弱内存模型通过允许内存操作重排序来提升性能。
3.  **内存屏障**是强制内存操作顺序的指令，是弱内存模型下保证多线程程序正确性的关键工具。
4.  常见的内存模型包括**完全存储定序（TSO）**、**部分存储定序（PSO）** 和**弱定序（Weak Ordering）**，它们对重排序的限制逐渐放宽。
5.  内存屏障的使用策略是“按需付费”，仅在需要保证顺序的同步点插入，从而在保证正确性的同时最大化性能。
