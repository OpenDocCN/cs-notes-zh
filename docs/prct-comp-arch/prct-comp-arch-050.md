# 050：静态分支预测

![](img/8406ff79747cea1279e1803dcae3cee8_0.png)

在本节课中，我们将要学习如何处理分支指令的结果预测。我们将首先聚焦于**静态**领域的技术，即不依赖于动态指令序列的观察，而是通过静态分析指令序列，以较高的概率预测分支的走向。

## 概述：静态预测技术

上一节我们介绍了分支预测的基本概念。本节中，我们来看看几种静态分支预测技术。静态预测不依赖于程序运行时的信息，而是在编译时或硬件设计时就确定预测策略。

### 延迟槽技术

首先介绍的技术实际上并非预测，而是**延迟槽**。我们在之前的课程中讨论过，其核心思想是在分支指令后插入若干条无论分支是否跳转都会执行的指令，以填充因等待分支结果而产生的空闲周期。

以下是延迟槽的示意图：
```
指令 B, E, Q, Z (地址 100)
延迟槽指令 1
延迟槽指令 2
```
无论分支跳转与否，延迟槽中的指令都会执行。

然而，找到合适的指令填充延迟槽的概率并不高。填充一个延迟槽的概率约为70%，填充两个则降至约40-50%。相比之下，现代动态分支预测的准确率可达95%以上。因此，对于追求高性能的系统，更复杂的预测机制通常是更好的选择。

### 静态预测的经验法则

接下来，我们看看关于静态预测的一些基本经验法则。

总体而言，在类似SPECint的基准测试中，分支被执行的总体概率约为60%至70%。但这个概率分布并不均匀。

以下是关键观察：
*   **向后跳转的分支**（如循环）被执行的**概率非常高**。因为循环的定义就是需要跳回之前的代码。
*   **向前跳转的分支**（如if-then-else语句）被执行的**概率接近50%**。这通常是数据依赖的条件判断。

程序倾向于在循环中执行较长时间，这是理解静态预测的关键。

### 编译器提示的静态预测

基于以上观察，一种技术是扩展指令集，允许**编译器向硬件提示**分支的预测方向。

具体实现是定义两种分支指令：
*   `branch.T`：预测为“执行”的分支。
*   `branch.NT`：预测为“不执行”的分支。

这只是一个性能提示，如果预测错误，处理器仍会确保正确执行，只是性能会下降。

许多真实架构采用了类似思想。如果编译器能结合**代码剖析**信息（即运行一次程序来收集分支历史），这种方法的准确率可以相当高。例如，编译器可以将循环的向后分支标记为`branch.T`，而将极少触发的错误检查分支也标记为`branch.T`（跳过错误处理代码）。

在流水线中，这种提示能带来好处：
*   对于预测“执行”的分支，虽然目标地址在解码阶段才知道，但我们可以提前开始从预测目标取指，减少空闲周期。
*   对于预测“不执行”的分支，我们可以直接顺序取指（PC+4, PC+8），完全没有停顿。

如果预测错误，则会产生**误预测惩罚**，需要清空流水线并重新取指。

### 硬件静态预测启发式

最后，我们看看完全由硬件实现的静态预测启发式方法，无需编译器提示。

以下是几种简单的硬件策略：

1.  **总是预测不执行**
    *   这是我们目前设计的流水线中默认采用的方法。
    *   实现简单，直接取顺序地址（PC+4）。
    *   准确率不高，会错误预测所有循环（向后分支）。

2.  **总是预测执行**
    *   实现上有困难，因为在解码阶段我们可能还不知道分支目标地址。
    *   对于许多向前跳转的if-then-else分支效果也不好。

3.  **向后执行，向前不执行**
    *   这是一种更优的启发式方法：所有**向后**分支预测为“执行”，所有**向前**分支预测为“不执行”。
    *   这很好地契合了“程序在循环中停留时间长”的特点，准确率比前两种高。
    *   然而，其准确率仍然低于由编译器提供提示的静态预测，因为编译器可以运用更复杂的算法进行分析。

## 总结

![](img/8406ff79747cea1279e1803dcae3cee8_2.png)

![](img/8406ff79747cea1279e1803dcae3cee8_3.png)

本节课中我们一起学习了**静态分支预测**技术。我们首先了解了通过填充**延迟槽**来隐藏分支延迟的方法及其局限性。接着，我们探讨了基于程序行为统计的静态预测经验法则，即向后分支（循环）的执行概率远高于向前分支。基于此，我们介绍了两种主要的静态预测实现方式：一是由**编译器提供预测提示**给硬件，这能获得较高的准确率；二是由**硬件实现简单的预测启发式规则**，如“向后执行，向前不执行”。静态预测虽然不如动态预测准确，但在简化硬件设计或特定架构中仍有其价值。下一节，我们将深入探讨更强大的动态分支预测技术。
