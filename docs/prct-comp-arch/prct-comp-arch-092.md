# 092：基于总线的多处理器与缓存一致性协议

在本节课中，我们将要学习缓存一致性协议的基本概念。我们将从一个单处理器系统中的并行I/O问题入手，理解为什么需要一致性机制，并最终将其概念扩展到多处理器系统。

## 从单处理器系统说起

上一节我们介绍了多处理器系统中的数据一致性问题。本节中我们来看看，实际上，类似的问题在单处理器系统中就已经存在了。

考虑一个单处理器系统，它包含一个处理器、一个缓存、一个主存，以及一个带有DMA引擎的磁盘。DMA引擎是一个自主控制器，可以直接在磁盘和主存之间移动数据。

当处理器正在运行时，如果磁盘需要将一个页面从磁盘传输到主存，或者从主存传输到磁盘，问题就会出现。

以下是可能发生的两种场景：

1.  **磁盘到内存的传输**：DMA控制器开始将数据从磁盘复制到主存。然而，缓存中可能持有相同地址的旧数据。处理器从缓存中读取该地址时，将永远无法获取到刚从磁盘写入主存的新值。
2.  **内存到磁盘的传输**：假设有一个写回式缓存。当DMA传输从主存向磁盘复制数据时，它可能错过缓存中已更新但尚未写回主存的“脏”数据。因此，磁盘最终得到的是过时的数据。

这个问题的核心在于：**DMA引擎只知道访问主存，它不知道缓存中可能存在同一地址的更新数据**。

## 解决方案的思路

那么，如何保证磁盘（或其他数据读取者）能看到缓存中最新的数据呢？

你需要一种机制，要么将缓存中的数据无效化并写回主存，让磁盘获取更新后的副本；要么将数据从特定缓存中“踢出”，以确保新的数据读取者能看到更新。

这引出了我们接下来要讨论的核心概念：**缓存一致性协议**。

## 引入缓存一致性协议

缓存一致性协议将为我们提供一些保证和机制，确保根据特定规则，一个组件对数据的更新能够被其他组件看到，从而避免它们读取到过时的数据。

即使在单处理器系统中，也可能需要缓存一致性协议。而在多处理器系统中，这个问题会变得更加复杂。

## 核心问题总结

无论是并行I/O还是多处理器带缓存访问数据，我们都面临一个根本性问题：一个处理器（或DMA控制器）的写入，需要能够被另一个处理器（或磁盘控制器）看到。

![](img/a66c00531becd1f8ac0a593d5c06c50e_3.png)

本节课中我们一起学习了缓存一致性问题的起源。我们从单处理器系统中的DMA与缓存冲突入手，理解了数据不一致的根本原因在于多个数据访问者（如CPU缓存和DMA引擎）持有同一数据的多个副本，且缺乏同步更新机制。这为后续深入探讨多处理器下的缓存一致性协议奠定了概念基础。
