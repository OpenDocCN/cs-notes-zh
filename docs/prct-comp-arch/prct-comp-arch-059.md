![](img/f770be99f6ac860238e19c0335b058af_0.png)

在本节课中，我们将继续探讨高级缓存技术。我们将重点讨论如何构建乱序内存系统，以及如何通过缓存分体化等技术来提升内存带宽和并行性。

上一讲我们介绍了流水线缓存、重排序缓冲区、多级缓存、受害者缓存以及硬件/软件预取。本节中，我们来看看如何通过**缓存多端口化**和**分体化**来进一步提升性能。

## 缓存多端口化 🚪

在传统的双发射处理器流水线中，每个周期通常只能执行一次内存操作。这限制了性能，尤其是在需要并行执行多个独立内存操作时。

一种直观的解决方案是为缓存增加多个端口，使其能同时处理多个读写请求。这被称为**真多端口缓存**。

然而，实现真多端口缓存代价高昂：
*   **逻辑复制**：至少需要复制地址解码逻辑。若要支持同时写入，还需复制写入逻辑。
*   **面积与功耗**：很多时候，为了增加一个端口，设计者甚至需要复制整个缓存存储阵列，导致面积和功耗大幅增加。
*   **命中时间**：如果尝试共享部分逻辑，会增加缓存读出路径的负载，可能导致命中时间（访问延迟）增加。

因此，真多端口缓存通常不是首选方案。

## 缓存分体化 🏗️

为了以更低的成本获得更高的带宽，我们可以采用**缓存分体化**技术。

其核心思想是：不构建一个大型的多端口缓存，而是将缓存划分为多个独立的、单端口的**存储体**。从外部看，它仍然像一个多端口缓存（例如，可以同时接收两个地址，输出两个数据），但内部通过一个交叉开关将请求路由到不同的存储体。

以下是缓存分体化的关键考量点：

### 存储体寻址策略
如何将内存地址映射到不同的存储体是关键。常见的策略有：
*   **低位交叉存取**：使用地址的低位比特来选择存储体。
*   **高位交叉存取**：使用地址的高位比特来选择存储体。

**公式示例（低位交叉存取）**：
`bank_index = address[log2(num_banks)-1:0] % num_banks`

**代码示例（伪代码）**：
```c
// 假设有 4 个存储体，使用地址的第 2、3 位来选择
int get_bank_index(uint64_t address) {
    return (address >> 2) & 0x3; // 取地址的 bit[3:2]
}
```

选择策略需要谨慎：
*   如果程序顺序访问一个大数组（步长为1），使用**高位交叉存取**会导致所有访问都命中同一个存储体，引发严重的**存储体冲突**，无法提升带宽。
*   如果程序访问一个结构体数组，且每次都访问结构体内的同一个字段（例如第一个字节），使用**低位交叉存取**同样会导致所有访问命中同一存储体。
*   因此，有时折中的方案是使用**中位比特**进行交叉存取，或者设计更复杂的哈希函数，但这可能增加关键路径的延迟。

### 分体化的优势与挑战
**优势**：
*   **高带宽**：通过增加存储体数量，可以显著提高并发访问能力。
*   **低成本**：相比真多端口缓存，使用多个单端口存储体在面积和时序上通常更优。

**挑战**：
*   **存储体冲突**：当两个请求被映射到同一个存储体时，会发生冲突，其中一个请求必须等待，降低了有效带宽。增加存储体数量（使其多于端口数）可以降低冲突概率。
*   **额外布线**：需要交叉开关（多路复用器/解复用器）来路由地址和数据，这会增加布线复杂性和些许延迟。
*   **不均衡的利用率**：如果地址映射策略不理想，可能导致数据集中存储在少数几个存储体中，造成存储空间和带宽利用不均衡，相当于实际可用的缓存容量变小了。
*   **存储体尺寸**：将缓存划分成许多小存储体，可能会因为无法很好地分摊行/列解码器、读出放大器等固定开销，而导致面积效率降低，或产生不理想的“长窄形”存储阵列布局。

尽管存在挑战，缓存分体化在实践中被广泛采用。例如，Sun的Niagara处理器和Intel的某些多核芯片，其末级缓存都采用了大量（如32或64个）存储体来提供极高的内存带宽。

## 总结 📝

本节课我们一起学习了提升缓存带宽的两种主要技术。
*   我们首先了解了**真多端口缓存**，它虽然能直接提供并行访问能力，但因其高昂的实现成本（面积、功耗、延迟）而较少使用。
*   接着，我们深入探讨了更实用的**缓存分体化**技术。它将大缓存划分为多个单端口存储体，通过交叉开关模拟多端口行为，以更低的成本获得了高带宽。我们重点讨论了存储体寻址策略（低位/高位/中位交叉存取）的选择及其对性能的影响，并分析了分体化带来的优势（高带宽、低成本）与挑战（存储体冲突、额外布线、不均衡利用率）。
