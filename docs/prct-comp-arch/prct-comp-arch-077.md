![](img/e5d7072d511a6ce9931d9b21a12e3dc0_0.png)

在本节课中，我们将要学习计算机体系结构中的一个新主题：多线程。我们将探讨为什么需要引入多线程技术，以及它如何帮助处理器更有效地工作。

上一节我们介绍了向量处理器，本节中我们来看看一种不同的并行技术——线程级并行。

## 概述

有时，我们无法从程序中提取指令级并行（ILP），这是超标量和超长指令字（VLIW）处理器试图利用的。有时，数据级并行（DLP）也难以利用，因为并非所有代码都像密集矩阵运算那样容易找到数据并行性。相反，某些工作负载天然具有**线程级并行**。

所谓线程级并行，是指存在需要执行的、独立的顺序性任务。处理这类问题最高效的方式，就是为每个需要处理的工作单元分配一个独立的进程或线程。

## 线程级并行的应用场景

以下是线程级并行发挥作用的几个典型场景：

**1. 网络数据包处理**
一个很好的例子是网络处理，例如构建防火墙。网络卡收到一个数据包，防火墙需要检查这个数据包以寻找恶意攻击、病毒或攻击代码序列。这并非数据级并行，因为对不同类型的数据包（如TCP与UDP，或不同端口的流量），处理的控制流程会截然不同。最高效的处理方式是为每个传入的数据包分配一个独立的作业或线程。

**2. 加速复杂搜索问题**
在某些应用中，使用并行线程可以更快地解决问题。例如，传统的旅行商问题。为这个问题分配多个线程，甚至可以获得**超线性加速**。因为线程可以并行探索搜索树的不同分支，并且线程之间可以共享数据。当一个线程到达某个搜索状态时，它可以利用其他线程已经计算过的结果，避免重复探索某些路径，这类似于动态规划的思想。

**3. 隐藏延迟**
使用线程还可以隐藏内存访问等操作的延迟。假设有一个程序经常发生缓存未命中，但同时程序内部存在并行性。一种处理方式是将问题分割，为不同的子问题分配线程。当一个线程因为缓存未命中而阻塞，等待内存响应时，处理器可以切换到另一个线程执行有用工作，从而利用原本会被浪费的周期。

## 从流水线视角看问题

让我们从处理器流水线的角度，看看如何回收被浪费的时钟周期。

考虑以下指令序列：
```assembly
LOAD R1, [A]
LOAD R2, [R1]   ; 依赖于第一条LOAD
ADD  R3, R2, #1 ; 依赖于第二条LOAD
STORE [B], R3   ; 依赖于ADD
```
这段代码存在一连串的数据依赖。在顺序执行的流水线中，即使采用旁路技术减少了一些气泡，图中大量的紫色时间段（空闲时间）仍然是处理器的“死时间”。超标量或乱序执行技术在这种情况下也无济于事，因为指令间存在真实的依赖链。向量处理器、更宽的发射宽度或VLIW对此也都没有帮助。

我们面临大量空闲周期，目标就是尝试**回收这些被浪费的周期**。
