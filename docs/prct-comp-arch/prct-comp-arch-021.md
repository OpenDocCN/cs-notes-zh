# 021：基础双发射顺序超标量处理器

![](img/fbe31a4e3d77a91b7e959a431e2c68d3_0.png)

在本节课中，我们将学习一种基础的“双发射顺序超标量”处理器设计。我们将探讨其与经典五级流水线的核心区别，并理解其架构中的关键组件与挑战。

## 概述

我们将从一个基础的“双发射顺序超标量”处理器模型开始。这个名称听起来有些复杂，但其核心思想是：它与你之前见过的流水线不同，因为它拥有两个算术逻辑单元，从而能够同时执行两条整数运算指令。这是其最显著的区别。

## 架构核心变化

上一节我们提到了双发射的核心概念，本节中我们来看看为了实现这一目标，处理器架构需要做出哪些具体改变。

### 1. 指令获取

首先，与经典的五级MIPS处理器相比，我们必须能够同时获取两条指令。如果我们希望同时执行两个操作，就需要从指令缓存或指令内存中一次性取出两条指令。

以下是实现双指令获取的关键点：
*   **程序计数器**：需要能够同时提供两个地址，例如PC和PC+4。
*   **指令寄存器**：需要两个指令寄存器来分别存放取出的两条指令。
*   **对齐问题**：需要考虑指令边界对齐问题。
*   **分支指令**：如果取出的第一条指令是分支指令，需要特殊处理。

### 2. 寄存器文件

在我们的基础流水线中，寄存器文件通常只有两个读端口。现在，由于需要同时处理两条指令，我们必须能够同时读取四个操作数。

以下是寄存器文件的增强需求：
*   **读端口**：需要四个读端口，以便同时为两条指令提供各自的源操作数。
*   **写端口**：为了能够同时提交（或退休）两条指令的结果，也需要增加写端口的数量。在示意图中，寄存器文件被逻辑上分开绘制以便理解，但实际上它是一个整体。

### 3. 非对称流水线与指令调度

我们的两条流水线（A和B）并非完全对称。假设流水线A可以执行整数运算和分支指令，而流水线B可以执行整数运算和内存访问（加载/存储）。但流水线A不能执行内存访问，流水线B不能执行分支。

这就引出了一个关键问题：如果从指令缓存取出的第一条指令（IR0）恰好是一条加载指令，而加载指令只能在流水线B中执行，该怎么办？

这确实是个问题。因此，在超标量设计中，我们需要引入**指令发射逻辑**或**指令调度逻辑**。这个逻辑模块的作用是：根据指令的类型，动态地将它们“引导”或“调度”到正确的功能流水线中。它可能需要交换两条指令在流水线中的位置。这个过程可能会引入额外的延迟，从而促使我们设计更深的流水线来容纳这些逻辑。

### 4. 控制逻辑复制

另一个必须做的改变是在控制方面：我们必须开始考虑复制控制逻辑。

以下是控制逻辑的复制需求：
*   **双解码器**：因为要同时解码两条指令，所以需要两个独立的解码器。指令寄存器0连接到解码器A，指令寄存器1连接到解码器B。
*   **信号驱动**：这两个解码器将分别驱动A和B两条数据路径上的控制信号。
*   **指令交换**：如果指令调度逻辑需要交换指令，那么解码器的输入也可能需要进行相应的交换和通信。

## 总结

![](img/fbe31a4e3d77a91b7e959a431e2c68d3_3.png)

本节课中，我们一起学习了基础的双发射顺序超标量处理器模型。我们了解到，为了实现同时执行两条指令的目标，处理器需要在多个方面进行增强：包括同时获取两条指令、扩展寄存器文件的端口、设计非对称的功能流水线、引入关键的指令调度逻辑以将指令分配到正确的流水线，以及复制控制逻辑（如解码器）。这些构成了一个基础超标量处理器的核心框架。
