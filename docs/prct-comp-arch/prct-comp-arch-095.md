# 095：互连网络简介 🖥️

在本节课中，我们将要学习处理器之间如何共享信息以及如何连接多个处理器。我们将对比总线与其他网络形式，并探讨为何需要改变连接方式。

## 总线通信的挑战

上一节我们介绍了总线作为共享信息的一种方式。本节中我们来看看当处理器数量增加时，总线面临的挑战。

假设总线上有两个核心。一个核心想与另一个核心通信。这相对简单，只需“喊话”即可。但存在一些挑战，例如两个核心可能同时尝试通信，因此需要仲裁机制。总的来说，仲裁逻辑比较简单。

现在考虑总线上有四个核心的情况。这就像四个人在一个房间里试图互相喊话。带宽可能成为瓶颈。总线的仲裁也变得更复杂。此外，互连网络的线延迟和电容负载也变得更差。

一个核心需要驱动这条共享的多负载总线。总线上的电容负载比两个核心时大得多，因为总线长度增加，并且负载也更多。

进一步考虑构建一个连接更多核心（例如12个）的总线。如果一个核心想与远端核心通信，信号必须传播整个总线长度。在高通信速率下，必须等待光从一端飞到另一端的时间。如果使用类似侦听或广播协议，每个周期都必须等待最坏情况下的通信时间。电容负载问题依然存在，这不是简单的传输线问题，还需要考虑驱动所有不同接收器的电容。这是一个多向总线，需要三态能力和驱动/接收控制。

当有12个人时，如果每个人同时喊一个数字，很难听清所有人说了什么。只能听到最近邻居的声音。如果需要保证一次只有一个人能在总线上“喊话”，就需要仲裁。但仲裁逻辑现在更慢，因为需要连接更多节点，决策逻辑包含更多组合逻辑层。

如果扩展到1000个处理器，情况会更糟。物理上，上千个节点之间的布线距离将显著降低总线速度。

## 交换式互连网络

上述问题促使我们思考连接这12个核心的其他方式。接下来介绍一种称为交换式互连或点对点链接的解决方案。

点对点并不意味着每个核心都能直接与其他所有核心通信（那是另一种拓扑，稍后讨论）。点对点仅意味着每条链路只有一个发送方和一个接收方。然后使用沿途的交换机进行决策和路由。

观察这种结构，可以实现多个近邻通信同时发生。通过增加交换功能，我们既能在所有不同节点之间建立连接，也能同时进行多个子对话。

这仍然允许远端处理器之间进行通信。我们需要决定通信路径，例如是走这条路、那条路还是一些曲折的路线。

与可以增加宽度的总线类似（但这无助于改善总线占用率），我们也可以增加更多网络，或者有效地添加多个并发的交换式互连网络。我们也可以增加这些链路的带宽。在总线上可以使用的增加带宽的技巧，同样可以应用于交换式互连网络。

这只是非常概括的概述。接下来我们将深入探讨一些具体概念。
