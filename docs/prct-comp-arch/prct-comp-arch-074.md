## 向量规约操作

上一节我们介绍了向量计算机的基本概念和优势。本节中我们来看看一种常见的向量操作：规约。

规约操作是指将一个向量的所有元素通过某种运算（如加法、求最大值）合并为单个标量值的过程。例如，计算一个向量中所有元素的总和。

在向量机上执行规约操作时，如果试图用一条指令访问所有元素，会破坏向量处理器的“车道”并行结构，导致性能下降。因此，需要一种更高效的实现方法。

以下是实现向量求和规约的一种常用算法：

1.  **初始向量**：假设我们有一个很长的向量。
2.  **分半相加**：将向量分成两半，使用向量加法指令将前半部分与后半部分对应元素相加，得到一个长度减半的新向量，其中每个元素是原向量两半部分对应元素的和。
3.  **递归执行**：对得到的新向量重复步骤2，继续分半相加。
4.  **得到标量**：重复此过程，直到最终得到一个单一的标量值，即原向量的总和。

这个算法本质上是一个**二进制树形算法**，它充分利用了向量指令的并行能力，在多个步骤中逐步将数据规模减小。

## 聚集与分散操作

之前讨论的向量加载和存储操作，其内存访问模式是连续且对齐的，可以很好地映射到具有多存储体的内存系统。然而，许多应用需要更灵活的访问模式。

例如，考虑以下操作：`C[D[i]]`。这里，`D`是一个向量，其元素是索引值。我们需要用这些索引去访问另一个数组`C`中的元素。这种模式称为**间接寻址**。

基本的向量架构可能不支持这种操作，但可以通过添加特殊指令来实现。

*   **聚集**：从一个内存数组中，根据一个索引向量指定的位置，收集数据到一个目标向量寄存器中。
    *   指令示例：`LVIRd, Rs1, Rs2` (Load Vector Indirect)
    *   含义：`Rd[i] = Memory[Rs1 + Rs2[i]]`
*   **分散**：聚集操作的逆操作。将一个源向量寄存器中的数据，根据一个索引向量指定的位置，分散存储到内存数组中。
    *   指令示例：`SVIRs1, Rs2, Rd` (Store Vector Indirect)
    *   含义：`Memory[Rd + Rs2[i]] = Rs1[i]`

实现聚集和分散操作对内存系统提出了更高要求。由于访问地址不再连续且不可预知，内存系统必须允许所有处理车道能访问到所有内存存储体，这增加了内存互连网络的复杂性。

## Cray-1 超级计算机案例研究

现在，让我们通过一个历史实例来具体了解向量计算机的设计。我们将目光投向Cray-1，这是一台标志性的早期超级计算机。

Cray-1以其独特的外观设计闻名（呈字母“C”形，据说为了缩短背板布线距离，同时也与创始人Seymour Cray的姓氏首字母呼应）。它采用水冷系统，甚至为操作员设计了一个温暖的座椅。

从体系结构角度看，Cray-1的设计有其鲜明特点：

*   **向量寄存器**：拥有8个向量寄存器，每个寄存器可容纳64个元素（向量长度最大为64）。
*   **功能流水线**：它没有采用现代的多“车道”设计，而是为不同的运算（如加法、乘法）设置了独立且较长的功能流水线。例如，浮点乘法流水线深度为6个周期，倒数运算约为14个周期。
*   **内存与缓存**：Cray-1没有采用缓存，也不支持虚拟内存。作为专用于解决大规模科学计算问题（如核武器模拟、密码破译）的超级计算机，它通常一次只运行一个大型任务，因此对多任务和虚拟化的需求较低。
*   **时钟频率**：在1976年，其约80MHz的主频是当时最快的。
*   **流水线互锁**：由于向量长度很长，它不需要复杂的流水线级间旁路技术。但它支持**链化**操作，即允许一个功能单元的结果直接作为另一个功能单元的输入，从而提高效率。

Cray-1的微架构展示了在晶体管资源有限的年代，如何通过深流水线和寄存器-寄存器架构来高效实现向量处理。
