## 缓存回顾：为何需要缓存？⚡

上一节我们介绍了缓存的基本思想。现在我们来回顾一下其核心优势。

处理器需要频繁访问内存。但大容量内存速度较慢，而处理器速度很快。为了解决这个速度不匹配的问题，我们可以在处理器附近放置一个**容量小、速度快**的存储器，即缓存。

其核心思想是：如果处理器要访问的数据有很大概率（高局部性）正好在这个小且快的缓存中，那么处理器就无需频繁访问慢速的主内存。这样可以**降低功耗、提升性能**。

处理器、缓存和主内存的关系如下：地址从处理器流向缓存和内存。数据可以双向流动，因为**存储**操作会将数据从左（处理器）写到右（内存），而**加载**操作会将数据从右（内存）读到左（处理器）。

## 缓存的内部视图与查找流程 🔍

缓存中存储的是主内存中数据的副本。为了减少管理开销，缓存中的数据被组织成**行**或**块**。

缓存中的每一行通常包含以下几个部分：
*   **地址标签**：用于标识该行数据来自主内存的哪个地址。它不一定是完整地址，只需足够区分可能映射到同一缓存位置的不同数据即可。
*   **有效位**：一个标记位，指示该行数据是否有效（例如，缓存初始化后或数据被驱逐后，该行无效）。
*   **数据块**：从主内存复制过来的实际数据。

以下是处理器执行**加载**操作时，缓存的基本工作流程：
1.  **地址解析**：处理器发出加载指令，给出内存地址。
2.  **索引与标签比较**：根据地址中的一部分（索引位）找到缓存中可能对应的一个或多个行。然后，将地址中的另一部分（标签位）与这些行中存储的标签进行比较。
3.  **有效性检查**：同时检查对应行的有效位是否为“有效”。
4.  **命中判断**：
    *   如果标签匹配且有效位为“有效”，则发生**缓存命中**。数据直接从缓存返回给处理器。
    *   如果不匹配或无效，则发生**缓存未命中**。
5.  **未命中处理**：缓存需要从主内存中获取所需数据。
6.  **块替换**：新数据取回后，需要放入缓存。如果目标位置已被占用，则需根据**替换策略**选择一个旧数据块驱逐出去。
7.  **数据返回**：将新数据填入缓存，并最终返回给处理器。一些高级缓存设计会尝试并行执行数据返回和旧数据驱逐，以降低加载延迟。

## 缓存的分类维度 🗂️

我们可以从以下几个关键维度对缓存进行分类：

*   **块放置策略**：一个特定的内存块可以被放置在缓存的哪些位置？
*   **块标识策略**：给定一个内存地址，如何在缓存中查找对应的块？（与放置策略相关）
*   **块替换策略**：当缓存未命中且需要放入新数据时，应该驱逐哪个旧块？
*   **写策略**：当处理器执行**存储**（写）操作时，缓存和内存如何更新？

---

### 块放置策略：三种主要类型

以下是三种基本的缓存组织结构，我们通过一个包含32个块的简化内存（例如，块0到块31）和一个小缓存（例如，8个块位置）来说明。

![](img/6b66ddaed552d86d921d644fc22e1f33_0.png)

**1. 直接映射缓存**
*   **原理**：每个内存块只能被放置在缓存中**唯一一个**特定位置。
*   **映射规则**：通常使用 `块编号 mod 缓存大小` 来计算位置。例如，内存块12在8个位置的缓存中，只能放在 `12 mod 8 = 4` 号位置。
*   **特点**：实现简单，速度快，但灵活性最差，容易发生冲突。

**2. 组相联缓存**
*   **原理**：缓存被分为若干组，每个内存块可以映射到**特定一组**内的**任何一个**位置。
*   **映射规则**：`块编号 mod 组数` 决定映射到哪一组。例如，在一个2路组相联缓存（共4组，每组2个位置）中，块12通过 `12 mod 4 = 0` 映射到第0组，然后可以放在该组内的任意一个空闲位置。
*   **特点**：平衡了灵活性和复杂度。现代处理器（如Intel Core i7）的L1缓存通常是8路组相联。

**3. 全相联缓存**
*   **原理**：任何内存块可以被放置在缓存中的**任何**一个位置。
*   **映射规则**：没有索引限制。块12可以放在8个位置中的任意一个。
*   **特点**：灵活性最高，冲突最少，但查找和替换逻辑最复杂，成本最高。

### 块标识：如何根据地址查找？🔎

我们以直接映射缓存为例，看看如何分解一个内存地址来在缓存中查找数据。

假设：
*   内存地址为32位。
*   缓存有4个条目（直接映射）。
*   每个缓存块大小为64字节。

地址被分解为三部分：
1.  **块内偏移**：用于定位数据在64字节块内的具体字节。`log2(64) = 6`位。
2.  **索引**：用于选择缓存中的哪一个条目。因为有4个条目，所以需要 `log2(4) = 2`位。
3.  **标签**：地址中剩余的所有高位。`32 - 6 - 2 = 24`位。

查找过程：
1.  使用 **索引** 位（2位）找到缓存中对应的那一个条目。
2.  将该条目中存储的**标签**（24位）与地址中的**标签**位进行比较。
3.  同时检查该条目的**有效位**。
4.  如果**标签匹配且有效位为1**，则缓存命中，再结合**偏移**位取出具体数据。

对于**组相联缓存**（如2路），过程类似，但索引位会先确定是哪一个组（例如，2路组相联，4组，则索引为 `log2(4)=2`位）。然后，需要**并行比较**该组内所有路（2个）的标签和有效位。如果任一匹配则命中，否则未命中。

### 块替换策略：当缓存满时怎么办？🔄

在直接映射缓存中，替换目标唯一，无需选择。但在组相联和全相联缓存中，当目标组已满时，就需要一个策略来决定驱逐哪个旧块。

以下是几种常见的替换策略：

*   **随机替换**：从候选块中随机选择一个驱逐。实现简单，有时效果出人意料地好。
*   **最近最少使用**：选择最久未被访问的块进行驱逐。这个策略基于**时间局部性**原理（最近被访问的数据很可能再次被访问），通常能获得较好的命中率。但精确实现LRU（尤其是高相联度时）硬件开销较大，因此常使用近似的伪LRU算法。
*   **先进先出**：选择最早进入缓存的块进行驱逐。实现比LRU简单，但它只考虑了进入时间，而没有考虑访问频率，可能不如LRU有效。
*   **非最近使用**：一种简化策略，确保不驱逐**最近刚被使用过**的块，而从其他块中挑选。这是对LRU的一种近似。

### 写策略：处理器存储数据时发生了什么？✍️

当处理器执行存储（写）操作时，缓存系统需要决定如何更新数据。这主要涉及两个关键决策：

**决策一：写命中时，是否立即更新下级存储？**
*   **写直达**：数据同时写入缓存**和**主内存（或下一级缓存）。
    *   **优点**：数据一致性维护简单，缓存行被驱逐时无需写回。
    *   **缺点**：每次写操作都消耗通往内存的带宽。
*   **写回**：数据只写入当前缓存，并标记该缓存行为“脏”。只有当这个“脏”行被驱逐时，才将其写回主内存。
    *   **优点**：显著减少写内存的带宽消耗。
    *   **缺点**：需要额外的“脏”位来跟踪状态，一致性协议更复杂。

**决策二：写未命中时，是否分配缓存行？**
*   **写分配**：将所写地址对应的整个数据块从主内存加载到缓存中，然后在缓存中合并新数据，最后按上述写命中策略处理。
    *   **优点**：利用了空间局部性，如果后续很快要读该数据，则已在缓存中。
*   **非写分配**：不将数据块载入缓存，直接将其写入主内存（或下一级缓存）。
    *   **优点**：避免加载可能不会被再次访问的数据，节省缓存空间和带宽。
    *   **缺点**：如果很快要读该数据，会发生缓存未命中。

**常见的组合策略：**
*   **写回 + 写分配**：高性能系统的常见选择，旨在最大化利用缓存，减少带宽消耗。
*   **写直达 + 非写分配**：实现简单的低成本方案，常用于对一致性要求严格或带宽相对充裕的场合。

在实际的多级缓存层次结构中，策略可能会混合使用。例如，L1缓存对L2缓存采用**写直达**以简化多处理器一致性协议，而L2缓存对主内存采用**写回**以节省片外带宽。
