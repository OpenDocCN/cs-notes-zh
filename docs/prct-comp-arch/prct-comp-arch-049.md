# 049：分支预测简介 🧠

![](img/d00ed86ad3b42a204c5b58ae5a3da9af_0.png)

在本节课中，我们将要学习分支预测技术。分支预测是现代处理器设计中的一项关键技术，用于解决控制依赖带来的流水线停顿问题。我们将探讨预测的两个核心方面：分支结果（是否跳转）和目标地址（跳转到哪里）。

## 分支预测概述

上一节我们讨论了分支指令带来的流水线冒险。本节中我们来看看如何通过预测来缓解这个问题。

分支预测包含两个必须预测的要素：
1.  **预测结果**：即判断条件分支指令是“跳转”（taken）还是“不跳转”（not taken）。
2.  **预测目标地址**：即预测如果跳转，程序计数器（PC）将指向的具体内存地址。

许多人认为分支预测只关乎第一点，但实际上两者都至关重要。预测结果相对直观，即使随机猜测也有50%的正确率。然而，预测目标地址则要求精确性。在32位地址空间中，你需要从2^32个可能地址中准确猜中一个，这极具挑战性。因此，我们将花时间讨论预测分支和跳转目标地址的策略。

## 指令解析时机回顾

这部分内容是对之前第2讲内容的回顾。我们来分析一下在流水线中，何时能确定分支的相关信息。

考虑一个稍长的六阶段流水线（包含取指、译码、执行等阶段）。以下是各信息变得已知的时间点：

*   **分支、跳转及跳转链接指令的目标地址**：通常在**译码（Decode）阶段**末尾可以计算得出。这是因为这些指令的目标地址通常由指令中的立即数偏移量加上当前PC值计算而来。
*   **分支结果（是否跳转）**：对于条件分支，结果通常要到**执行（Execute）阶段**末尾才能确定，因为需要等待比较运算完成。虽然对于像“与零比较”这类简单分支，可以通过在译码阶段添加特殊比较器来提前判断，但这并非通用解决方案。对于需要比较两个通用寄存器的分支（如 `beq $r1, $r2`），必须等待执行阶段完成完整的32位比较（相当于一次减法）。
*   **跳转寄存器与跳转链接寄存器指令的目标地址**：这类指令（如 `jr $ra`）的目标地址存储在寄存器中。因此，在译码阶段结束时，我们甚至无法获得备选地址。我们需要从寄存器文件中读取值，并可能经过旁路网络，因此必须等待到**执行阶段**才能获得目标地址。

综上所述，在简单的流水线设计中，关键信息（尤其是条件分支的结果）要到流水线较深阶段才可知，这会导致流水线出现停顿（气泡）。分支预测的目的就是尝试在流水线更早的阶段（如取指或译码阶段）对这些信息做出尽可能准确的猜测。
