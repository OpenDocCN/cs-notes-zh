# 070：向量处理器入门 🚀

![](img/64f45004de8ed4230d3b547b6a29d121_0.png)

在本节课中，我们将要学习向量计算机的基本概念。向量处理器是一种利用数据级并行性来处理大规模数组运算的架构。我们将从编程模型开始，了解其核心思想，并探讨其相对于传统标量处理器的优势。

---

## 软件模型：向量架构的抽象视图

上一节我们介绍了向量处理器的基本概念，本节中我们来看看它的抽象编程模型。在讨论具体硬件实现之前，理解软件如何与向量处理器交互至关重要。

在传统的向量架构中，处理器包含两种寄存器：
*   **标量寄存器**：与普通微处理器中的寄存器类似，每个寄存器保存一个值（例如32位或64位）。
*   **向量寄存器**：每个向量寄存器代表**多个值**。例如，Cray-1处理器的向量寄存器最多可容纳64个元素，每个元素为64位。

一个重要的硬件/编程模型组件是**向量长度寄存器**。它指定了当前操作实际使用的向量元素数量。为简化理解，我们可以先假设它总是等于向量的最大长度（例如64）。

---

## 向量指令集

为了支持向量操作，需要在原有标量指令集基础上增加新的指令。标量指令（如分支、地址计算）仍然在标量寄存器上执行。

以下是新增的向量指令类型：
*   **V.V 操作**：在两个向量寄存器之间执行操作，例如向量加法 `ADDV V1, V2, V3`。
*   **V.S 操作**：在一个向量寄存器和一个标量寄存器之间执行操作，例如向量与标量相加 `ADDSV V1, V2, R1`，这会将标量值加到向量中的每个元素上。
*   **向量加载/存储**：可以从内存中加载或存储大块数据到向量寄存器。

需要强调的是，这只是一个**抽象编程模型**，尚未涉及具体有多少个ALU或功能单元。

---

## 向量加载与存储

现在，我们来看看如何将数据移入和移出向量寄存器。核心指令是**加载向量**。

基本的加载向量指令格式为：`LV V1, (R1)`。其中，`V1`是目标向量寄存器，`R1`是一个标量寄存器，它保存了内存中向量数据的**基地址**。

更复杂的加载可以支持**跨步**访问。例如，可以指定一个跨步值（如5），从而加载内存中每隔4个元素的第5个元素。指令可能类似于 `LV V1, (R1), R2`，其中 `R2` 存放跨步值。

在本课程入门阶段，我们主要关注**单位跨步**，即顺序访问内存中的连续元素。

## 代码示例：标量 vs 向量

为了直观感受向量架构的优势，我们来看一个简单的代码示例：对两个数组进行逐元素相乘。

**标量C代码**：
```c
for (i=0; i<64; i++) {
    C[i] = A[i] * B[i];
}
```

**对应的标量汇编（MIPS风格）**：
需要循环、加载、相乘、存储、指针递增和边界检查。

**对应的向量汇编**：
1.  `LVI VL, #64`        // 设置向量长度寄存器为64
2.  `LV V1, (Ra)`        // 从地址Ra加载向量A到V1
3.  `LV V2, (Rb)`        // 从地址Rb加载向量B到V2
4.  `MULV V3, V1, V2`    // V3 = V1 * V2 （64次乘法）
5.  `SV (Rc), V3`        // 将结果向量V3存储到地址Rc

可以看到，向量代码**无需显式循环**，一条指令即可完成64次操作。向量长度寄存器提供了灵活性，可以处理长度不等于最大向量长度的循环。

## 向量架构的优势

那么，向量处理到底带来了哪些好处？

1.  **高效利用深流水线**：早期向量机（如Cray-1）采用很深的流水线以实现高时钟频率（如80MHz在当时非常快）。由于向量指令一次性处理大量数据，**无需在相邻数据元素之间进行数据旁路**，简化了硬件设计。
2.  **隐藏内存延迟**：向量加载/存储访问连续的内存块。通过使用**多体交叉存储器**，并确保访问是单位跨步，可以保证连续访问分布到不同的存储体上。
    *   **公式**：`访问地址` 对 `存储体数量` 取模，决定使用哪个存储体。
    *   这样，在第一个存储体还在为第一次访问服务时（占用时间），就可以启动对第二个、第三个存储体的访问。对于长向量，当循环回到第一个存储体时，它早已准备就绪，从而有效地**重叠了多个内存访问的延迟**，提高了内存带宽利用率。
