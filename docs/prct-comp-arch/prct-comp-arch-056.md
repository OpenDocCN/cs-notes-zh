# 056：多级缓存 🏗️

在本节课中，我们将要学习一种重要的缓存优化技术：**多级缓存**。我们将探讨为什么现代处理器普遍采用多级缓存设计，以及这种设计如何影响我们之前讨论过的各种缓存参数。

## 概述

现代处理器通常包含一级（L1）、二级（L2），甚至三级（L3）或四级（L4）缓存。本节课程将解释这种多级缓存设计的优势，并分析其对缓存性能指标的影响。

## 多级缓存的基本原理

上一节我们讨论了缓存优化的各种思路，本节中我们来看看如何通过增加缓存层级来提升性能。

核心问题在于，我们很难同时拥有**容量大**和**速度快**的单一存储器。根据物理定律，大容量意味着更远的物理距离，而更远的距离必然导致更长的访问延迟。

解决这一矛盾的方法是引入多级缓存。我们可以设计一个**容量小、速度快**的L1缓存，紧邻CPU。在其后方，再放置一个**容量较大、速度稍慢**的L2缓存。这样，大部分访问请求可以由快速的L1缓存满足，而L2缓存则能捕获更多L1未命中的数据，从而减少访问主内存（DRAM）的次数。

## 缓存性能指标辨析

在讨论多级缓存时，理解不同的性能指标至关重要。仅仅看某个缓存的“未命中率”可能无法反映其真实性能，因为多级缓存之间存在数据过滤效应。

以下是三个核心的性能指标概念：

*   **本地未命中率**：这是针对单个缓存的指标。其计算公式为：
    `本地未命中率 = (该缓存的未命中次数) / (访问该缓存的次数)`
    它衡量的是该缓存自身的工作效率。

*   **全局未命中率**：这在多级缓存体系中更有意义。其计算公式为：
    `全局未命中率 = (该缓存的未命中次数) / (CPU发出的总内存访问次数)`
    它衡量的是该缓存在整个内存访问路径中对CPU性能的影响。

*   **每指令未命中数**：这是一个更综合的指标。其计算公式为：
    `每指令未命中数 = (缓存未命中次数) / (执行的指令总数)`
    这个指标的优势在于，它同时考虑了程序的访存密集度（即load/store指令的比例）和缓存的未命中情况。即使某个缓存的本地未命中率很高，但如果程序本身访存操作很少，那么它对整体性能的影响也可能很小。

## 多级缓存对设计的影响

增加L2缓存不仅本身带来好处，还会深刻影响L1缓存的设计决策，两者并非独立存在。

### L1缓存容量与能耗

由于L2缓存可以作为后备，我们**可以设计更小的L1缓存**。更小的L1缓存可以放置得离CPU更近，从而获得更快的访问速度。虽然这会导致L1缓存的未命中率上升，但快速的L2缓存可以缓解这一问题。

从能耗角度看，这是一个显著的胜利。访问更小的L1缓存本身消耗的能量就更少。

### 简化L1缓存设计：写直达策略

在单一缓存设计中，我们通常采用**写回**策略，以避免每次写操作都访问慢速的主存。但这带来了复杂性：需要处理脏数据的替换和写回。

有了L2缓存后，L1缓存可以采用**写直达**策略。每次写操作都同时更新L1和L2。L2缓存可以作为写操作的缓冲区，吸收写流量，避免其立即冲击片外主存。脏数据的管理和复杂的替换逻辑可以交给L2缓存处理。

这极大地简化了L1缓存的控制逻辑，使其更容易集成到处理器流水线中。

### 简化缓存一致性

缓存一致性是多核处理器中的一个核心挑战，我们将在课程后期详细讨论。它涉及如何保持多个处理器核心的缓存数据同步。

采用**写直达的L1缓存**配合**写回的L2缓存**，可以简化一致性协议。当其他核心需要使本核心L1缓存中的某行数据失效时，由于L1是写直达的，它通常不持有唯一的脏数据副本（最新数据在L2中），因此只需简单地使L1中的对应行失效即可，无需触发复杂的写回操作。这减少了对处理器流水线的干扰。

### 简化错误恢复

在遇到软错误（如宇宙射线导致的比特翻转）时，写直达的L1缓存也更有优势。因为L1中不保存唯一的脏数据，当检测到错误（例如奇偶校验错误）时，可以直接将该缓存行标记为无效。我们知道L2缓存中保存着该数据的正确副本，因此无需宣告内存数据损坏，简化了错误恢复流程。

## 包含性与排他性缓存

当我们拥有多级缓存时，就面临一个策略选择：数据在不同层级间是**包含**还是**排他**的？

*   **包含性缓存**：较低层级（如L1）缓存中的数据，一定也存在于较高层级（如L2）缓存中。公式表示为：`L1 ⊆ L2`。
    *   **优点**：简化缓存一致性操作。外部的一致性查询（如侦听）只需要检查最外层的缓存（如L2），如果L2中没有，则L1中肯定也没有。
    *   **缺点**：存储空间利用率较低，因为同一份数据在多个层级中保存了副本。

*   **排他性缓存**：不同层级的缓存尽量保存不同的数据。当数据从L1被替换出去时，通常会放入L2。
    *   **优点**：有效存储容量更大。总可用缓存容量近似等于`L1容量 + L2容量`。
    *   **缺点**：硬件设计更复杂，需要在层级间交换数据行；一致性协议也更复杂。

## 现代处理器实例分析

让我们通过两个处理器的例子，看看实际设计中如何权衡。

### Intel Itanium 2 处理器
*   **L1缓存**：16KB，4路组相联，单周期延迟。
*   **L2缓存**：256KB，4路组相联，5周期延迟。
*   **L3缓存**：3MB，12路组相联，12周期延迟，采用更大的128字节行大小。
*   **分析**：从L1到L2，容量增加了16倍，符合“经验法则”。越远离核心，缓存容量和关联度增加，行大小也变大，以利用空间局部性。

### IBM Power7 处理器
*   **L1缓存**：32KB指令/数据缓存，3周期延迟。
*   **L2缓存**：256KB私有缓存，8周期延迟（容量为L1的8倍）。
*   **L3缓存**：32MB共享缓存，由嵌入式DRAM实现，25周期延迟。
*   **分析**：展示了多核共享大容量末级缓存的设计，以及嵌入式DRAM技术的应用。

## 经验法则：8倍规则

在缓存设计中，一个常见的经验法则是：**为了增加一个缓存层级是值得的，新层级的容量至少应是下一级缓存的8倍**。
这个规则源于一个权衡：增加缓存层级会引入额外的访问延迟。如果新缓存不够大，其带来的未命中率下降收益，可能无法抵消因检查该缓存而增加的延迟成本。大多数实际处理器设计都遵循或超过这一比例。

## 性能记分板总结

最后，让我们更新我们的性能记分板，总结多级缓存带来的影响：

*   **对于L1缓存**：未命中惩罚**降低**。因为未命中时只需访问较快的L2，而非遥远的主存。
*   **对于整个缓存体系（L1+L2+...）**：**平均访存时间**得到改善。这是因为：
    1.  **未命中率降低**：更大的末级缓存减少了访问主存的频率。
    2.  **未命中惩罚降低**：层次化设计使得每次未命中需要走的“距离”更短。
