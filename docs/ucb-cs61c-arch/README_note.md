# 🧠 UCB CS61c 组成原理中文笔记：第1课 - 计算机抽象与指令集架构

在本节课中，我们将学习计算机科学中两个最核心的抽象概念：**计算机抽象**与**指令集架构**。理解这些概念是理解计算机如何工作的基础。

## 计算机抽象

上一节我们介绍了课程目标，本节中我们来看看什么是计算机抽象。计算机系统由多个层次构成，每一层都为其上层提供了一个简化的“抽象”接口，隐藏了下层的复杂实现细节。这使得我们可以专注于当前层的逻辑，而无需理解所有底层的硬件或软件是如何工作的。

以下是计算机系统中一些关键的抽象层次：

*   **高级语言层**：如Python、Java，让程序员用接近人类语言的语法编写程序。
*   **操作系统层**：管理硬件资源（CPU、内存、磁盘），为应用程序提供统一的系统调用接口。
*   **指令集架构层**：定义了一台计算机能够理解和执行的所有基本指令的集合，是硬件与软件之间的关键契约。
*   **微架构层**：指处理器（CPU）内部的具体实现，例如如何取指、译码、执行。
*   **逻辑电路层**：由门电路（与、或、非门）构成，实现具体的计算功能。
*   **器件层**：最底层，由晶体管等物理器件组成。

## 指令集架构

理解了抽象层次后，我们聚焦于连接软件与硬件的关键层——指令集架构。ISA是硬件提供给软件的一套指令集合和编程模型。它规定了软件可以使用的寄存器、指令格式、寻址模式以及如何访问内存。

一个典型的ISA指令格式可以表示为：
`操作码 + 操作数1 + 操作数2 + 目标地址`

以下是ISA的两个主要分类：

*   **复杂指令集计算机**：指令复杂，功能强大，单条指令能完成较多工作。指令长度可变，寻址模式丰富。代表是x86架构。
*   **精简指令集计算机**：指令简单，格式固定，只完成基本操作，复杂功能由多条指令组合实现。追求高时钟频率和流水线效率。代表是MIPS、ARM、RISC-V架构。

## CISC与RISC对比

为了更清晰地理解CISC与RISC的区别，以下是它们主要特性的对比：

*   **设计哲学**：CISC希望用最少的指令完成任务，RISC希望用简单的指令快速执行。
*   **指令复杂度**：CISC指令复杂，RISC指令精简。
*   **指令长度**：CISC长度可变，RISC长度固定（例如4字节）。
*   **寻址模式**：CISC支持多种，RISC支持较少。
*   **执行时间**：CISC指令间执行时间差异大，RISC大多数指令单周期完成。
*   **控制单元**：CISC多采用微码控制，RISC多采用硬连线控制。
*   **寄存器使用**：CISC操作可直接针对内存，RISC强调`加载/存储`架构，即运算只在寄存器间进行，与内存交互只有专门的加载和存储指令。

## 总结

本节课中我们一起学习了计算机系统的抽象层次，并深入探讨了作为软硬件界面的**指令集架构**。我们了解了ISA的定义、基本格式，并对比了**CISC**与**RISC**两种主要设计哲学的特点与区别。掌握这些概念是后续学习具体指令集和计算机组成细节的坚实基础。
