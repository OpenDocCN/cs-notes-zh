## 回顾乘数权重法 📚

上一讲我们介绍了乘数权重法，这是一个非常简单的算法。虽然我们今天不需要详细记住其更新规则，但请记住它是一个轻量级且易于实现的算法。其分析虽然不简单，但我们已经完成，现在我们将把它作为黑盒使用。

以下是乘数权重法的保证：假设对手选择奖励向量，您每天根据乘数权重法选择一个动作分布。对手产生的奖励向量中，所有奖励值在 -1 到 1 之间。您的目标是最大化总奖励，而基准是事后最佳固定动作。乘数权重法保证，其期望奖励与基准之间的误差项随时间增长是次线性的。

## 扩展与解释 🔧

为了今天的应用，我们需要对上述保证进行一些简单的扩展。

### 扩展一：与混合策略的比较

乘数权重法不仅与事后每个固定动作相比具有保证，还与每个混合策略相比具有相同的保证。换句话说，对于任何动作分布或混合策略 P，乘数权重法的期望奖励至少与始终按照该分布选择动作的期望奖励一样好。这是因为事后最佳固定动作已经是最优的，混合策略不会提供额外的优势。

以下是具体说明：对于任何固定分布 P，如果每天按照该分布选择动作，则期望奖励不会超过最佳固定动作的奖励。因此，乘数权重法的保证自然扩展到混合策略。

### 扩展二：时间平均遗憾

另一种理解乘数权重法保证的方式是通过时间平均遗憾。如果我们希望每步遗憾不超过 ε，那么需要玩游戏的次数至少为 4 log n / ε²。这意味着乘数权重法的收敛速度非常快，仅与动作数量的对数相关。

### 扩展三：奖励范围的扩展

如果奖励值在 -M 到 M 之间，我们可以通过缩放将其转换为 -1 到 1 的范围。具体来说，将奖励除以 M，运行乘数权重法，然后再缩放回来。这样，为了达到每步遗憾不超过 ε，需要的迭代次数为 M² / ε² * log n。

## 零和博弈中的应用 🎲

接下来，我们将讨论乘数权重法在零和博弈中的应用。我们将展示如何利用乘数权重法证明最小最大定理，并推导相关算法。

### 博弈设置

考虑一个零和博弈，行玩家使用乘数权重法选择行分布，而列玩家在观察到行玩家的分布后，选择最佳响应列。这种设置对行玩家不利，因为它必须首先公开其策略。

### 行玩家与列玩家的表现

在这种设置下，列玩家至少获得其最小最大值，甚至可能更好。而行玩家尽管处于劣势，但其期望收益与最小最大值之间的差距不超过 ε。以下是证明的关键步骤：

1. 行玩家使用乘数权重法生成分布。
2. 列玩家根据行玩家的分布选择最佳响应列。
3. 通过乘数权重法的保证，行玩家的时间平均收益至少与任何固定混合策略一样好。
4. 结合最小最大值的定义，可以证明行玩家的收益接近最小最大值。

### 最小最大定理的证明

通过上述分析，我们可以得到另一个最小最大定理的证明。当 ε 趋近于零时，行玩家和列玩家的期望收益都收敛到最小最大值，从而证明最小最大定理成立。

## 线性规划中的应用 📐

接下来，我们将探讨乘数权重法在线性规划中的应用，特别是线性分类器问题。

### 问题描述

假设我们有一些正点和负点，目标是找到一个线性函数，使得所有正点在一侧，所有负点在另一侧。通过预处理，我们可以将问题简化为寻找一个通过原点且系数非负的半空间。

### 算法设计

我们将乘数权重法应用于线性分类器问题。具体步骤如下：

1. 将坐标作为动作，运行乘数权重法生成系数分布。
2. 检查当前系数是否满足所有点的约束。
3. 如果满足，则找到解；否则，选择一个违反约束的点，将其坐标作为奖励向量反馈给乘数权重法。
4. 重复直到找到解或达到最大迭代次数。

### 算法正确性

算法的正确性基于以下观察：
- 如果当前系数不满足约束，则乘数权重法的期望奖励非正。
- 假设存在一个满足边际 ε 的解，则该解对应的期望奖励至少为 ε。
- 通过乘数权重法的遗憾保证，当迭代次数足够多时，必须找到一个满足约束的解。

### 与零和博弈的联系

线性分类器问题可以视为一个零和博弈，其中行玩家选择系数分布，列玩家选择数据点。乘数权重法在这个博弈中的应用与上述算法完全一致。

## 最大流问题中的应用 🌊

最后，我们将简要介绍乘数权重法在最大流问题中的应用。通过将最大流问题转化为零和博弈，我们可以利用乘数权重法近似求解。

### 问题转化

将最大流问题转化为零和博弈：
- 行玩家选择边分布，对应对偶可行解。
- 列玩家选择路径，对应原始可行解。
- 支付矩阵定义为边是否在路径中。

### 算法步骤

1. 行玩家使用乘数权重法选择边分布。
2. 列玩家根据边分布计算最短路径作为最佳响应。
3. 重复直到遗憾降至 ε 以下。
4. 将所有列玩家选择的路径叠加，得到近似最大流。

### 算法优势

这种方法不仅适用于最大流问题，还可以扩展到多商品流问题，这是传统算法难以高效解决的。
