# 斯坦福大学《网络优化算法》课程笔记：第3讲：最大流的推送-重标算法

在本节课中，我们将继续学习最大流问题。我们将介绍一种与增广路径算法完全不同的新范式——推送-重标算法。这种算法在实践中通常非常高效，是现代解决最大流问题的常用方法之一。

## 算法动机：为何需要新范式？

上一节我们介绍了基于增广路径的算法。本节中，我们来看看为何需要一种不同的方法。

考虑一个特定的网络实例。假设有一个参数K（例如10000）。网络包含一条从源点S到中间节点X的路径，该路径由K条边组成，每条边的容量为K。同时，节点X通过K条互不相交的路径连接到汇点T，但每条路径的容量仅为1。

在这个网络中，最大流值显然是K（受限于右侧的单位容量边）。然而，如果我们使用Ford-Fulkerson算法，每次迭代只能沿长路径推送1单位流量，每次迭代需要O(K)时间扫描路径，总共需要K次迭代，导致总运行时间为O(K²)。

直观上，我们更希望有一种算法能“淹没”从S到X的路径，一次性将K单位流量推到X，然后再花线性时间将其分配到右侧的K条边中。这种“推送”思想，而非沿着完整路径“增广”，是推送-重标算法的核心直觉。当然，如果右侧路径少于K条，我们可能会在X处产生“超额流”，需要将其退回源点。关键在于，如何系统化地实现这种思想，使其适用于任何网络结构。

## 核心概念：预流

为了允许“推送”和可能产生的“超额”，我们首先需要放宽流定义中的守恒约束。

**预流** 是一个定义在边上的向量，它满足两个条件：
1.  **容量约束**：对于每条边 `e`，满足 `0 ≤ f(e) ≤ u(e)`。
2.  **松弛的守恒约束**：对于除源点 `S` 和汇点 `T` 外的每个顶点 `v`，流入的流量可以**大于或等于**流出的流量。即允许“超额”存在。

用公式表示，对于所有 `v ∈ V \ {s, t}`：
```
∑_{e into v} f(e) ≥ ∑_{e out of v} f(e)
```
而在流中，我们要求严格相等。

## 剩余图与超额量

与流一样，对于预流 `f`，我们同样可以定义**剩余图** `G_f`。构造方式完全相同：对于原图中的每条边 `(u, v)`，如果 `f(e) < u(e)`，则在剩余图中添加一条正向边 `(u, v)`，剩余容量为 `u(e) - f(e)`；如果 `f(e) > 0`，则添加一条反向边 `(v, u)`，剩余容量为 `f(e)`。

为了衡量预流违反守恒约束的程度，我们定义顶点 `v` 的**超额量** `excess(v)`：
```
excess(v) = ∑_{e into v} f(e) - ∑_{e out of v} f(e)
```
对于预流，除 `S` 和 `T` 外，所有顶点的超额量都**非负**。一个预流是流（即可行），当且仅当所有顶点的超额量均为0。因此，算法的目标就是将预流转化为流，即消除所有超额量。

## 基本操作：推送

在增广路径算法中，我们通过沿整条 `S-T` 路径增加流量来修改流。现在，我们允许违反守恒约束，因此可以在单步操作中只沿**一条边**推送流量。

**推送(Push)** 子程序作用于一条边 `(v, w)`，该边必须存在于当前剩余图 `G_f` 中（即剩余容量 > 0）。设：
-   `excess(v)` 为顶点 `v` 的当前超额量。
-   `r(v, w)` 为边 `(v, w)` 在剩余图中的剩余容量。

我们可以推送的流量 `Δ` 受限于这两个值：
```
Δ = min( excess(v), r(v, w) )
```
然后，我们沿边 `(v, w)` 推送 `Δ` 单位的流量。这与增广路径中的更新方式一致：如果 `(v, w)` 是原图中的正向边，则增加其流量；如果是反向边（对应于原图中反向推送），则减少原图中对应边的流量。

推送操作可能会在终点 `w` 创造新的超额量。

## 挑战与解决方案：高度函数

如果只是随意推送，算法可能会陷入循环，例如让超额量在一个环上无限循环。为了系统性地引导推送并保证正确性与终止性，我们引入**高度函数**的概念。

为每个顶点 `v` 维护一个高度 `h(v)`。可以将其想象为顶点的“海拔”。算法将始终保持以下三个**不变性**：
1.  **源点高度固定**：`h(s) = n`（n为顶点总数），且永不改变。
2.  **汇点高度固定**：`h(t) = 0`，且永不改变。
3.  **高度差限制**：对于剩余图 `G_f` 中的任意边 `(v, w)`，满足 `h(v) ≤ h(w) + 1`。这意味着边只能**上山**、**持平**或**最多下山1个单位**。

这些不变性的关键在于：**它们保证了在剩余图中，从源点 `S` 到汇点 `T` 没有路径**。因为任何从 `S`（高度n）到 `T`（高度0）的路径最多有 `n-1` 条边，即使每条边都允许下降1，总下降高度最多为 `n-1`，无法从n下降到0。这与最大流的最优性条件（剩余图中无增广路径）相关联。

因此，推送-重标算法的策略与增广路径算法形成对称：
-   **增广路径算法**：始终保持可行性（是一个流），努力达到最优性条件（剩余图中无 `S-T` 路）。
-   **推送-重标算法**：始终保持最优性条件（通过高度函数保证无 `S-T` 路），努力恢复可行性（将预流转化为流）。

## 推送-重标算法描述

以下是算法的完整描述。

### 初始化

我们不能从全零流开始，因为那会违反高度差限制（从高度n的S到高度0的邻居，下山超过1）。因此，初始化如下：
-   **设置高度**：`h(s) = n`；对于所有其他顶点 `v`，`h(v) = 0`。
-   **初始化预流**：**饱和**所有从源点 `S` 出发的边，即对于每条边 `(s, v)`，设置 `f(s, v) = u(s, v)`。所有其他边上的流量设为0。
这样，从S出发的边在剩余图中被移除（因为饱和了），从而满足了高度差限制。同时，这会在S的邻居处产生初始的超额量。

### 主循环

只要存在除 `S` 和 `T` 外具有正超额量 `(excess(v) > 0)` 的顶点，就重复以下步骤：
1.  **选择活动顶点**：在所有具有正超额量的顶点中，选择**高度最高**的一个，记为 `v`。若高度相同，则任意选择。
2.  **尝试推送**：检查在剩余图 `G_f` 中，是否存在从 `v` 出发的边 `(v, w)`，满足 `h(v) = h(w) + 1`（即“下山”边）。
    -   如果存在这样的边：
        -   执行 **推送(Push)** 操作：沿该边推送 `min(excess(v), r(v, w))` 单位的流量。
    -   如果不存在这样的边：
        -   执行 **重标(Relabel)** 操作：将顶点 `v` 的高度增加1，即 `h(v) = h(v) + 1`。

### 终止与正确性

当主循环退出时，意味着没有顶点具有正超额量。根据定义，此时的预流 `f` 已经是一个可行的流。同时，由于算法始终维护高度不变性，剩余图中没有 `S-T` 路径。根据最大流最小割定理，这正是一个最大流。因此，算法终止时输出最大流。

## 算法示例

让我们通过一个简单网络来演示算法的执行过程。考虑以下网络（此处用文字描述，建议对照视频中的图示）：
-   S连接到A和B，容量分别为1和100。
-   A连接到T，容量为1。
-   B连接到A和T，到A的容量为1，到T的容量为1。
已知最大流值为3。

**初始化**：
-   高度：`h(S)=4`, `h(A)=h(B)=h(T)=0`。
-   预流：饱和S出发的边，`f(S,A)=1`, `f(S,B)=100`。其他边流量为0。
-   超额量：`excess(A)=1`, `excess(B)=100`。

**迭代过程简述**：
1.  选择最高超额顶点（A和B同高，选A）。A无下山边，重标：`h(A)=1`。
2.  选择最高超额顶点（A，高度1）。A有下山边至T，执行非饱和推送1单位到T。A超额归零。
3.  选择唯一超额顶点B（高度0）。B无下山边，重标：`h(B)=1`。
4.  选择B。B有下山边至T，饱和推送1单位到T。B超额变为99。
5.  选择B。B无下山边（至A是持平，至S是上山），重标：`h(B)=2`。
6.  选择B。B有下山边至A（原边及反向边），推送流量...（后续步骤会逐步将超额从B推回A，再推回S或T，最终经过一系列推送和重标后，得到最大流3）。

此例展示了算法如何通过推送和重标操作，最终消除所有超额，得到最大流。

## 运行时间分析（概要）

算法的时间复杂度分析基于对推送和重标操作次数的限制。以下是核心结论（Goldberg & Tarjan, 1980s）：
-   **重标次数**：`O(n²)`。
-   **推送次数**：`O(n²m)`（饱和推送 `O(nm)`，非饱和推送 `O(n³)`，合起来 `O(n²m)`）。

**分析思路**：
1.  **关键引理**：任何具有正超额量的顶点 `v`，在剩余图中都存在一条从 `v` 回到源点 `s` 的路径。
2.  **重标次数界**：由关键引理和高度差限制可证，任何顶点的高度不会超过 `2n-1`。每次重标增加高度，因此每个顶点最多重标 `O(n)` 次，总计 `O(n²)` 次。
3.  **饱和推送次数界**：对于一条边 `(v,w)`，在两次同方向的饱和推送之间，顶点 `v` 和 `w` 都至少被重标了2次。由于重标总次数有限，每条边的饱和推送次数为 `O(n)`，总计 `O(nm)`。
4.  **非饱和推送次数界**：在两次重标操作之间，最多发生 `O(n)` 次非饱和推送。因为非饱和推送会清零一个顶点的超额，且在该重标周期内，由于我们总是选择最高顶点推送，被清零的超额不会再次出现（除非有更高的顶点推送下来，但这需要重标）。结合重标次数 `O(n²)`，得非饱和推送 `O(n³)` 次。

综合以上，总操作次数（推送+重标）为多项式级别，因此算法在多项式时间内终止。通过适当的数据结构（如使用桶来维护最高超额顶点），可以实现 `O(n³)` 的总运行时间。

## 总结

本节课中我们一起学习了最大流问题的推送-重标算法。我们首先看到了增广路径算法在某些实例上的低效性，从而引出了“推送”流量的新想法。为了系统化这一想法，我们引入了**预流**的概念以放宽守恒约束，并定义了**超额量**来衡量不可行性。算法的核心在于使用**高度函数**来引导推送方向，并通过维护高度不变性来隐含保证最优性条件。算法通过两种基本操作——沿下山边**推送**和增加顶点高度的**重标**——不断消除超额量，最终在获得一个可行流的同时，该流也是最大流。我们还概述了算法的多项式时间复杂度证明。推送-重标算法因其在实践中出色的性能而成为解决最大流问题的重要方法。
