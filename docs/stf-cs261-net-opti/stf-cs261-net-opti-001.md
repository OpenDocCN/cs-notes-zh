# 001：课程目标与最大流问题介绍

在本节课中，我们将学习这门课程的整体目标，并深入探讨网络优化中的一个核心问题：最大流问题。我们将了解其定义、一个直观但不完善的贪心算法，以及通过引入“残差图”概念进行修正后得到的正确算法——Ford-Fulkerson算法。

## 课程目标概述

本课程有两个主要目标，各占课程内容的约一半。

首先，课程将延续CS161（算法导论）的精神，专注于研究基础且重要的问题及其高效算法。这类问题通常具有两个特点：它们本身足够基础且灵活，能够为众多应用建模；同时，它们恰好存在高效的精确求解算法。在CS161中，最短路径问题就是一个典型例子。本课程的目标之一是让大家避免“重复造轮子”。当你遇到实际问题时，如果能识别出它本质上是某个已有高效算法的基础问题，就可以直接应用现成工具快速解决。我们将练习识别那些看似不同、实则是基础问题“变体”的能力。

在课程的前五周，我们将具体学习以下几个更复杂的问题：最大流问题、最小割问题（与最大流紧密相关）、匹配计算问题，并最终学习线性规划。线性规划是已知在理论和实践中都高效的、最具通用性的问题之一。我们将讨论这些问题的算法及其实际应用背景。虽然这些算法的运行时间可能不像CS161中的许多算法那样接近线性时间（例如可能是二次时间），但它们通常仍然足够快，足以解决实际问题。

其次，现实中的许多重要问题并不属于上述“有高效精确解”的范畴。因此，本课程的第二个目标是提供处理这类“难解”问题的工具。我们将主要关注两类问题：第一类是NP难问题，对于这类问题，我们通常认为不存在多项式时间的精确算法（除非P=NP）。我们将学习如何妥协，例如研究启发式算法的设计与分析技巧。第二类是在线算法，这里的“在线”并非指互联网，而是指算法必须在不知道未来输入的情况下做出决策，这通常迫使你设计某种贪心策略。这类问题在实际中也非常常见。

本课程面向两类听众：对于计划将此作为最后一门算法课的同学，课程将涵盖算法领域的精华内容；对于计划深入研究算法（如考虑攻读研究生或学习更高级课程）的同学，课程将提供对前沿领域的一瞥，并作为通往高级算法研究的桥梁。

## 最大流问题介绍

现在，让我们开始学习第一个主题：最大流算法。这是一个非常经典的算法问题。

### 问题定义与直观理解

首先通过一个例子直观地理解最大流问题。我们有一个**有向图**，可以将其视为一个网络。每条边上都有一个数字，称为该边的**容量**，表示这条边能承载的“流量”上限。最大流问题的目标是找出从指定的**源点** 到指定的**汇点** ，能通过网络推送的最大流量。

考虑下图示例：
```
     s --3--> a --2--> t
     |        |        |
     2        1        2
     v        v        v
     b --3--> c --2--> t
```
（假设这是一个有向图，边指向下方和右方）

在这个网络中，边上的数字表示容量。一个可能的方案是：从上路径 `s->a->t` 推送2单位流量，从下路径 `s->b->c->t` 推送2单位流量，再从曲折路径 `s->a->c->t` 推送1单位流量。这样总共推送了5单位流量，且没有超过任何边的容量。可以证明，无法推送超过5单位的流量，因为从源点 `s` 出发的所有边的总容量只有5（3+2）。因此，最大流值为5。

下面给出更正式的定义。

**输入**：
*   一个有向图 `G = (V, E)`。
*   两个特殊顶点：源点 `s` 和汇点 `t`。
*   每条边 `e ∈ E` 有一个非负整数容量 `c(e)`。

**可行解（流）**：
一个流 `f` 为每条边 `e` 分配一个非负实数 `f(e)`，称为该边的流量。它必须满足两类约束：
1.  **容量约束**：对于每条边 `e`，流量不能超过其容量。公式表示为：`0 ≤ f(e) ≤ c(e)`。
2.  **守恒约束**：对于除 `s` 和 `t` 外的任意顶点 `v`，流入 `v` 的总流量必须等于流出 `v` 的总流量。即：`∑_{e into v} f(e) = ∑_{e out of v} f(e)`。

**目标**：
在所有满足约束的流中，找到一个流，使得从源点 `s` 流出的总流量（即流的值）最大。流的值定义为：`v(f) = ∑_{e out of s} f(e)`。可以证明，这也等于流入汇点 `t` 的总流量。

为简化问题，通常假设没有边进入 `s`，也没有边离开 `t`，因为这样的边对最大化从 `s` 到 `t` 的流量没有帮助。

### 一个简单的贪心算法及其缺陷

面对新问题，一个自然的起点是尝试贪心算法。以下是针对最大流问题的一个朴素贪心思路。

**算法1（朴素增广）**：
1.  从所有边流量为0的流（零流）开始。
2.  在当前的流 `f` 中，寻找一条从 `s` 到 `t` 的路径 `P`，使得路径上的每条边都有剩余容量（即 `f(e) < c(e)`）。这可以通过图搜索算法（如BFS或DFS）在 `O(|E|)` 时间内完成。
3.  如果找不到这样的路径，算法终止，输出当前流 `f`。
4.  如果找到路径 `P`，则计算该路径上每条边的剩余容量 `c(e) - f(e)` 的最小值，记为 `Δ`。然后，将路径 `P` 上每条边的流量增加 `Δ`。这保证了容量约束仍然满足，并且由于路径上每个中间节点的流入和流出都增加了 `Δ`，守恒约束也得以保持。
5.  返回步骤2。

然而，这个算法并不正确。它可能提前终止于一个非最大流。考虑之前例子的一种变体：如果算法首先选择了曲折路径 `s->a->c->t` 并推送了3单位流量（该路径的最小容量），那么上边 `s->a` 和下边 `c->t` 将被饱和。此时，图中不再存在所有边都有剩余容量的 `s-t` 路径，算法将终止，输出一个值为3的流。但我们知道最大流值是5。算法陷入了局部最优，因为它无法“撤销”先前做出的可能不好的流量分配决定。

### 引入残差图与Ford-Fulkerson算法

上述贪心算法的缺陷在于它一旦分配了流量就无法更改。修正的思路是允许某种形式的“撤销”操作。具体来说，如果我们之前在某条边上推送了流量，现在应该允许为了整体利益，沿相反方向推送流量来抵消它。

为了系统化地描述这种“撤销”操作，我们引入了**残差图**的概念。

**残差图 `G_f`**：
给定图 `G` 和一个流 `f`，残差图 `G_f` 定义如下：
*   顶点集与 `G` 相同。
*   对于 `G` 中的每条边 `e = (u, v)`，且 `f(e) < c(e)`，则在 `G_f` 中添加一条**正向边** `(u, v)`，其**残差容量**为 `c(e) - f(e)`。这条边表示还可以沿原方向推送多少流量。
*   对于 `G` 中的每条边 `e = (u, v)`，且 `f(e) > 0`，则在 `G_f` 中添加一条**反向边** `(v, u)`，其**残差容量**为 `f(e)`。这条边表示可以沿相反方向推送多少流量来“撤销”原有的流量。

残差图编码了所有可能的流量调整操作：沿正向边增加流量，沿反向边减少流量。

基于残差图，我们得到了正确的 **Ford-Fulkerson算法**。

**算法2（Ford-Fulkerson算法）**：
1.  初始化：设 `f` 为零流。
2.  构建当前流 `f` 对应的残差图 `G_f`。
3.  在残差图 `G_f` 中，寻找一条从 `s` 到 `t` 的路径 `P`（同样可用BFS/DFS）。
4.  如果找不到这样的路径，算法终止，输出当前流 `f`。
5.  如果找到路径 `P`，则计算该路径上所有边残差容量的最小值，记为 `Δ`。
6.  **增广**：对于路径 `P` 上的每条边：
    *   如果它是 `G_f` 中的正向边，对应 `G` 中的边 `(u, v)`，则令 `f(e) += Δ`。
    *   如果它是 `G_f` 中的反向边，对应 `G` 中边 `(u, v)` 的反向，则令 `f(e) -= Δ`（即减少原边的流量）。
7.  返回步骤2。

回到之前的例子，当朴素算法在推送3单位流量后停滞时，其残差图中包含一条路径 `s->b->c->a->t`，其中 `c->a` 是反向边。沿这条路径增广2单位流量，相当于在原始图中：在 `s->b` 和 `b->c` 上增加流量，在 `a->c` 上减少流量（即撤销部分流量），在 `a->t` 上增加流量。最终得到了值为5的最大流。

### 算法正确性分析框架

为什么Ford-Fulkerson算法是正确的？我们需要一个框架来证明当算法终止时，它确实找到了最大流。

一个通用的算法分析范式包含两步：
1.  **寻找最优性条件**：找出一个**充分条件**，如果某个解满足该条件，则它一定是最优解。这部分是结构性的，不涉及具体算法。
2.  **设计满足条件的算法**：设计一个算法，确保它**只在**满足上述最优性条件时才终止。

如果完成了这两步，那么算法的正确性自然得证。

对于最大流问题，其最优性条件由以下关键事实给出（我们将在下节课证明）：

**定理（最优性条件）**：设 `f` 是图 `G` 中的一个流。如果在残差图 `G_f` 中**不存在**从 `s` 到 `t` 的路径，那么 `f` 就是 `G` 中的一个最大流。

观察Ford-Fulkerson算法：它的终止条件恰恰是“在残差图中找不到 `s-t` 路径”。因此，它完美地满足了上述算法设计范式的第二步。只要我们接受这个定理（第一步），就可以立即得出以下推论：

**推论**：Ford-Fulkerson算法在终止时，总能输出一个最大流。

关于算法终止性的简要说明：由于我们假设容量为整数，且每次增广的流量 `Δ` 是残差容量的最小值（也是整数且至少为1），因此每次迭代流的值至少增加1。流的值存在上界（例如所有从 `s` 出发的边容量之和），故算法必然在有限步内终止。
