# 斯坦福大学《网络优化算法》课程笔记：第19讲：超越暴力搜索

在本节课中，我们将探讨如何为NP难问题设计比暴力搜索更快的精确算法。我们将坚持算法的完全正确性，但允许运行时间超过多项式级别。我们将通过三个经典问题——顶点覆盖、旅行商问题和3-SAT——来具体展示如何实现这一点。

---

## 顶点覆盖问题：19.1：参数化算法与核化

上一节我们介绍了本课程的目标。本节中，我们来看看第一个例子：顶点覆盖问题。

我们考虑一个特殊情况：给定一个无向图，每个顶点的代价为1（即寻找最小基数的顶点覆盖）。更具体地说，我们给定一个目标值K，任务是判断图中是否存在一个大小不超过K的顶点覆盖。我们假设K是一个较小的值（例如10到20），而图的规模（节点数n）可能很大（例如成百上千）。

暴力搜索需要检查所有大小为K的顶点子集，其时间复杂度约为 **O(n^K)**。对于稍大的K，这完全不可行。我们的目标是设计一个算法，其运行时间形式为 **O(f(K) * poly(n))**，其中f(K)是仅依赖于K的函数（通常是指数级），而poly(n)是关于n的多项式。这类算法被称为**固定参数可解**算法。

以下是算法的步骤：

1.  **预处理高度数顶点**：将所有度数大于K的顶点加入集合S。因为如果存在一个大小不超过K的顶点覆盖，它必须包含所有这些顶点，否则无法覆盖它们关联的大量边。
2.  **删除已选顶点**：从原图G中删除集合S中的所有顶点及其关联的边，得到子图G‘。剩余的任务是在G’中寻找一个大小不超过 **K - |S|** 的顶点覆盖。
3.  **删除孤立点**：删除G‘中所有度数为0的孤立顶点，得到图G‘’。
4.  **边数检查**：如果G‘’中的边数超过 **K²**，则直接判定不存在大小不超过K的顶点覆盖。因为G‘’中每个顶点的度数最多为K，最多K个顶点最多只能覆盖K²条边。
5.  **暴力搜索**：现在，图G‘’的规模（顶点和边数）是 **O(K²)**。我们在这个小图上，对所有大小不超过 **K - |S|** 的顶点子集进行暴力搜索，检查其是否为顶点覆盖。

**算法分析**：
*   **正确性**：上述步骤保证了，原图存在大小不超过K的顶点覆盖，当且仅当能在缩减后的小图G‘’中找到大小不超过 **K - |S|** 的顶点覆盖。
*   **运行时间**：预处理步骤（1-4）可以在 **O(m)** （线性）时间内完成。步骤5的暴力搜索在大小为 **O(K²)** 的图上进行，因此时间复杂度为 **O(2^(K²))**。总运行时间为 **O(m + 2^(O(K²)))**，成功地将对n的依赖（多项式）与对K的依赖（指数）分离开来。

在图G‘’上求解顶点覆盖问题，这个缩减后的小图被称为原问题的**核**。通过核化技术，我们将一个大实例化简为一个仅与参数K相关的小实例，从而显著提升了算法效率。

## 旅行商问题：19.2：动态规划与状态压缩

上一节我们介绍了通过参数化方法超越暴力搜索。本节中，我们来看看一个不同的思路：使用动态规划来优化完全NP难问题的指数时间算法，即使不引入小参数假设。

我们考虑一般的旅行商问题：给定一个完全图，每条边有非负代价，需要找到一个访问所有顶点恰好一次的最小代价回路（哈密顿回路）。暴力搜索需要检查所有n!个顶点排列，这是不可行的。

我们将展示一个动态规划算法，其运行时间为 **O(n² * 2^n)**，远优于O(n!)。

**算法核心：定义子问题**

我们使用动态规划。令顶点编号为1到n。定义子问题：
`dp[S][j]`：其中S是顶点集合的一个子集，且必须包含起点1；j是S中的一个顶点。这个子问题的值是：从顶点1出发，访问集合S中所有顶点恰好一次，最后停在顶点j的**最小代价路径**的代价。

**子问题数量**：S有2^n种可能，j有n种可能，所以子问题总数为 **O(n * 2^n)**。

**动态规划转移方程**

1.  **基础情况**：当 `S = {1}` 且 `j = 1` 时，`dp[{1}][1] = 0`。
2.  **转移方程**：对于 `|S| > 1` 且 `j != 1` 的情况，我们考虑到达j之前最后一个顶点是谁。假设是k (k ∈ S, k ≠ j)。那么，这条最优路径的代价等于：从1出发访问集合 `S \ {j}` 中所有顶点并最终停在k的最优代价，再加上从k到j的边代价 `c(k, j)`。我们需要遍历所有可能的k，取最小值。
    **公式**：`dp[S][j] = min_{k ∈ S, k ≠ j} { dp[S \ {j}][k] + c(k, j) }`

**求解原问题**

在计算完所有子问题后，最优TSP回路的代价可以通过以下方式得到：考虑所有可能的、在最优回路中位于顶点1之前的顶点j。
**公式**：`min_{j=2 to n} { dp[V][j] + c(j, 1) }`
其中V是所有顶点的集合。`dp[V][j]` 给出了从1到j访问所有顶点的最优路径代价，加上边(j,1)就构成一个回路。

**算法分析**：
*   **运行时间**：共有O(n * 2^n)个子问题。计算每个子问题`dp[S][j]`需要遍历S中所有可能的k（最多O(n)个）。因此总时间复杂度为 **O(n² * 2^n)**。
*   **空间复杂度**：需要存储整个DP表，空间复杂度为 **O(n * 2^n)**。

这个算法通过状态压缩（用二进制位掩码表示集合S）和动态规划，巧妙地将阶乘级的搜索空间降低到了指数级（2^n），是解决中小规模TSP实例的有效精确算法。

## 3-SAT问题：19.3：随机局部搜索算法

上一节我们利用动态规划改进了TSP的精确算法。本节中，我们转向可满足性问题（SAT），并展示一个简单的随机算法如何显著超越暴力搜索。

我们考虑3-SAT问题：给定n个布尔变量和m个子句，每个子句是三个文字的析取（逻辑或）。问是否存在一个真值赋值，使得所有子句被满足。暴力搜索需要尝试所有2^n种赋值。

我们将介绍一个由Schöning在1999年提出的随机算法。该算法非常简单，但通过巧妙的概率分析，可以证明其运行时间远优于2^n。

**算法描述**：

1.  重复以下过程T次（T稍后确定）：
    *   a. **随机初始化**：随机生成一个真值赋值A（每个变量独立地以1/2概率设为True或False）。
    *   b. **局部搜索**：重复以下步骤3n次：
        i.   如果当前赋值A满足所有子句，则**成功返回**。
        ii.  否则，任意选择一个未被满足的子句C。
        iii. 在子句C的三个变量中，**均匀随机地**选择一个变量。
        iv.  翻转（取反）该变量的值，更新赋值A。

2.  如果T次尝试都未找到解，则**宣称该公式不可满足**。

**算法分析（思路）**：

我们只分析公式可满足的情况。设a*是某个固定的满足赋值。
*   **单步分析**：设当前赋值A与a*在d个变量上取值不同。考虑一个未被满足的子句C。因为a*满足C，而A不满足，所以C中至少有一个变量，A与a*的赋值不同。因此，当我们随机选择C中的一个变量翻转时，至少有 **1/3** 的概率选中这个变量，使得A向a*靠近一步（距离d减1）；至多有 **2/3** 的概率选中其他变量，使得A远离a*一步（距离d加1）。
*   **成功路径**：一种算法成功的可能方式是：从初始赋值A0开始，设其与a*的距离为k。在后续的3k步中，如果“靠近”的步数比“远离”的步数恰好多k步（例如，2k步靠近，k步远离），那么最终也能达到距离0（即找到a*或另一个解）。
*   **概率下界**：通过精细的概率计算（涉及二项式系数和斯特林公式近似），可以证明，每一次外层尝试（随机初始化+3n步搜索）成功找到一个解的概率P至少是 **Ω((3/4)^n / √n)**。
*   **设定尝试次数T**：为了使算法在可满足时失败的概率小于一个小的常数（如1/n），我们只需要设置 **T = O( (4/3)^n * √n * log n )**。

**结论**：
这个简单的随机算法，其期望运行时间约为 **O( (4/3)^n * poly(n) )**。虽然仍是指数时间，但底数4/3 ≈ 1.333远小于2，这意味着对于相同规模的n，它能处理的问题实例比暴力搜索大得多。这展示了随机性在解决NP难问题中的强大力量。

---

**本节课总结**：
在本节课中，我们一起学习了三种超越暴力搜索来求解NP难问题精确解的策略：
1.  **参数化算法与核化**（顶点覆盖）：针对解的大小K很小的情况，通过预处理将问题实例化简为仅与K相关的“核”，从而分离复杂度。
2.  **动态规划与状态压缩**（旅行商问题）：利用二进制位掩码表示状态集合，设计动态规划算法，将阶乘级复杂度降为指数级(2^n)。
3.  **随机局部搜索**（3-SAT）：通过简单的随机翻转操作，并结合概率分析，证明算法可以在远快于2^n的期望时间内找到解。

这些例子表明，即使对于NP难问题，精心设计的算法也能在可接受的时间内解决比暴力搜索大得多的实例，拓展了可计算问题的边界。
