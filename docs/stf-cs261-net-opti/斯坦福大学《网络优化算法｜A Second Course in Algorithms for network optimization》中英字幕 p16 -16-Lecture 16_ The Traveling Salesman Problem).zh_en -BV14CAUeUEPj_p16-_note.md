# 网络优化算法：16：旅行商问题 🧳

在本节课中，我们将要学习一个非常著名的问题——旅行商问题。我们将了解其定义、为什么它难以求解，并探讨在特定限制条件下（如满足三角不等式）的一些经典近似算法。

---

## 问题定义与挑战

旅行商问题通常简称为 TSP。其输入是一个**完全无向图**，每条边都有成本。目标是计算一个**最小成本的旅行商环路**。该环路是一个简单环，需要访问图中的每个顶点恰好一次。

例如，考虑一个图，其最优环路成本可能是 13。这个问题之所以著名，是因为它在过去约 70 年里，一直是推动人们研究如何处理 NP 完全问题的主要挑战性问题。

**核心概念：**
- **输入**：完全图 `G=(V, E)`，成本函数 `c: E -> R+`
- **输出**：一个访问所有顶点恰好一次的环路 `H`，最小化 `sum_{e in H} c(e)`

然而，TSP 非常难以求解，甚至难以近似。假设 P ≠ NP，那么对于任何近似比 α，都不存在多项式时间的 α-近似算法。这包括 2-近似、10-近似、log n 近似，甚至 2^n 近似。这是一个根本性的障碍。

为了证明其难度，我们可以从**哈密顿环路问题**进行归约。给定一个无向图，判断是否存在一个访问每个顶点恰好一次的简单环是 NP 完全的。我们可以构造一个 TSP 实例：原图中存在的边成本设为 1，不存在的边成本设为一个很大的值 M（例如 α * n）。如果原图有哈密顿环路，则最优 TSP 环路成本为 n；如果没有，则任何 TSP 环路都必须至少使用一条成本为 M 的边，因此成本至少为 n-1+M。这样，一个 α-近似算法就能用来判断原图是否有哈密顿环路，从而证明了 TSP 近似的难度。

---

## 度量旅行商问题 📏

鉴于一般 TSP 的难度，我们转向研究一个具有实际意义且更易处理的特殊案例：**度量 TSP**。在这个问题中，我们要求边成本满足**三角不等式**。这意味着对于任意三个顶点 i, j, k，有 `c(i, j) ≤ c(i, k) + c(k, j)`。直观上，直达路径的成本不会高于经过中间点的路径成本。

三角不等式的实用性在于它允许我们进行“捷径”操作：如果我们有一条路径，可以用一条直达边替代它，新成本不会更高。这将在算法设计中起到关键作用。

需要指出的是，度量 TSP 本身仍然是 NP 难的。我们可以修改上述归约：将不存在的边成本设为 2，存在的边成本设为 1。这样，边成本只包含 1 和 2，自然满足三角不等式。此时，最优 TSP 环路成本为 n（如果存在哈密顿环路）或至少为 n+1（如果不存在）。因此，判断最优解是 n 还是 n+1 就能解决 NP 完全的哈密顿环路问题。

## 下界与最小生成树启发式算法 🌳

为了设计近似算法，我们需要找到最优解成本的下界。对于 TSP，一个简单而有效的下界是**最小生成树的成本**。

**引理 1**：对于任何 TSP 实例（即使不满足三角不等式），最优环路 `OPT` 的成本至少等于图的最小生成树 `MST` 的成本。
**证明**：考虑最优 TSP 环路。如果移除其中任意一条边，我们就得到一条访问所有顶点的路径，这本质上是一个生成树（实际上是一条路径）。因此，`OPT` 的成本至少等于这个生成树的成本。而最小生成树是所有生成树中成本最小的，所以 `MST` 的成本 ≤ `OPT` 的成本。

这个下界引出了我们的第一个算法：**最小生成树启发式算法**。

**算法步骤：**
1.  计算图 G 的最小生成树 `T`。
2.  将 `T` 中的所有边复制一份，构造一个新图 `H`。这样 `H` 中每个顶点的度数都是偶数，因此 `H` 是一个欧拉图。
3.  在 `H` 中找到一个欧拉环路 `C`。
4.  对欧拉环路 `C` 进行“捷径”处理：按 `C` 中首次访问的顺序列出所有顶点，跳过重复访问的顶点，直接从前一个顶点走到下一个首次出现的新顶点。根据三角不等式，这种捷径操作不会增加总成本。最终得到一个访问每个顶点恰好一次的 TSP 环路。

**分析**：
-   设算法输出的环路成本为 `ALG`。
-   根据三角不等式，`ALG` ≤ `C` 的成本。
-   欧拉环路 `C` 的成本等于 `H` 中所有边的成本之和，即 `2 * cost(MST)`。
-   根据引理 1，`cost(MST) ≤ OPT`。
-   因此，`ALG ≤ 2 * cost(MST) ≤ 2 * OPT`。

这证明了最小生成树启发式算法是度量 TSP 的一个 **2-近似算法**。该分析是紧的，存在实例使得算法解的成本恰好达到 `2 * OPT`。

## 克里斯托菲德斯算法 🏆

上一节我们介绍了一个简单的 2-近似算法。本节中，我们来看看如何通过一个巧妙的改进，将近似比提升到 3/2。这就是著名的**克里斯托菲德斯算法**。

改进的核心思想在于：为了将生成树变为欧拉图，我们不一定需要将所有边加倍。我们只需要修正那些度数为奇数的顶点。而修正这些顶点的一种廉价方式是添加一个**最小成本完美匹配**。

首先，我们需要第二个下界。

**引理 2**：设 `S` 是顶点集的一个偶数大小子集，`M` 是 `S` 上（在原图边成本下）的最小成本完美匹配。那么，`cost(M) ≤ OPT / 2`。
**证明**：考虑最优 TSP 环路 `OPT`。将其限制在顶点集 `S` 上，通过捷径操作可以得到一个访问 `S` 中所有顶点的环路 `C‘`，且 `cost(C’) ≤ OPT`。因为 `S` 是偶数大小的，`C‘` 是一个偶环。我们可以将其边交替着上红色和蓝色，从而得到两个 `S` 上的完美匹配。这两个匹配的成本之和等于 `cost(C’)`。因此，其中成本较小的那个匹配的成本至多为 `cost(C‘) / 2 ≤ OPT / 2`。最小成本完美匹配 `M` 的成本不会大于这个值，故得证。

**克里斯托菲德斯算法步骤：**
1.  计算图 G 的最小生成树 `T`。
2.  令 `W` 为 `T` 中所有度数为奇数的顶点集合。图论中的握手引理保证了 `W` 的顶点数是偶数。
3.  在由 `W` 导出的子图（完全图）上，计算一个最小成本完美匹配 `M`。这是一个非二分图上的最小权匹配问题，可以在多项式时间内求解（例如使用带花树算法的扩展）。
4.  将树 `T` 和匹配 `M` 的边合并，得到新图 `H = T ∪ M`。
5.  由于我们为每个奇度顶点添加了一条匹配边，`H` 中所有顶点的度数都变为偶数，因此 `H` 是欧拉图。
6.  在 `H` 中找到一个欧拉环路 `C`。
7.  对 `C` 进行捷径处理，得到最终的 TSP 环路。

**分析**：
-   算法输出环路成本 `ALG ≤ cost(C)`。
-   `cost(C) = cost(H) = cost(T) + cost(M)`。
-   根据引理 1，`cost(T) ≤ OPT`。
-   根据引理 2，`cost(M) ≤ OPT / 2`。
-   因此，`ALG ≤ OPT + OPT/2 = (3/2) * OPT`。

这证明了克里斯托菲德斯算法是度量 TSP 的一个 **3/2-近似算法**。这也是目前已知最好的近似比，该记录自 1976 年保持至今，能否突破 3/2 是近似算法领域最重要的开放问题之一。

## 非对称旅行商问题 🔄

前面我们讨论了无向图上的度量 TSP。本节中，我们来看看有向图上的版本，即**非对称旅行商问题**。在 ATSP 中，输入是一个完全有向图，每条弧都有成本，且从 i 到 j 的成本可能与从 j 到 i 的成本不同。我们同样假设成本满足有向图上的三角不等式。

ATSP 包含了度量 TSP 作为特例，因此至少同样困难。我们的算法策略依然是：构造一个**有向欧拉图**（强连通且每个顶点的入度等于出度），然后通过捷径得到哈密顿环路。

我们需要一个类似于最小生成树的下界。在这里，我们使用**最小成本环覆盖**。

**定义**：一个有向图的环覆盖是一组顶点不相交的有向环的集合，这些环覆盖了图中的所有顶点。每个环的长度至少为 2。
**性质**：一个访问所有顶点的 TSP 环路本身就是一个环覆盖（只包含一个环）。因此，最小成本环覆盖的成本 `CC` 是 `OPT` 的一个下界：`CC ≤ OPT`。并且，最小成本环覆盖可以在多项式时间内求解（例如通过归约到二分图最小权完美匹配）。

基于此，我们介绍一个 ATSP 的近似算法（Frieze, Galbiati, Maffioli, 1982）。

**算法步骤：**
1.  初始化边集 `F` 为空。
2.  当当前图 `G‘` 的顶点数大于 1 时，循环：
    a. 计算 `G‘` 的最小成本环覆盖。
    b. 将该环覆盖中的所有弧加入 `F`。
    c. 从 `G‘` 中删除每个环中除一个“代表”顶点外的所有其他顶点。
3.  循环结束后，令最终图 `H` 的顶点为原图所有顶点，边集为 `F`。
4.  可以证明 `H` 是一个有向欧拉图。在 `H` 中找到一个有向欧拉环路 `C`。
5.  对 `C` 进行捷径处理，得到最终的 TSP 环路。

**算法理解与正确性**：
-   **入度等于出度**：在每次迭代中，环覆盖为每个顶点贡献 1 的入度和 1 的出度。因此，最终 `H` 中每个顶点的入度等于其出度。
-   **强连通性**：算法维持一个不变性：任何已被删除的顶点 `v`，都存在一个当前仍未删除的顶点 `w`，使得在 `H` 中存在从 `v` 到 `w` 和从 `w` 到 `v` 的路径。在最后一次迭代中，环覆盖只包含一个环（否则循环不会结束），这个环上的剩余顶点彼此强连通。结合不变性，所有顶点都强连通。因此 `H` 是有向欧拉图。

**近似比分析**：
-   在每次循环迭代中，设当前子图上的最优 TSP 环路成本为 `OPT_i`。根据三角不等式和捷径操作，有 `OPT_i ≤ OPT`（原问题最优解）。
-   我们计算的环覆盖成本 `CC_i ≤ OPT_i ≤ OPT`。
-   每次迭代，我们至少删除每个环中一半的顶点（因为环长≥2，只保留一个）。因此，循环最多执行 `log₂ n` 次。
-   最终欧拉图 `H` 的成本 `cost(H) = sum(CC_i) ≤ (log₂ n) * OPT`。
-   算法输出成本 `ALG ≤ cost(H) ≤ O(log n) * OPT`。

因此，该算法是 ATSP 的一个 **O(log n)-近似算法**。目前最好的多项式时间算法可以达到 `O(log n / log log n)` 的近似比，但寻找常数近似比的算法仍然是该领域另一个重大的开放问题。

## 总结

本节课中我们一起学习了旅行商问题及其变体。
1.  我们首先了解了经典的旅行商问题及其 NP 难近似性。
2.  接着，我们转向了更实际的度量 TSP（满足三角不等式），并学习了两个经典近似算法：
    -   **最小生成树启发式算法**，这是一个简单的 2-近似算法。
    -   **克里斯托菲德斯算法**，通过结合最小生成树和奇点最小权匹配，将近似比提升到了 3/2，这也是当前最佳结果。
3.  最后，我们探讨了非对称旅行商问题，介绍了一个基于最小环覆盖迭代合并的 O(log n)-近似算法，并了解了该问题的最新进展和开放挑战。

这些算法展示了对于难解的组合优化问题，通过利用问题本身的结构（如三角不等式）和寻找巧妙的下界，我们仍然可以在多项式时间内找到高质量的近似解。
