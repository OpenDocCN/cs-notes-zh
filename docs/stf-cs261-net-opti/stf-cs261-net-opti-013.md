# 013：在线调度与在线斯坦纳树

在本节课中，我们将学习两个经典的在线算法问题：最小化完工时间调度和在线斯坦纳树问题。我们将探讨针对这些问题的简单贪心算法，并分析它们在最坏情况下的性能保证。

---

## 调度问题与贪心算法

上一节我们介绍了在线问题的基本概念。本节中，我们来看看一个具体的在线调度问题。

我们面对的场景是：有 `m` 台完全相同的机器。作业 `j` 一个接一个地在线到达，每个作业 `j` 有一个处理时间 `p_j`。每当一个新作业到达时，算法必须立即且不可撤销地将其分配到一台机器上。

我们的目标是**最小化完工时间**。一台机器 `i` 的负载 `L_i` 是分配给它的所有作业的处理时间之和。完工时间 `makespan` 则是所有机器负载中的最大值，即 `max_i L_i`。我们希望这个最大值尽可能小。

一个自然的贪心策略是：当新作业 `j` 到达时，将其分配给**当前负载最小**的机器。这个算法由 Ron Graham 在 50 年前提出。

为了衡量这个在线算法的性能，我们将其与一个强大的基准进行比较：**事后最优解** `OPT`，即已知所有作业后能计算出的最小完工时间。

以下是分析算法性能的两个简单下界：
1.  **下界 1**：`OPT` 至少等于最大作业的处理时间，即 `OPT >= max_j p_j`。
2.  **下界 2**：`OPT` 至少等于平均负载，即 `OPT >= (1/m) * sum_j p_j`。

**定理**：上述贪心调度算法是 **2-竞争** 的。也就是说，对于任何作业序列，算法产生的完工时间最多是 `OPT` 的两倍。

**证明思路**：
考虑算法最终负载最大的机器 `i`。设 `j` 是最后分配给机器 `i` 的作业。在分配 `j` 之前，机器 `i` 的负载是所有机器中最小的。因此，在分配 `j` 之前的时刻，机器 `i` 的负载不超过所有机器的平均负载。分配 `j` 之后，机器 `i` 的最终负载满足：
`L_i <= (分配j前的总负载/m) + p_j <= OPT + OPT = 2 * OPT`
其中，第一项利用了平均负载下界（下界2），第二项利用了最大作业下界（下界1）。

## 在线斯坦纳树问题

在了解了在线调度之后，我们转向另一个图论问题：在线斯坦纳树。

在这个问题中，我们预先知道一个无向图 `G=(V, E)` 以及每条边 `e` 的非负代价 `c(e)`。终端顶点 `t_1, t_2, ..., t_k` 在线依次到达。每当一个新终端到达时，算法必须扩展当前已构建的子图，以确保所有已到达的终端相互连通，并且希望总代价尽可能小。

为了使分析可行，我们考虑一个特例：图是完全图，且边代价满足**三角不等式**，即对于任意三点 `u, v, w`，有 `c(u, w) <= c(u, v) + c(v, w)`。这实际上不失一般性，因为一般情况可以规约到此特例。

一个简单的贪心算法如下：
1.  初始化树 `T` 为空。
2.  当第一个终端 `t_1` 到达时，无事可做。
3.  当第 `i` 个终端 `t_i` (`i > 1`) 到达时，查看所有先前到达的终端 `t_j` (`j < i`)，将 `t_i` 通过代价最小的边 `(t_i, t_j)` 连接到树 `T` 上。

然而，这个简单贪心算法在最坏情况下的竞争比可能高达 `Ω(log k)`，其中 `k` 是终端总数。有趣的是，这个上界是紧的：**任何**在线算法（包括随机算法）在最坏情况下的竞争比都至少是 `Ω(log k)`。

**定理**：上述贪心算法是 `O(log k)`-竞争的。

证明依赖于一个关键引理。设算法最终构建的树为 `T`，其边按代价从高到低排序。

**关键引理**：对于任意 `i`，`T` 中第 `i` 贵的边的代价至多是 `2 * OPT / i`，其中 `OPT` 是事后最优斯坦纳树的代价。

**定理证明（基于关键引理）**：
贪心算法的总代价是 `T` 中所有边的代价之和。根据引理，最贵的边代价不超过 `2*OPT`，第二贵的边代价不超过 `2*OPT/2`，依此类推。因此总代价不超过：
`2 * OPT * (1 + 1/2 + 1/3 + ... + 1/(k-1)) ≈ 2 * OPT * ln k`

**关键引理证明思路**：
1.  考虑事后最优斯坦纳树 `T*`，其代价为 `OPT`。
2.  **边加倍技巧**：将 `T*` 中的每条边复制一份，得到一个多重图 `H`。`H` 中所有顶点度数为偶，因此存在欧拉回路 `C`，且 `C` 的总代价为 `2 * OPT`。
3.  固定 `i`，设 `S` 是贪心算法中连接代价最高的 `i` 个终端集合。
4.  利用三角不等式，将欧拉回路 `C` “短路”，得到一个只经过 `S` 中 `i` 个终端的简单圈 `C‘`，其总代价不超过 `2 * OPT`。
5.  圈 `C‘` 有 `i` 条边，因此其中至少有一条边 `(s_h, s_j)` 的代价不超过 `2 * OPT / i`。
6.  考虑 `s_h` 和 `s_j` 中较晚到达的那个终端（比如 `s_j`）。当它到达时，连接它到较早到达的 `s_h` 的边是一个可选连接，其代价不超过 `2 * OPT / i`。贪心算法会选择代价最小（可能更小）的连接方式，因此连接 `s_j` 的实际代价也至多是 `2 * OPT / i`。由于 `s_j` 属于代价最高的 `i` 个终端之一，这就证明了引理。

## 在线二分图匹配问题预告

最后，我们简要介绍将在下一讲深入讨论的在线二分图匹配问题，这是在线广告投放等应用的核心模型。

问题设定如下：二分图的左部顶点集合 `L` 是预先已知的。右部顶点 `r_1, r_2, ...` 在线到达。当一个右部顶点 `r_i` 到达时，会同时揭示它连接到 `L` 中哪些顶点的边。算法必须立即决定是否将 `r_i` 与一个当前尚未匹配的左部邻居匹配，且匹配决定不可撤销。目标是最大化最终匹配的基数。

一个简单的贪心算法是：每当一个右部顶点到达时，如果它有未匹配的邻居，就将其与任意一个这样的邻居匹配。这个算法是 `1/2`-竞争的（即匹配数至少是最优解的一半）。然而，存在一个更巧妙的**随机化贪心算法**，其竞争比可以达到 `1 - 1/e ≈ 0.632`，并且这个比值是最优的。

在下一讲中，我们将看到如何利用线性规划对偶理论来分析并证明这个算法的性能保证。

本节课中我们一起学习了两个经典的在线算法案例。对于最小化完工时间调度，一个简单的贪心策略就能获得 2 倍的竞争比。对于在线斯坦纳树问题，尽管简单贪心算法不是常数竞争的，但其 `O(log k)` 的竞争比已经是最优的了。我们还预览了在线二分图匹配问题，它将在下一讲中展示对偶理论在在线算法分析中的强大应用。
