# 017：线性规划与近似算法

在本节课中，我们将学习如何利用线性规划这一强大的工具来设计和分析近似算法。我们将看到，线性规划不仅能在算法分析中提供理论下界，有时甚至可以直接作为算法的一部分。我们将通过两个经典问题——集合覆盖和顶点覆盖——来具体说明这些思想。

## 集合覆盖问题：重温与扩展

上一节我们讨论了集合覆盖问题的一个变体（Set Coverage），本节我们来看看它的另一个经典版本——集合覆盖问题。

**集合覆盖问题**的输入与Set Coverage相同：一个全集 `U` 和一系列子集 `S1, S2, ..., Sm`。不同之处在于，这里没有预算 `K` 的限制。相反，**硬性约束**是必须覆盖全集 `U` 中的所有元素，而**目标**是使用尽可能少的子集来完成覆盖。

### 贪心算法及其分析

一个自然的想法是沿用Set Coverage的贪心策略：在每一步，选择能覆盖**最多新元素**的子集，直到所有元素都被覆盖。

以下是该贪心算法的伪代码：
```
初始化已选集合 C = 空集
当 C 的并集未覆盖所有元素时：
    选择子集 Si，使其能覆盖最多尚未被 C 覆盖的新元素
    将 Si 加入 C
返回 C
```

对于所有子集成本相同（例如均为1）的情况，我们可以利用之前对Set Coverage的分析，快速证明该算法是一个 `O(log n)` 的近似算法，其中 `n = |U|`。

**证明思路**：
1.  假设最优解使用了 `OPT` 个子集。
2.  根据Set Coverage的分析，贪心算法在前 `OPT` 次迭代中，至少能覆盖 `(1 - 1/e)` 比例的元素。
3.  这意味着每进行 `OPT` 次迭代，未被覆盖的元素数量至少减少至原来的 `1/e`。
4.  从 `n` 个元素开始，经过 `OPT * ln(n)` 次迭代后，未被覆盖的元素数将小于1，即所有元素已被覆盖。
5.  因此，贪心算法使用的子集数最多为 `OPT * ln(n)`。

### 带权重的集合覆盖

在实际应用中，每个子集 `Si` 可能具有不同的成本 `ci`。此时，目标变为寻找总成本最小的集合覆盖。

我们可以自然地扩展贪心算法：在每一步，不再选择覆盖新元素最多的子集，而是选择**性价比最高**的子集，即**单位新元素成本最低**的子集。

扩展的贪心算法伪代码如下：
```
初始化已选集合 C = 空集
当 C 的并集未覆盖所有元素时：
    对于每个子集 Si，计算其性价比：成本 ci / (Si 中尚未被 C 覆盖的元素数量)
    选择性价比最高（该比值最小）的子集 Si
    将 Si 加入 C
返回 C
```

令人欣慰的是，即使存在不同成本，该算法仍然是一个 `O(log n)` 的近似算法。其证明比无权情况更复杂，但核心思想是定义每个元素 `e` 一个“价格” `q_e`，即首次覆盖 `e` 的那个子集的性价比。通过分析这些 `q_e` 值的性质，可以证明算法总成本与最优成本之间满足对数近似比。

## 线性规划视角下的分析

上述对带权贪心算法的分析虽然有效，但显得有些特殊。现在，让我们从线性规划的角度重新审视它，这将提供一个更系统化的理解框架。

### 集合覆盖的线性规划松弛

首先，我们为集合覆盖问题建立一个整数规划模型。对于每个子集 `Si`，引入一个0-1决策变量 `xi`：`xi = 1` 表示选择该子集，`xi = 0` 表示不选。目标是最小化总成本 `∑ ci * xi`。约束条件是每个元素 `e` 至少被一个选中的子集覆盖：`∑_{i: e ∈ Si} xi ≥ 1`。

如果我们放宽 `xi ∈ {0, 1}` 的限制，允许 `xi` 取 `[0, 1]` 之间的任意值，就得到了一个**线性规划松弛**。这个松弛问题的最优值 `OPT_f` 是原整数规划最优值 `OPT` 的一个下界，即 `OPT_f ≤ OPT`。

### 对偶规划与弱对偶性

根据线性规划理论，上述松弛问题（称为原问题）有一个对应的**对偶问题**。对偶问题为每个元素 `e` 引入一个变量 `p_e`（可以理解为覆盖元素 `e` 的“价格”），目标是最大化所有元素价格之和 `∑ p_e`。约束条件是，对于每个子集 `Si`，其包含的所有元素的价格之和不能超过该子集的成本：`∑_{e ∈ Si} p_e ≤ ci`。

**弱对偶性**指出，任何对偶可行解的目标函数值，都是原问题最优值（即 `OPT_f`）的一个下界，从而也是原整数问题最优值 `OPT` 的下界。

### 重新解释贪心算法的证明

现在，我们可以用对偶语言重新表述之前贪心算法的分析：
1.  算法运行时为每个元素 `e` 定义的 `q_e`，在除以 `ln n` 之后，恰好构成了对偶问题的一个可行解 `p_e = q_e / ln n`。
2.  贪心算法的总成本等于所有 `q_e` 之和，即 `ln n * (∑ p_e)`。
3.  由于 `p` 是对偶可行解，根据弱对偶性，`∑ p_e ≤ OPT_f ≤ OPT`。
4.  因此，贪心算法的成本 `≤ ln n * OPT`。

这个视角揭示了贪心算法成功的本质：它隐式地构造了一个对偶可行解，并通过弱对偶性保证了其近似比。

## 顶点覆盖问题：线性规划舍入算法

顶点覆盖是集合覆盖的一个特例，但因其特殊的结构，我们可以设计出更好的近似算法。给定一个无向图 `G=(V, E)` 和顶点成本 `c_v`，目标是选择一个顶点子集 `S`，使得每条边至少有一个端点在 `S` 中，并且最小化 `∑_{v∈S} c_v`。

### 基于线性规划舍入的2-近似算法

与集合覆盖类似，我们可以为顶点覆盖建立线性规划松弛。对于每个顶点 `v`，引入变量 `x_v ∈ [0, 1]`。约束条件为对每条边 `(u, v)`，有 `x_u + x_v ≥ 1`。目标是最小化 `∑ c_v * x_v`。

以下是一个简单的2-近似算法：
1.  **求解松弛**：求解上述线性规划，得到最优分数解 `x*`。
2.  **舍入**：令 `S = { v | x*_v ≥ 1/2 }`。即，将所有分数值不小于1/2的顶点加入覆盖集。

**算法分析**：
*   **可行性**：对于任意边 `(u, v)`，由于 `x*_u + x*_v ≥ 1`，则 `x*_u` 和 `x*_v` 中至少有一个 `≥ 1/2`，因此该边至少有一个端点被 `S` 包含。
*   **近似比**：算法成本 `∑_{v∈S} c_v ≤ ∑_{v∈V} c_v * (2 x*_v) = 2 * (∑ c_v x*_v) = 2 * OPT_f ≤ 2 * OPT`。

这个算法直接使用了线性规划作为子程序，并通过简单的阈值舍入获得了2倍近似解。

## 顶点覆盖：原始-对偶算法

虽然上述舍入算法是多项式时间的，但求解线性规划仍然不够快。我们可以利用原始-对偶的思想，设计一个**线性时间**的2-近似算法，而无需显式求解线性规划。

顶点覆盖的对偶问题是一个**最大边权包装问题**：为每条边 `e` 分配一个权重 `p_e`，最大化 `∑ p_e`，并满足对于每个顶点 `v`，其关联边的权重之和不超过 `c_v`：`∑_{e incident to v} p_e ≤ c_v`。

以下是原始-对偶算法：
```
初始化：顶点覆盖 S = 空集，对偶变量 p_e = 0 对所有边 e。
当 S 不是一个顶点覆盖时：
    选择一条未被 S 覆盖的边 (u, v)（即 u, v ∉ S）。
    同时增加 p_{(u,v)} 的值，直到顶点 u 或 v 的对偶约束变紧（即关联边权重和等于其成本 c_u 或 c_v）。
    将变紧的那个顶点加入 S。
返回 S。
```

**算法分析**：
该算法始终保持对偶解 `p` 的可行性。它满足以下性质：
1.  **对偶可行性**：`p` 始终是对偶问题的可行解。
2.  **松弛的互补松弛条件**：若顶点 `v ∈ S`，则其对应的对偶约束是紧的（`∑_{e incident to v} p_e = c_v`）。若边 `e` 的权重 `p_e > 0`，则其至少有一个端点在 `S` 中（但可能两个都在，这是与精确互补松弛的差距）。

基于这些性质，可以证明算法是2-近似的：
```
算法成本 = ∑_{v∈S} c_v
          = ∑_{v∈S} (∑_{e incident to v} p_e)   // 因为对 v∈S，其对偶约束是紧的
          = ∑_{e} p_e * (e 的端点中属于 S 的个数)
          ≤ 2 * ∑_{e} p_e                       // 每条边最多被它的两个端点计入两次
          ≤ 2 * OPT_f                           // ∑ p_e 是对偶可行解的目标值，是 OPT_f 的下界
          ≤ 2 * OPT
```
这个算法巧妙地在维持对偶可行性的同时，逐步构建原始可行解（顶点覆盖），并通过近似满足互补松弛条件来保证近似比。

## 总结

本节课我们一起学习了线性规划在近似算法设计中的核心作用：
1.  **提供下界**：通过线性规划松弛及其对偶，我们可以为NP难问题的最优值获得可计算的下界，这是分析近似算法性能的基准。
2.  **指导算法设计**：对集合覆盖贪心算法的对偶分析，揭示了其成功背后的原理。
3.  **作为算法组件**：在顶点覆盖问题中，线性规划松弛的解可以直接通过舍入转化为近似解。
4.  **启发高效算法**：原始-对偶方法利用对偶问题的结构，引导我们设计出了无需求解线性规划、运行速度极快（线性时间）且性能优良（2-近似）的算法。

这些技巧是设计近似算法的强大工具箱，它们表明，经典算法设计范式与线性规划理论结合，能产生解决实际计算难题的有效方案。
