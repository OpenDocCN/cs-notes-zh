# 斯坦福大学《网络优化算法》课程笔记：第2讲：最大流的增广路径算法

在本节课中，我们将要学习最大流问题的核心概念、证明其最优性的关键定理，并介绍基于增广路径的经典算法。

---

## 回顾：流与残差图

上一节我们介绍了流网络的基本概念。本节中，我们来看看如何判断一个流是否是最优的。

一个流网络包含一个有向图、一个源点 `s` 和一个汇点 `t`。每条边 `e` 有一个容量 `u(e)`。流 `f` 是一个从边到实数的映射，它满足两个约束：
*   **容量约束**：对于每条边 `e`，有 `0 ≤ f(e) ≤ u(e)`。
*   **守恒约束**：对于除 `s` 和 `t` 外的每个顶点 `v`，流入 `v` 的流量等于流出 `v` 的流量。

流 `f` 的值 `|f|` 是从源点 `s` 流出的净流量。

一个核心概念是**残差图** `G_f`。给定图 `G` 和流 `f`，残差图 `G_f` 的构建方式如下：
*   对于 `G` 中的每条边 `(u, v)`，如果 `f(e) < u(e)`，则在 `G_f` 中添加一条正向边 `(u, v)`，其剩余容量为 `r(u, v) = u(e) - f(e)`。
*   如果 `f(e) > 0`，则在 `G_f` 中添加一条反向边 `(v, u)`，其剩余容量为 `r(v, u) = f(e)`。

Ford-Fulkerson 算法非常简单：只要在残差图 `G_f` 中存在一条从 `s` 到 `t` 的路径（称为**增广路径**），就沿着该路径尽可能多地推送流量（受限于路径上的最小剩余容量），然后更新残差图。当残差图中不存在 `s-t` 路径时，算法终止。

我们上次证明了该算法会终止，但尚未证明它终止时得到的是最大流。这就引出了一个更广泛的问题：我们如何知道算法没有过早停止？如何验证一个给定的流就是最大流？

---

## 最大流最小割定理

为了回答上述问题，我们需要引入**割**的概念。

一个 `s-t` 割是将顶点集 `V` 划分为两个集合 `A` 和 `B`，使得源点 `s ∈ A`，汇点 `t ∈ B`。

割 `(A, B)` 的**容量**定义为从 `A` 指向 `B` 的所有边的容量之和，记作：
`cap(A, B) = Σ_{e ∈ δ⁺(A)} u(e)`
其中 `δ⁺(A)` 表示尾在 `A` 中、头在 `B` 中的边集。

我们关心的是具有最小容量的 `s-t` 割，即**最小割**。

现在，我们可以陈述并证明一个关键定理，它将流的“最优性”、“值等于某割容量”以及“残差图中无 `s-t` 路径”这三个条件等价起来。

**定理**：对于流网络 `G` 中的一个流 `f`，以下三个陈述等价（TFAE）：
1.  `f` 是一个最大流。
2.  存在一个 `s-t` 割 `(A, B)`，使得 `|f| = cap(A, B)`。
3.  在残差图 `G_f` 中，不存在从 `s` 到 `t` 的路径。

**证明**：我们将通过证明一个循环蕴含关系来完成证明：`(2) ⇒ (1)`, `(1) ⇒ (3)`, `(3) ⇒ (2)`。

**步骤 1: (2) ⇒ (1)**
首先，我们证明一个更一般的不等式：对于**任意**流 `f` 和**任意** `s-t` 割 `(A, B)`，都有：
`|f| ≤ cap(A, B)`
这个不等式意味着，任何割的容量都是任何流值的上界。

**证明**：
`|f| = Σ_{e ∈ δ⁺(s)} f(e) - Σ_{e ∈ δ⁻(s)} f(e)` （流值的定义，假设没有边进入 `s`，第二项为0）
`= Σ_{v ∈ A} ( Σ_{e ∈ δ⁺(v)} f(e) - Σ_{e ∈ δ⁻(v)} f(e) )` （对 `A` 中所有顶点求和，利用守恒约束，除 `s` 外其他顶点此项为0）
`= Σ_{e ∈ δ⁺(A)} f(e) - Σ_{e ∈ δ⁻(A)} f(e)` （从顶点视角转换为边视角）
`≤ Σ_{e ∈ δ⁺(A)} f(e)` （减去一个非负数）
`≤ Σ_{e ∈ δ⁺(A)} u(e) = cap(A, B)` （应用容量约束）

现在，如果存在一个割 `(A, B)` 和一个流 `f` 使得 `|f| = cap(A, B)`（即条件2），那么根据上述不等式，不可能存在值更大的流，因此 `f` 必然是最大流。同时，该割 `(A, B)` 也必然是最小割。

**步骤 2: (1) ⇒ (3)**
我们证明其逆否命题：如果残差图 `G_f` 中存在一条 `s-t` 路径（非条件3），那么 `f` 不是最大流（非条件1）。
这很直接：如果在 `G_f` 中存在 `s-t` 路径，那么 Ford-Fulkerson 算法就可以沿着这条路径进行一轮增广，从而得到一个值严格更大的流。因此，`f` 不可能是最大的。

**步骤 3: (3) ⇒ (2)**
假设在残差图 `G_f` 中不存在 `s-t` 路径。我们需要构造一个割 `(A, B)`，使得 `|f| = cap(A, B)`。

构造方法如下：在 `G_f` 中从源点 `s` 开始进行广度优先搜索（BFS）或深度优先搜索（DFS）。令 `A` 为所有能从 `s` 到达的顶点集合。由于假设没有 `s-t` 路径，所以 `t ∉ A`。显然 `s ∈ A`。那么 `(A, V \ A)` 就构成了一个 `s-t` 割。

这个割具有关键性质：
*   对于原图 `G` 中任何从 `A` 指向 `V \ A` 的边 `e`，必有 `f(e) = u(e)`（饱和）。因为如果 `f(e) < u(e)`，那么在 `G_f` 中就会存在一条从 `A` 指向 `V \ A` 的正向边，`BFS` 就会继续探索，`A` 会更大，矛盾。
*   对于原图 `G` 中任何从 `V \ A` 指向 `A` 的边 `e`，必有 `f(e) = 0`。因为如果 `f(e) > 0`，那么在 `G_f` 中就会存在一条从 `A` 指向 `V \ A` 的反向边（用于撤销流量），同样会导致 `A` 更大，矛盾。

现在回顾我们在证明 `(2) ⇒ (1)` 时推导的不等式：
`|f| = Σ_{e ∈ δ⁺(A)} f(e) - Σ_{e ∈ δ⁻(A)} f(e)`
根据我们刚刚得出的性质，对于这个特定的流 `f` 和割 `(A, V\A)`，有：
`Σ_{e ∈ δ⁺(A)} f(e) = Σ_{e ∈ δ⁺(A)} u(e) = cap(A, V\A)`
`Σ_{e ∈ δ⁻(A)} f(e) = 0`
因此，`|f| = cap(A, V\A)`。条件2得证。

至此，定理证明完毕。

---

### 推论与应用

从该定理可以直接得出著名的**最大流最小割定理**：
`max_{f 是流} |f| = min_{(A,B) 是 s-t 割} cap(A, B)`
即，最大流的值等于最小割的容量。

该定理还有一个重要的算法意义：**给定一个最大流，我们可以在线性时间内找到一个最小割**。
方法正是证明中 `(3) ⇒ (2)` 的构造过程：在得到最大流 `f` 后，在残差图 `G_f` 中从 `s` 开始做 BFS，所有能到达的顶点构成集合 `A`，则 `(A, V\A)` 就是一个最小割。在实践中，这通常是求解最小割问题最快的方法之一。

## Edmonds-Karp 算法

Ford-Fulkerson 算法在边容量为整数时保证终止，但如果容量很大，它可能需要进行非常多次增广（伪多项式时间）。问题在于它可能反复选择低效的增广路径。

Edmonds-Karp 算法是 Ford-Fulkerson 算法的一个特例，它通过一个简单的策略来保证多项式时间复杂度：**在残差图中，总是选择边数最少的（即最短的）`s-t` 增广路径**。

以下是该算法的关键引理，它刻画了算法的进展：

**关键引理**：设 `d_f` 为残差图 `G_f` 中从 `s` 到 `t` 的最短路径距离（以边数计）。
1.  在 Edmonds-Karp 算法的执行过程中，`d_f` 永远不会减少。
2.  `d_f` 至少每进行 `m` 次增广就会严格增加 1，其中 `m` 是图的边数。

**直观理解与证明思路**：
*   **部分 1**：算法沿着一条最短路径增广。新增的反向边都是从路径上较晚的顶点指向较早的顶点，这不会创建出比当前最短路径更短的 `s-t` 路径。
*   **部分 2**：考虑 `d_f` 保持不变的一段时间。我们定义**分层图** `L_f`，它是 `G_f` 的一个子图，只包含在 BFS 树中从 `s` 出发、一层层向前推进的边。`L_f` 中的 `s-t` 路径恰好就是 `G_f` 中的最短 `s-t` 路径。
    *   每次增广都会饱和（或清零）`L_f` 中的至少一条边（瓶颈边）。
    *   只要 `d_f` 不变，被删除的边（正向边饱和或反向边清零）就不会重新出现，因为算法只沿着正向路径增广，不会重新激活那些反向边。
    *   由于 `L_f` 最多有 `m` 条边，因此在最多 `m` 次增广后，`L_f` 中就不再存在 `s-t` 路径，这意味着在原始的 `G_f` 中，`s` 到 `t` 的最短距离 `d_f` 必须增加。

**时间复杂度**：由于 `d_f` 从至少 1 开始，最多增加到 `n-1`（之后 `s` 和 `t` 就不连通了），而每次增加前最多进行 `m` 次增广，所以总增广次数为 `O(mn)`。每次增广需要一次 BFS 来寻找最短路径，耗时 `O(m)`。因此，Edmonds-Karp 算法的总时间复杂度为 **`O(m²n)`**。

## 阻塞流与 Dinic 算法

Edmonds-Karp 算法每次只增广一条路径。我们可以做得更好：一次增广一批路径。这就是 Dinic 算法的思想。

Dinic 算法的高层结构如下：
1.  从零流开始。
2.  **While** 在残差图 `G_f` 中存在 `s-t` 路径 **Do**:
    a.  通过 BFS 构建分层图 `L_f`。
    b.  在分层图 `L_f` 中找到一个**阻塞流** `g`。
    c.  将当前流 `f` 增加 `g`（即沿阻塞流中的所有路径增广）。
3.  输出 `f`。

**阻塞流**的定义是：在图 `G`（这里指分层图 `L_f`）中，一个流 `g` 被称为阻塞流，如果 `G` 中不存在这样的 `s-t` 路径：该路径上的每条边都有正的剩余容量（即未被 `g` 完全占用）。简单说，就是贪心算法在当前分层图中无法再找到任何可增广路径时得到的流。

Dinic 算法的关键性质是：**每次增广一个阻塞流后，残差图中 `s` 到 `t` 的最短距离 `d_f` 一定会严格增加**。（其证明思路与 Edmonds-Karp 的关键引理相似，将在习题中探讨。）

这意味着 Dinic 算法最多只需要进行 `n-1` 次阻塞流计算。

**时间复杂度**：如果有一个子程序能在 `T(m, n)` 时间内计算出一个阻塞流，那么 Dinic 算法的总时间就是 `O(n * T(m, n))`。
*   一个简单的基于 DFS 的阻塞流算法可以在 `O(mn)` 时间内完成。将其代入，得到 Dinic 算法的一个实现时间复杂度为 **`O(mn²)`**，这比 Edmonds-Karp 的 `O(m²n)` 更好。
*   使用更高级的数据结构，可以更快地计算阻塞流，从而得到接近 `O(mn)` 的 max-flow 算法。

## 总结

本节课中我们一起学习了：
1.  **最大流最小割定理**及其证明，该定理给出了判断流最优性的充要条件，并揭示了最大流与最小割之间的对偶关系。
2.  **Edmonds-Karp 算法**，通过总是增广最短路径，将 Ford-Fulkerson 算法改进为多项式时间算法 `O(m²n)`。
3.  **Dinic 算法**的基本思想，它通过批量处理（计算阻塞流）来进一步减少迭代轮数，理论复杂度可达 `O(mn²)`，并为更快的算法奠定了基础。

这些基于增广路径的算法是理解最大流问题的基础。下一讲我们将介绍另一种截然不同但实践中非常高效的范式：推送-重贴标签算法。
