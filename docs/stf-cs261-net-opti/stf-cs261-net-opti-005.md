# 005：最小成本二分图匹配

在本节课中，我们将要学习**最小成本完美二分图匹配**问题。我们将从回顾最大流应用中的二分图匹配开始，然后引入边成本的概念，探讨如何找到总成本最小的完美匹配。课程的核心是理解最优性条件，并介绍著名的**匈牙利算法**，该算法通过维护节点价格和匹配，逐步构建最优匹配。

---

## 第5讲：最小成本二分图匹配

### 问题定义与假设

上一节我们介绍了无成本的二分图最大基数匹配问题，本节中我们来看看当每条边都附有成本时，如何找到成本最小的完美匹配。

**输入**：
*   一个二分图 `G = (V, W, E)`，其中 `V` 和 `W` 是左右两个顶点集合。
*   每条边 `e ∈ E` 都有一个成本 `c(e)`。

**目标**：
*   在所有**完美匹配**（即匹配了所有顶点的匹配）中，找到总成本最小的一个。即，最小化 `Σ_{e ∈ M} c(e)`。

为了简化讨论，我们做出以下**不失一般性**的假设：
1.  **两侧顶点数相等**：`|V| = |W| = n`。如果不等，可以向较小的一侧添加虚拟顶点。
2.  **图存在完美匹配**。如果原图不存在，可以添加成本极高的虚拟边来保证存在性，而不影响最优解的成本排序。
3.  **所有边成本非负**。如果存在负成本，可以对所有边成本加上一个足够大的常数，这会使所有完美匹配的总成本增加相同的量，从而不改变它们之间的相对优劣顺序。

---

### 最优性条件：负环定理

与最大流问题类似，我们需要一个判断给定完美匹配是否最优的条件。这个条件将指导我们设计算法。

首先，我们定义两个关键概念。假设我们有一个匹配 `M`（不一定是完美的）。

1.  **M-交错环**：图中的一个环 `C`，如果其边在匹配 `M` 内和匹配 `M` 外交替出现，则称 `C` 为 `M-交错环`。
2.  **负环**：一个 `M-交错环` `C`，如果环中**属于匹配 `M` 的边**的成本之和，**大于**环中**不属于匹配 `M` 的边**的成本之和，则称 `C` 为一个**负环**。

**核心定理（最优性条件）**：
一个完美匹配 `M` 是**最小成本**的，**当且仅当**图中**不存在**关于 `M` 的负环。

**证明思路**：
*   **（必要性）简单方向**：如果存在一个负环 `C`，我们可以通过“翻转”环中的边（将匹配边移出匹配，非匹配边加入匹配）得到一个新的完美匹配 `M‘ = M ⊕ C`。计算可知，`M‘` 的总成本严格小于 `M` 的成本，因此 `M` 不可能是最优的。
*   **（充分性）困难方向**：假设 `M` 没有负环，但存在另一个更优的完美匹配 `M‘`。考虑两个匹配的对称差 `M ⊕ M‘`，它由若干个不相交的偶环组成（因为每个顶点在 `M` 和 `M‘` 中的度数均为1）。每个环都是 `M-交错环`。由于 `M` 没有负环，每个环都是“非负”的（匹配边成本 ≤ 非匹配边成本）。因此，通过逐个翻转这些环，可以从 `M` 变换到 `M‘`，且总成本不会减少，这与 `M‘` 成本更低的假设矛盾。

这个定理类似于最大流中的“无增广路”条件，为我们提供了算法终止的判据。

---

### 算法框架：维护最优性条件

借鉴推流重标号算法的思想，我们不希望在算法结束时才满足最优性条件，而是希望**在算法运行过程中始终维持**某种能推导出最优性条件的**不变性**。然后，算法只需专注于逐步达到**可行性**（即得到一个完美匹配）。

为此，我们引入**节点价格**的概念。算法将同时维护：
1.  一个（部分）匹配 `M`。
2.  为每个顶点 `v` 分配一个价格 `p(v)`。

基于价格，我们定义**约化成本**：对于边 `(v, w)`，其约化成本为 `c_p(v, w) = c(v, w) - p(v) - p(w)`。

我们将维护以下两个**不变性**：
1.  **非负约化成本**：对于所有边 `(v, w) ∈ E`，有 `c_p(v, w) ≥ 0`。
2.  **匹配边紧致**：对于所有匹配边 `(v, w) ∈ M`，有 `c_p(v, w) = 0`。

**关键引理**：如果算法最终得到一个**完美匹配** `M`，并且上述两个不变性成立，那么 `M` 就是最小成本完美匹配。
*   **证明概要**：根据不变性1和2，可以证明对于任何 `M-交错环`，其匹配边的总成本不大于非匹配边的总成本（即无非负环）。根据最优性条件定理，`M` 即是最优解。

因此，算法的任务变得清晰：**初始化**时令 `M` 为空集，所有价格 `p(v) = 0`（在边成本非负的假设下，不变性成立）。然后，在**保持两个不变性**的前提下，不断增大匹配 `M` 的规模，直至其成为完美匹配。

### 匈牙利算法：两种“好情况”

匈牙利算法的主循环检查两种“好情况”。只要其中一种成立，我们就可以在保持不变性的前提下取得进展。

#### 好情况一：存在增广路

**定义**：一条从左侧未匹配顶点到右侧未匹配顶点的路径 `P`，满足：
*   路径是交替的（边在匹配内和匹配外交替出现）。
*   路径上的所有边都是**紧致的**（约化成本为0）。

**操作——增广**：
*   计算新匹配 `M‘ = M ⊕ P`（对称差/异或操作）。
*   这会将匹配的基数增加1。
*   **不变性保持**：因为路径 `P` 全由紧致边构成，增广操作只是将一些紧致边加入匹配，将另一些紧致边移出匹配，所有边的约化成本未变，故两个不变性依然成立。

#### 好情况二：存在“好集合”

**定义**：一个左侧顶点的子集 `S`，满足：
1.  `S` 包含至少一个未匹配顶点。
2.  令 `N(S)` 为通过**紧致边**与 `S` 中顶点相连的右侧顶点集合。则 `N(S)` 中的每个顶点，都在当前匹配 `M` 中与 `S` 内的某个顶点相匹配。

**性质**：这样的集合 `S` 类似于霍尔定理中的“受限集”，因为 `|N(S)| < |S|`（由于存在未匹配顶点且匹配边一一对应）。

**操作——价格更新**：
*   令 `Δ = min{ c_p(v, w) | v ∈ S, w ∉ N(S) }`。这是从 `S` 出发，指向 `N(S)` 之外顶点的所有边中，最小的约化成本。
*   更新价格：
    *   对所有 `v ∈ S`，`p(v) := p(v) + Δ`。
    *   对所有 `w ∈ N(S)`，`p(w) := p(w) - Δ`。
*   **影响与不变性保持**：
    *   对于两个端点都在 `S ∪ N(S)` 内或都在其外的边，约化成本不变。
    *   对于从 `S` 到 `N(S)` 之外的边，其约化成本减少 `Δ`。我们选择的 `Δ` 保证了至少有一条这样的边约化成本变为0（成为新的紧致边），且没有边的约化成本变为负值。
    *   对于从 `S` 之外到 `N(S)` 的边，其约化成本增加 `Δ`，这没有影响（可能从紧致变为非紧致）。
    *   因此，不变性1（非负约化成本）依然成立。不变性2（匹配边紧致）也成立，因为所有匹配边要么两端点都在 `S ∪ N(S)` 内（价格一增一减，约化成本保持0），要么都在其外（价格不变）。

**进展**：价格更新操作增加了紧致边的数量，这为后续找到增广路创造了条件。

---

### 算法流程与示例

**算法主循环**：
1.  **初始化**：`M = ∅`，对所有顶点 `v`，`p(v) = 0`。
2.  **循环**：当 `M` 不是完美匹配时：
    *   尝试寻找一条**增广路**（好情况一）。
    *   如果找到，则执行**增广**操作。
    *   如果未找到，则必然存在一个**好集合** `S`（好情况二）。执行**价格更新**操作。

**如何寻找增广路或好集合？**
这可以通过一个修改版的广度优先搜索（BFS）来完成。从所有左侧未匹配顶点开始搜索，只沿着紧致边进行。如果搜索过程到达了一个右侧的未匹配顶点，我们就找到了一条增广路（好情况一）。如果搜索过程终止而未找到这样的路径，那么所有被搜索到的左侧顶点集合 `S`，就构成了一个好集合（好情况二）。

**示例简述**：
考虑一个简单二分图，边成本矩阵如下：
```
    W1  W2
V1 [7,  3]
V2 [5,  2]
```
1.  初始化：`M = ∅`, `p = [0, 0; 0, 0]`。无边紧致。
2.  未找到增广路。取 `S = {V1}`（未匹配），`N(S) = ∅`。进行价格更新：`Δ = min(7, 3) = 3`。更新后 `p(V1)=3`。边 `(V1, W2)` 变为紧致。
3.  找到增广路 `V1 -> W2`。增广后 `M = {(V1, W2)}`。
4.  未找到增广路。取 `S = {V1, V2}`（V2未匹配），通过紧致边 `(V1, W2)` 得 `N(S) = {W2}`。进行价格更新：`Δ = min(c_p(V1, W1)=4, c_p(V2, W1)=5) = 4`。更新后 `p(V1)=7, p(V2)=4, p(W2)=-4`。边 `(V1, W1)` 变为紧致。
5.  （后续步骤）继续交替进行价格更新和增广，最终将得到最小成本完美匹配 `{(V1, W2), (V2, W1)}`，总成本为 `3 + 5 = 8`。

### 总结与下节预告

本节课中我们一起学习了：
1.  **最小成本完美二分图匹配**问题的定义。
2.  判断匹配最优性的**负环定理**。
3.  设计算法的核心思路：通过维护**节点价格**和**约化成本**，使算法始终满足能推导出最优性的**不变性**，然后专注于构建完美匹配。
4.  **匈牙利算法**的高层框架，包括两种取得进展的操作：**沿紧致增广路增广**和**基于好集合的价格更新**。
