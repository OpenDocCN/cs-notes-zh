# Python 异步编程入门：01：理解异步编程基础

在本节课中，我们将要学习异步编程的核心概念。我们将探讨什么是异步编程，它与同步编程的区别，以及为什么在特定场景下使用异步编程能显著提升效率。

## 什么是异步编程？

上一节我们介绍了课程目标，本节中我们来看看异步编程的定义。

同步编程意味着代码按顺序执行。每一行代码必须等待前一行代码执行完毕后才能开始运行。这就像在只有一个收银台的超市排队，每个人必须等待前面的人结账完毕。

异步编程允许代码在等待某个操作（如从网络下载文件）完成时，去执行其他任务。这就像在超市里有多个收银台，当一个收银台忙碌时，你可以去另一个收银台结账，从而减少总的等待时间。

## 核心概念：事件循环

理解了基本定义后，我们需要认识异步编程的引擎：事件循环。

事件循环是异步编程的核心。它负责管理和调度所有任务。你可以把它想象成一个任务调度员。它持续运行，检查哪些任务已经完成了等待（例如，文件下载好了），然后安排这些任务继续执行。

在Python中，`asyncio`库提供了事件循环的实现。以下是一个最简化的代码示例，展示了事件循环如何运行一个异步任务：

```python
import asyncio

async def main():
    print('Hello')
    await asyncio.sleep(1)
    print('World')

asyncio.run(main())
```

## 关键语法：async 和 await

认识了事件循环，接下来我们学习驱动它的两个关键字。

`async`和`await`是Python中用于异步编程的关键字。
*   `async`：用于声明一个函数是“异步函数”。异步函数在被调用时不会立即执行，而是返回一个“协程”对象。
*   `await`：用于在异步函数内部“等待”另一个异步操作完成。在等待期间，事件循环可以切换到其他任务。

它们的关系可以用一个简单的公式描述：**在一个 `async def` 函数内，使用 `await` 来调用其他 `async` 函数**。

以下是这两个关键字的使用要点：

*   只有用 `async def` 定义的函数才能使用 `await` 关键字。
*   `await` 后面通常跟一个“可等待对象”，如另一个异步函数或特定任务。
*   调用异步函数时，必须在前面加上 `await`，或者在事件循环中运行它。

## 为何使用异步编程？

学习了基本语法，你可能会问：为什么要用异步？本节将解释其优势。

异步编程的主要优势在于处理“I/O密集型”任务时的高效率。I/O操作包括从网络请求数据、读写磁盘文件、与数据库交互等，这些操作的主要时间是“等待”外部系统响应，而不是消耗CPU进行计算。

考虑一个需要从10个不同网站下载页面的程序。
*   **同步方式**：顺序下载，总耗时是10次下载时间的总和。
*   **异步方式**：几乎同时发起所有下载请求，然后在等待每个响应时切换，总耗时接近于单次最长的下载时间。

其效率提升的原理可以概括为：**异步编程通过避免不必要的等待，最大化利用CPU时间，从而在I/O密集型场景下提升吞吐量**。

## 一个简单对比示例

理论需要实践来巩固。让我们通过一个代码示例来直观感受同步与异步的区别。

假设我们有两个任务，每个都需要“等待”1秒钟（模拟I/O操作）。

**同步版本**：
```python
import time

def task(name, delay):
    time.sleep(delay)
    print(f'{name} done')

start = time.time()
task("A", 1)
task("B", 1)
print(f'Total time: {time.time() - start:.2f} seconds')
```
输出会是：
```
A done
B done
Total time: 2.00 seconds
```

**异步版本**：
```python
import asyncio
import time

async def async_task(name, delay):
    await asyncio.sleep(delay)
    print(f'{name} done')

async def main():
    start = time.time()
    # 创建两个任务并并发执行
    task1 = asyncio.create_task(async_task("A", 1))
    task2 = asyncio.create_task(async_task("B", 1))
    await task1
    await task2
    print(f'Total time: {time.time() - start:.2f} seconds')

asyncio.run(main())
```
输出会是：
```
A done
B done
Total time: 1.00 seconds
```
可以看到，异步版本的总耗时接近1秒而非2秒，因为两个任务的等待时间重叠了。

## 总结

本节课中我们一起学习了异步编程的基础知识。

我们首先明确了异步编程是一种通过让程序在等待时执行其他任务来提升效率的范式。然后，我们认识了其核心机制——**事件循环**，以及两个关键语法——**`async`** 和 **`await`**。通过对比示例，我们了解到异步编程特别适用于**I/O密集型**场景，能有效减少总等待时间，提升程序性能。记住，异步不是让单个任务变快，而是让整个系统在同时处理多个任务时更高效。
