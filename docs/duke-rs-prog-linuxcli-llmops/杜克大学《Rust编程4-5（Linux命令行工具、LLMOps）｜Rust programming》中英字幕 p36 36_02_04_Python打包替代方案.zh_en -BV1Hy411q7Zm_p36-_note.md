# Rust编程4-5（Linux命令行工具、LLMOps）：36：Python打包替代方案 🐍

![](img/98f0367abc5379f968350df5477b9f96_0.png)

在本节课中，我们将探讨Python打包生态中一些替代传统`setuptools`的工具。我们之前提到，Python的打包方式存在一些问题，有多种不同的工具可以实现相同的打包目标。我们之前创建了一个二进制文件，并使用`setuptools`来完成打包所需的多种操作。现在，我们将介绍一些新的工具作为潜在的替代方案。

## 安装`build`工具 🔧

首先介绍的新工具是`build`。它被提议作为未来可能替代`setuptools`的工具。在使用之前，我们需要先安装它。

安装命令如下：
```bash
pip install build
```
这里的`build`就是工具的名称。

安装完成后，我们就可以使用它了。例如，我们可以运行：
```bash
python -m build
```
为了有更多空间，我们移除了文件浏览器视图。运行上述命令后，我们可以查看帮助菜单，了解`build`工具提供的所有功能。这确实是一个命令行工具，专门用于构建包。

## 使用`build`构建Wheel包 📦

如果我们想构建一个Wheel包，可以使用`bdist_wheel`命令。这与`setuptools`中的命令类似，但使用的是不同的工具。

具体命令是：
```bash
python -m build --wheel
```
这个命令的执行过程略有不同。你会看到它为我们创建了一个虚拟的隔离环境。当我们运行这个命令时，它会创建一个虚拟环境，并在其中安装必要的包，包括`setuptools`和`wheel`。它还会获取所有依赖项，并完成所有必需的步骤。

之前我们需要手动安装`wheel`包，但使用`build`时则不需要。命令完成后，所有生成的文件都会放在同一个位置。你可以看到它创建了一个类似`build/on a bid_mac_1011_arm64/wheel/`的子目录，并将我们所有的包和所需文件放在那里。这无疑是一个可行的替代方案。

## 可编辑模式安装的替代方案 🔄

过去，我们还使用过`python setup.py develop`命令。这个命令仍然可用且有效。但我们现在有一个替代方案，即使用`pip install`的`-e`标志。

以下是查看`-e`标志帮助信息的方法：
```bash
pip install --help
```
然后我们可以自己查找`-e`标志。不过，这种方法可能会遇到一些问题，因为帮助信息中有很多标志。但`-e`标志确实存在，它的描述是：“以可编辑模式安装项目”。这与`setuptools`的`develop`模式功能相同，可以从本地项目路径或版本控制系统（VCS）URL安装。

因此，我们可以运行：
```bash
pip install -e .
```
这意味着在当前目录以可编辑模式安装包。它会构建依赖项，并实现与之前`develop`模式相同的行为。这些都是你可以考虑的替代方案。

## 使用`pyproject.toml`配置文件 ⚙️

最后一个要介绍的工具是`pyproject.toml`。它允许我们通过一个配置文件来实现之前`setuptools`的功能。

`pyproject.toml`文件看起来很像一个配置文件。它实际上不是可执行文件，但如果我们没有`setup.py`，它仍然可以让我们仅通过一个`.toml`文件来安装依赖项并确保一切就绪。

在文件中，我们可以描述项目所需的一切。例如，我们可以指定所需的Python版本和包依赖。以下是一个示例内容：
```toml
[project]
name = "my_project"
version = "0.1.0"
requires-python = ">=3.8"
dependencies = [
    "click",
]
```
在这个例子中，我们不仅指定了库（例如`click`），还指定了希望使用的Python版本。这提供了一种更现代、更声明式的项目配置方式。

## 总结 📝

![](img/98f0367abc5379f968350df5477b9f96_2.png)

本节课我们一起学习了Python打包的几种替代方案。我们首先介绍了新的`build`工具，它可以用于构建Wheel包并自动管理环境。然后，我们探讨了使用`pip install -e .`作为`python setup.py develop`的替代命令，以实现可编辑模式安装。最后，我们了解了`pyproject.toml`配置文件，它提供了一种声明式的方法来管理项目元数据和依赖关系，是未来Python打包的重要方向。这些工具共同为Python开发者提供了更灵活、更现代的打包选择。
