# 035：为Python命令行工具创建包 📦

![](img/58a4e4544e7b5a87d341361729f26e18_0.png)

在本节课中，我们将学习如何将一个Python项目打包成可供分发的格式。我们将重点介绍使用 `setup.py` 文件创建二进制包（wheel）的方法，以及如何在开发模式下安装包以便快速测试修改。

## 概述

Python包不仅仅是包含代码的目录。当开发者谈论“包”时，通常指的是经过打包、可以发布和安装的软件单元。本节课将指导你使用 `setup.py` 和 `wheel` 工具来创建这样的包。

## 打包基础与 `setup.py`

上一节我们介绍了Python项目的基本结构。本节中我们来看看如何通过 `setup.py` 文件来描述和构建一个可发布的包。

`setup.py` 文件是Python传统打包生态系统的核心，它定义了项目的元数据和构建指令。需要注意的是，Python的打包生态存在一些挑战，例如 `setuptools` 未来可能不再支持某些交互方式。不过，目前它仍然是可用的方法之一，我们稍后会探讨其他替代方案。

## 准备虚拟环境

在开始打包之前，创建一个独立的虚拟环境是至关重要的。这能确保依赖隔离，并且是打包过程的标准实践。

以下是创建和激活虚拟环境的步骤：

1.  在项目根目录下创建虚拟环境。使用 `.venv` 作为目录名是一个好习惯。
    ```bash
    python -m venv .venv
    ```
2.  激活虚拟环境。在VS Code中，选择该环境作为工作区解释器后，新打开的终端会自动激活它。
3.  如果需要在终端手动激活，可以运行：
    ```bash
    source .venv/bin/activate
    ```
    激活后，终端提示符前通常会显示 `(.venv)`。

## 构建二进制分发包（Wheel）

虚拟环境准备就绪后，我们就可以开始构建包了。构建二进制包（wheel）是一种高效的分发方式，用户无需在本地编译即可安装。

首先，必须确保在虚拟环境中安装了 `wheel` 包，它是构建二进制分发格式的基础。
```bash
pip install wheel
```

安装完成后，可以使用 `setup.py` 来查看可用的构建命令。
```bash
python setup.py --help-commands
```

在帮助命令的输出中，找到 `bdist_wheel`，这个命令就是用来创建wheel格式的二进制分发包的。运行以下命令进行构建：
```bash
python setup.py bdist_wheel
```

命令执行成功后，会在项目目录下生成一个 `dist` 文件夹，里面包含一个 `.whl` 文件。这个文件就是打包好的二进制分发包，可以直接用于发布和安装。

## 理解构建产物

构建过程完成后，让我们查看一下生成的文件，以理解其命名和用途。

生成的 `.whl` 文件名中包含了系统平台和架构的标签（例如 `macosx_10_11_arm64`）。这是因为wheel是预编译的二进制分发格式，它针对特定的操作系统和处理器架构进行了优化。这样，用户在其对应的系统上安装时，就无需本地编译，安装速度更快，也避免了编译依赖的问题。

## 开发模式安装

在开发过程中，频繁地重新构建和安装包效率很低。为此，Python提供了开发模式安装，它允许你直接链接到源代码进行修改和测试。

使用以下命令进行开发模式安装：
```bash
python setup.py develop
```

这个命令会将你的包以“开发模式”安装到环境中。它不会将文件复制到Python的 `site-packages` 目录，而是创建一个指向你项目源代码的链接。这样，你对代码所做的任何修改都会立即生效，无需重新安装。你可以通过 `which` 命令验证命令行工具是否链接到了你的项目路径。

当你完成开发并希望进行永久性安装时，则应使用标准的 `pip install .` 或 `python setup.py install` 命令。

## 总结

![](img/58a4e4544e7b5a87d341361729f26e18_2.png)

本节课中我们一起学习了为Python命令行工具创建包的核心流程。我们首先了解了 `setup.py` 文件的作用，然后通过创建虚拟环境来隔离依赖。接着，我们安装了 `wheel` 包并使用 `python setup.py bdist_wheel` 命令成功构建了二进制分发包。最后，我们探讨了开发模式安装（`python setup.py develop`）的便利性，它允许开发者在修改代码后立即看到效果，极大地提升了开发效率。掌握这些步骤，你就能有效地打包和分发自己的Python工具了。
