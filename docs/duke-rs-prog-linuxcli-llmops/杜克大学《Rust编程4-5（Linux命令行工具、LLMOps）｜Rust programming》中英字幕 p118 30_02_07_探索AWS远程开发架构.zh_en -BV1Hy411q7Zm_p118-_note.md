## 概述
在本节课中，我们将学习如何构建一个基于Rust和Hugging Face Candle的异步开发工作流。我们将重点介绍如何利用AWS云服务，以高效、低成本的方式搭建一个适用于大型语言模型（LLM）开发和部署的远程开发架构。

---

## 开发环境搭建 💻

上一节我们概述了课程目标，本节中我们来看看如何搭建开发环境。

首先，你需要一个EC2 G5实例。该实例的优势在于它具备执行现代GPU推理任务的能力，并且你可以使用Spot实例来节省成本。

在开发过程中，你可以随时停止和启动这个实例。你可以通过Visual Studio Code Remote连接到该实例，这是一种非常高效的开发方式。

以下是搭建步骤：
1.  在VS Code中安装远程开发插件。
2.  配置插件以连接到你的EC2 G5实例。
3.  连接成功后，你的本地笔记本电脑即可访问这个强大的云端开发环境。

这意味着你可以在一个功能强大的云端环境中开始开发代码，并且按使用量付费，而无需购买昂贵的硬件，例如价值4万美元的GPU。

---

## 生产环境部署与协同工作流 🔄

上一节我们介绍了开发环境的搭建，本节中我们来看看如何将其扩展到生产环境，并实现协同工作。

这里的情况变得更有趣。你可以结合EFS（弹性文件系统）进行协同开发。

以下是该工作流的核心环节：
*   **共享存储**：EFS的挂载点可以同时连接到你的开发机和生产环境。
*   **模型管理**：你可以将庞大的模型和权重文件（数据量可能达到数十GB）下载到EFS中。
*   **数据中转**：EFS可以作为这些数据从开发到生产环境的中转站。

当你向AWS Batch部署任务时，Batch服务可以自动配置EC2 G5实例。这样，你就能以一种非常高效、优雅且低成本的方式，在这些强大的GPU上执行基于Rust的现代推理任务。

你不需要使用Notebook，也不需要Python，整个过程采用纯Rust的方式实现。

## 架构扩展与成本优化 💡

上一节我们探讨了生产部署流程，本节中我们来看看如何进一步扩展架构并优化成本。

另一个需要考虑的方向是，如果你将代码编译为同时兼容CPU的版本。

你的AWS Lambda函数可以直接从EFS读取该二进制文件，并同样将其作为生产实例提供服务。

因此，我认为这是一个非常令人兴奋的、新兴的Rust在LLMOps和MLOps领域的故事。这真正意味着你可以探索新事物、新想法，并追求效率。

对于许多希望部署LLM服务的公司而言，我认为这个特定的架构是一个很好的起点。

## 总结
本节课中，我们一起学习了如何利用AWS云服务构建一个基于Rust的远程开发与生产架构。我们从搭建EC2 G5开发实例开始，探讨了通过VS Code Remote进行远程开发的方法。接着，我们引入了EFS来实现开发与生产环境间的数据共享与协同工作，并介绍了如何使用AWS Batch进行高效、低成本的GPU推理任务部署。最后，我们还看到了该架构通过AWS Lambda服务CPU版本二进制文件的扩展可能性。这套架构为希望在LLM领域采用高效、现代化Rust技术栈的团队提供了一个强大的起点。
