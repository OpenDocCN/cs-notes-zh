# Python日志记录：4-5：在Python中使用不同类型的日志

在本节课中，我们将学习如何在Python中配置和使用不同类型的日志处理器，以实现更灵活、更强大的日志记录功能。我们将超越基础的`basicConfig`方法，探索如何为根日志记录器添加多个处理器，从而将日志信息同时输出到控制台和文件，并设置不同的日志级别。

## 概述

之前我们使用`basicConfig`来配置日志记录。虽然这种方法简单，但它存在局限性。本节我们将采用一种更灵活、可扩展的方式，手动配置根日志记录器，并为其添加控制台处理器和文件处理器。这种方法允许我们为不同的输出目标设置不同的日志级别，例如，在文件中记录详细的调试信息，而在控制台仅显示关键的信息。

## 配置根日志记录器

首先，我们需要获取并配置根日志记录器。根日志记录器是所有其他日志记录器的父级，通过配置它，我们可以影响整个应用程序的日志行为。

![](img/56b3e4f97e268e0fa43630dc5d78113c_1.png)

```python
import logging

# 获取根日志记录器
root_logger = logging.getLogger()
# 设置根日志记录器的默认级别为DEBUG
root_logger.setLevel(logging.DEBUG)
```

在上面的代码中，我们使用`logging.getLogger()`获取了根日志记录器，并将其默认日志级别设置为`DEBUG`。这意味着所有低于或等于`DEBUG`级别的日志消息最初都会被捕获。

## 创建并配置处理器

接下来，我们将创建两个处理器：一个用于控制台输出，另一个用于文件输出。处理器负责将日志记录器产生的消息发送到指定的目的地。

以下是创建和配置这两个处理器的步骤：

### 1. 控制台处理器

控制台处理器（`StreamHandler`）将日志消息输出到终端（默认是标准错误输出`stderr`）。

```python
# 创建控制台处理器
console_handler = logging.StreamHandler()
# 设置控制台处理器的日志级别为INFO
console_handler.setLevel(logging.INFO)
```

### 2. 文件处理器

文件处理器（`FileHandler`）将日志消息写入到指定的文件中。

```python
# 创建文件处理器，指定日志文件名
file_handler = logging.FileHandler('blockpy.log')
# 设置文件处理器的日志级别为DEBUG
file_handler.setLevel(logging.DEBUG)
```

通过以上设置，我们实现了差异化的日志记录：文件`blockpy.log`将记录所有`DEBUG`及以上级别的详细信息，而控制台只显示`INFO`及以上级别的关键信息。这种配置在开发时非常有用，既保留了完整的调试轨迹，又保持了用户界面的简洁。

## 将处理器添加到根日志记录器

创建好处理器后，我们需要将它们添加到根日志记录器中，这样日志消息才能被正确地分发到各个处理器。

```python
# 将控制台处理器添加到根日志记录器
root_logger.addHandler(console_handler)
# 将文件处理器添加到根日志记录器
root_logger.addHandler(file_handler)
```

现在，根日志记录器配置了两种处理器。任何通过这个记录器（或其子记录器）生成的日志消息，都会同时被发送到控制台和日志文件，并根据各自处理器的级别设置进行过滤。

## 测试配置效果

让我们来测试一下配置是否生效。运行一个简单的日志记录脚本：

```python
# 记录不同级别的日志消息
root_logger.debug('这是一条调试信息。')
root_logger.info('这是一条普通信息。')
root_logger.warning('这是一条警告信息。')
```

执行上述代码后，你可能会注意到两个现象：
1.  **控制台输出**：由于控制台处理器级别为`INFO`，你将只看到“这是一条普通信息。”和“这是一条警告信息。”，而不会看到`DEBUG`级别的消息。
2.  **文件输出**：打开`blockpy.log`文件，你会看到所有三条消息（`DEBUG`、`INFO`、`WARNING`）都被记录在内。

此外，你可能还会发现，日志消息的格式（如时间戳）与使用`basicConfig`时不同。这是因为我们没有配置格式化器。你可以通过为每个处理器设置`formatter`属性来自定义日志输出的格式，例如添加时间戳、日志级别和模块名称。

```python
# 创建一个格式化器
formatter = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
# 将格式化器应用到处理器
console_handler.setFormatter(formatter)
file_handler.setFormatter(formatter)
```

## 总结

在本节课中，我们一起学习了如何在Python中构建更高级的日志记录系统。我们首先介绍了根日志记录器的概念，然后逐步创建了控制台处理器和文件处理器，并为它们设置了不同的日志级别。最后，我们将这些处理器添加到根日志记录器，并验证了日志消息能够被正确地分流到不同的输出目标。

![](img/56b3e4f97e268e0fa43630dc5d78113c_3.png)

这种方法比简单的`basicConfig`提供了更大的灵活性和控制力，允许你根据需求精细地调整日志行为，例如为不同环境（开发、生产）或不同用户（开发者、最终用户）定制日志输出。你可以进一步探索，例如通过命令行参数动态配置日志级别，或者为不同的模块创建子记录器并分配特定的处理器。
