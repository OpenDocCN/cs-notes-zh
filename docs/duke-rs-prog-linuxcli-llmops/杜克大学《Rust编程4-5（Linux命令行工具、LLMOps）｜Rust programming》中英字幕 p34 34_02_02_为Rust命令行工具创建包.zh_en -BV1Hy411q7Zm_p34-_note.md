# Rust编程4-5：34_02_02：为Rust命令行工具创建包 📦

![](img/9eaec9e678aa2e472e914ccbc3c0f0ca_0.png)

在本节课中，我们将学习如何为Rust命令行工具创建和构建一个可发布的包。我们将重点理解`cargo build`命令在调试（debug）和发布（release）模式下的区别，以及如何生成最终的可执行文件。

到目前为止，我们一直在使用Rust开发一个命令行工具。我们所有的代码都位于`src`目录下，主要包含`lib.rs`和`main.rs`文件。这个工具是一个`else`块的包装器，我们将其命名为`block_rs`。

现在，是时候深入理解构建一个包（package）所涉及的具体步骤了。

## 理解Cargo的构建过程

在此之前，我们通常使用`cargo run`命令来运行程序。这个命令实际上一次性完成了两件事：首先构建（build）项目，然后运行生成的可执行文件。

让我们先检查一下项目结构。在项目根目录下，你会发现一个`target`文件夹。这是Cargo存放所有构建产物的目录。如果你查看里面的内容，会发现大量文件，包括项目本身编译后的库、所有依赖项（例如我们使用的`clap`库）等。

为了更清晰地理解，我们先看看`cargo build`命令。运行它（如果代码没有改动，它会很快完成，因为Cargo会利用缓存）。`cargo run`命令内部就包含了`cargo build`这一步。

现在，让我们进入`target`目录看看。在`target/debug`目录下，我们可以找到`block_rs`（或`block_rs.exe`，取决于你的操作系统）。这就是我们命令行工具的调试版本可执行文件。我们可以直接运行它，例如：
```bash
./target/debug/block_rs --help
```
这个文件的大小大约是5MB。目前，我们只需关注这个可执行文件本身。

## 构建发布版本（Release Build）

那么，如何构建一个准备发布的、经过优化的包呢？答案是使用`cargo build --release`命令。

在运行这个命令之前，我们先看看`cargo build --help`，会发现其中提到了`--release`选项。构建发布版本意味着几件事：
1.  **启用优化**：编译器会进行大量优化，使最终代码运行速度更快。
2.  **移除调试信息**：例如测试代码、调试符号等会被剥离，使包更精简。
3.  **更长的编译时间**：因为编译器需要进行复杂的分析和优化，所以构建过程会比调试模式慢得多。

Rust团队这样设计（区分调试和发布模式）是为了让我们在开发时能快速迭代（使用快速的调试构建），而在准备发布时，再花时间生成一个高度优化的最终产品。

现在，让我们运行`cargo build --release`。你会立刻注意到，与之前快速的调试构建相比，发布构建需要花费更长的时间（在我的机器上大约花了一分多钟）。

构建完成后，再次查看`target`目录。现在除了`debug`文件夹，还多了一个`release`文件夹。

## 调试版与发布版的对比

让我们来对比一下两个版本的可执行文件：

以下是关键区别：
*   **文件大小**：发布版的可执行文件（约1.1MB）通常比调试版（约4.9MB）更小，因为移除了调试信息。
*   **构建速度**：调试版的构建速度极快，适合开发。发布版的构建速度慢，因为它进行了全面的优化。
*   **运行性能**：发布版的可执行文件由于启用了优化，运行时性能会更好。

## 总结

本节课中，我们一起学习了如何为Rust命令行工具创建包。我们掌握了：
1.  `cargo run` 命令同时完成了构建和运行。
2.  `cargo build` 命令专门用于构建项目，产物位于 `target/debug/` 目录。
3.  `cargo build --release` 命令用于构建优化的发布版本，产物位于 `target/release/` 目录。发布版本构建更慢，但生成的可执行文件更小、运行更快，适合最终分发。
