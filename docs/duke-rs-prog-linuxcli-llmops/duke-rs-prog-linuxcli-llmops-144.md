# 144：使用AWS Bedrock API 🚀

![](img/f9a2b510795ec4a1a5d264c0c616140a_0.png)

在本节课中，我们将探讨AWS Bedrock API，这是一个允许开发者灵活使用多种大型语言模型（LLM）的服务。我们将了解其核心功能，包括如何查询不同公司的模型、如何微调自定义模型，以及如何在实际应用中调用这些模型。

---

## 概述

AWS Bedrock API为LLM操作（LLMOps）提供了强大的支持。它允许开发者构建能够访问多种大型语言模型的应用程序，而不仅仅依赖于单一公司的模型。这为应用开发带来了前所未有的灵活性和选择空间。

## 核心功能

上一节我们概述了Bedrock API的价值，本节中我们来看看它的具体功能。

### 1. 多模型查询

AWS Bedrock API的核心优势之一是能够访问来自不同公司的多种基础模型。这意味着你的应用程序可以集成一个多样化的模型生态系统。

以下是其工作方式：
*   你的应用程序（例如一个移动应用）可以向Bedrock API发起查询。
*   API可以访问来自五、六家不同公司提供的基础模型。
*   你的应用程序甚至可以将这些模型选择呈现给最终用户，让他们自行决定使用哪个模型。

### 2. 自定义模型微调

除了使用现成的基础模型，你还可以基于Bedrock系统中的这些模型，创建和微调属于你自己的定制化模型。

以下是你可以实现的目标：
*   基于Bedrock中的基础模型构建你自己的定制模型。
*   为用户提供高度定制化的体验。
*   构建仅供企业内部使用的应用程序，确保只使用你自己的数据进行微调，模型完全为你所有。

### 3. 提示与模型调用

对于最终用户而言，他们可以通过简单的操作来利用这些强大的模型。

用户的核心操作流程如下：
*   用户提供提示（`prompt`）。
*   应用程序调用（`invoke`）所选模型来处理该提示。

## 未来展望与总结

本节课中我们一起学习了AWS Bedrock API的关键特性。我们看到，大型语言模型的未来并非盲目接受由单一公司主导的、受严格监管的AI。

未来的趋势将是：
*   **竞争涌现**：不同公司和模型之间将展开竞争。
*   **能力多样化**：各种模型将发展出不同的专长和能力。

像AWS这样的云服务提供商将成为一个绝佳的选择，原因在于：
*   你不仅可以从**众多大型语言模型**中进行选择。
*   还可以使用**你自己微调的模型**。

因此，AWS Bedrock API是一个非常值得关注和尝试的新特性，它为构建下一代AI应用打开了大门。
