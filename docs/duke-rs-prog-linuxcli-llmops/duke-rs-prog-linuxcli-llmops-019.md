## 处理导入：绝对导入与相对导入

现在，让我们通过一些更实际的例子，来看看如何处理导入，以及如何进行绝对导入和相对导入。

为了搭建一个简单框架，我们只关注`modules`这个目录。假设这是你的项目，里面有一个`__init__.py`文件、一个`constants.py`文件和一个名为`sub`的子目录，子目录里有一个`command.py`文件，其中包含一个函数。

让我们探索在Python项目中工作时可能会遇到的一些情况。

### 探索导入行为

在`modules`目录中，我们有`__init__.py`、`up.py`和`constants.py`。让我们看看如果导入`modules`会发生什么。

导入`modules`后，你可以调用`modules.__file__`来查看它来自哪里。这非常有用，可以让你知道这个模块存在于文件系统的哪个位置。你也可以直接打印它，在交互式终端中会得到输出。

现在，假设你在`modules`中，导入了`modules`，并且想要访问`sub`子目录下的`run`函数。你可能会尝试`modules.sub`，但会得到错误提示：`modules`没有`sub`属性。

问题在于，我们有一个名为`sub`的子目录，里面有一个`__init__.py`文件，按理说应该可以工作，比如`modules.sub.command.run()`。但为什么不行呢？

这里有两种主要的处理方式。我们可以使用`import modules.sub`，然后通过`modules.sub`来访问。这样，导入系统就将`sub`作为`modules`的一部分了。

但接着，如果我们尝试`modules.sub.command`，又会发现没有`command`属性。如果你不确定如何继续，或者不想使用完整的导入语句，这会变得很烦人。

我们可以尝试`import modules.sub.command`，然后`modules.sub.command.run()`就能工作了。但如果你不想一直输入完整路径，有什么办法可以简化呢？

### 在`__init__.py`中添加导入

我们可以在`__init__.py`文件中添加一些导入语句。进入`modules`目录下的`__init__.py`文件，尝试添加`import sub`。

保存后，需要重启Python shell（因为我们在动态修改文件，需要重新加载）。然后尝试`import modules`，但会得到一个`ModuleNotFoundError`。为什么这不行呢？`sub`明明存在。

问题在于我们尝试进行**绝对导入**。这意味着`sub`应该被视为命名空间中一个独立的、绝对的模块。但它并不是一个独立的模块，它是这个名为`modules`的项目目录的一部分。

### 使用相对导入

为了解决这个问题，我们需要使用**相对导入**。具体做法是，在`__init__.py`中写`from . import sub`。这个点`.`表示“从这里”，即从当前路径、当前目录导入`sub`。

让我们清除之前的尝试，重启Python，然后执行`import modules`。现在，`modules.sub`就能正常工作了。这是需要牢记的一点。

你可以灵活决定将哪些内容作为模块的一部分。这里的`modules`可以是你的项目、命令行工具、库、子模块或目录。你可以在里面放入任何你想要的东西。

### 导入常量并暴露到顶层

我们可以进一步扩展。例如，在`constants.py`中，我们可能有`verbose`和`log`这样的常量。你可能会有上百个这样的常量，将它们组织在`constants.py`中是可以的，但你可能也希望其中一些能在库的顶层直接访问，而不仅仅是在`constants`模块内部。

你可以在`__init__.py`中这样写：`from .constants import verbose, log`。注意，这里我们再次使用了相对导入（`.`）。

让我们再次退出并重启Python解释器（因为文件已被修改）。然后执行`import modules`，现在`modules.log`和`modules.verbose`就可以正常工作了。
