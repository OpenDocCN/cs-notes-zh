# 004：动态随机存取存储器（DRAM）基础

在本节课中，我们将要学习动态随机存取存储器（DRAM）的基本工作原理。这是一种与静态RAM（SRAM）不同的内存技术，虽然实现起来更复杂，但成本更低，能实现更高的存储密度。

上一节我们介绍了完全由晶体管构成的静态RAM。本节中，我们来看看另一种更便宜但也更具挑战性的内存——动态RAM。

## 核心组件：电容与MOSFET

动态RAM的基础是两个关键电子元件：**电容**和**MOSFET**。

![](img/c9b5c4f63f07371e9774ea7918b11419_1.png)

![](img/c9b5c4f63f07371e9774ea7918b11419_2.png)

*   **电容**：一种能存储电荷的电子元件。当连接到电压源时，它会充电并储存能量，即使断开电源后，也能短暂地像一个小电池一样工作。
*   **MOSFET**：一种特殊类型的晶体管，因其高效率而更适合计算机应用和微型化。它有三个引脚：栅极、源极和漏极。在栅极施加一个小电压，就能控制电流从源极流向漏极，或在特定条件下反向流动。

通过将MOSFET和电容组合，我们可以构建一个基础的存储单元。利用MOSFET的栅极控制电流，可以为电容充电（存储“1”）或放电（存储“0”）。这个简单的电路就是一个**动态RAM存储单元**，能够存储1比特信息。

## DRAM vs. SRAM：结构与成本

如果你看过关于静态RAM的视频，可能会觉得动态RAM的实现看起来简单得多。

*   在静态RAM中，我们使用一系列逻辑门以巧妙但更复杂的方式“锁存”电信号来存储数值。
*   相比之下，对一个电容进行充放电看起来容易得多。

![](img/c9b5c4f63f07371e9774ea7918b11419_4.png)

![](img/c9b5c4f63f07371e9774ea7918b11419_5.png)

一个静态RAM存储单元（锁存器）需要多个晶体管，而一个动态RAM存储单元只需要**一个晶体管和一个很小的电容**。这使得在相同面积内可以集成更多的动态RAM存储单元，从而实现更高的存储容量和更低的制造成本。

![](img/c9b5c4f63f07371e9774ea7918b11419_7.png)

## DRAM的核心挑战与读取操作

然而，动态RAM并非完美，它有几个主要缺点。最关键的问题是：**读取数据会破坏数据**。

考虑一个存储了“1”（电容已充电）的动态存储单元。要读取其值，唯一的方法是打开MOSFET的栅极，检查是否有电流从电容流出。这意味着检查电容是否带电的过程会导致其放电，因此读取操作是破坏性的。

以下是动态RAM读取数据而不丢失数据的步骤概述：

1.  **预充电位线**：首先，将连接每一列存储单元的**位线**预充电到工作电压的一半（例如，0.5V）。
2.  **激活字线**：通过**行地址解码器**选中特定行，打开该行所有存储单元中MOSFET的栅极。
3.  **感应电压变化**：
    *   对于存储“1”的单元，电容电压高于位线，电荷流出，导致位线电压微升。
    *   对于存储“0”的单元，电容电压低于位线，电荷流入电容，导致位线电压微降。
4.  **放大与锁存信号**：位于每条位线末端的**感应放大器**检测电压变化。如果电压高于参考值（如0.5V），则输出代表“1”的电压（如1V）；反之则输出“0”。感应放大器会立即将读取到的值存入一个**锁存器**中暂存。
5.  **选择输出**：通过一个**多路复用器**，根据列地址从该行所有锁存的值中选择目标比特位输出。
6.  **重写（刷新）**：由于读取过程改变了电容的电荷状态，系统会利用锁存器中保存的值，通过位线将正确的电压值写回该行的所有电容中，恢复其原始状态。

通过这一系列精密操作，我们实现了从动态RAM中非破坏性地读取数据。

## DRAM的写入操作

写入操作建立在读取操作的基础之上。目标是向特定存储单元写入新值（0或1），同时不影响同行其他单元的数据。

写入过程与读取非常相似，但数据流向相反：

1.  **预充电与读取行数据**：同样先预充电位线，然后激活目标行字线。感应放大器读取并锁存该行所有单元的当前值。
2.  **输入新数据**：此时，我们**输入**要写入的新数据值。
3.  **路由数据**：一个**多路分配器**根据列地址，将输入的新数据定向到目标列对应的锁存器中，覆盖其原有值。
4.  **写回整行**：接着，所有锁存器（包括刚被修改的那个）将它们当前的值通过位线写回对应的存储单元电容中。

这样，我们就完成了对指定单元的写入，并同时刷新了该行所有其他单元的数据。

## DRAM的额外挑战：刷新与定时

![](img/c9b5c4f63f07371e9774ea7918b11419_13.png)

![](img/c9b5c4f63f07371e9774ea7918b11419_15.png)

动态RAM还有两个固有难题：

1.  **电荷泄漏**：即使MOSFET栅极关闭，电容的电荷也会缓慢泄漏，导致数据随时间丢失。
2.  **操作时序复杂**：读取、写入和刷新操作都涉及多个精密步骤（预充电、激活、感应、放大、重写），每个步骤都需要精确的定时控制。

为了解决电荷泄漏问题，动态RAM必须定期执行**刷新操作**。这本质上就是周期性地对每一行执行一次“读取后立即重写”的操作，在数据丢失前恢复电容的电荷。刷新操作由专用电路管理，会暂时阻止对内存的访问，因此引入了延迟。

尽管有这些额外步骤和延迟，但由于每次读写操作仅需几纳秒，而刷新周期以毫秒计，因此在两次刷新间隔内可以执行成千上万次内存访问。

## 从比特到字节：数据总线

到目前为止，我们讨论的是单个比特的存取。计算机通常以字节（8比特）为单位操作内存。这可以通过更复杂的多路复用器和多路分配器来实现，它们能够处理整个**数据总线**，一次性对一组（例如8个）存储单元进行寻址和读写。

## 总结

本节课中我们一起学习了动态随机存取存储器（DRAM）的基础知识。

![](img/c9b5c4f63f07371e9774ea7918b11419_17.png)

![](img/c9b5c4f63f07371e9774ea7918b11419_19.png)

*   **核心原理**：利用**电容**存储电荷（代表数据0/1），通过**MOSFET**控制访问。
*   **主要优势**：结构简单，存储密度高，**成本低廉**。
*   **关键挑战与解决方案**：
    *   **破坏性读取**：通过“**读取-放大-重写**”的复杂时序操作解决。
    *   **电荷泄漏**：通过定期的**自动刷新**操作解决。
    *   **时序复杂**：需要精密的控制电路来协调所有步骤。
*   **与SRAM对比**：DRAM速度较慢（因为操作步骤多），但成本更低、容量更大；SRAM速度极快但结构复杂、成本高、密度低。这就是为什么CPU内部的高速缓存使用SRAM，而主内存使用DRAM。

![](img/c9b5c4f63f07371e9774ea7918b11419_21.png)

希望本视频能帮助你理解，处理器内部的缓存（一种SRAM）比你电脑中的内存条（DRAM）更快，不仅是因为它离CPU更近，更是由于其根本不同的工作原理。所以，下次听到“速度”这个词时，不要过于从字面理解——把硬盘放得离CPU再近，它也不会比内存更快。
