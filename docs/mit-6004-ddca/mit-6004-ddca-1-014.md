# 数字系统与计算机架构：1.2.12：补码加法运算示例 🧮

![](img/c7eb6eeaad547efc5f7a66f111aab486_0.png)

在本节课中，我们将学习如何使用二进制补码表示法进行加法运算。我们将通过具体的例子，演示如何将减法问题转化为加法问题，并理解运算过程中可能出现的溢出情况。

---

![](img/c7eb6eeaad547efc5f7a66f111aab486_2.png)

二进制补码表示法的一个优点是，它允许你将加法和减法都作为加法问题来处理。具体方法是将减法问题 `A - B` 转化为加法问题 `A + (-B)`。由于我们已经知道如何轻松地对补码数值进行取反操作，这个过程变得非常直接。

![](img/c7eb6eeaad547efc5f7a66f111aab486_3.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_4.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_5.png)

上一节我们介绍了补码的取反操作，本节中我们来看看如何利用它进行实际的加减法运算。

![](img/c7eb6eeaad547efc5f7a66f111aab486_7.png)

## 示例一：计算 15 - 18

![](img/c7eb6eeaad547efc5f7a66f111aab486_9.png)

让我们使用6位二进制补码表示法来计算 `15 - 18`。

![](img/c7eb6eeaad547efc5f7a66f111aab486_11.png)

首先，将减法转化为加法：
`15 - 18 = 15 + (-18)`

![](img/c7eb6eeaad547efc5f7a66f111aab486_13.png)

**步骤1：获取操作数的补码表示**

![](img/c7eb6eeaad547efc5f7a66f111aab486_15.png)

*   `15` 的二进制表示：`001111`
*   `18` 的二进制表示：`010010`

为了得到 `-18`，我们对 `18` 的表示执行“按位取反并加1”的操作：
1.  按位取反 `010010` 得到 `101101`。
2.  加 `1`：`101101 + 1 = 101110`。
因此，`-18` 的补码表示为 `101110`。

![](img/c7eb6eeaad547efc5f7a66f111aab486_17.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_18.png)

**步骤2：执行加法运算**

![](img/c7eb6eeaad547efc5f7a66f111aab486_20.png)

现在，我们将 `15` 和 `-18` 的补码相加：
```
   001111  (15)
 + 101110  (-18)
 ------------
   111101
```
在二进制加法中，`1 + 1` 会产生一个 `0` 和一个进位 `1`。按照这个规则，我们得到最终的和为 `111101`。

**步骤3：解读结果**

![](img/c7eb6eeaad547efc5f7a66f111aab486_22.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_23.png)

为了解读 `111101` 代表什么数值，我们再次对其执行“按位取反并加1”的操作（即求其相反数）：
1.  按位取反 `111101` 得到 `000010`。
2.  加 `1`：`000010 + 1 = 000011`。
`000011` 等于十进制数 `3`。

这意味着 `111101` 等于 `-3`。这正是我们执行 `15 - 18` 时期望得到的结果。

![](img/c7eb6eeaad547efc5f7a66f111aab486_25.png)

## 示例二：计算 27 - 6

![](img/c7eb6eeaad547efc5f7a66f111aab486_28.png)

让我们尝试另一个例子：`27 - 6`。

![](img/c7eb6eeaad547efc5f7a66f111aab486_30.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_31.png)

同样，转化为加法：`27 - 6 = 27 + (-6)`。

**步骤1：获取操作数的补码表示**

![](img/c7eb6eeaad547efc5f7a66f111aab486_33.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_34.png)

*   `27` 可以分解为 `16 + 8 + 2 + 1`，即 `2^4 + 2^3 + 2^1 + 2^0`。其6位二进制表示为 `011011`。
*   `6` 可以分解为 `4 + 2`，即 `2^2 + 2^1`。其6位二进制表示为 `000110`。

为了得到 `-6`，我们对 `6` 的表示执行取反操作：
1.  按位取反 `000110` 得到 `111001`。
2.  加 `1`：`111001 + 1 = 111010`。
因此，`-6` 的补码表示为 `111010`。

![](img/c7eb6eeaad547efc5f7a66f111aab486_36.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_37.png)

**步骤2：执行加法运算**

将 `27` 和 `-6` 的补码相加：
```
   011011  (27)
 + 111010  (-6)
 ------------
 1 010101
```
注意，这里产生了一个额外的进位（第7位）。因为我们使用的是6位表示法，这个最高位的进位会被丢弃。

![](img/c7eb6eeaad547efc5f7a66f111aab486_39.png)

**步骤3：解读结果**

![](img/c7eb6eeaad547efc5f7a66f111aab486_40.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_41.png)

所以，最终结果是 `010101`。这个二进制数等于 `2^4 + 2^2 + 2^0 = 16 + 4 + 1 = 21`。这正是 `27 - 6` 的正确结果。

上一节我们完成了两个成功的运算示例，本节中我们来看看运算中一个重要的异常情况：溢出。

## 溢出检测 ⚠️

如果你尝试相加的两个数，其结果超出了你的N位表示法所能表示的数字范围，就意味着发生了溢出。

![](img/c7eb6eeaad547efc5f7a66f111aab486_46.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_47.png)

溢出可以通过观察被加数的符号和结果的符号来检测：

![](img/c7eb6eeaad547efc5f7a66f111aab486_48.png)

以下是判断溢出的两条规则：

1.  **正数相加得负数**：如果你尝试将两个最高位为 `0` 的正数相加，得到的结果最高位却是 `1`（即结果为负数），那么发生了溢出。
2.  **负数相加得正数**：如果你尝试将两个最高位为 `1` 的负数相加，得到的结果最高位却是 `0`（即结果为正数），那么同样发生了溢出。

![](img/c7eb6eeaad547efc5f7a66f111aab486_50.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_51.png)

当一个正数和一个在表示范围内的负数相加时，不会发生溢出。

![](img/c7eb6eeaad547efc5f7a66f111aab486_53.png)

## 溢出示例

![](img/c7eb6eeaad547efc5f7a66f111aab486_55.png)

让我们看一个例子。假设我们尝试使用6位补码计算 `31 + 12`。

*   `31` 的补码：`011111`
*   `12` 的补码：`001100`

![](img/c7eb6eeaad547efc5f7a66f111aab486_57.png)

![](img/c7eb6eeaad547efc5f7a66f111aab486_58.png)

执行加法：
```
   011111  (31)
 + 001100  (12)
 ------------
   101011
```
结果 `101011` 的最高位是 `1`，这意味着结果被解释为一个负数。然而，我们相加的是两个正数。这表明**溢出发生了**。

溢出发生的原因是，这两个数的和是 `43`，它大于 `2^(5) - 1 = 31`（这是6位补码能表示的最大正数）。

![](img/c7eb6eeaad547efc5f7a66f111aab486_60.png)
