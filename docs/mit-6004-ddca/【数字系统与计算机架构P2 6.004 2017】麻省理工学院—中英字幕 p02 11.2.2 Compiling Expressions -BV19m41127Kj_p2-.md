A compiler is a program that translates a high level language program into a functionally equivalent sequence of machine instructions。

 In other words， an assembly language program。

![](img/115bee56a3e07bed3de526bef8b8141d_1.png)

A compiler first checks that the high level program is correct。 In other words。

 that the statements are well formed。 The programmer isn't asking for nonsensical computations。

 For example， adding a string value in an integer or attempting to use the value of a variable before it has been properly initialized。

The compiler may also provide warnings when operations may not produce the expected results。

 For example， when converting from a floating point number to an integer。

 where the floating point value may be too large to fit into the number of bits provided by the integer。

If a program passes scrutiny， the compiler then proceeds to generate efficient sequences of instructions。

 often finding ways to rearrange the computation so that the resulting sequences are shorter and faster。

It's hard to beat a modern optimizing compiler at producing assembly language。

 since the compiler will patiently explore alternatives and deduce properties of the program that may not be apparent to even diligent assembly language programmers。

We'll focus on the compilation technique for four types of statements。

An unconditional statement is simply an expression that's evaluated once。

A compound statement is simply a sequence of statements to be executed in turn。

Conditional statements， sometimes called if statements， compute the value of a test expression。
