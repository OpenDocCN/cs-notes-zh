# 007：变长编码 📊

![](img/7560caf09fd3003683e51951fa93c91c_1.png)

在本节课中，我们将要学习变长编码。当所有可能选项的信息量不同时，固定长度编码的效率不高。我们将探讨如何通过为不同概率的符号分配不同长度的编码，来获得更短的**期望编码长度**，并了解其工作原理。

![](img/7560caf09fd3003683e51951fa93c91c_2.png)

上一节我们介绍了固定长度编码，它适用于所有选项概率均等的情况。本节中我们来看看当选项出现的概率不同时，如何设计更高效的编码方案。

## 期望编码长度

![](img/7560caf09fd3003683e51951fa93c91c_4.png)

![](img/7560caf09fd3003683e51951fa93c91c_5.png)

如果各个选项的信息量不同，我们可以做得更好。衡量编码效率的一种方法是计算其**期望长度**。期望长度的计算方法是：将每个符号 `x_i` 的编码长度 `L_i`，乘以其出现的概率 `P_i`，然后对所有符号求和。

公式如下：
**期望长度 = Σ (P_i * L_i)**

![](img/7560caf09fd3003683e51951fa93c91c_7.png)

我们的目标是找到一种编码，其期望长度比固定长度编码更短。理想情况下，我们希望编码的期望长度尽可能接近信息源 `X` 的**熵 `H(X)`**，熵代表了信息的平均信息量。

## 变长编码的设计原则

![](img/7560caf09fd3003683e51951fa93c91c_9.png)

我们知道，如果一个符号 `x_i` 出现的概率 `P_i` 较高，其信息量就较小，因此我们希望为其分配较短的编码。

![](img/7560caf09fd3003683e51951fa93c91c_11.png)

反之，如果一个符号 `x_i` 出现的概率 `P_i` 较低，其信息量就较大，我们可以为其分配较长的编码。

![](img/7560caf09fd3003683e51951fa93c91c_13.png)

我们将构建这种编码长度可变的方案，称之为**变长编码**。

## 一个变长编码的例子

以下是一个我们之前见过的例子。有四个待编码的符号 A、B、C、D，每个都有指定的出现概率。

以下是建议的编码方案，它遵循了上一张幻灯片的原则：为信息量小的高概率符号（例如 B）分配短编码，为信息量大的低概率符号（例如 C 或 D）分配长编码。

![](img/7560caf09fd3003683e51951fa93c91c_15.png)

| 符号 | 概率 | 编码 |
| :--- | :--- | :--- |
| A    | 1/4  | 10   |
| B    | 1/2  | 0    |
| C    | 1/12 | 110  |
| D    | 1/12 | 111  |

我们可以将这个编码表示为一棵二叉树。由于所有符号都出现在树的叶节点上，可以证明这个编码是**无歧义的**。

![](img/7560caf09fd3003683e51951fa93c91c_17.png)

让我们尝试解码一段编码数据：`0 10 0 10 111`。

以下是解码步骤：
1.  从树根开始，第一个编码位是 `0`，沿左分支到达叶节点 **B**。所以解码的第一个符号是 B。
2.  再次从树根开始，下一个位 `1` 沿右分支，接着 `0` 沿左分支，再 `0` 沿左分支，到达叶节点 **C**。第二个符号是 C。
3.  继续，`0` 解码为 **B**。
4.  `10` 解码为 **A**。
5.  `111` 解码为 **D**。

因此，整个解码后的消息是：**B, C, B, A, D**。

## 计算编码效率

现在计算这个变长编码的期望长度：
*   A 的编码长度（2 位） × 其概率（1/4） = 0.5
*   B 的编码长度（1 位） × 其概率（1/2） = 0.5
*   C 的编码长度（3 位） × 其概率（1/12）≈ 0.25
*   D 的编码长度（3 位） × 其概率（1/12）≈ 0.25

**期望长度总和 ≈ 1.5 位**

![](img/7560caf09fd3003683e51951fa93c91c_19.png)

我们做得怎么样？如果对这四个符号使用固定长度编码，每个符号需要 2 位。那么编码 1000 个符号需要 **2000 位**。

![](img/7560caf09fd3003683e51951fa93c91c_21.png)

使用我们的变长编码，编码 1000 个符号的期望长度约为 **1500 位**。

![](img/7560caf09fd3003683e51951fa93c91c_22.png)

编码所需位数的理论下限是 1000 乘以信息源 `X` 的熵 `H(X)`，计算后约为 **1626 位**。可见，变长编码让我们更接近理论最优值，但尚未完全达到。

![](img/7560caf09fd3003683e51951fa93c91c_24.png)
