# 011：纠错码 🔧

![](img/e798b2e8bf32dd33c0093ca113686aea_0.png)

在本节课中，我们将要学习如何不仅检测单个比特错误，还能纠正错误以恢复原始数据。我们将探讨通过增加有效码字之间的汉明距离来实现纠错的基本原理。

![](img/e798b2e8bf32dd33c0093ca113686aea_2.png)

## 增加汉明距离以实现纠错 💡

![](img/e798b2e8bf32dd33c0093ca113686aea_4.png)

上一节我们介绍了通过汉明距离来检测错误。本节中我们来看看，如何通过进一步增加汉明距离来实现纠错。

![](img/e798b2e8bf32dd33c0093ca113686aea_5.png)

通过将有效码字之间的最小汉明距离增加到 **3**，我们可以保证由单个比特错误产生的码字集合不会重叠。

## 码字集合与纠错原理 🧩

![](img/e798b2e8bf32dd33c0093ca113686aea_7.png)

![](img/e798b2e8bf32dd33c0093ca113686aea_8.png)

以下是理解纠错的关键：当两个有效码字的汉明距离至少为3时，任何一个码字发生单个比特错误后，变成的新码字都不会与另一个有效码字发生单个比特错误后产生的码字相同。

例如，考虑两个有效码字 **000** 和 **111**（汉明距离为3）：
*   码字 **000** 发生单个比特错误可能变成：**100**、**010** 或 **001**。
*   码字 **111** 发生单个比特错误可能变成：**110**、**101** 或 **011**。

可以看到，这两个集合没有任何共同的码字。

![](img/e798b2e8bf32dd33c0093ca113686aea_10.png)

![](img/e798b2e8bf32dd33c0093ca113686aea_11.png)

## 错误推断与纠正 🛠️

基于上述原理，并假设最多发生一个错误，我们可以从接收到的任何码字推断出原始的码字。

![](img/e798b2e8bf32dd33c0093ca113686aea_13.png)

![](img/e798b2e8bf32dd33c0093ca113686aea_14.png)

例如，如果我们接收到码字 **001**，我们可以推断原始的码字是 **000**，并且发生了一个比特错误。因为 **001** 只出现在 **000** 的错误集合中，而不在 **111** 的错误集合里。

![](img/e798b2e8bf32dd33c0093ca113686aea_16.png)

## 通用纠错条件 📐

我们可以将这一见解推广：如果我们想要纠正最多 **E** 个错误，那么有效码字之间的最小汉明距离必须至少为 **2E + 1**。

用公式表示即：
**最小汉明距离 ≥ 2E + 1**

![](img/e798b2e8bf32dd33c0093ca113686aea_18.png)

![](img/e798b2e8bf32dd33c0093ca113686aea_19.png)

![](img/e798b2e8bf32dd33c0093ca113686aea_20.png)

例如，要纠正单个比特错误（E=1），我们需要有效码字的最小汉明距离为 **3**。
