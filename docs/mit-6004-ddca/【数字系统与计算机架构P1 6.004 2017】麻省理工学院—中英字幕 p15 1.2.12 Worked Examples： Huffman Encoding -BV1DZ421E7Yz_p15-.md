This problem presents Huffman encoding， which produces variable lengthencodings based on the probabilities of the occurrence of each different choice。

 More likely choices will end up with shorter encodings than less likely choices。



![](img/ccb12e8dcee09a26326b5448558e1b78_1.png)

The goal of this problem is to produce a Huffman code to encode student choices of majors。

 There are a total of four majors， and each has a probability associated with it。



![](img/ccb12e8dcee09a26326b5448558e1b78_3.png)

To produce a Huffman en coating， one begins with the two choices with lowest probability。



![](img/ccb12e8dcee09a26326b5448558e1b78_5.png)

In this example， Major 67 has a probability of 0。06， and Major 61 has a probability of 0。09。



![](img/ccb12e8dcee09a26326b5448558e1b78_7.png)

Since these are the two lowest probabilities， these two choices are selected as the starting point for building our encoding tree。

The root node that combines the two choices has a probability equal to the sum of the leaf nodes or 0。

15 in this example。We then label one side of this tree with a0 and the other with a one。

The next step is to find the next two smallest probabilities out of the remaining set of probabilities where majors 6。

![](img/ccb12e8dcee09a26326b5448558e1b78_10.png)

This new node has probability 0。56。Again， we labeled the two branches。

 one with a0 and the other with a one。We now repeat this process one last time。

With the only two remaining choices which are now node B and major 62。



![](img/ccb12e8dcee09a26326b5448558e1b78_12.png)

This means that we should make a new node C that merges node B and major 62。



![](img/ccb12e8dcee09a26326b5448558e1b78_14.png)

Note that the probability of this node is 1。0 because we have reached the top of our tree。

Our final step is to label these last two branches。



![](img/ccb12e8dcee09a26326b5448558e1b78_18.png)

Now that all the branches are labeled， we can traverse the tree from the root node to each leaf node in order to identify the encoding that has been assigned to the major associated with that leaf node。



![](img/ccb12e8dcee09a26326b5448558e1b78_20.png)

These encodings make sense because we expect the major with the highest probability。 In this case。

 So it ends up with a second shortest encoding and so on。

Note that the Huffman encoding tree for this problem could have also been drawn like this。

These two trees are identical in structure and result in the same encodings for the four majors。

Furthermore， a Huffman tree can result in more than one valid encoding。

 The only constraint in labeling the edges is that from each node。

We now add one more column to our table， which gives P times log base2 of1 over P for each of the majors using this information。

![](img/ccb12e8dcee09a26326b5448558e1b78_32.png)

This is calculated by taking the sum of P times log base2 of1 over P for all choices of majors。

 For this problem， the entropy is 1。6。
