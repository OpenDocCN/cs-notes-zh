## 从CMOS电路到逻辑函数

上一节我们介绍了CMOS电路的基本结构，本节中我们来看看如何分析一个给定的CMOS下拉网络，并写出它对应的逻辑函数。

![](img/43376a6108b325c45a0bd099a4501f3c_5.png)

给定如下所示的CMOS下拉网络电路图，我们可以推导出该电路所代表的逻辑函数。

![](img/43376a6108b325c45a0bd099a4501f3c_6.png)

![](img/43376a6108b325c45a0bd099a4501f3c_0.png)

![](img/43376a6108b325c45a0bd099a4501f3c_8.png)

该函数为 **F = NOT( (A OR B) AND (C OR D) )**。

用逻辑表达式表示为：
```
F = ¬((A + B) · (C + D))
```

这个函数可以用真值表来描述，真值表列出了所有可能的输入组合及其对应的输出值。

![](img/43376a6108b325c45a0bd099a4501f3c_10.png)

![](img/43376a6108b325c45a0bd099a4501f3c_11.png)

## 构建真值表

真值表枚举了所有可能的输入组合，并为每一种组合指定了输出F的值。

![](img/43376a6108b325c45a0bd099a4501f3c_13.png)

以下是构建该函数真值表的过程：

![](img/43376a6108b325c45a0bd099a4501f3c_15.png)

![](img/43376a6108b325c45a0bd099a4501f3c_16.png)

*   当输入 **A=0, B=0, C=0, D=0** 时：
    *   (A OR B) = 0， (C OR D) = 0。
    *   (0 AND 0) = 0。
    *   最后取反，得到 **F = 1**。

*   当输入 **A=0, B=0, C=0, D=1** 时：
    *   (A OR B) = 0， (C OR D) = 1。
    *   (0 AND 1) = 0。
    *   最后取反，得到 **F = 1**。

按照同样的方法，我们可以完成整个真值表。

![](img/43376a6108b325c45a0bd099a4501f3c_18.png)

![](img/43376a6108b325c45a0bd099a4501f3c_19.png)

![](img/43376a6108b325c45a0bd099a4501f3c_13.png)

最终得到的输出序列为：**1, 0, 1, 0, 0, 0, 1, 0, 0, 0, 1, 0, 0, 0**。

## 从真值表到逻辑表达式

给定一个真值表，我们可能会被问到：这个真值表定义的函数F能否用单个CMOS门实现？要解决这个问题，我们首先需要将真值表转换为F的逻辑方程。

![](img/43376a6108b325c45a0bd099a4501f3c_18.png)

根据此真值表，函数F可以表示为**积之和**形式，即列出所有使F等于1的项，然后将它们用OR运算连接起来。

![](img/43376a6108b325c45a0bd099a4501f3c_25.png)

![](img/43376a6108b325c45a0bd099a4501f3c_26.png)

对于这个真值表，F的表达式为：
```
F = (¬A·¬B·¬C) + (¬A·B·¬C) + (A·¬B·¬C)
```

![](img/43376a6108b325c45a0bd099a4501f3c_27.png)

![](img/43376a6108b325c45a0bd099a4501f3c_28.png)

## 逻辑化简与CMOS实现判断

![](img/43376a6108b325c45a0bd099a4501f3c_30.png)

为了判断该函数能否作为单个CMOS电路实现，我们需要对函数F进行逻辑化简，目标是看 **NOT F** 能否被写成仅由输入变量（A, B, C）通过AND和OR运算组成的函数，而不包含这些输入变量的取反。

![](img/43376a6108b325c45a0bd099a4501f3c_31.png)

我们从F的表达式开始化简：
```
F = ¬A·¬B·¬C + ¬A·B·¬C + A·¬B·¬C
```

![](img/43376a6108b325c45a0bd099a4501f3c_33.png)

**第一步：提取公因子¬C**
```
F = ¬C · (¬A·¬B + ¬A·B + A·¬B)
```

**第二步：观察并重组括号内的项**
我们可以重写括号内的表达式，并重复第一项：
```
F = ¬C · (¬A·¬B + A·¬B + ¬A·B + ¬A·¬B)
```

![](img/43376a6108b325c45a0bd099a4501f3c_35.png)

**第三步：应用逻辑恒等式**
注意到对于任意变量X，有 `X + ¬X = 1`。但这里我们应用吸收律和合并律。观察发现，`¬A·¬B + A·¬B = ¬B`，`¬A·¬B + ¬A·B = ¬A`。实际上，更直接的化简是：
括号内 `(¬A·¬B + ¬A·B)` 可以提取 `¬A`，得到 `¬A·(¬B+B) = ¬A·1 = ¬A`。
而 `A·¬B` 保持不变。但根据真值表，更精确的化简路径是合并最小项。
最终化简结果为：
```
F = ¬C · (¬A + ¬B)
```

![](img/43376a6108b325c45a0bd099a4501f3c_37.png)

**第四步：求NOT F并应用德摩根定律**
现在，我们对F取反以获得NOT F：
```
NOT F = ¬( ¬C · (¬A + ¬B) )
```
应用德摩根定律：
*   `¬(A · B) = ¬A + ¬B`
*   `¬(A + B) = ¬A · ¬B`
因此：
```
NOT F = ¬(¬C) + ¬(¬A + ¬B) = C + (A · B)
```
所以：
```
NOT F = C + (A · B)
```

现在我们得到了 `NOT F = C OR (A AND B)`。这表明F可以用单个CMOS门实现，因为我们成功地将NOT F表达为一个仅由未取反的输入（A, B, C）通过AND和OR运算组成的函数。

![](img/43376a6108b325c45a0bd099a4501f3c_39.png)

![](img/43376a6108b325c45a0bd099a4501f3c_40.png)

## 绘制对应的CMOS电路

![](img/43376a6108b325c45a0bd099a4501f3c_42.png)

根据化简结果，我们可以绘制出对应的CMOS电路。

*   **下拉网络**：实现 `NOT F` 的函数，即 `C + (A·B)`。这对应着**C**与**（A和B串联）** 两部分并联。
*   **上拉网络**：是下拉网络的对偶（即AND和OR互换，变量取反），应为 **（A和B并联）** 与 **C** 串联。

## 总结

本节课中我们一起学习了CMOS电路分析的完整流程：
1.  **从电路到函数**：根据CMOS下拉网络写出逻辑函数。
2.  **从函数到真值表**：通过枚举输入组合计算输出，构建真值表。
3.  **从真值表到表达式**：使用积之和（SOP）方法，根据真值表写出逻辑表达式。
4.  **逻辑化简与实现判断**：通过代数化简（如提取公因子、应用德摩根定律）将表达式转化为 `NOT F` 的形式，并据此判断能否用单个CMOS门实现以及设计对应的电路。

![](img/43376a6108b325c45a0bd099a4501f3c_48.png)

掌握这一流程对于理解和设计数字逻辑电路至关重要。
