The load and store instructions access main memory note that it's the same main memory that holds the instructions。

 even though for drafting convenience， we show it as two separate boxes in our data path diagram。



![](img/918668b543f324449c0396c3305617e6_2.png)

In the form we show it here， main memory has three ports。

 two read ports for fetching instructions and reading load data。

 and one write port used by the store instruction to write data into main memory。

The address calculation is exactly the same computation as performed by the Add C instruction。

 The contents of the RA register are added to the sign extended 16 bit literal from the low order 16 bits of the instruction。



![](img/918668b543f324449c0396c3305617e6_4.png)

So we'll simply reuse the existing datapath hardware to compute the address。For the load instruction。

 the output of the AOU is routed to the main memory as the address of the location we wish to access。

After the memory's propag delay， the contents of the address location is returned by the memory。

 and we need to route that value back to the register file to be written into the RC register。



![](img/918668b543f324449c0396c3305617e6_6.png)

The memory has two control signals，O memoryory output enable。

 which we set to one when we want to read a value from the memory。



![](img/918668b543f324449c0396c3305617e6_8.png)

And MWE memoryory right enable， which is set to one when we want main memory to store the value on its right data WD port into the addressed memory location。

We need to add another mus to select which value to write back to the register file。

 the output of the LU or the data returning from main memory。



![](img/918668b543f324449c0396c3305617e6_10.png)

We've used a3 to1 mugs and we'll see the use for the otherMX input when we get to the implementation of branches and jumps。



![](img/918668b543f324449c0396c3305617e6_12.png)

The2 bit WD cell signal is used to select the source of the right back value。

Let's follow the flow of data when executing the load instruction。



![](img/918668b543f324449c0396c3305617e6_14.png)

The ALU operas are chosen just as they were are for the ADC instruction。

 and the ALU is requested to perform an ad operation。



![](img/918668b543f324449c0396c3305617e6_16.png)

The AOU result is connected to the address port of main memory。

 whose control signals are set for a read operation。The WD cell control signal。

Is set to two to route the returning data to the register file。

 execution of the store instruction is very similar to the execution of the load instruction with one extra complication。



![](img/918668b543f324449c0396c3305617e6_18.png)

The value to be written to memory comes from the RC register。

 but the RC instruction field is not connected to a Reg file read address。Happily。

 the RB Reg address isn't being used by the store instructions since the second AOU operaant comes from the literal field。

So we'll use a muX to enable the RC field to be selected as the address for the Reg file second Readport。

When the RA2 cell control signal is 0， the RB field is selected as the address。 When RA2 cell is1。

 the RC field is selected。The output from the second Read data port is connected to the right data port of main memory。



![](img/918668b543f324449c0396c3305617e6_20.png)

The store instruction is the only instruction that does not write a result into the register file。

So the Wf control signal will be zero when executing store。

Here is the flow of data when executing store。 The opera ends are selective as for load。

 and the A U performs the address computation with the results sent to main memory as the address。

Meanwhile， the RC field is selected as the address for the second Register file Report。

 and the value from the RC register becomes the right data for main memory。



![](img/918668b543f324449c0396c3305617e6_22.png)

![](img/918668b543f324449c0396c3305617e6_23.png)

By setting the MWR control signal to1， the main memory will write the WD data into the selected memory location at the end of the cycle。



![](img/918668b543f324449c0396c3305617e6_25.png)

The Word control signal is set to zero since we won't be writing a value into the register file。

 and since we're not writing to the register file， we don't care about the value for the WD cell signal。

Of course， the logic will need to supply some value for WD cell。

 The don't care annotation is telling the logic designer that she's welcome to supply whatever value is most convenient。

This is particularly useful when using kernel maps to optimize the control logic。

 where the value can be chosen as either 0 or 1， whichever results in the best minimization of the logic equations。
