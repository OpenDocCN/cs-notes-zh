The problem with our simple multico system is that there is no communication when the value of a shared variable is changed。

The fix is to provide the necessary communications over a shared bus this watched by all the caches。

A cache can then snoop on what's happening in other caches and then update its local state to be consistent。

The required communications protocol is called a Ca coherence Pro。In designing the protocol。

 we'd like to incur the communications overhead only when there's actual sharing and progress。

 in other words， when multiple caches have local copies of a shared variable。

When the cache lines data is exclusive， this cache has the only copy of those memory locations and indicates that the local data is the same as that in main memory。

To deal with sharing issues， there's a fourth state called Sha that indicates when other caches may also have a copy of the same unmodified memory data。

When filling a cache from main memory， other caches can snoop on their read request and participate in fulfilling their read request。

If some other cache has the requested cash line in the exclusive or shared state。

The key to success is that each cache now knows when a cache line may be shared by another cache。

No attempt is made to update shared values， they are simply invalidated and the other caches will issue reuquests if they need the value of the shared variable at some future time。
