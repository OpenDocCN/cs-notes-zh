## 数据加密的核心概念

上一节我们介绍了加密数据写入的目标，本节中我们来看看数据加密的几个核心状态。

![](img/83a54578c6d30dd768c629fcffadb293_1.png)

数据加密主要涉及三种状态：静态数据、传输中数据和使用中数据。

![](img/83a54578c6d30dd768c629fcffadb293_3.png)

**静态数据**是一个需要了解的重要概念。它意味着数据在已经存在于文件系统或数据库中时进行加密。

**传输中数据**是指在私有或公共通信通道上进行加密。这确保了数据在移动过程中仍然是安全的。

**使用中数据**意味着数据在仍处于内存或CPU缓存中时进行加密，这样即使有人能够访问该机器，也无法拦截数据。

## 列级加密与始终加密

现在，让我们深入看看数据库列级别的加密。

![](img/83a54578c6d30dd768c629fcffadb293_7.png)

您可以指定希望加密特定的列。当启用“始终加密”时，它可以设计存储敏感信息。在SQL Server数据库中，它使客户端能够在客户端应用程序内部加密数据，并且永远不会通过密钥向SQL Server泄露。
