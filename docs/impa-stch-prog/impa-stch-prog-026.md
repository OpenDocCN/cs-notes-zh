## 近端束方法回顾 🔄

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_8.png)

上一节我们介绍了近端束方法的基本框架，本节中我们来看看其核心组成部分。我们感兴趣的问题是**最小化一个凸函数**。

**问题定义：**
```
min f(x)
s.t. x ∈ X
```
其中，`f` 是一个凸函数，`X` 是一个凸集（例如整个 R^n）。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_10.png)

近端束方法的关键组成部分之一是**模型** `f_M^k`，它是多个线性化函数的最大值。这些线性化函数来自一个信息束，其索引集包含在 `{1, 2, ...}` 中。此外，束中还包含一个**聚合线性化** `f_a^k`。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_12.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_14.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_16.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_18.png)

**近端子问题：**
迭代点 `x^{k+1}` 是以下二次规划（QP）问题的唯一解：
```
min f_M^k(x) + (1/(2t_k)) * ||x - x̂^k||^2
s.t. x ∈ X
```
其中，`x̂^k` 是当前的稳定性中心。

如果我们用原函数 `f` 替换模型 `f_M^k`，那么上述问题就变成了标准的近端方法。但在随机优化中，函数 `f`（例如两阶段或多阶段问题）可能难以精确评估，因此我们用简单的模型来近似它，这正是“近端”一词的由来。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_20.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_22.png)

我们还有**预测下降量** `δ_k`，定义为 `f(x̂^k) - f_M^k(x^{k+1})`。利用聚合线性化，可以证明预测下降量可以写成特定形式，并且满足一个重要的不等式。

**重要不等式：**
对于所有可行的 `x`，有：
```
f(x) ≥ f_M^k(x) ≥ f_a^k(x) = f(x̂^k) + <g_a^k, x - x̂^k> - e_a^k
```
其中，`g_a^k` 是聚合次梯度，`e_a^k` 是聚合误差。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_24.png)

我们的目标是生成一个点列，使得聚合误差 `e_a^k` 和方向范数 `||x^{k+1} - x̂^k||` 都趋于零。如果这两者都为零，那么稳定性中心 `x̂^k` 就是最优解。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_26.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_27.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_29.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_31.png)

**算法流程简述：**
1.  从初始点开始，调用Oracle（黑箱）获取函数值和次梯度信息。
2.  构建或更新模型。
3.  求解上述QP子问题，得到候选点 `x^{k+1}`。
4.  计算预测下降量。
5.  进行**严肃步检验**：如果函数值有足够下降，则更新稳定性中心（严肃步）；否则，保持稳定性中心不变（零步）。
6.  丰富模型信息（添加新的线性化），并循环。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_33.png)

## 收敛性分析 📈

对于收敛性分析，我们需要考虑两种互斥的情况。第一种情况是算法生成了**无限多个严肃步**，即稳定性中心序列 `{x̂^k}` 是无限的。第二种情况是严肃步的数量**有限**。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_35.png)

我们将依赖于以下假设：函数 `f` 下有界，并且近端参数 `t_k` 远离零（有正下界）。

### 情况一：无限多个严肃步

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_37.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_39.png)

假设算法生成了无限多个严肃步，我们希望证明聚合误差和方向范数趋于零。

根据严肃步的判定规则，当第 `k` 次迭代被声明为严肃步时，满足：
```
f(x̂^{k+1}) ≤ f(x̂^k) - κ * δ_k
```
其中，`κ` 是介于0和1之间的参数，`δ_k` 是预测下降量。

定义严肃步的索引集合 `K_s`。由于严肃步无限，且函数 `f` 下有界，通过对上述不等式从 `k=0` 到 `∞` 求和，可以推导出当 `k ∈ K_s` 且 `k → ∞` 时，有 `δ_k → 0`。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_41.png)

进一步，利用预测下降量的定义和QP问题的最优性条件，可以证明当 `δ_k → 0` 时，有 `e_a^k → 0` 且 `||x^{k+1} - x̂^k|| → 0`。这就完成了第一种情况的证明。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_43.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_45.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_47.png)

### 情况二：有限个严肃步

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_49.png)

第二种情况的证明更为复杂。其核心依赖于一个关于**束管理**的技术性引理。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_51.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_53.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_55.png)

**关键引理（束管理）：**
如果我们选择的束信息（即用于构建模型的线性化集合）满足一定条件，特别是确保新的线性化和聚合线性化都在束中，那么我们可以证明，对于足够大的迭代次数 `k`，聚合误差满足：
```
e_a^k ≥ (1/(2t_k)) * ||x^{k+1} - x̂^k||^2
```

现在，假设严肃步只发生有限次。那么存在一个索引 `K_bar`，使得对于所有 `k ≥ K_bar`，稳定性中心 `x̂^k` 固定不变（即为 `x̂^{K_bar}`）。

我们从这个引理出发。对于足够大的 `k`，利用该引理和模型的定义，经过一系列推导（类似于情况一），最终可以证明，对于零步的索引集合 `K_n`，当 `k ∈ K_n` 且 `k → ∞` 时，同样有 `e_a^k → 0` 和 `||x^{k+1} - x̂^k|| → 0`。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_57.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_59.png)

### 主要收敛定理

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_61.png)

综合以上两种情况，我们得到近端束方法的主要收敛定理。

**定理：**
假设函数 `f` 下有界，近端参数 `t_k` 有正下界。那么：
1.  存在一个索引集合 `K`，使得当 `k ∈ K` 且 `k → ∞` 时，有 `e_a^k → 0` 且 `||x^{k+1} - x̂^k|| → 0`。
2.  函数值序列 `{f(x̂^k)}` 收敛到 `f` 在 `X` 上的下确界 `f^*`。
3.  稳定性中心序列 `{x̂^k}` 的任何聚点（如果存在）都是原优化问题的最优解。

如果进一步假设函数 `f` 是强制的（例如可行域有界），那么整个序列 `{x̂^k}` 会收敛到一个最优解。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_63.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_65.png)

## 水平束方法介绍 🔽

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_67.png)

现在，我们转向另一种束方法变体——**水平束方法**。该方法由 Lemaréchal, Nemirovskii 和 Nesterov 在1995年提出。

水平束方法与近端束方法密切相关。主要区别在于，它将模型移到了约束中，并使用一个**水平参数**来代替近端参数。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_69.png)

**水平子问题：**
迭代点 `x^{k+1}` 是以下投影问题的解：
```
min (1/2) * ||x - x̂^k||^2
s.t. f_M^k(x) ≤ l_k
      x ∈ X
```
其中，`l_k` 是水平参数。这可以理解为将稳定性中心 `x̂^k` 投影到模型 `f_M^k` 的水平集 `{x ∈ X: f_M^k(x) ≤ l_k}` 上。

**两种方法的联系：**
事实上，通过拉格朗日对偶，可以证明在一定条件下，近端子问题的解也是某个特定水平参数下的水平子问题的解，反之亦然。参数 `t_k` 和 `l_k` 通过最优拉格朗日乘子相联系。但在实际中，两种方法的行为有所不同。

水平束方法的一个显著特点是，**稳定性中心的选择具有更大的灵活性**。它不必是当前已知的最好点，可以是初始点或前一个迭代点。稳定性通过水平集约束来实现，该约束确保了新迭代点不会偏离“好”的函数值区域太远。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_71.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_73.png)

### 水平参数更新

水平参数 `l_k` 的更新至关重要。一个常见的策略是将其设置为当前最优值上界 `f_k^{up}` 和下界 `f_k^{low}` 的凸组合：
```
l_k = f_k^{low} + γ * (f_k^{up} - f_k^{low})， 其中 γ ∈ (0, 1)
```
这里，`f_k^{up}` 通常是当前最好函数值 `f(x̂^k)`，`f_k^{low}` 是目标函数最优值的一个下界估计。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_75.png)

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_77.png)

**下界估计的获取：**
有两种主要方式：
1.  **积极更新**：在每次迭代时，通过求解一个线性规划（LP）来最小化当前模型，从而得到一个下界 `f_k^{low} = min_{x∈X} f_M^k(x)`。这能提供更紧的下界，可能减少迭代次数，但每次迭代需要多解一个LP。
2.  **惰性更新**：保持一个固定的下界估计不变。当尝试求解水平子问题（投影）时，如果发现该水平集是空的（即 `{x ∈ X: f_M^k(x) ≤ l_k} = ∅`），则意味着当前的水平参数 `l_k` 已经低于模型所能达到的最小值，从而也低于真实函数的最优值。此时，我们可以将 `l_k` 设置为这个模型最小值（通过求解一次LP获得），并更新下界。这种方法在Oracle调用非常耗时，而求解LP相对便宜时比较有效。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_79.png)

### 算法与性质

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_81.png)

**水平束算法框架（简化版）：**
1.  **初始化**：给定初始点 `x̂^0`，获取初始上界 `f_0^{up}=f(x̂^0)`，计算或给定初始下界 `f_0^{low}`，设定参数 `γ ∈ (0,1)`，容忍度 `ε > 0`。
2.  **迭代**：
    a. 计算当前间隙 `gap = f_k^{up} - f_k^{low}`。若 `gap ≤ ε`，则停止。
    b. 设置水平参数 `l_k = f_k^{low} + γ * gap`。
    c. **尝试**求解投影子问题（水平子问题），得到候选点 `x^{k+1}`。
        - 若问题不可行（水平集为空），则按上述“惰性更新”方式更新 `f_k^{low}` 和 `l_k`，然后返回步骤c重试。
    d. 调用Oracle计算 `f(x^{k+1})` 和次梯度。
    e. 更新上界：`f_{k+1}^{up} = min{f_k^{up}, f(x^{k+1})}`。
    f. 更新模型（束管理），添加新的线性化。
    g. （可选）按一定规则更新稳定性中心 `x̂^{k+1}`（例如，使用类似严肃步的规则，或当最优间隙减少到一定比例时更新）。
    h. 令 `k = k+1`，返回步骤a。

**一个重要性质：**
与近端束方法类似，水平束方法中产生的**聚合线性化** `f_a^k` 包含了模型的关键信息。可以证明，如果只使用这个聚合线性化（而丢弃束中所有其他线性化）来定义水平集并做投影，得到的点与使用完整模型得到的点 `x^{k+1}` 是相同的。这为束管理（删除旧线性化）提供了理论依据。

### 收敛性直观与优势

水平束方法的收敛性分析通常通过将迭代划分为**循环**来进行。一个循环是指连续多次迭代中稳定性中心保持不变的阶段。

可以证明，如果使用特定的稳定性中心更新规则（例如，当最优间隙减少到初始间隙的 `δ^l` 倍时更新，其中 `δ ∈ (0,1)`），那么算法会生成无限多个这样的循环，从而驱动最优间隙趋于零。

**实践优势：**
当应用于两阶段随机规划问题时，水平束方法被称为**水平分解法**。与传统的L形方法（割平面法）相比，水平束方法通常能显著加快收敛速度（有文献报告可达70%-80%）。其关键在于通过水平集约束提供的稳定性，以及可以灵活控制Oracle调用的次数（例如，只在必要时精确计算函数值）。

## 总结 📝

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_83.png)

本节课中我们一起学习了近端束方法的收敛性分析，并介绍了水平束方法。

![](img/d6d67e06dec3bfe2707ec81ed3f6db77_85.png)

*   我们首先回顾了近端束方法，分析了其在生成无限多严肃步和有限严肃步两种情况下的收敛性，证明了关键的最优性度量会趋于零。
*   接着，我们引入了水平束方法，它通过将模型作为水平集约束，并使用投影来产生新迭代点。我们讨论了水平参数的更新策略、算法的基本流程，以及该方法在理论和实践上的特点，特别是其灵活性、与近端方法的联系，以及在随机规划中应用时的效率优势。

水平束方法为求解复杂的凸优化问题，特别是那些函数求值昂贵的随机规划问题，提供了一个强大而高效的框架。
