# 操作系统原理：01：程序与进程的区别 🔄

在本节课中，我们将要学习计算机科学中两个核心概念——“程序”与“进程”的区别。我们将从历史背景入手，逐步剖析两者的定义、内存布局以及在不同编程语言环境下的表现，帮助你清晰地理解为何一个“程序”在运行时被称为“进程”。

---

## 概述

计算机科学中，一个常见的问题是：如何称呼CPU执行的所有活动？早期的批处理系统执行的工作单元被称为“作业”。随着分时操作系统的出现，设计目标变为在多个用户间共享CPU，因此人们说这些系统运行的是“用户程序”。最终，家用计算机市场普及了并发技术，允许单个用户同时运行多个程序。然而，在计算机科学领域，我们并不将这些执行实体称为“程序”，而是称之为“进程”。本节课将详细探讨这两个概念的区别。

---

## 程序是什么？📁

上一节我们介绍了课程目标，本节中我们来看看“程序”到底是什么。

非正式地说，进程通常被定义为一个“正在执行的程序”。虽然这个定义在技术上是正确的，但它不足以帮助初学者理解进程的本质。因此，我们首先需要明确“进程不是什么”。

![](img/dad846e8b48d900d0ed6f1e891e36a71_1.png)

不同的程序显然有不同的用途，但在执行时，它们在许多方面是相似的，因为它们最终都归结为CPU执行的基本操作。从这里，我们可以开始区分程序与进程。

![](img/dad846e8b48d900d0ed6f1e891e36a71_3.png)

![](img/dad846e8b48d900d0ed6f1e891e36a71_4.png)

**程序**是CPU为完成特定任务所需的一系列指令和数据的集合。仔细分析这个描述，你会发现它同样可以描述一个可执行文件，因为程序本质上就是如此。

> 程序是存储在你设备中的被动实体，等待你点击它来通知计算机开始运行。

---

## 从程序到进程：内存视角 🧠

上一节我们定义了程序，本节中我们来看看程序是如何转变为进程的。

正如过去多次提到的，要运行一个程序，首先需要将其加载到内存中，因为CPU是从内存中获取指令和数据的。当程序被加载到内存后，包含可执行代码的部分被称为**文本段**，而像全局变量和常量值这样的数据则被加载到**数据段**。

![](img/dad846e8b48d900d0ed6f1e891e36a71_6.png)

然而，运行程序仅靠这两个段是不够的，它还需要额外的内存空间来存储运行时产生的所有数据，例如用户输入、临时结果或变量。

我们对**栈**和**堆**已经有很多了解，例如它们的大小会动态增长和收缩。文本段是唯一一个在大小和内容上永不改变的部分。数据段虽然大小不变，但其内容可能会根据运行程序的操作而变化。

![](img/dad846e8b48d900d0ed6f1e891e36a71_8.png)

![](img/dad846e8b48d900d0ed6f1e891e36a71_9.png)

可以看到，尽管最初我们将程序加载到内存中，但在运行时，这个内存布局已不能再被视为一个“程序”。它现在变成了一个**进程**。

![](img/dad846e8b48d900d0ed6f1e891e36a71_11.png)

请注意，这只是**进程的内存布局**，而不是进程本身。

> 进程的内存布局 = **文本段**（代码） + **数据段**（全局/静态数据） + **堆**（动态内存） + **栈**（局部变量/函数调用）

## 理解进程：一个生动的例子 🖥️

![](img/dad846e8b48d900d0ed6f1e891e36a71_13.png)

上一节我们从内存角度看到了进程的构成，本节中我们通过一个具体例子来理解进程的独立性。

![](img/dad846e8b48d900d0ed6f1e891e36a71_15.png)

考虑操作系统自带的那个用于打开文本文件的小程序。现在想象我们有两个文件需要阅读。如果我们同时打开这两个文件，操作系统会启动两次文本编辑器应用程序，这会产生两个独立的窗口，每个窗口显示不同的文本文件。

![](img/dad846e8b48d900d0ed6f1e891e36a71_17.png)

两者是同一个程序，但却是两个完全不同的进程。每个进程都有自己独立的内存空间。虽然两个进程内存中文本段的内容是相同的（因为是同一个程序），但它们正在处理的数据是不同的。

在这个例子中，第一个进程处理的文本文件比第二个大得多，因此它自然需要更多内存来管理其数据。

## 编译型语言与解释型语言的差异 ⚙️

![](img/dad846e8b48d900d0ed6f1e891e36a71_19.png)

上一节我们通过例子理解了进程的独立性，本节中我们来看看在不同编程语言环境下，程序与进程的关系有何不同。

![](img/dad846e8b48d900d0ed6f1e891e36a71_20.png)

到目前为止，我们所讨论的一切对于编译型语言（如C或Rust）是成立的。编译的结果是一个可执行文件，那就是程序本身。

![](img/dad846e8b48d900d0ed6f1e891e36a71_22.png)

但对于解释型语言（如Python或JavaScript）呢？那里没有编译过程。例如在Python中，每次我们想运行代码，都必须先指示计算机运行解释器，然后告诉解释器执行我们的代码。

换句话说，这里涉及两个被动实体：
1.  Python解释器（它是一个程序）。
2.  我们的Python文件（它本质上是一个文本文件）。

这里需要强调一个事实：**Python源文件本身不是一个程序**。请记住，源代码只是文本，计算机无法理解，因此无法直接执行。

所以在这种情境下，我们实际要求计算机运行的程序是**解释器**，而不是我们编写的Python代码。一旦这个新进程被创建，它就像任何其他进程一样，拥有自己的内存区域，包括文本段、数据段、栈和堆。

这里的关键点是：加载到文本段中的不是我们的Python源代码，而是**解释器的可执行代码**。我们的源代码被解释器进程加载到其某个数据段（很可能是堆）中，作为解释器将要读取和解释的**数据**来处理。

> 对于解释型语言：**进程** = 解释器程序（在文本段） + 用户源代码（作为数据在堆/数据段）

---

## 总结

本节课中我们一起学习了“程序”与“进程”的核心区别。

*   **程序**是一个被动的、静态的实体，是存储在磁盘上的指令和数据的集合，通常表现为一个可执行文件。
*   **进程**是一个主动的、动态的实体，是程序被加载到内存并执行时的实例。它拥有独立的内存空间（文本段、数据段、堆、栈）。
*   同一个程序可以同时对应多个进程（例如打开多个记事本窗口），每个进程独立运行，处理不同的数据。
*   在编译型语言中，程序直接对应可执行文件；在解释型语言中，被执行的“程序”是解释器本身，用户的源代码是解释器进程处理的数据。

![](img/dad846e8b48d900d0ed6f1e891e36a71_24.png)

理解程序与进程的区别是学习操作系统、并发编程和系统资源管理的基础。记住：**程序是菜谱，进程是照着菜谱做饭的整个过程**。
