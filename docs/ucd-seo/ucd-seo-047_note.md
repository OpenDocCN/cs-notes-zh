## 概述

大多数人在浏览互联网时都遇到过错误代码。事实上，所有人都可能遭遇过令人头疼的“404文件未找到”错误。在我们继续探索技术性SEO的过程中，下一个主题就是这些恼人的错误代码。这些错误不仅让网站访问者感到沮丧，也会对SEO产生深远影响。本节课程将检视用户在浏览网页时可能遇到的一些常见错误代码，并描述这些错误如何影响SEO。

![](img/93ee88f1cf0f424853555e50445ff087_3.png)

我们都经历过这样的问题：在浏览网页时点击一个链接加载页面，结果却发现页面是空白的或显示一个错误。这不仅给用户带来糟糕的体验，也可能引发各种SEO问题。

![](img/93ee88f1cf0f424853555e50445ff087_5.png)

接下来，让我们讨论在分析网站时可能遇到的一些常见错误，以及这些错误对SEO意味着什么。

## HTTP状态码简介

每个错误都有一个特定的状态码与之关联。这有助于搜索引擎和开发人员理解尝试加载页面时出了什么问题。这些状态码被称为HTTP（超文本传输协议）响应状态码。

![](img/93ee88f1cf0f424853555e50445ff087_7.png)

当搜索引擎或用户请求查看Web服务器上的特定页面时，会返回这些代码。如果Web服务器无法返回该页面，它将返回一个三位数的代码作为响应。

![](img/93ee88f1cf0f424853555e50445ff087_9.png)

主要有两类代码：以数字4开头的代码和以数字5开头的代码。

以下是您应该了解的一些常见状态码及其对SEO的影响。

## 常见状态码解析

![](img/93ee88f1cf0f424853555e50445ff087_11.png)

### 状态码 200

状态码 **`200`** 表示“成功”。这意味着页面已正确加载。

### 状态码 404

状态码 **`404`** 是您可能熟悉的代码。当页面无法找到时返回此代码。这意味着页面已从服务器上移除，不再存在，因此返回404代替。404页面很常见。

当出现404代码时，搜索引擎会识别出这些页面已不可用，并知道不应将该页面纳入其索引中。

然而，一个需要注意的常见技术问题是处理不当的404页面，也称为“软404”。这意味着页面已不存在，但仍返回状态码200或“成功”。这可能会给您的网站带来各种问题。

当网站存在大量软404页面时尤其如此，因为这些页面被视为活动页面。这种情况可能发生在页面内容被移除，但页面本身仍保留在原处时。在电子商务网站（产品已下架）或房地产网站（房源已不可用）等场景中经常看到此类问题。

![](img/93ee88f1cf0f424853555e50445ff087_13.png)

这种类型的问题对SEO非常不利，因为搜索引擎现在看到的是一个没有信息或只有简单错误信息（如“此产品已不再可用”）的页面。在这种情况下，对于每个不再可用的产品，网站都会包含多个显示相同错误信息的页面。由于这些页面都返回状态码200而非404错误，搜索引擎会将其视为重复内容，这可能损害您的网站。即使错误信息略有不同，页面上的内容也被认为非常单薄，对网站或用户毫无帮助。

这类错误可能更难发现，因为如果您访问该网站，它会返回200状态码。在分析网站时，您需要对此类错误保持警惕。

![](img/93ee88f1cf0f424853555e50445ff087_15.png)

识别这些错误代码的一种方法是通过Google Search Console（谷歌搜索控制台），它会通知您任何它识别为可能是软404页面的页面。如果您正在处理一个容易出现软404的网站（如房地产或电子商务网站），最好询问网站所有者他们如何处理诸如缺货产品之类的页面。您也可以在Google Search Console的网站管理员工具中查看返回404错误代码的页面列表。

### 状态码 500

另一个常见的状态码是 **`500`**，表示“服务器错误”。这是一个更通用的错误，发生在尝试返回页面时出现问题，但服务器无法更具体地说明导致错误的确切原因。很多时候，这些错误是暂时的，会自行解决。

![](img/93ee88f1cf0f424853555e50445ff087_17.png)

您会在Google Search Console中看到这些错误的列表。当您检查时，它们可能已经被纠正。在这种情况下，只需标记为“已纠正”并继续。如果此错误持续出现，网站开发人员可能需要查看服务器日志，以尝试确定问题所在并进行纠正。

![](img/93ee88f1cf0f424853555e50445ff087_19.png)

### 状态码 503

状态码 **`503`** 表示“服务不可用”。当服务器宕机、您正在进行维护或服务器暂时过载时，应呈现此代码。这可以让搜索引擎知道存在问题，但问题是暂时的，它们应该稍后重新访问。无论出于何种原因使服务器下线时，确保返回503代码而非500代码非常重要。

![](img/93ee88f1cf0f424853555e50445ff087_21.png)
