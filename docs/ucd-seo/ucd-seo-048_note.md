# 搜索引擎优化：048：重定向基础 🔄

在本节课中，我们将学习重定向的基础知识。重定向是技术性SEO的重要组成部分，它能在网页被移除或替换时，将用户和搜索引擎引导至新的正确页面。我们将介绍几种主要的重定向类型，讨论它们对SEO的影响，并学习如何在特定场景下恰当地使用它们。

## 重定向简介

在之前的课程中，我们讨论了各种HTTP状态码。状态码的另一大类就是重定向。

有时，网站管理员会决定从网站上移除某个页面，或用更新、更完善的页面来替换它。当这种情况发生时，最好能将访问旧页面的用户引导至新的页面。实现这一目标的一个简便方法就是通过重定向，它可以自动将访问者转移到新页面。大多数时候，用户甚至不会察觉到重定向的发生。

然而，当一个页面被重定向时，会呈现一个HTTP状态码。这个代码向搜索引擎提供了关于应如何处理该页面的指令。

## 主要的重定向类型

以下是SEO需要熟悉的主要重定向类型。

*   **永久重定向**：即 **301 HTTP状态码**。这告诉搜索引擎原页面已不存在，它重定向到的页面是其永久替代品。这意味着搜索引擎最终会将原页面积累的信任度（权威性）传递给新页面，有助于新页面在搜索结果中获得与原页面相近的排名。
*   **临时重定向**：即 **302 HTTP状态码**。302的一个较新版本是307重定向，但你通常仍会看到302或301。临时重定向告诉搜索引擎，此次重定向只是临时性的，不应将任何信任度传递给重定向后的版本。在某些情况下，临时重定向是最佳选择，例如当页面只是暂时被重定向到另一个页面时（如进行网站维护）。不过，这类情况通常很少见，因此在SEO中，我们通常总是推荐使用301重定向。
*   **Meta Refresh**：这是另一种很少使用但你应该了解的重定向类型。与前述重定向不同，Meta Refresh不会呈现HTTP状态码，因为重定向是在页面级别执行的，而非由服务器本身完成。这类重定向通常速度较慢，用户也更有可能察觉到他们被重定向了（例如看到“如果您在5秒内未被重定向，请点击这里”的提示）。通常不推荐使用Meta Refresh，因为它无法向搜索引擎清晰地传递原页面发生了什么的信号，并且可能造成较差的用户体验。

## 重定向与页面权威性

推荐使用301重定向而非302重定向的一个关键原因是，永久重定向会将所谓的“链接权重”或“页面权威性”从第一个页面传递给第二个页面。这意味着指向旧页面的链接所传递的权威性会转移给新页面。

但需要注意的是，虽然重定向传递了大部分权威性，但仍会有一部分损失。据估计，重定向时大约会损失页面总权威性的 **5%**。虽然这个比例很小，但在建议使用重定向时，你应该意识到这一点。

例如，如果你移除或重定向了网站内的一个页面，最佳实践是更新网站内指向该页面的链接，使其直接指向新页面，而不是继续链接到已被重定向的旧页面。请注意，重定向仍然是必要的，以确保任何外部链接仍能被正确引导，但内部链接应始终进行相应更新。

另一方面，临时重定向（302）几乎不传递或完全不传递权威性，并且会使旧页面保留在索引中。这也是为什么302重定向对SEO不利的原因之一。

## 重定向的最佳实践

由于重定向可能导致权威性损失，你还需要注意避免“重定向链”。这是指你可能在过去将页面A重定向到页面B，后来又决定页面B不再有效，于是将页面B重定向到页面C。在这种情况下，最好的做法是将页面A和页面B都直接重定向到页面C，而不是在中间形成重定向链。因为每次重定向都会损失一些权威性，避免链式重定向可以保留更多的权威性。这在经历过多次重新设计的旧网站中很常见，是在审核网站时需要留意的地方。

当将旧页面重定向到新页面时，最好将旧页面重定向到与旧页面内容和主题最匹配的页面上。许多人倾向于简单地将所有旧页面重定向到主页，但从搜索引擎的角度来看，这缺乏语义相关性。此外，这也会导致糟糕的用户体验，因为用户原本期望访问某个特定主题的页面，却被送到了主页，且没有任何下一步的提示。

只有在没有更合适的页面可以重定向时，才建议重定向到主页。通常，优先级的顺序是：
1.  重定向到特定的相关页面。
2.  如果不存在特定页面，则重定向到类别页面。
3.  如果类别页面也不存在，那么才考虑重定向到主页。

例如，假设我们有一个关于礼品篮的网站，其中曾有一个产品页面是“带有鲜花的早日康复泰迪熊”。如果这个产品不再提供，理想的重定向方式是：
1.  首先寻找另一个类似的产品，例如“带有气球的早日康复泰迪熊”。
2.  如果没有匹配的产品，则次优选择是将页面重定向到“早日康复礼品”类别页。
3.  如果网站完全移除了所有早日康复礼品，并且不再有相关的类别页面，那么最后的选择才是主页。
