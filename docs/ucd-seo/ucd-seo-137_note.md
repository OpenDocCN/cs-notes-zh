# SEO基础：137：执行内容审核与技术审查

在本节课中，我们将学习如何通过分析竞争对手和自身网站的内容来优化SEO策略。我们将分步进行内容审核，并检查可能影响网站排名的技术因素。

上一节我们基于竞争分析确定了目标关键词和主要竞争对手。本节中，我们来看看如何分析竞争对手的内容类型与质量。

## 分析竞争对手的内容

你制作的内容类型和质量对你的SEO活动成功至关重要。评估竞争对手的内容表现，将帮助你确定哪种内容类型可能最受你的受众欢迎，了解现有内容格局，思考如何改进，并发现尚未被利用的内容机会。

以下是分析时需要关注的几个方面。但请注意，不必局限于这些领域。如果在浏览网站时获得了绝佳的内容创意，请务必记录下来以备将来参考。在课程中，我们为此使用了一个模板，建议你也使用它。该模板包含了你应查看和评估的网站各个区域的记录项。

记录某些信息很重要，以便日后需要时可以轻松查阅。需要跟踪的信息包括：
*   **页面URL**：记录你正在查看的页面的网址。
*   **内容类型**：注明你正在查看的内容类型。例如，这是博客页面、美学文章，还是PDF等其他形式。
*   **内容描述**：简要描述内容是什么。例如，这只是一个信息页面，还是针对长尾关键词的“如何做”文章、信息图或其他内容。
*   **备注**：备注部分是记录任何不适合放在其他栏目的特定笔记的好地方。例如，如果你注意到正在查看的页面与竞争对手网站上的另一个页面非常相似，这值得记录下来，因为它可能暗示着重复内容的问题。
*   **社交参与度**：建议花时间查看相关指标，例如人们在社交媒体上分享和参与此页面的程度。这有助于你确定针对所制作的特定内容类型应使用哪些平台，以及你的受众可能对此类内容的参与度如何。

现在，我们对竞争对手生产的内容类型以及受众对该内容的反应有了清晰的了解。下一步是查看我们自己网站上的现有内容，并确定如何改进这些内容。

## 执行内部内容审核

这让你能够看到你制作的内容类型与竞争对手制作的内容类型是否一致。完成上一部分和本部分后，你应该对如何改进内容以提升SEO价值和用户参与度，以及可以制作哪些新内容有了清晰的思路。

此外，执行内部内容审核可能会让你发现机会，将某些内容合并成更有价值的SEO资源，或者识别出过时、应从网站中删除的内容。

对于这部分，建议至少查看所选网站内的10个页面。从顶级页面开始，例如主导航项链接的页面，或你认为应该排名良好的重要产品页面。同时查看网站的“关于我们”页面。你的客户是否真正在主题上提供了真实感和权威感？如果你的网站此类页面数量有限，可以转向其他资源，例如博客。

你需要了解正在审核的内容类型。需要评估的方面包括：
*   **内容形式**：此页面是基于文本、基于图像、是可下载资源还是其他形式？
*   **季节性**：内容是否具有季节性（如果有的话）？
*   **可改进性**：内容是否可以通过添加图像或其他资源来改进？
*   **目标受众匹配度**：此内容与目标受众的匹配程度如何？例如，它是直接使用具有商业意图的关键词来定位他们，还是使用更长的长尾关键词来回答他们可能有的问题？
*   **行动号召**：还要查看内容是否包含行动号召。这虽然不影响SEO，但我们不仅希望吸引流量，还希望帮助确保流量有最佳的转化机会。
*   **其他备注**：包含你可能对内容的其他任何备注，例如标题标签是否需要改进，或标题标签是否应包含焦点关键词。

请记住，对于这部分或一次简短的SEO审核，我们只查看最少量的页面。但如果你是在公司内部工作或签订了长期的SEO合同，保持一份持续更新的内部内容文档非常重要。根据你网站的规模，这可能是一项需要每月分块进行以实现渐进式更改的工作。

完成内容审核和建议的下一步是创建关键词矩阵。

## 创建关键词矩阵

关键词矩阵是一个大纲，概述了应将哪些关键词分配到网站的特定页面。

我们在完成关键词矩阵之前对竞争对手内容和自身内容进行彻底分析的原因是，这能为我们可能想要创建的新内容提供思路。这些新内容仍然需要关键词焦点，以确保其具有SEO价值且不与网站上的其他页面竞争。此外，在我们的分析过程中，我们可能发现了内容合并的机会，或者你决定完全删除某个页面。在这些情况下，应在关键词矩阵中注明，以便我们只将关键词分配给我们建议保留或建议创建的页面。

对于这部分，请包含10个当前页面，注明你是否合并了现有页面，并包含对5个新页面的建议。对于每个页面，提供你对**主要关键词**以及任何**次要关键词**的建议。除了关键词，请务必记下该关键词的**搜索量**和**当前排名**。

好的，本里程碑的最后一部分，你将查看可能对网站在搜索中良好排名产生负面影响的**技术因素**。

## 进行技术审查

虽然内容和关键词对任何SEO策略都很重要，但确保搜索引擎机器人能够有效地抓取网站（错误有限或没有错误）并读取网站内的内容也同样重要。这是一个你可以执行的基本技术审查，以发现任何危险信号。

为了分析这些基本领域，请查看网站上的 `robots.txt` 文件。以下是具体步骤：
*   **检查 `robots.txt`**：查看网站上的 `robots.txt` 文件。如果不存在，请撰写建议说明为何应创建该文件，并包含你建议机器人不要抓取的任何页面。解释你提出此建议的原因，因为客户很可能会询问为什么这些页面不应被抓取。如果 `robots.txt` 文件确实存在，请查看该文件并将其与网站上的页面进行比较。你有什么建议的更改吗？请提供你的理由。
*   **检查网站错误**：除了检查 `robots.txt` 文件，还要查看网站是否遇到任何错误。你可能希望使用我们在课程中展示的 Screaming Frog 工具来抓取网站。列出你看到的任何错误、错误是什么以及你建议如何改进。例如，是否可以设置任何重定向，将用户引导至没有错误的更好页面。
