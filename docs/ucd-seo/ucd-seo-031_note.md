# 031：元数据与元标签

在本节课中，我们将要学习页面SEO的核心概念之一：元数据与元标签。理解这些概念是进行更深入SEO优化的基础。

上一节我们介绍了SEO的不同关注领域，本节中我们来看看构成搜索结果页面的具体元素。

## 搜索结果剖析

当你在搜索引擎中进行搜索时，会得到一个类似这样的结果页面。下图展示的是其中一个单独的搜索结果。

![](img/4d16f2c5a3bd518c9653e4a0d0113ab6_1.png)

为了便于讲解，我已经将结果中的不同区域用颜色标出。你所看到的每一个区域都与SEO相关，并且是我们在页面优化过程中需要重点关注的领域。

所有这些区域统称为一个网站的**元数据**，但每个独立的区域都有其特定的名称。**元数据**本质上是指描述其他数据的数据。

## 什么是元数据与元标签？

网站的每个页面都包含一个元数据区域。这个区域由一个个独立的**元标签**构成。

元标签是简短的文本片段，用于帮助搜索引擎识别页面的重要信息，例如页面是关于什么的，或者搜索引擎机器人是否应该忽略该页面。

这些信息包含在页面的源代码中，这意味着除非你特意查看构成页面的代码，否则这些信息不会被公开看到。

在源代码中，每个页面都有一个`<head>`元素，它是元数据的容器，位于页面的顶部。

## 元数据元素详解

让我们回到第一个示例，以便查看各种元数据元素。这是针对加州大学戴维斯分校葡萄酒酿造项目的搜索结果。为了显示这个结果，我在谷歌上搜索了“online wine making course”。

![](img/4d2c5a3bd518c9653e4a0d0113ab6_2.png)

现在我们来仔细分析一下这个结果。

以下是构成此搜索结果的主要元数据元素：

*   **标题标签**：图中蓝色区域。标题标签描述了网站管理员赋予这个特定页面的标题或名称。
*   **URL**：图中绿色区域。这是页面的网站地址。严格来说，它并非你可以定义的元数据的一部分，搜索引擎会从它分析的页面中提取URL。
*   **元描述**：URL下方的文本块。它描述了你在页面上将找到的内容。

请注意，在元描述中，有些词被加粗了。这些词之所以被加粗，是因为它们匹配了我们搜索查询中使用的词语。这些词就是我们所说的**关键词**。

你可以看到，并非所有关键词都需要连续出现才能被加粗。例如，我搜索“online wine making course”，结果中“courses”、“online”和“wine making”都被加粗了，因为谷歌能识别“course”的复数形式以及相关词组。

此外，句子中的“class”一词也被加粗了。这是因为谷歌能识别“courses”和“class”在语义上是相关的。

当我们提到元数据时，我们指的是构成搜索结果展示方式的各个元素。
