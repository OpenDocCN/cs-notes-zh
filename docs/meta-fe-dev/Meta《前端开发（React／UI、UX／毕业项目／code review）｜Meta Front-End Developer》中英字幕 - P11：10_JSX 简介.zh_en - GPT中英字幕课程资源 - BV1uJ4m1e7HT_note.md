那么，是什么让 JSX 如此特别？

一言以蔽之，是**表现力**。开发者可以使用一种表现力极强的语法来表达他们希望 React 渲染的内容，这种语法几乎与 HTML 或 XML 相同。

![](img/921471368b50c177dcd90eb8edb48581_13.png)

![](img/921471368b50c177dcd90eb8edb48581_14.png)

![](img/921471368b50c177dcd90eb8edb48581_15.png)

换句话说，JSX 允许开发者直接在 JavaScript 代码中编写 HTML。例如，你可以通过创建一个导航组件来为网站添加导航功能。

![](img/921471368b50c177dcd90eb8edb48581_16.png)

然后，在其中放置 HTML 语义化的 `nav` 元素和一个无序列表。

并且，因为这是 React，列表的值可以通过传递 `props` 来变得动态。

![](img/921471368b50c177dcd90eb8edb48581_18.png)

![](img/921471368b50c177dcd90eb8edb48581_19.png)

![](img/921471368b50c177dcd90eb8edb48581_20.png)

![](img/921471368b50c177dcd90eb8edb48581_21.png)

重要的是要知道，一个普通的 JavaScript 函数被用来定义 React 应如何渲染组件，无论它在哪里通过 `Nav` 这个 JSX 元素被引用。这正是 JSX 如此强大的原因，比如你可以插入特定的 JSX 表达式，如变量和 `props`。

![](img/921471368b50c177dcd90eb8edb48581_22.png)

这种方式之所以非常棒，是因为任何放在代码花括号 `{}` 内的内容本质上都是常规的 JavaScript 代码。例如，你可以使用 `li`（列表项）标签来计算一系列简单的表达式。这意味着你可以将花括号视为一个特殊区域，可以在其中编写任何你喜欢的 JavaScript 代码，而其余代码则写在 `return` 语句内部。

---

上一节我们介绍了 JSX 表达式，现在让我们聚焦于函数 `return` 语句内部的一些 JSX 规则。

![](img/921471368b50c177dcd90eb8edb48581_24.png)

`return` 语句可以被视为表达性语法的区域，它允许你按照希望在渲染的应用中呈现的样子来编写常规的 HTML 代码。但是，如果你需要返回跨越多行的 HTML 代码，则必须将其放在括号 `()` 内。

![](img/921471368b50c177dcd90eb8edb48581_25.png)

![](img/921471368b50c177dcd90eb8edb48581_26.png)

![](img/921471368b50c177dcd90eb8edb48581_27.png)

![](img/921471368b50c177dcd90eb8edb48581_28.png)

![](img/921471368b50c177dcd90eb8edb48581_29.png)

这允许开发者在常规 HTML 标签和用花括号 `{}` 表示的动态 JSX 表达式之间交替使用。

![](img/921471368b50c177dcd90eb8edb48581_30.png)

![](img/921471368b50c177dcd90eb8edb48581_31.png)

同样重要的是要记住，HTML 代码必须包裹在一个顶级元素中，例如 `div` 标签。如果你不想向 DOM 添加额外的 `div` 元素，可以使用一种叫做 **Fragment** 的东西来代替。

这就像一个没有标签名的 HTML 开闭标签。虽然你可以将 `return` 语句内的代码视为常规 HTML，但值得记住的是，你是在一个 JavaScript 文件中，因此存在一些差异。

![](img/921471368b50c177dcd90eb8edb48581_33.png)

![](img/921471368b50c177dcd90eb8edb48581_34.png)

例如，你不能在 HTML 元素中使用关键字 `class` 来处理 CSS 类。这是因为 `class` 是 JavaScript 中的保留关键字。你必须使用一个稍有不同的关键字 `className` 来代替。

![](img/921471368b50c177dcd90eb8edb48581_35.png)

![](img/921471368b50c177dcd90eb8edb48581_36.png)

同时请注意，`className` 采用**驼峰命名法**，即通过单个大写字母来分隔单词。

![](img/921471368b50c177dcd90eb8edb48581_38.png)

![](img/921471368b50c177dcd90eb8edb48581_39.png)

你可以使用 `className` 来列出任意数量的 CSS 类，以样式化组件内的特定 JSX 元素。

![](img/921471368b50c177dcd90eb8edb48581_40.png)

![](img/921471368b50c177dcd90eb8edb48581_41.png)

这就像你在常规 HTML 中使用 `class` 属性一样，并且这只是你可以为 React 应用添加样式的方式之一。你将在以后学习更多关于如何做到这一点的方法。目前，只需知道这是最接近你在处理 HTML 和 CSS 网站布局时会做的事情。

还存在一些其他差异，你将在以后学习它们。但在实际应用中，你可以认为这些代码几乎就是常规的 HTML。
