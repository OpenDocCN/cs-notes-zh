# 23：什么是状态 🧠

在本节课中，我们将要学习React中一个核心概念——**状态**。我们将探讨状态是什么，它与Props有何不同，以及如何在组件中使用它来控制UI的显示和行为。通过理解状态，你将能够构建更具交互性和动态响应的React应用。

![](img/f310724086207f0e55eb67f246b01a66_1.png)

## 状态的概念与类比

考虑闹钟的不同模式以及你会在何种情况下使用它们。

![](img/f310724086207f0e55eb67f246b01a66_1.png)

![](img/f310724086207f0e55eb67f246b01a66_3.png)

通常，闹钟有几种模式：**闹钟开启**用于设置起床时间，**闹钟关闭**用于不需要此功能时，以及**贪睡**用于再偷睡几分钟。

![](img/f310724086207f0e55eb67f246b01a66_3.png)

设置这些模式并不需要给你的闹钟添加任何额外的东西，它们是内置功能，只需按一下按钮即可设置。

## 在React中实现类似功能

如果你在React应用中创建这个功能，可以创建一个名为 `Clock` 的组件，然后通过**Props**传入状态值。

回忆一下，**Props**是React的一个特性，本质上允许你在浏览器中保存关于UI的信息。

![](img/f310724086207f0e55eb67f246b01a66_5.png)

![](img/f310724086207f0e55eb67f246b01a66_5.png)

然而，在React中，你还有另一种方式来实现这个功能，那就是使用一个类似的概念——**状态**。

状态同样允许你轻松改变组件的行为以适应特定需求。通过本视频的学习，你将能够描述React中的状态是什么，以及为什么开发者使用它来控制组件在浏览器中的显示内容。

![](img/f310724086207f0e55eb67f246b01a66_7.png)

## 状态的定义与重要性

![](img/f310724086207f0e55eb67f246b01a66_7.png)

将状态理解为组件的内部数据会很有帮助，它决定了组件的当前行为。状态通常用于存储影响组件行为的数据。

![](img/f310724086207f0e55eb67f246b01a66_9.png)

状态之所以重要，是因为它允许组件之间保持同步，并确保你的应用按预期运行。

![](img/f310724086207f0e55eb67f246b01a66_11.png)

![](img/f310724086207f0e55eb67f246b01a66_9.png)

例如，如果一个组件更新了它的状态，所有依赖于该状态的其他组件也会自动更新。

## 状态与Props的关系

![](img/f310724086207f0e55eb67f246b01a66_11.png)

这意味着一个组件通过使用**Props**将其状态传递给它的子组件。

![](img/f310724086207f0e55eb67f246b01a66_13.png)

![](img/f310724086207f0e55eb67f246b01a66_13.png)

如果子组件还有自己的孙组件，那么子组件可能也有一些状态，它们会通过Props传递给那些孙组件。

## 状态的存储与初始化

在React中，状态保存在**状态变量**中。改变状态的主要方式就是修改这些变量。

![](img/f310724086207f0e55eb67f246b01a66_17.png)

![](img/f310724086207f0e55eb67f246b01a66_17.png)

当一个组件被创建时，它会获得一个**初始状态**。这个状态用于初始化组件的属性。

![](img/f310724086207f0e55eb67f246b01a66_19.png)

## 有状态组件与无状态组件

组件可以分为**有状态**或**无状态**，但这到底意味着什么？为了更好地理解，让我们来探讨每种类型的一个例子。

![](img/f310724086207f0e55eb67f246b01a66_21.png)

首先是一个没有定义状态的App组件。它执行一个单一动作：渲染文本“一个无状态组件”。

```jsx
function App() {
  return <div>一个无状态组件</div>;
}
```

![](img/f310724086207f0e55eb67f246b01a66_23.png)

然后，我们有一个有状态的函数组件。

![](img/f310724086207f0e55eb67f246b01a66_25.png)

![](img/f310724086207f0e55eb67f246b01a66_23.png)

这个组件也渲染一些文本，但它引用了一个变量来实现。

![](img/f310724086207f0e55eb67f246b01a66_25.png)

```jsx
function App() {
  const [greet, setGreet] = React.useState("Hello");
  return <div>{greet}</div>;
}
```

![](img/f310724086207f0e55eb67f246b01a66_27.png)

![](img/f310724086207f0e55eb67f246b01a66_27.png)

## 理解useState Hook

你稍后会探索它是如何工作的。现在，请注意App函数体第一行使用的语法。

如果你熟悉普通JavaScript中的数组解构，这行代码可能对你来说已经很清楚了。但为了更清晰，让我们看一个例子。

考虑一个名为 `fruits` 的数组，它包含三个字符串：`apple`、`pear` 和 `plum`。

```javascript
const fruits = ["apple", "pear", "plum"];
```

![](img/f310724086207f0e55eb67f246b01a66_29.png)

![](img/f310724086207f0e55eb67f246b01a66_29.png)

在ES6版本中，JavaScript引入了**数组解构**的概念，允许你使用一行代码从数组中分配多个变量。

换句话说，你可以快速地将 `apple`、`pear` 和 `plum` 分配给 `fruitOne`、`fruitTwo` 和 `fruitThree` 变量，而不是一次分配一个。

```javascript
const [fruitOne, fruitTwo, fruitThree] = fruits;
// fruitOne = "apple", fruitTwo = "pear", fruitThree = "plum"
```

![](img/f310724086207f0e55eb67f246b01a66_31.png)

![](img/f310724086207f0e55eb67f246b01a66_31.png)

记住这一点，让我们回到有状态组件中的那一行代码。

注意，使用的语法与你刚刚检查的数组解构示例类似，但包含一个有趣的代码片段：`React.useState`。

**Hook** 允许开发者“钩入”一些原本无法访问的功能。

![](img/f310724086207f0e55eb67f246b01a66_33.png)

例如，要访问状态对象，你会使用 `useState` Hook。

![](img/f310724086207f0e55eb67f246b01a66_33.png)

![](img/f310724086207f0e55eb67f246b01a66_35.png)

## 解构useState的返回值

![](img/f310724086207f0e55eb67f246b01a66_35.png)

为了更好地理解在App组件中被解构的是什么，让我们对 `useState` Hook 调用一个 `console.log`。

```jsx
console.log(React.useState("Hello"));
```

![](img/f310724086207f0e55eb67f246b01a66_37.png)

![](img/f310724086207f0e55eb67f246b01a66_37.png)

输出显示一个包含两样东西的数组：字符串 `"Hello"` 和一个**函数**。

![](img/f310724086207f0e55eb67f246b01a66_39.png)

![](img/f310724086207f0e55eb67f246b01a66_39.png)

在这个例子中，`"Hello"` 是分配给 `greet` 状态变量的**状态值**。那个函数是一个内置的、未声明的函数。

![](img/f310724086207f0e55eb67f246b01a66_41.png)

![](img/f310724086207f0e55eb67f246b01a66_41.png)

这个函数可以用任何你喜欢的名字来解构，但有一个约定需要遵循。

![](img/f310724086207f0e55eb67f246b01a66_43.png)

![](img/f310724086207f0e55eb67f246b01a66_43.png)

如果你将状态变量命名为 `greet`，那么解构出来的状态更新函数应该命名为 `setGreet`。这是因为第二个解构出来的变量是一个用于更新状态变量的函数。

![](img/f310724086207f0e55eb67f246b01a66_45.png)

![](img/f310724086207f0e55eb67f246b01a66_46.png)

![](img/f310724086207f0e55eb67f246b01a66_45.png)

## 总结

本节课中我们一起学习了React中的**状态**。具体来说，我们探讨了：
*   状态是决定组件当前行为的内部数据。
*   状态与Props的关系：状态可以通过Props传递给子组件。
*   组件分为**有状态组件**和**无状态组件**。
*   如何使用 `useState` Hook 在函数组件中创建和管理状态，包括状态初始化和解构状态值及其更新函数。

理解状态是掌握React交互式UI开发的关键一步。在接下来的课程中，你将学习如何通过事件（如点击）来触发状态更新，从而使你的应用真正“活”起来。
