## PyCon 2019 会议笔记：1：深度学习与创意应用 🧠

上一节我们列出了所有演讲的目录，本节中我们来看看几个关于深度学习和创意应用的演讲。

**P1：用三种 Python 深度学习创建一部肥皂剧剧本**
这个演讲展示了如何利用 Python 的深度学习库（如 TensorFlow 或 PyTorch）来生成戏剧性的文本内容。核心是使用循环神经网络（RNN）或 Transformer 模型进行文本生成。一个简单的文本生成循环可以表示为：

```python
# 伪代码示例：基于模型预测生成下一个词
```

**P29：放下深度学习 - 何时不应使用神经网络**
这个演讲提供了一个重要的平衡视角。它指出，对于许多问题，传统的机器学习方法（如决策树、线性回归）或更简单的规则系统可能比复杂的神经网络更有效、更易解释且计算成本更低。核心建议是：不要盲目追求深度学习，应先评估问题的性质和数据量。

## PyCon 2019 会议笔记：2：Python 安全与工程实践 🔒

在了解了深度学习的应用与局限后，我们转向 Python 在安全和软件工程领域的实践。

以下是几个关于安全开发和工程实践的演讲：

*   **P2：位于比特中的蛇 - 使用 Pyt 进行安全自动化**：探讨了使用 Python 脚本自动化安全任务，例如漏洞扫描和日志分析。
*   **P10：Python 安全工具**：介绍了 Python 生态系统中用于安全评估和分析的常用工具库。
*   **P11：为安全环境构建可重现的 Python 应用程序**：强调了在安全敏感场景下，使用虚拟环境、依赖锁定（如 `Pipfile.lock`）和容器化（Docker）来确保应用环境的一致性至关重要。

**P12：发布你的第一个 Python 包并自动化未来的发布**
这个演讲是面向初学者的实用指南。它讲解了如何将你的代码打包，上传到 PyPI（Python 包索引），并利用 CI/CD 工具（如 GitHub Actions）自动化测试和发布流程。关键步骤包括配置 `setup.py` 或 `pyproject.toml` 文件。

## PyCon 2019 会议笔记：3：核心语言特性与性能 ⚙️

掌握了工程实践后，我们需要深入理解 Python 语言本身的一些高级特性和性能相关话题。

**P4：浮点数是朋友 - 充分利用 IEEE754**
这个演讲解释了计算机中浮点数的表示方式（IEEE 754 标准），以及为什么会出现 `0.1 + 0.2 != 0.3` 这样的情况。它教导开发者如何正确比较浮点数（例如使用 `math.isclose`）和理解精度限制。

**P5：高级 asyncio - 解决真实世界的生产问题**
讲解了 Python 的异步编程库 asyncio 在解决高并发 I/O 密集型问题时的应用。核心是理解事件循环、协程 (`async def`)、任务 (`asyncio.create_task`) 和 `await` 关键字。

**P6：点上发生的 8 件事 - 属性访问与描述**
深入剖析了当你访问一个对象属性 `obj.attr` 时，Python 解释器背后执行的复杂查找链，包括描述符协议 (`__get__`, `__set__`)。这对于理解属性、方法、`@property` 装饰器的工作原理至关重要。

**P31：GIL 该不该存在 - 多核（C）Python 的未来**
讨论了 Python 的全局解释器锁（GIL）对多线程并行计算的影响，以及绕过 GIL 的方法（如使用多进程 `multiprocessing` 模块、C 扩展或将计算任务转移到用 Cython 或 NumPy 编写的库中）。

## PyCon 2019 会议笔记：4：数据科学、系统与公平性 📊

Python 在数据分析和系统构建方面同样强大。本节涉及相关主题。

**P7：爬取一百万场宝可梦战斗 - 通过实例了解分布式系统**
通过一个有趣的项目，演示了如何使用 Python 工具（如 Scrapy、Celery）构建分布式爬虫和数据收集系统，以处理大规模任务。

**P8：统计分析（以及与 sys 模块的其他乐趣）**
可能介绍了如何使用 Python 的 `statistics` 模块进行基本统计分析，并探索了 `sys` 模块中一些不为人知但有用的功能。

**P9 与 P13：算法公正的度量**
这两个演讲都关注数据科学中的伦理问题。**P9：算法公正的度量与误度量** 和 **P13：测量模型公平性** 探讨了如何评估机器学习模型是否存在对特定群体的偏见，并介绍了公平性指标，例如：
*   人口统计平等：`P(\hat{Y}=1 | A=a) = P(\hat{Y}=1 | A=b)`
*   机会均等：`P(\hat{Y}=1 | A=a, Y=1) = P(\hat{Y}=1 | A=b, Y=1)`
