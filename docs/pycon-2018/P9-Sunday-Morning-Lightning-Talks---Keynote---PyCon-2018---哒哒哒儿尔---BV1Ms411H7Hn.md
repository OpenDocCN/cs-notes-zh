每段代码背后总有一个人。现在，贡献如此之多。这也可能有点棘手。我喜欢说，这就像有人试图给你一只你没有要求的小狗。你可能会把这只小狗视为这只可爱、迷人的东西，然后说，嘿，你想要这只可爱的小狗吗？而我在看的是。

养活和遛狗十年的时间。因为你真的需要仔细考虑这一点。假设你给了我一个 PR。我会审查它，假设我立刻接受它。没有任何修改和变动。那段代码现在就得我来维护，可能要至少十年。记住，Python 已经 28 岁了，早于 Unicode。而我认为没有人会在。

这个房间希望它明天就消失。因此，希望我们至少还有 28 年的美好 PyCon 时光。这意味着我必须担心这段代码能否持续到 28 年后的 PyCon。因此，这需要认真考虑这些事情，好的。如果我接受这个，我会破坏某人的代码吗？顺便提一下，这里有超过 700 万。

我将会破坏某人的代码。他们会非常不高兴我破坏了他们的代码吗？

这会合理吗？我可能刚刚让一堆印刷在死树上的书籍因为这个变化而变得过时。我的意思是，这真是让人震惊。我们开始考虑这种影响。因此，有时候我们不得不说，对不起。我会在下一张幻灯片中讨论这个问题。另一个可能出错的方式是，有时候人们会说。

这就是一个 PR，因为这个真的坏了或者这个真的错了。再一次，消极情绪是没有必要的。基本上，你不需要告诉我你在修复我的问题。你可以简单地说，“我尽力帮助，因为我认为我可以改进这个。” 再一次，所有这些内容都可以以积极的方式表达，而不是消极。

对任何人表现出外在的消极态度。现在，我看到当人们提交某些内容时，我们也需要以积极的方式反馈。所以当你给我善意，给我一个 PR 时，我至少应该以感谢的方式回馈你，而不是说，“嘿，你做错了。” “这到底怎么回事？这不对。算了吧。” 

我曾经提交过大约六到十个拼写错误的 PR，涉及项目文档，对吧？没什么大不了的，很快就完成了。在 GitHub 上，我只需点击那个小铅笔，快速修正，然后继续阅读，发现另一个错误，继续这样做。我的第一个回应是，“你不知道可以只提交一个 PR 吗？” 我说，“哦，是的，但我在阅读。”

通过 GitHub，这样做更快，只需点击铅笔就能完成。而且，我以为你会喜欢。圣诞节真的必要吗？如果你发现两个 PR，可以忽略它们，但我的意思是，真的只是指出来，“来吧，新手，你在干什么？” 这完全是多余的。另一个问题是，当你给出反馈时。

尤其是当你说，“抱歉，不，我不能接受这个，因为我认为破坏至少十万个用户的代码不值得你要求的改变。” 有些人可能会对此感到个人受伤，而这从来不是出于个人的原因。只是感觉像这样。所以可能会想，“你为什么不爱我？你为什么不接受这个？” 举个例子，当我在 2016 年经历职业倦怠时。

发生这种情况的原因之一，我应该先说，故事中的那个人已经道歉了，他说，“是的，我对你有意见，因为你妨碍我贡献。” 我说，“抱歉，这并不是故意要成为你的障碍。只是，我也有我的责任，对这个项目和 Python 的七百万用户负责。”

确保我保持理智。所以抱歉，但不，我不能接受这个。所以这真的很棘手。最后，维护可能就像和残疾人争论政治。我们应该都知道更好，但我们并不总是如此。我实际上几乎因争论而流泪，与核心开发者争论。接下来的 Python 人问，“你们还在和他们说话吗？”

因为我们都如此关心，所以事情可能会变得相当激烈，而我们没有时间退一步，离开电子邮件客户端，心想，“好吧，没关系。我认识这个人，他们是好人，他们并不是那个意思，没关系。我可以理性地讨论，一切都很好。每个人都可以友善相待。” 我知道停下来休息是很困难的。

但老实说，如果我们在发送推文之前都能停下来思考一下，我们可能会整体上更快乐。因此，这不仅仅是新成员进入项目，或者因为你在外面，也可能是在项目内部。因此，基本上，重点是我们都需要改进。我知道这听起来有点偏向于维护者，但老实说，确实是这样。

规模因素在维护者遭受更多伤害方面起着重要作用。比如说，举手。谁在使用开源软件？顺便说一下，Python 是开源的，所以每个人都应该举手。好的，保持举手，抱歉。现在，如果你实际上在维护一个开源项目，请继续举手。

好的，举手的人数明显减少。现在，谁在维护 Python？是的，所以如果大家能看到，我现在大约数到三到五个。好吧，这里大约有三个人。所以在这个房间里，维护 Python 的人数不到十个。所以我的观点是，你们所有人都可以为 Python 贡献，而在这个房间里帮助维护的人不到十个。所以确实存在一个现实的规模问题。

这里的问题在于，即使社区中只有十分之一或百分之一的人持消极态度，也很容易产生重大影响，因为新成员的数量在增加。我相信我们很多人都听过这样一句话：需要十个善举才能弥补一个消极行为。我不知道这是否经过心理研究的验证，但这感觉是正确的。

在 Twitter 上乞求善意，期待人们在你经历了糟糕互动后说声谢谢，确实有点艰难。有时候我承认我这样做，只是为了不让自己整天都感到糟糕。基本上，每个人都需要像善意不需要任何回报一样来行动。我认为这是一个我们没有做到的关键点。

例如，当你给我发送那个 PR 时，请记得我可能因为各种原因不得不拒绝。这并不是个人问题。请把这看作是你对我和社区的善意，至少尝试一下这个拉取请求并告诉我这个更改。如果我不能接受，抱歉，但这就是现实。如果我们能够意识到这就是互动的方式，那就好了。

这没问题。这是双向的，对吧？就像如果你给我发送一个拉取请求，而我请你进行更改，而你没有去做，那也没关系，对吧？我向你提出这个请求是出于善意，请你更新你的 PR，以便我可以接受。如果你没有时间，那也没关系。这就是现实。但这并不是交易，对吧？并不是说，嘿，如果我给你发送这个 PR，那么……

我希望你能接受这个请求，或者如果我请求你进行更改，那我希望你能做出更改。事情并不是这样的。这就是我们最终陷入困境的原因，人们感到非常沮丧，而这并不利于可持续性。而且这是一种双向的关系，对吧？

如果你在对我好，我需要表现得好像一切都是出于对我的好意。 如果你把它发给我，我也需要意识到你在对我好。 你在这里试图表现得很友好，我应该对你表现出同样的同情和理解。 这在某些情况下可能会变得复杂。

一种交流风格并不一定能传达出友好的感觉。 所以，始终尝试给人们一个怀疑的好处是好的，完全可以说，嘿，知道吗，你的措辞给人的感觉有些不对，我认为你并不是想这样说的。 所以，请下次能否试着用不同的方式表述？

说实话，大多数时候人们会说，哦，我很抱歉，我不知道。 糟糕的一天。 这不是我的第一语言。 什么的。 这是完全可以接受的。 让人们知道他们的措辞并不好是合理的，但没关系。 我没有放在心上，我们都很好，我们会继续前进。
