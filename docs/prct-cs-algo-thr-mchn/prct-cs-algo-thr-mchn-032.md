## 数据类型的概念

上一节我们介绍了计算机处理信息的基本方式。本节中，我们来看看数据类型的定义。

数据类型是一组值以及对这些值的一组操作的集合。在计算机中，所有数据都以二进制形式编码，因此关键在于我们如何解释这些二进制位。

TOY计算机的数据类型被称为**16位二进制补码整数**。这意味着每个数据单元由16个二进制位组成，并采用二进制补码的规则来表示整数（包括正数和负数）。

TOY可以对这种数据类型执行两种主要操作：
1.  **算术运算**：例如加法。在实际计算机中，还包括乘法、除法等。
2.  **位运算**：将整数视为一系列比特位，并逐位执行操作。我们在课程开始的密码学示例中见过，通过按位异或操作来实现加密协议。

所有其他类型的数据（如32位整数、浮点数、字符、字符串）和操作都必须通过软件来实现。这反映了早期计算机的情况：硬件只提供最基本的操作，更复杂的功能由软件构建。如今，许多常用操作已直接集成到硬件中以提高性能，但硬件与软件之间的界限本质上是抽象的——我们只是在不同的层面上实现值和操作。

对于TOY，所有值都表示为**16位字**，我们通过开关和指示灯与它们交互。

## 无符号整数与二进制补码

上一节我们定义了数据类型。本节中，我们来看看TOY最初的设计以及一个关键的改进。

TOY最初的设计（许多早期计算机也是如此）是处理无符号整数，即从0到 `2^16 - 1` 范围内的值，用二进制编码，并用十六进制表示。

例如，十进制数6375的二进制表示为：
`0001100011100111`
其十六进制表示为 `0x18E7`。

我们可以对这些数字执行加、减或判断是否为零等操作，就像处理普通二进制数一样。

然而，人们很快意识到，无需改变硬件，只需改变对数据的解释方式，就能让计算机处理负整数。这就是**二进制补码**表示法。如今，所有计算机都使用二进制补码来表示整数。

二进制补码的规则很简单：
*   对于一个正数 `x`，直接使用其16位二进制表示。
*   对于一个负数 `-|x|`，使用 `2^16 - |x|` 的16位二进制表示。

可表示的范围是 `-2^15` 到 `2^15 - 1`（即-32768到32767）。这样，正数的数量大约只有无符号表示时的一半，因为另一半编码空间用于表示负数。

以下是二进制补码表示的一些关键属性：
*   **符号位**：最高位（最左边）表示符号。0代表正数或零，1代表负数。
*   **唯一的零**：0的表示是唯一的（所有位都是0）。如果简单地用最高位表示符号（原码），就会产生+0和-0两种表示，使运算复杂化。
*   **运算便利性**：加法和减法操作与无符号整数完全相同，无需修改硬件。

## 十进制与二进制补码的转换

理解了二进制补码的原理后，我们来看看如何进行具体的转换操作。

将十进制数转换为二进制补码表示，步骤如下：
1.  检查数字是否在 `-2^15` 到 `2^15 - 1` 的范围内。如果超出，则无法用16位表示。
2.  取该数的绝对值，并转换为16位二进制。
3.  如果原数是0或正数，转换完成。
4.  如果原数是负数，则对第二步得到的二进制位**按位取反**（0变1，1变0），然后**加1**。

**示例：将 -13 转换为二进制补码**
1.  13的16位二进制表示为：`0000000000001101`
2.  按位取反：`1111111111110010`
3.  加1：`1111111111110011`
4.  十六进制表示为：`0xFFF3`

将二进制补码转换回十进制数，步骤如下：
1.  检查最高位。如果是0，表示正数，直接转换为十进制。
2.  如果是1，表示负数。先对**所有位取反**，然后**加1**，得到其绝对值的二进制，再转换为十进制并加上负号。

**示例：将 `0xFFFF` (即所有位都是1) 转换为十进制**
1.  最高位是1，所以是负数。
2.  所有位取反：`0000000000000000`
3.  加1：`0000000000000001` (值为1)
4.  因此，`0xFFFF` 表示 -1。

## 算术运算与溢出问题

学会了数值表示，现在我们可以看看对它们进行运算时会发生什么。

使用二进制补码表示法，加法和减法运算可以直接进行，就像处理无符号整数一样。硬件电路无需任何改变。

然而，这里存在一个关键问题：**溢出**。当运算结果超出数据类型所能表示的范围时，就会发生溢出。

考虑16位二进制补码的最大正数：`2^15 - 1 = 32767`，其二进制表示为 `0111111111111111` (0后跟15个1)。
如果对其加1：
```
  0111111111111111 (32767)
+ 0000000000000001 (1)
-------------------
  1000000000000000 (-32768)
```
结果变成了最小的负数 `-32768`。这是因为最高位的进位改变了符号位。

许多编程初学者都会遇到这个问题：当一个数不断增大，超过最大值后，突然变成了一个很大的负数。这就是溢出现象。

## 位运算

除了算术运算，TOY还支持一系列位运算，这些运算将数据视为比特位序列进行处理。

位运算对两个操作数的每一个对应位执行基本的布尔函数。以下是TOY支持的几种位运算：

**按位与 (AND)**
运算规则：只有当两个对应位都为1时，结果位才为1，否则为0。
代码表示：`result = x & y`
用途：常用于使用“掩码”来提取（隔离）一个字中的特定位。例如，用一个在特定位置为1、其余位置为0的掩码进行AND操作，可以清零其他位，只保留目标位。

**按位异或 (XOR)**
运算规则：如果两个对应位不同，则结果位为1；如果相同，则结果位为0。
代码表示：`result = x ^ y`
用途：我们在课程开始的加密示例中见过，用于加密和解密。也常用于切换特定位的状态。

**移位运算**
*   **左移 (Shift Left)**：将所有位向左移动指定的位数，右侧空出的位用0填充。
    代码表示：`result = x << n`
    效果：相当于乘以 `2^n`。
*   **逻辑右移 (Shift Right)**：将所有位向右移动指定的位数，左侧空出的位用0填充。
    代码表示（对于无符号数）：`result = x >> n` (在Java等语言中，`>>` 对于有符号数是算术右移)
    效果：相当于除以 `2^n`（向下取整）。
*   **算术右移**：对于有符号数（二进制补码），右移时左侧空出的位用符号位（最高位）填充，以保持负数的符号。这是移位运算与数据类型的交叉点。

## 总结

本节课中，我们一起学习了计算机数据类型的核心概念。我们了解到TOY计算机使用**16位二进制补码整数**作为其基本数据类型，这允许它表示正数、负数和零，并使用相同的硬件电路进行加法和减法运算。

我们探讨了：
*   数据类型的定义：一组值及对其的操作。
*   二进制补码的表示方法和转换规则。
*   算术运算及其潜在的**溢出**问题。
*   各种**位运算**（AND, XOR, 移位）及其用途。
