---

## 回顾基本组件

上一节我们介绍了程序计数器的工作原理。本节中，我们来看看构建一个完整处理器所需的所有核心组件。

以下是构成我们“玩具8号”（Toy 8）计算机CPU的基本组件：
*   **算术逻辑单元（ALU）**：执行算术和逻辑运算。
*   **内存（Memory）**：存储指令和数据。
*   **寄存器（Register，R）**：存储临时数据。
*   **程序计数器（PC）**：存储下一条要执行的指令地址。
*   **指令寄存器（IR）**：存储当前正在执行的指令。
*   **控制单元（Control）**：生成控制信号序列。
*   **时钟（Clock）**：产生规律的脉冲信号。

此外，由于某些组件有多个输入源，我们还需要**多路复用器（Mux）**。例如，寄存器（R）有三个可能的输入源，内存地址（Memory Address）有两个可能的输入源，因此需要相应的多路复用器来选择。

---

## 建立数据通路连接

了解了组件之后，下一步是确定如何将它们连接起来。这取决于我们想要实现的指令集。我们需要为每条指令建立正确的数据通路。

以下是实现所有指令所需的关键总线连接：

*   **取指（Fetch）**：所有指令的第一步。需要将PC中的地址送入内存，并将内存的输出（即指令）送入指令寄存器（IR）。因此，需要连接：
    *   `PC` -> `Memory Address Mux` -> `Memory`
    *   `Memory Output` -> `Instruction Register (IR)`

*   **算术指令（如ADD）**：需要从内存中取出一个操作数，并与寄存器中的值进行运算。
    *   操作数地址来自指令寄存器（IR）的地址位。因此需要连接：`IR Address Bits` -> `Memory Address Mux`。
    *   从内存取出的操作数需要送入ALU。因此需要连接：`Memory Output` -> `ALU`。
    *   寄存器（R）的值也需要送入ALU。因此需要连接：`Register (R)` -> `ALU`。
    *   ALU的计算结果需要写回寄存器。因此需要连接：`ALU Output` -> `Register Mux` -> `Register (R)`。

*   **加载指令（LOAD）**：需要将内存中指定地址的数据加载到寄存器。
    *   地址来自IR。连接：`IR Address Bits` -> `Memory Address Mux`。
    *   内存数据需要写入寄存器。连接：`Memory Output` -> `Register Mux` -> `Register (R)`。

*   **存储指令（STORE）**：需要将寄存器中的数据存入内存的指定地址。
    *   地址来自IR。连接：`IR Address Bits` -> `Memory Address Mux`。
    *   寄存器数据需要写入内存。连接：`Register (R)` -> `Memory Input`。

*   **零跳转指令（BRANCH-IF-ZERO）**：如果寄存器值为零，则将PC设置为IR中的地址。
    *   需要将IR中的地址加载到PC。连接：`IR Address Bits` -> `PC Mux` -> `PC`。

通过系统性地分析每条指令的需求，我们可以确定所有必要的总线连接。当多个信号源需要接入同一个组件时，就使用多路复用器（Mux）进行选择。

## 控制信号

数据通路建立了组件之间的“高速公路”。现在我们需要“交通信号灯”来指挥数据在正确的时间流向正确的地方，这就是控制信号。

控制信号作用于每个组件，决定其当前行为。以下是主要组件的控制信号：

*   **时钟（Clock）**：接收“运行/停止”信号。
*   **控制单元（Control）**：接收来自时钟的四个阶段信号（取指、取指右、执行、执行右），并根据当前指令（来自IR的操作码）和寄存器状态，生成所有其他控制信号。
*   **算术逻辑单元（ALU）**：选择执行哪种运算（加、异或、与）。
*   **多路复用器（Muxes）**：选择哪个输入信号被输出。
*   **指令寄存器（IR）、寄存器（R）、内存（Memory）**：各自的“写入”使能信号。
*   **程序计数器（PC）**：选择“递增”还是“加载”地址，以及“写入”使能信号。

控制单元本身是一个**组合逻辑电路**。它根据输入（时钟阶段、操作码、寄存器值）查表，然后输出相应的控制信号组合。其核心是**解复用器（Demultiplexer）**，它根据操作码将“执行”或“执行右”信号路由到对应的控制线上，从而激活特定指令所需的操作。

## 指令执行示例

让我们通过一个简单的程序示例，看看控制信号如何协同工作。假设程序是：1) 从内存加载一个数到寄存器；2) 将寄存器值与另一个内存数相加；3) 将结果存回内存；4) 停机。

1.  **取指（Fetch & Fetch Right）**：
    *   `Fetch`信号激活，控制单元点亮“地址Mux选择PC”的控制线。内存地址来自PC（假设为1）。
    *   `Fetch Right`信号激活，控制单元点亮“IR写入”控制线。内存地址1中的指令（例如`LOAD`）被存入指令寄存器（IR）。

2.  **执行LOAD指令（Execute & Execute Right）**：
    *   `Execute`信号激活，控制单元为`LOAD`指令设置控制线：PC准备递增，地址Mux选择IR的地址位，寄存器Mux选择内存输入。
    *   `Execute Right`信号激活，同时触发“PC写入”和“寄存器写入”。结果：PC递增到2，同时内存中指定地址的数据被加载到寄存器。

3.  **取指并执行ADD指令**：
    *   类似步骤1，取出下一条指令（`ADD`）到IR。
    *   `Execute`信号为`ADD`设置控制线：PC准备递增，ALU选择加法，寄存器Mux选择ALU输出。
    *   `Execute Right`信号激活，触发“PC写入”和“寄存器写入”。结果：PC递增到3，同时寄存器中的原值与从内存取出的新值在ALU中相加，结果写回寄存器。

4.  **取指并执行STORE指令**：
    *   取出`STORE`指令到IR。
    *   `Execute`信号为`STORE`设置控制线：PC准备递增，地址Mux选择IR的地址位。
    *   `Execute Right`信号激活，触发“PC写入”和“内存写入”。结果：PC递增到4，同时寄存器中的值被写入内存的指定地址。

5.  **取指并执行HALT指令**：
    *   取出`HALT`指令到IR。
    *   `Execute`信号激活，控制单元点亮“停机”信号，时钟停止。

通过这一系列由时钟驱动的、由控制单元编排的控制信号序列，程序得以正确执行。

## 总结与现实意义

本节课中我们一起学习了如何从基本组件构建一个简单的CPU。

*   **核心概念**：CPU是一个由开关和导线构成的电路，处理用二进制编码的信息（包括其自身的指令），这体现了**冯·诺依曼架构**的核心思想。
*   **实现方式**：我们使用带有反馈的电路实现存储（内存、寄存器），用基于布尔逻辑的小型元件构建大型电路（如ALU）。指令的本质是在存储单元间移动信息并指定操作。
*   **驱动力**：一切工作的基础是**时钟脉冲**。规律的时钟激活一系列控制信号，这些信号引发状态变化，从而最终实现机器指令。
*   **抽象的力量**：我们构建的这个基础机器相对简单。现代计算机的惊人能力、规模和速度，几乎都是通过层层**软件抽象**实现的。每一层抽象都赋予了系统更强大的功能和更广的适用范围。

我们绘制的“玩具8号”CPU原理图，在概念上与真实的微处理器芯片（如下图）没有本质区别，区别主要在于规模（我们可能是1比特/平方厘米，而现代芯片是数百亿比特/平方厘米）和实现细节。理解了这个简单模型，你就揭开了计算机如何工作的神秘面纱。
