# 027：P类与NP类 🧩

![](img/c4362e93c9364fd54e75322b6eda47e3_0.png)

在本节课中，我们将学习计算复杂性理论中的两个核心概念：P类和NP类。我们将从严谨的定义出发，探讨这两类问题的区别与联系，并理解“P是否等于NP”这一悬而未决的世纪难题的深刻含义。

## 定义：搜索问题与NP类

首先，我们需要从严谨的定义开始，我们的“角色”是不同类别的问题，我们希望通过效率来对问题进行分类。

我们从一个概念开始：**搜索问题**。我将一个搜索问题定义为：任何存在**高效算法**来**验证**一个给定解是否正确的问题。也就是说，至少当你拿到一个问题的一个候选解时，你希望知道它是否真的是一个解。

例如，在**旅行商问题**中，给定一组城市、城市间的距离和一个阈值M。如果有人给你一个解（即一个城市的排列顺序，代表你应该访问它们的顺序），那么你可以简单地通过累加距离来轻松检查总长度是否小于M。只要验证解的算法保证在多项式时间内完成，这就是一个搜索问题。

因此，我们有了**NP类**。我们将所有这类问题（所有搜索问题）的集合称为NP类。我稍后会解释这个名称的由来，但现在我们先接受这个名称。

以下是NP类问题的例子：
*   **旅行商问题**：如上所述，它是一个搜索问题，因为存在算法可以检查一个候选解是否有效。
*   **整数线性规划**：给定一组不等式，需要找到一个满足这些不等式的二进制向量。同样，如果给你一组声称是解的x值，你可以将这些值代入不等式并检查每个方程是否成立。
*   **布尔可满足性问题**：给定一组逻辑方程，需要找到一组变量赋值使其全部为真。如果给你一组候选赋值，你可以将它们代入方程并检查。
*   **因数分解**：给定一个整数M，需要找到一个非平凡因子（除了1和M本身）。如果有人声称一个数是因子，你可以用长除法来检查。

所有这类性质的问题都称为搜索问题。我们认为，这些正是我们**希望用计算机解决**的问题。这是一个非常重要的问题类别。

## 暴力搜索与P类

对于一个搜索问题，你总是可以通过检查所有可能性来找到一个解，我们称之为**暴力搜索**。

*   对于旅行商问题，你可以尝试所有城市的排列，以找到一个长度小于M的路径。
*   对于整数线性规划，如果你有n个二进制变量，每个变量可以取0或1，那么就有2^n种可能性，你需要尝试所有这些可能性。
*   对于因数分解，你可以尝试从2到该数平方根的所有整数，看是否能整除。

一个初级程序员也能编写出这样的暴力搜索代码。但问题是，当n很大时，它**效率低下，速度太慢**。即使对于中等规模的n，当n达到实际应用中可能出现的大小时，即使是超级计算机也无法完成。

现在，让我们看看**P类**。这是所有**可处理的搜索问题**的类别，即那些无论输入是什么，我们都能保证在**多项式时间**内解决的问题。

例如：
*   **排序**：我们有插入排序、归并排序等算法。
*   **三数和问题**：在一组数中寻找三个和为0的数。只需尝试所有三元组，其数量是多项式级别的。
*   **解线性方程组**：使用高斯消元法的高效版本可以解决。
*   **线性规划**：自20世纪80年代以来，我们已经知道存在解决它的高效算法。

P类的意义在于，这是我们**实际上能够解决**的问题类别。我们有保证的多项式时间算法，我们可以改进它们，并解决更大规模的问题实例。我们不会被指数级障碍所阻挡，那是连超级计算机都无法处理的。

显然，所有P类问题也都在NP类中。如果我们能在多项式时间内找到一个解，那么我们当然能在多项式时间内检查它是否是一个解。这是需要记住的第一点：P ⊆ NP。

**总结一下定义**：
*   **NP**：如果你有一个解，你可以高效地检查它是否是一个好解。
*   **P**：你可以用一个高效算法在多项式时间内找到一个解。

## P vs. NP：核心问题

在继续之前，需要说明的是，有多种不同的方式来描述这些问题类别。我们一直在讨论**搜索问题**，其目标是找到一个解。还有其他表述方式，比如**判定问题**（是否存在一个解？）和**优化问题**（找到最优解）。一些问题和研究者会在不同的领域工作。例如，旅行商问题通常被表述为优化问题：找到连接所有城市的最短路径。

我提到这一点是为了避免混淆。从技术上讲，这些表述方式并不完全等价，但主要的结论适用于所有三种。这有点像选择一种编程语言，你必须选择一种，然后围绕这个选择发展一切。我们给出的P和NP定义与通常讨论判定问题的经典定义略有不同，但我们发现搜索问题的表述方式更自然，便于在一节课内解释清楚。

现在，我们来到核心问题。我们有两个类别：
*   **NP**：我们可以高效验证解的问题。
*   **P**：我们可以高效找到解的问题。

问题是：**这两个问题类别是相同的吗？**

这似乎应该有一个答案，只要你看看它的含义：
*   如果 **P ≠ NP**，这意味着存在至少一个（可能很多）**难解**的问题。我们可以证明，你无法保证在多项式时间内解决它。对于这些问题，如果我们想保证解决任何输入的问题，我们能做的最好方法似乎就是暴力搜索。
*   如果 **P = NP**，那么所有NP类中的问题，即使是那些看起来很难的问题（如旅行商问题），实际上都是可处理的。实际上存在多项式时间算法，只是我们尚未发现而已，就像线性规划问题最终被解决一样。

这是一个巨大的差异：暴力搜索是我们能做的最好方法，还是存在我们尚未发现的高效算法？

令人沮丧的是，**大多数研究者相信P ≠ NP**，即那些问题实际上是难解的，但**至今无人能够证明这一点**。这个悬而未决的状态已经持续了几十年。这是我们想要探讨的核心问题。

## 从不同视角看P vs. NP

这个问题如此重要，我们将从几个不同的角度来审视它。

**视角一：非确定性**

我们考虑**非确定性机器**的概念，这种机器在每一步都可以在多个选项中进行选择。这与我们考虑过的所有抽象机器和真实计算机都不同。在我们讨论过的任何计算机中，如果你知道它的状态，你就知道接下来会发生什么，它是**确定性的**。

非确定性机器则不同。特别地，由于它可以在多个选项中选择，一个非确定性机器可以**直接“猜出”** 那个能导向解的选项。

这听起来像是一个幻想，但从数学上讲并不复杂。例如，想象一下Java有一个“either-or”语句。如果你写`either x = 0 or x = 1`，它会选择这两个中的一个。对于整数线性规划问题，无论实例是什么，机器都会选择“either-or”语句的正确选项来进行我们关心的赋值，从而得到正确的解。真正的问题是：是否存在一条路径让机器通过这些选择到达答案？

当然，即使是非确定性也不会改变我们可以计算的问题范围，它关乎的是**效率**。我们在讨论正则表达式和克林定理时见过一个例子，该定理将非确定性有限自动机转换为确定性有限自动机，但代价是确定性自动机的状态数可能呈指数级增长。这种指数级的膨胀，正是与我们正在讨论的效率问题相关的。

无论如何，非确定性看起来像是一个完全的幻想，我们无法想象一台总能猜出正确答案的机器。但事实是：
*   如果 **P ≠ NP**，那么难解的搜索问题存在，非确定性机器将为我们提供高效的算法（如果我们能建造它的话）。
*   如果 **P = NP**，那么非确定性机器将毫无帮助，或者根本不会让事情变得更高效。

因此，P等于NP的问题实际上是：**即使我们能建造非确定性机器，它会有帮助吗？** 这似乎是一个我们应该很容易解决的理论问题，因为没人相信我们能建造出真正计算答案的非确定性机器，但多年来（实际上是几十年来）无人能够证明这一点。

**视角二：创造与鉴赏**

另一种看待这个情况的角度是围绕**创造性**。即创造某物与能够欣赏或理解它之间的区别。

*   莫扎特创作一首乐曲，我们认为这很困难，但听众欣赏它则相对容易。
*   数学家证明一个深奥的定理（如费马大定理），证明过程是困难的，但同事检查他的证明则相对容易。
*   波音公司设计一个机翼，设计是一回事，验证它则是另一回事，且相对容易。
*   任何科学家提出一个理论，然后由实验人员验证。

这就像是莫扎特和《美国偶像》评委之间的区别。**P与NP的类比**是：**找到一个解是一回事**（我们认为这很困难），**检查一个解是否正确是另一回事**（我们认为这相对容易）。但至今无人能够证明，**创造一个问题解比检查它是否正确更困难**。

---

**本节课总结**

在本节课中，我们一起学习了计算复杂性理论的核心基石：
1.  我们定义了**搜索问题**，即存在高效验证算法的问题。
2.  所有搜索问题的集合称为**NP类**。
3.  在NP类中，那些存在高效求解算法的问题构成了**P类**。
4.  我们探讨了**P是否等于NP**这个悬而未决的世纪难题。如果相等，则所有易于验证的问题都易于求解；如果不相等，则存在本质上难以求解但易于验证的问题。
5.  我们从**非确定性计算**和**创造 vs. 鉴赏**两个角度，深入理解了这一问题的深刻内涵。
