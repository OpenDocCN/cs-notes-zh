# 010：实现方法 🔗

![](img/390b2dfc7a67e1280ae9005b3e585a78_0.png)

在本节课中，我们将学习如何使用**链表**这种强大的数据结构，来实现一个满足特定性能要求的**下压栈**。我们将详细探讨其实现细节，确保每一步都清晰易懂。

---

## 概述

下压栈是一种理想化的后进先出存储模型。我们有一个抽象数据类型，允许我们编写操作栈的Java程序，这就是我们的API。同时，我们还有性能要求：所有操作必须是常数时间，内存使用必须与集合大小成线性关系，并且代码中不能对集合大小设限。如果一个实现不满足这些规范，我们就不能说它实现了一个真正的下压栈。本节我们将使用链表来实现它。

---

## 链表的选择与结构

上一节我们介绍了栈的抽象概念，本节我们来看看如何用链表具体实现它。我们选择使用链表来保存集合。栈中的对象将成为链表中的节点，我们只需要一个变量 `first` 来指向链表中的第一个节点。

以下是支持此用途的实例变量和构造函数：

```java
public class Stack<Item> {
    private Node first; // 指向链表的第一个节点
    private int n;      // 栈的大小

    // 构造函数
    public Stack() {
        first = null;
        n = 0;
    }

    // 私有节点类
    private class Node {
        Item item;
        Node next;
    }
}
```
我们的实例变量包括一个指向第一个节点的引用和栈的大小。这里使用了泛型，使得栈可以存储任何类型的数据。每个节点都有一个 `next` 字段，用于引用下一个节点。

> 注意：在某些更复杂的场景中，如果需要泛型数组，不能直接声明，而需要从对象数组进行转换。这一点留作后续参考。

## 测试客户端

我们的测试客户端与之前相同，但现在可以使用泛型栈。这意味着无论我们放入什么类型的数据，实现都应该正常工作。

```java
public static void main(String[] args) {
    Stack<String> stack = new Stack<String>();
    // ... 测试操作
}
```

## 方法的实现

现在，剩下的就是实现各个操作方法。

### 判断栈是否为空

我们可以通过检查 `first` 是否为 `null` 来判断栈是否为空。因为我们有变量 `n`，也可以测试 `n` 是否等于0，两者是等价的。

```java
public boolean isEmpty() {
    return first == null; // 或 return n == 0;
}
```

### 入栈操作

要将新项压入栈，我们需要在链表的开头添加一个新节点。

以下是实现入栈操作的步骤：

1.  将当前 `first` 节点保存为 `second`。
2.  创建一个新的 `first` 节点。
3.  将新节点的 `item` 字段设置为要入栈的项。
4.  将新节点的 `next` 字段指向 `second`。
5.  增加栈的大小 `n`。

```java
public void push(Item item) {
    Node second = first;
    first = new Node();
    first.item = item;
    first.next = second;
    n++;
}
```

### 出栈操作

栈中最近添加的项是链表的第一项。出栈操作需要移除并返回这第一项。

以下是实现出栈操作的步骤：

1.  保存 `first` 节点的项。
2.  将 `first` 指向 `first.next`，从而跳过原第一个节点，使其可被垃圾回收。
3.  减少栈的大小 `n`。
4.  返回保存的项。

```java
public Item pop() {
    Item item = first.item;
    first = first.next;
    n--;
    return item;
}
```

### 获取栈的大小

获取栈的大小非常简单，直接返回变量 `n` 即可。

```java
public int size() {
    return n;
}
```

这个实现比之前看到的数组表示法稍微复杂一些，但关键区别在于，**这个实现能够满足我们的性能规范**。

---

## 性能评估与总结

让我们根据规范来评估这个实现：

*   **完全实现了给定的API**：所有操作都按预期工作。
*   **常数时间操作**：添加项到开头或从开头移除项都只需几条语句。
*   **线性内存使用**：空间使用量只是栈中项数的倍数。栈有一百万项，就有一百万个节点；栈有三项，就只有三个节点。
*   **无集合大小限制**：代码中没有对集合大小设限。

在本节课中，我们一起学习了如何使用链表实现一个满足严格性能规范的下压栈。你可能一开始觉得这是个相对简单的问题，但深入思考后会发现，能够满足此类性能规范是编程中至关重要的一步。正如你所见，这种能力对于Java虚拟机或PostScript等实际应用极其重要，而这一切都得益于引入了**链式数据结构**的概念。

> 同样，也可以使用单链表实现队列抽象，你可以在教材中看到相关内容。

总结来说，栈和队列是两种基本的数据结构抽象，它们的主要区别在于移除项的顺序。它们的性能规范真正定义了它们。你会发现有些实现自称是栈或队列，却不满足这些性能规范，这些通常无法用于工业级应用。

这向我们指出，**链式结构是数组之外的一种根本性替代方案**，它使我们能够做到仅用数组无法完成的事情。具体来说，我们在本讲座中展示了它们实现了栈和队列的抽象，而这是仅用数组无法满足的。
